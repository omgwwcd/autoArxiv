## 1. Empowering GUI Agents via Autonomous Experience Exploration and Hindsight Experience Utilization for Task Planning

**Topics:** GUI Agent, Task Planning, Compositional Generalization, Experience Exploration
**Authors:** Tianyi Men, Zhuoran Jin, Pengfei Cao, Yubo Chen, Kang Liu, Jun Zhao
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27330v1) | [PDF](http://arxiv.org/pdf/2606.27330v1.pdf)

### 1. 一句话结论

提出PEEU方法，通过自主探索和事后经验利用，使7B小模型在跨网站规划任务中超越32B大模型。

### 2. 为什么重要

解决了小模型在GUI任务规划中泛化能力弱的问题，通过经验驱动的数据合成显著提升跨网站零样本泛化，为低成本、隐私保护的智能体部署提供了可行路径。

### 3. 研究问题

1. 小模型如何通过自主探索和事后经验利用提升跨网站任务规划能力？
2. 不同粒度的任务训练（原子级 vs. 高层级）对组合泛化有何影响？

### 4. 背景与问题设定

多模态Web智能体需将复杂GUI任务分解为可执行动作。小开源MLLM虽成本低、保护隐私，但规划能力弱、跨网站泛化差。现有方法或训练原子级任务（如点击、输入），或使用粗粒度高层任务，但存在轨迹与任务不对齐、约束不足的问题。

### 5. 方法概览

- 任务 / 环境: 真实在线网站（Allrecipes等7个未见网站），使用WebVoyager基准评估。
- 变量 / 干预设计: 两阶段框架：1）规划树探索：智能体自主设定目标并探索网站，构建探索树；2）规划经验利用：从轨迹中提取原子经验，融合生成对齐且严格约束的高层任务，用于SFT和GRPO训练。
- 评价指标: 轨迹级成功率（accuracy）。
- 模型比较: 基线包括Atomic-Prompt、Trajectory-Prompt、Coarse-SFT/GRPO、Atomic-SFT/GRPO，均在相同数据量下比较。
- 我的理解: 该方法模仿人类学习过程：先自由探索积累经验，再事后反思将经验转化为高质量训练数据，从而提升小模型的规划泛化能力。

### 6. 关键发现

- PEEU显著提升跨网站泛化: 基于Qwen2.5-VL-7B的PEEU-SFT在2k轨迹训练下达到30.6%准确率，超越Qwen2.5-VL-32B的22.7%，且在所有设置中均优于基线。
- 高层级任务训练优于原子级任务: TDHAF分析表明，掌握原子技能不能保证高层规划能力，而高层任务训练带来更强的OOD泛化。PEEU-SFT（30.6%）显著高于Atomic-SFT（21.7%）。
- 直接训练比检索更有效: 基于检索的Atomic-Prompt和Trajectory-Prompt在7B模型上仅得3.7%，低于基线的7.8%，而PEEU-GRPO达到19.9%。

左侧展示传统方法：粗任务（如'找4.5星蓝色滑雪板'）产生错误轨迹（黄色、3星）。右侧PEEU流程：先探索构建树，再提取经验（如'点击、3星'），最后利用事后经验将任务重新对齐为'找3星黄色滑雪板'，从而获得严格约束的高质量数据。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 提出自主探索+事后经验利用的范式，可推广至其他交互式智能体系统，降低对人工标注的依赖。
- 对 Skill: 强调高层级任务训练对组合泛化的关键作用，原子技能掌握不保证复杂规划能力。
- 对 Memory: 事后经验提取和融合机制可作为智能体记忆模块的核心，将探索经验转化为可复用的规划知识。
- 对评测: TDHAF框架为评估智能体组合泛化提供了多粒度分析工具，可推广至其他任务分解场景。

### 11. 局限性

- 探索阶段依赖GPT-4o，可能引入额外成本；
- 仅在Web导航任务上验证，未测试其他GUI环境（如桌面应用）；
- 事后经验合成依赖MLLM的总结能力，可能引入噪声。

### 12. 我的看法 / 个人笔记

PEEU通过事后经验对齐任务与轨迹，巧妙解决了粗任务训练中的数据质量问题，且TDHAF分析为理解组合泛化提供了新视角。未来可探索更高效的经验提取方式，并扩展到多模态交互的更多场景。

### 14. 总结

本文提出PEEU方法，通过自主探索和事后经验利用，为小MLLM生成高质量高层任务训练数据，显著提升跨网站规划泛化能力。7B模型在WebVoyager上达到30.6%准确率，超越32B模型。TDHAF分析证实高层任务训练对OOD泛化至关重要。

## 2. CORTEX: A Structured Reasoning Benchmark for Trustworthy 3D Chest CT MLLMs

**Topics:** Medical MLLM, 3D Chest CT, Structured Reasoning, Benchmark
**Authors:** Hashmat Shadab Malik, Anees Ur Rehman Hashmi, Numan Saeed, Muzammal Naseer, Salman Khan, Christoph Lippert
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27264v1) | [PDF](http://arxiv.org/pdf/2606.27264v1.pdf) | [Code](https://anonymous.4open.science/r/CORTEX-2F16/README.md)

### 1. 一句话结论

CORTEX 构建了首个结构化四阶段推理基准，用于 3D 胸部 CT 的可信多模态大模型评估，包含 76,177 条经临床验证的推理轨迹。

### 2. 为什么重要

现有 3D 胸部 CT MLLM 缺乏结构化推理监督和分阶段评估协议，导致诊断过程不可解释、不可验证。CORTEX 通过放射科医生工作流启发的四阶段推理轨迹和五维度评估标准，填补了这一空白，为构建可信赖的 3D 放射学推理模型奠定了基础。

### 3. 研究问题

1. 如何为 3D 胸部 CT 构建结构化、可验证的推理基准？
2. 如何将放射科医生的假设-演绎工作流形式化为多阶段推理轨迹？

### 4. 背景与问题设定

多模态大语言模型在医学影像推理中多采用自由文本链式思维，仅通过最终答案评判，难以解释和验证。现有 3D 胸部 CT VQA 数据集将专家报告简化为问答对，丢弃了连接发现与结论的推理过程，且缺乏患者病史。这阻碍了具备推理能力的 3D 胸部 CT MLLM 的发展，因为既缺少结构化监督训练数据，也缺少推理验证协议。

### 5. 方法概览

- 任务 / 环境: 基于 CT-RATE 验证集（1,304 名患者，3,039 次检查，76,177 个 QA 对），构建 3D 胸部 CT 结构化推理基准。
- 变量 / 干预设计: 将每个样本重构为四阶段推理轨迹：任务理解（重述问题、检查类型和临床上下文）、视觉观察（以第一人称按解剖结构叙述发现）、诊断推理（形成并评估候选假设）、答案合成（给出最终答案和简洁理由）。整合临床上下文（患者病史和检查指征）。使用五个前沿 LLM（MedGemma 27B、Qwen-3 32B、Baichuan-M2 32B、Gemini-3-Flash、Gemini-3.1-Flash）在五个温度（0.0, 0.2, 0.5, 0.7, 1.0）下生成候选轨迹，经规则门控（检查四阶段标签顺序）和五维度评估筛选。
- 评价指标: 五维度评估标准：任务理解（1-10分，评估是否准确重述问题和上下文）、观察忠实度（1-10分，评估观察是否基于报告且不虚构）、假设评估（1-10分，评估候选假设的合理性和证据引用）、推理逻辑（1-10分，评估推理路径的连贯性和完整性）、答案正确性（1-10分，评估最终答案与参考答案的一致性）。前两项作为硬门控（≥8分），后三项均值排序，并设质量阈值θ。
- 模型比较: 图3显示各模型在三个排序维度（观察忠实度、假设评估、推理逻辑）上的平均得分：Gemini-3.1-Flash 在观察忠实度上最高（约9.2），Qwen-3 32B 在假设评估和推理逻辑上领先（约8.8和8.9），MedGemma 27B 整体较低（约8.0-8.3）。这些平均分用于表征模型行为，而非用于逐样本选择。
- 我的理解: 该方法通过放射科医生参与设计的提示模板和评估标准，将临床推理过程结构化，并利用多模型多温度生成和自动筛选确保轨迹质量，最终经放射科医生验证达到93%一致性。

### 6. 关键发现

- 大规模结构化推理数据集: CORTEX 包含 76,177 条验证推理轨迹，其中开放问答 64,224 条，封闭问答 8,914 条，报告生成 3,039 条，从超过 190 万候选轨迹中筛选得出。
- 临床上下文整合减少幻觉: 将患者病史和检查指征整合到问题和推理轨迹中，使诊断基于发现与临床表现的交互，减少模型幻觉。
- 放射科医生验证的高一致性: 三位认证放射科医生独立审查 1,000 个随机样本，推理正确性评估的评分者间一致性达 93%。

### 7. 最重要的图

- 图来源: 图 1：CT-RATE 验证集重构为 CORTEX 三种 VQA 类型，以及 CT-RATE 答案与 CORTEX 推理轨迹的 token 长度对比。
- 为什么重要: 展示了 CORTEX 如何将原有答案对重新组织为结构化推理，并量化了推理轨迹的显著长度增加，体现了多阶段推理的丰富性。

### 8. 如何读这张图

左图：内圈为 CT-RATE 原始类别，外圈为 CORTEX 三种 VQA 类型（开放、封闭、报告生成），覆盖全部 76,177 对。右图：柱状图对比 CT-RATE 答案（短）与 CORTEX 推理轨迹（长）的 token 长度，显示推理轨迹因包含多阶段内容而远长于原始答案。

### 9. 次重要图

- 为什么也重要: 图 3 展示了不同模型在三个排序维度上的平均得分，揭示了模型在观察忠实度、假设评估和推理逻辑上的差异，为选择生成模型提供参考。例如，Gemini-3.1-Flash 在观察忠实度上表现最佳，而 Qwen-3 32B 在假设评估和推理逻辑上领先。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: CORTEX 的结构化推理轨迹可作为智能体系统的训练数据，使其能够执行可解释的诊断步骤，并支持分阶段验证。
- 对 Skill: 四阶段推理框架可迁移至其他医学影像任务，培养模型遵循临床工作流的能力。
- 对 Memory: 临床上下文的整合要求模型在推理中利用外部记忆（患者病史），减少对图像信息的过度依赖。
- 对评测: 五维度评估标准为推理模型提供细粒度反馈，超越传统最终答案准确率，可推广至其他需要可解释性的领域。

### 11. 局限性

- 当前工作仅构建基准，未训练或评估实际推理模型，模型训练和评估留待未来。
- 推理轨迹生成依赖报告作为视觉内容的文本代理，可能丢失图像中的细微视觉信息。
- 评估标准虽经临床设计，但自动化评分仍可能受 LLM 判断偏差影响。

### 12. 我的看法 / 个人笔记

CORTEX 通过结构化推理和临床上下文整合，显著提升了 3D 胸部 CT MLLM 的可解释性和可信度。其放射科医生参与的设计确保了临床相关性，但未来需验证该框架对实际模型性能的提升效果。

## 3. A Process Harness for Uplifting Legacy Workflows to Agentic BPM: Design and Realization in CUGA FLO

**Topics:** Agentic BPM, Process Harness, LLM Agents, Workflow Automation
**Authors:** Fabiana Fournier, Lior Limonad
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27188v1) | [PDF](http://arxiv.org/pdf/2606.27188v1.pdf) | [Code](https://github.com/cuga-project/cuga-agent)

### 1. 一句话结论

提出过程缰绳（Process Harness）机制，通过策略治理的LLM代理层增强传统工作流引擎，实现向Agentic BPM的渐进式升级，无需替换底层引擎。

### 2. 为什么重要

该工作解决了传统BPM在动态环境中缺乏适应性的根本矛盾，通过分离结构合规性与运行时适应性，为遗留工作流系统提供了一条可逆、增量的现代化路径，有望推动企业级流程自动化从刚性执行向治理型智能演进。

### 3. 研究问题

1. 如何在不替换现有工作流引擎的前提下，将传统工作流系统升级为Agentic BPM？
2. 如何设计一个策略治理的代理层，使其既能提供开放世界的推理与适应能力，又能保证对流程模型的结构性遵从？

### 4. 背景与问题设定

传统BPM系统通过BPMN等标准提供严格的结构保证，但面对未预见的场景（如监管变更、新型申请）时只能依赖人工干预或设计时编码的异常路径，缺乏开放世界推理能力。LLM代理虽能提供灵活推理，但直接作为规划器使用无法保证执行符合模型、审计或合规要求。因此，需要一种新机制，在保留确定性执行骨架的同时，引入策略约束的代理智能。

### 5. 方法概览

- 任务 / 环境: 企业级业务流程，以贷款审批工作流为例，包含任务节点、网关节点和序列流，由工作流引擎（如Camunda）驱动执行。
- 变量 / 干预设计: 在流程引擎的指定控制点（任务、网关、序列流）插入回调，由过程缰绳层调用策略治理的LLM代理：TaskAgent处理知识密集型任务，DecisionAgent进行逐案例网关路由，FlowAgent通过钩子机制实现运行时流适应。每个代理的推理受显式策略（FRAME）约束。
- 评价指标: 未明确给出定量指标，但定性标准包括：策略合规性、领域输出正确性、路由确定性、FRAME符合性及干预可审计性。
- 模型比较: 与经典BPM（高结构合规性、低适应性）、LLM-as-Planner（低结构合规性、高适应性）相比，CUGA FLO在结构合规性和运行时适应性两个维度均达到高水平。
- 我的理解: 该方法的核心创新在于将LLM推理分解为三种角色，每种角色绑定到特定的流程元素类型，并通过策略（FRAME）和访问控制函数（ϕ）双重约束，实现了结构性与灵活性的调和。钩子机制允许FlowAgent在运行时对流程拓扑进行策略允许的修改，而无需在设计时枚举所有异常路径。

### 6. 关键发现

- 过程缰绳范式实现结构合规性与运行时适应性的统一: 通过将流程拓扑固定（由引擎强制执行）与流程适应动态化（由FRAME策略治理的FlowAgent在运行时实施）分离，解决了传统BPM的封闭世界假设与LLM代理的开放世界推理之间的张力。
- TDF模型提供三种代理角色的原则性分解: TaskAgent、DecisionAgent和FlowAgent分别对应任务执行、网关路由和流适应，每种代理都有明确的定义、策略集和质量标准，确保了关注点分离和策略治理的可管理性。
- CUGA FLO实现可逆、增量的Agentic BPM升级: 三个自治级别（任务委托、路由委托、流适应）可独立激活或停用，无需修改底层引擎，支持从代理增强BPM到完全Agentic BPM的渐进式转型。

## 4. HarmVideoBench: Benchmarking Harmful Video Understanding in Large Multimodal Models

**Topics:** 多模态理解, 内容安全, 基准测试
**Authors:** Jiajun Wu, Haoyu Kang, Yining Sun, Jiacheng Hou, Heng Zhang, Danyang Zhang, Zhenjun Zhao, Haochi Zhang, Leixin Sun, Eric Hanchen Jiang, Yushan Li, Ruiyu Li, Mengkai Huang, Yan Gao, Xu Zhang, Guancheng Wan
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27187v1) | [PDF](http://arxiv.org/pdf/2606.27187v1.pdf)

### 1. 一句话结论

HarmVideoBench 通过三层推理评估揭示大模型在有害视频理解中表面识别强但深层推理弱的瓶颈，并提出 BCR 方法将宏平均准确率从 61.7% 提升至 84.4%。

### 2. 为什么重要

现有有害视频基准多简化为二分类，忽略多层危害特性且缺乏解释性。HarmVideoBench 首次系统评估模型在可观察证据、片段内含义和片段外推理三个层次的理解能力，为内容审核模型诊断和提升提供了透明、可操作的框架。

### 3. 研究问题

1. 大模型在有害视频理解中，能否超越表面线索进行深层语义和上下文推理？
2. 如何设计基准以区分模型在感知、语义理解和外部知识整合三个层面的失败？

### 4. 背景与问题设定

视频平台有害内容激增，大模型被用于自动审核。但现有基准存在两大缺陷：一是将评估简化为二分类，忽略有害视频的多层特性（如隐含恶意、文化隐喻）；二是缺乏解释性，模型可能利用表面捷径（如特定词汇）而非真正理解危害意图，导致评估黑箱化。

### 5. 方法概览

- 任务 / 环境: 有害视频理解任务，输入为视频帧和转录文本，输出为多选答案。
- 变量 / 干预设计: 基准包含 1379 个视频，每个配 3 个问题，分别对应可观察证据、片段内含义、片段外推理三个层次。BCR 方法通过预测推理边界并仅在需要时检索外部上下文来增强模型。
- 评价指标: 每个层次的分类准确率及宏平均准确率。
- 模型比较: 评估 19 个模型，包括闭源（如 Gemini-3.1-Pro）和开源（如 Qwen3-VL 系列），并对比随机和人类基线。
- 我的理解: 该基准通过分层设计，将有害视频理解从单一分类拆解为递进推理过程，类似认知诊断测试，能精准定位模型在哪个环节失败。BCR 方法则像给模型加了一个“边界感知器”，避免其过度依赖局部线索或随意调用外部知识。

### 6. 关键发现

- 模型在可观察证据上接近人类，但在深层推理上差距显著: 在可观察证据层，最佳模型（Gemini-3.1-Pro）准确率 96.0%，接近人类 98.9%；但在片段内含义层降至 84.6%，片段外推理层仅 65.8%，与人类（97.5% 和 96.2%）差距巨大。
- BCR 方法显著提升深层推理能力: 基于 Qwen3-VL-8B 的 BCR 方法将宏平均准确率从 61.7% 提升至 84.4%，尤其在片段外推理层从 48.1% 提升至 70.1%，接近闭源模型水平。

### 7. 最重要的图

- 图来源: Figure 1: 对比传统二分类评估与 HarmVideoBench 三层推理评估
- 为什么重要: 直观展示了 HarmVideoBench 的核心创新：从简单判断“是否有害”转向诊断模型在三个推理层次上的表现，揭示了传统评估忽略的深层危害理解能力。

左侧传统方法：输入视频后直接输出二分类结果（有害/无害），无法解释原因。右侧 HarmVideoBench：将评估分解为三个递进层次——可观察证据（识别表面线索）、片段内含义（理解内部语义）、片段外推理（结合外部上下文），每个层次输出具体答案，从而定位模型失败环节。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 内容审核智能体需具备分层推理能力，不能仅依赖表面特征；BCR 方法可集成到智能体中，动态决定是否需要外部知识检索。
- 对 Skill: 模型需训练“边界感知”技能，区分问题是否依赖外部上下文，避免过度泛化或遗漏关键信息。
- 对 Memory: 片段外推理依赖外部知识，模型需有效存储和检索文化、历史等上下文信息，BCR 的选择性检索策略为此提供了轻量级方案。
- 对评测: 评估应从单一分类转向分层诊断，以识别模型在感知、语义和推理各环节的弱点，推动可解释的内容安全评估。

### 11. 局限性

- 基准视频主要来自公开数据集和平台，可能未覆盖所有危害类型（如新兴的深度伪造或隐蔽性操纵）。
- BCR 方法依赖训练集划分，在无训练数据场景下可能无法直接应用。
- 人类基线来自熟悉标注政策的标注者，不代表普通用户表现。

### 12. 我的看法 / 个人笔记

HarmVideoBench 是内容安全评估领域的重要进步，其分层设计不仅揭示了模型在深层推理上的短板，还为改进提供了明确方向。BCR 方法简洁有效，但未来需探索更通用的边界预测机制。该工作对推动可解释、负责任的内容审核系统具有实际价值。

## 5. Semantic Early-Stopping for Iterative LLM Agent Loops

**Topics:** LLM Agent, Early Stopping, Semantic Convergence, Multi-Agent Systems, Retrieval-Augmented Generation
**Authors:** Sahil Shrivastava
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27009v1) | [PDF](http://arxiv.org/pdf/2606.27009v1.pdf) | [Code](https://github.com/SahilShrivastava-Dev/semantic-halting-problem)

### 1. 一句话结论

基于语义收敛的无判断器早停策略可节省38%运营token且质量无损，但全质量门控版本因每轮调用判断器而成本激增129%；批评家批准信号在级联中优先级最高但实际触发极少，oracle差距表明识别最佳轮次是开放问题。

### 2. 为什么重要

该研究揭示了多智能体LLM循环中固定迭代次数的低效性，提出内容感知的语义早停方法，并提供了诚实理论（仅证明确定性终止而非收缩）、高效评估协议（运营/评估token分离揭示隐藏成本）和实证结果，为智能体系统的自适应计算和公平比较开辟了新方向。

### 3. 研究问题

1. 如何设计内容感知的早停规则以替代固定迭代次数？
2. 语义收敛信号与质量信号结合能否提升早停效果？
3. 如何公平比较不同停止策略并揭示隐藏成本？

### 4. 背景与问题设定

多智能体LLM循环（如Writer-Critic）通常使用固定迭代次数作为停止条件，这是一种语法层面的终止开关，无法感知内容是否仍在改进，导致简单输入浪费token而复杂输入被截断。本文研究语义早停：当连续草稿的嵌入在语义上停止变化（余弦距离低于阈值且持续k轮）且质量停止提升时停止循环。

### 5. 方法概览

- 任务 / 环境: 多跳检索增强问答（HotpotQA），使用distractor设置，过滤出多跳困难问题，每个问题提供约4个上下文段落（支持段落+干扰段落）。
- 变量 / 干预设计: 比较多种停止策略：固定迭代次数（max_iterations，6轮）、仅熵信号（entropy_only，仅余弦距离）、仅批评家（critic_only）、固定轮次（fixed_k，k=1,3,6）、全SHP（shp full，包含批评家批准、熵收敛、质量无增益、故障安全四级优先级级联）以及oracle（选择最佳轮次）。采用轨迹回放和缓存判断实现严格配对比较。
- 评价指标: 运营token数（Writer+Critic每轮消耗，不含判断器）、评估token数（判断器调用，单独计量）、信息得分（IS，基于RAGAS指标：忠实度、答案相关性、上下文精确度、上下文召回率的加权和）、配对t检验和Wilcoxon符号秩检验、TOST非劣效性检验（margin δ=0.02）。
- 模型比较: 所有智能体（Writer、Critic、Judge）均使用llama-3.1-8b-instruct模型，嵌入使用本地句子嵌入模型。基线为max_iterations（固定6轮）。
- 我的理解: 该方法通过将草稿映射到嵌入空间并监测余弦距离，实现了对语义变化的量化。结合质量信号（IS）形成四级优先级级联：批评家批准（最高优先级，若批评家批准则立即停止）、熵收敛（余弦距离低于阈值持续k轮）、质量无增益（IS不再提升）、硬性故障安全（达到最大轮次无条件停止）。评估协议通过轨迹回放（一次生成全轨迹，各策略回放选择停止轮次）和缓存判断（每个草稿仅判断一次）消除了生成噪声和重复判断成本，并严格区分运营token（策略运行消耗）和评估token（测量消耗），使比较公平且高效。

### 6. 关键发现

- 诚实理论：证明确定性终止而非收缩，批评家批准信号优先级最高但实际触发极少: 论文放弃了之前声称的Banach收缩定理（因LLM生成无Lipschitz常数保证），仅证明确定性终止（Theorem 1：故障安全无条件保证在T_max轮内停止）和良定义性（Lemma 1：IS在[0,1]内，距离在[0,2]内），并通过机器可执行测试验证。语义非扩张性作为经验猜想，通过测量发现距离平均递减但仅5%轨迹严格单调，解释了耐心窗口（k=2）的必要性。批评家批准信号在级联中优先级最高，但实际中批评家很少直接批准（通常提供反馈而非批准），因此该信号极少触发，主要作用被熵信号覆盖。
- 无判断器熵信号节省38%运营token且质量无损: 在60个问题的测试集上，仅使用余弦距离信号的entropy_only策略相比max_iterations节省38%运营token，信息得分差异仅为-0.004（p=0.81），统计上无显著质量损失。fixed_k3同样节省53%且质量持平。
- 全SHP策略因每轮调用判断器而成本最高（+129%），批评家批准信号效果有限: 全SHP策略（包含质量门控和批评家批准）消耗的运营token比基线多129%（2.3倍），且质量无提升（ΔIS=-0.004，p=0.78）。成本激增源于每轮调用LLM判断器计算IS，而判断器调用成本远高于Writer/Critic生成。批评家批准信号在级联中优先级最高，但实际触发极少，验证了无判断器设计的必要性。
- 固定一轮策略在质量上最优，但oracle显示巨大提升空间: fixed_k1（返回第一轮草稿）在质量上最高（IS=0.700），但oracle（选择最佳轮次）达到0.785，高出0.115（p≈4e-11），表明识别最佳轮次是未解决的开放问题。迭代并未提升质量，反而可能引入噪声。

### 7. 最重要的图

- 图来源: 图5：平均语义距离d_t随轮次变化（测试集，N=60），带95%置信带。
- 为什么重要: 该图直观展示了语义收敛趋势：距离在首次修订后急剧下降，随后稳定在阈值附近，验证了语义非扩张性猜想，并解释了为何需要耐心窗口（k=2）来容忍噪声。

### 8. 如何读这张图

横轴为轮次，纵轴为平均语义距离d_t（余弦距离）。实线为均值，阴影为95%置信区间。虚线为停止阈值ε=0.06。距离在轮次2后显著下降并保持低于阈值，表明语义收敛发生在前几轮，后续轮次变化微小。

### 9. 次重要图

- 为什么也重要: 图4展示了各策略相对于基线的运营token节省百分比，清晰揭示了无判断器策略（entropy_only、fixed_k）节省token，而全SHP和oracle因每轮调用判断器而成本激增，突出了运营/评估token分离这一方法论贡献的价值。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 语义早停可显著降低多智能体循环的token成本，且无需额外判断器，易于集成到现有系统中。批评家批准信号可保留但优先级需谨慎设置，因其实际触发极少。
- 对 Skill: 智能体应具备自适应停止能力，根据内容收敛性动态调整迭代次数，而非固定轮次。
- 对 Memory: 嵌入缓存和轨迹回放机制可减少重复计算，提升系统效率。
- 对评测: 区分运营token和评估token的协议揭示了隐藏成本，为公平比较不同策略提供了方法论，应成为未来智能体评估的标准实践。

### 11. 局限性

- LLM判断器存在噪声，即使N=60时非劣效性检验也未严格通过，需更强或人工验证的判断器。
- HotpotQA答案简短，单轮草稿即可回答，未充分测试迭代质量提升；长文本生成任务可能更适用。

### 12. 我的看法 / 个人笔记

该研究以诚实和严谨的态度处理了多智能体循环的早停问题，理论证明仅保留可验证部分，实验设计巧妙（轨迹回放、缓存判断、运营/评估token分离），结果清晰揭示了无判断器策略的实用性和全SHP的陷阱。批评家批准信号在级联中优先级最高但实际效果有限，这一发现对系统设计有指导意义。oracle差距的量化将问题从“何时停止”转向“选择最佳轮次”，为未来研究提供了明确目标。

### 14. 总结

本文提出语义早停（SHP）方法，通过监测草稿嵌入的余弦距离和质量信号，替代固定迭代次数。在HotpotQA上，无判断器熵信号策略节省38%运营token且质量无损，而全SHP因每轮调用判断器成本最高（+129%）。批评家批准信号在级联中优先级最高但实际触发极少。oracle差距表明识别最佳轮次是开放问题。研究提供了诚实理论（证明确定性终止而非收缩）、高效评估协议（运营/评估token分离）和实证结果，为智能体自适应计算奠定基础。
