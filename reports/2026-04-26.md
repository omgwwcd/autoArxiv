## 1. When Prompts Override Vision: Prompt-Induced Hallucinations in LVLMs

**Topics:** 大视觉语言模型, 幻觉, 多模态评估, 偏好优化, 指令跟随
**Authors:** Pegah Khayatan, Jayneel Parekh, Arnaud Dapogny, Mustafa Shukor, Alasdair Newson, Matthieu Cord
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.21911v1) | [PDF](http://arxiv.org/pdf/2604.21911v1.pdf) | [Code](https://pegah-kh.github.io/projects/prompts-override-vision/)

### 1. 一句话结论

本文提出诊断基准HalluScope和微调框架HalluVL-DPO，首次量化证明LVLM幻觉主要源于对文本指令预设的过度依赖，并提出基于语义差距加权的偏好优化方法进行针对性缓解。

### 2. 为什么重要

LVLM的幻觉严重制约其在安全关键场景（如自动驾驶、医疗诊断）的可靠部署。厘清幻觉根源并开发有效缓解技术，是推动多模态AI走向实用的核心挑战。

### 3. 研究问题

1. LVLM幻觉的主要驱动因素是什么？是视觉感知局限、语义共现先验，还是文本指令预设？
2. 如何构建一个能系统分离不同幻觉成因的诊断性评估基准？
3. 如何设计一个能有效缓解由文本指令先验引发幻觉的微调框架？

### 4. 背景与问题设定

尽管LVLM取得显著进展，但其输出仍常出现与视觉输入不符的幻觉。先前研究将幻觉归因于视觉主干限制或语言主导，但各因素相对贡献不明。现有评估基准（如POPE、CHAIR）将幻觉视为单一现象，无法区分错误源于感知失败、共现先验还是指令预设，阻碍了针对性诊断与改进。

### 5. 方法概览

- 任务 / 环境: 评估和缓解大型视觉语言模型在视觉问答任务中的幻觉，重点关注由文本指令诱导的失败模式。
- 变量 / 干预设计: 1. 诊断阶段：提出HalluScope基准，包含四类问题——正例物体识别（Rec_pos）、随机物体识别（Rec_rnd）、对抗性物体识别（Rec_adv）、对抗性预设属性（AdP），以分离感知、共现先验和指令预设的影响。2. 缓解阶段：提出HalluVL-DPO微调框架，使用基于样本信息量的加权直接偏好优化。框架利用构建的偏好数据集（包含正确接地回答与幻觉回答的对），并引入基于语义差距的权重（1-3分），使模型更关注信息量大的样本对。
- 评价指标: 正例识别准确率（Rec_pos）、随机识别准确率（Rec_rnd）、对抗性识别准确率（Rec_adv）、对抗性预设性能（AdP，即模型正确拒绝错误预设的比例）。
- 模型比较: 在HalluScope上评估了InternVL3-5-8B、Molmo-7B-D、Qwen2-VL-7B、Qwen2.5-VL-7B、LLaVA-1.5-7B等多个开源LVLM。
- 我的理解: 研究遵循“先诊断，后治疗”的逻辑。诊断发现，随着视觉主干能力提升，幻觉主因已从“看错”（感知失败）转变为“想偏”（被文本指令预设和语义先验带跑）。治疗则针对此主因，使用加权的偏好优化进行对齐微调。

### 6. 关键发现

- 幻觉主因是文本指令预设，而非视觉感知缺陷: 所有测试模型在Rec_pos和Rec_rnd上准确率均高于85%，表明视觉感知可靠。但在Rec_adv上准确率显著下降（8%-37%），在AdP上下降更剧烈（25%-85%）。这证明现代LVLM的幻觉主要由文本指令中引入的预设和模型自身的语义共现知识驱动。
- HalluVL-DPO能有效缓解目标幻觉且保持通用能力: 基于加权DPO的微调框架显著降低了由指令预设引发的幻觉（AdP任务）。同时，模型在其它幻觉基准（如POPE）和通用视觉能力评估上保持或提升了性能，表明缓解具有针对性且未损害原有能力。
- 样本信息量加权能提升偏好优化效果: 与标准DPO相比，根据响应对之间的语义差距分配权重（低、中、高信息量），能更有效地利用训练数据，聚焦于学习信号清晰的对比样本，从而提升对齐效果。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 多模态智能体必须增强对指令中误导性预设的“免疫力”，确保决策严格基于环境感知，而非被文本先验带偏，这对安全至关重要。
- 对 Skill: 评估LVLM时，需将其“视觉感知技能”与“抵抗文本诱导偏差的对齐技能”分开评估。HalluScope提供了这样的分离评估工具。
- 对 Memory: 模型从训练数据中学到的语义共现知识（一种“世界知识记忆”）可能成为幻觉源，需开发机制使其能与实时视觉证据进行动态校准。
- 对评测: 未来多模态评估应走向“诊断式”，不仅报告整体性能，更要揭示错误根源（感知、先验、指令），为模型改进提供明确方向。

### 11. 局限性

- HalluScope基准的构建依赖外部视觉模型（如Grounding-DINO、Florence-2）进行物体标注与验证，其自身误差可能影响基准的纯净度。
- HalluVL-DPO的缓解效果依赖于构建的偏好数据集的质量与多样性，其在更开放、复杂指令下的泛化能力有待进一步验证。
- 当前工作主要关注物体存在性相关的幻觉，对属性、关系等更细粒度幻觉的成因诊断与缓解涉及较少。

### 12. 我的看法 / 个人笔记

这是一项非常扎实且思路清晰的研究。其核心贡献在于通过精巧的基准设计，将长期模糊的“幻觉”问题分解量化，明确指出现代LVLM的“阿喀琉斯之踵”在于文本模态的过度主导。这不仅更新了领域认知，也为后续研究指明了重点——多模态对齐。HalluVL-DPO框架的加权设计也体现了对训练数据质量的精细考量。整体而言，论文为构建更可靠、更忠实的LVLM提供了重要的诊断工具和有效的干预思路。

### 13. 这篇工作引用的近期高影响力相关文章

  为什么重要：广泛使用的闭端幻觉评估基准。

  与当前论文关系：HalluScope可视为POPE的扩展与深化，不仅规模更大，其核心创新在于通过问题设计（如AdP）分离了不同幻觉成因，提供了诊断性洞察。

  为什么重要：偏好优化的基础性方法。

### 14. 总结

本文对LVLM的幻觉问题进行了系统性研究。首先，提出诊断性基准HalluScope，首次量化揭示现代LVLM的幻觉主要源于对文本指令预设和语义共现先验的过度依赖，而非视觉感知缺陷。基于这一洞见，作者提出HalluVL-DPO微调框架，采用基于语义差距加权的偏好优化，有效缓解了目标幻觉模式，且不损害通用能力。这项工作标志着LVLM可靠性研究的焦点从提升感知能力转向解决多模态对齐挑战。

## 2. A Multimodal Text- and Graph-Based Approach for Open-Domain Event Extraction from Documents

**Topics:** 事件抽取, 开放域, 多模态学习, 图神经网络, 大语言模型
**Authors:** Praval Sharma
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.21885v1) | [PDF](http://arxiv.org/pdf/2604.21885v1.pdf)

### 1. 一句话结论

提出MODEE方法，通过图神经网络（GraphSAGE）与大语言模型（T5）的多模态融合，实现开放域文档级事件抽取，性能超越现有方法。

### 2. 为什么重要

现有开放域事件抽取方法依赖启发式规则，泛化性差，且未充分利用大语言模型（LLM）的潜力。LLM自身存在‘中间丢失’和注意力稀释问题，难以建模文档级结构语义，而这对准确抽取跨句事件至关重要。

### 3. 研究问题

1. 如何设计一个能显式建模文档级上下文、结构与语义关系的开放域事件抽取框架？
2. 如何有效融合图表示与文本表示，以弥补纯LLM方法在文档级推理上的不足？

### 4. 背景与问题设定

事件抽取旨在从文本中识别事件及论元（如时间、地点、人物）。封闭域方法受限于预定义事件模式，无法泛化到未见类型。开放域方法可处理任意事件类型，但多依赖规则或简单微调LLM，未能显式建模文档级信息。LLM在长文档处理中面临注意力稀释挑战。

### 5. 方法概览

- 任务 / 环境: 输入为一个描述主事件的文档，输出为该事件的5Ws要素（where, when, what, who, why）。遵循‘一篇文档一个主事件’设定。
- 变量 / 干预设计: MODEE包含四个模块：1) 文本编码器（T5编码器）生成令牌级上下文嵌入；2) 图编码器（基于GraphSAGE，使用LSTM聚合）为文档令牌构建的全连接图生成节点嵌入；3) 注意力门控多模态融合模块，计算门控向量并加权融合两种嵌入；4) 文本解码器（T5解码器）基于融合表示生成格式化的5Ws序列。
- 评价指标: 使用精确匹配（EM）计算的精确率、召回率、F1分数，以及ROUGE-L和BERTScore。
- 模型比较: 与微调T5、零样本/五样本提示T5、以及现有开放域方法（如Liu et al., 2019）进行对比。MODEE在开放域和封闭域数据集上均表现最优。
- 我的理解: 核心创新在于从单一文本源衍生出‘文本’和‘图’两种模态，通过GNN显式捕获文档级令牌关系结构，再通过可学习的门控机制与LLM的深层语义动态融合，使模型能聚焦于事件相关令牌，实现更精准的端到端抽取。

### 6. 关键发现

- 多模态融合显著提升性能: 结合图结构信息（捕捉文档级关系）与LLM文本表示（提供深层语义）的注意力门控融合机制，使MODEE在开放域事件抽取任务上全面超越纯文本LLM基线及传统方法。
- 强大的跨领域泛化能力: MODEE虽为开放域设计，但在封闭域事件抽取数据集上经过评估，其性能也优于专门的封闭域方法，证明了其框架的通用性和适应性。
- 端到端生成式设计的有效性: 统一的T5编码器-解码器框架实现了端到端训练与推理，无需依赖外部实体识别等预处理步骤，简化流程的同时提升了整体协同性能。

### 7. 最重要的图

- 图来源: Figure 1: Overview of event extraction in MODEE.
- 为什么重要: 该图完整展示了MODEE的四阶段架构：从文档输入，到并行的文本编码与图编码，再经注意力门控融合，最后解码生成5Ws。直观体现了其多模态、端到端的核心设计思想与信息流。

从左至右阅读：1) 输入文档；2) 上方路径：文本经T5编码器得到文本嵌入；下方路径：文档被构建为全连接图，经GraphSAGE编码得到图节点嵌入；3) 中间：两种嵌入通过注意力门控融合模块进行整合；4) 右侧：融合后的表示输入T5解码器，生成结构化的5Ws输出序列。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 增强了智能体从长文档中快速、准确提取结构化事件信息的能力，为基于事件的决策、规划与知识库构建提供了更可靠的工具。
- 对 Skill: 示范了如何将图结构推理（GNN）与序列生成模型（LLM）深度融合，形成一种新的、可处理复杂文档结构的多模态理解与生成技能。
- 对 Memory: 提取的标准化5Ws事件表示可作为紧凑、语义丰富的事件记忆单元，便于后续的索引、关联、追溯和推理。
- 对评测: 强调了对于生成式事件抽取，需结合表面形式匹配（EM）与语义相似度（BERTScore）的多维度评估体系。

### 11. 局限性

- 基于‘一篇文档一个主事件’的假设，对包含多个同等重要事件的复杂文档处理能力有限。
- 图编码器使用全连接图，虽能建模任意令牌间依赖，但计算复杂度为O(n²)，可能引入噪声并限制处理更长文档的 scalability。
- 实验数据集集中于2015-2019年的印度新闻领域，在其他领域或体裁（如学术论文、社交媒体）的泛化性能有待验证。

### 12. 我的看法 / 个人笔记

这项工作提出了一个优雅且有效的解决方案：通过从单一文本源构建图来创造‘结构模态’，与‘文本模态’互补。其注意力门控融合机制设计精巧，使模型能自适应地聚焦关键信息。使用对比损失（每5W类采样5个节点）训练图编码器是一个重要细节，有助于学习有判别力的结构表示。‘从文本中衍生多模态’的思路兼具创新性与实用性，为开放域信息抽取提供了新范式。

### 13. 这篇工作引用的近期高影响力相关文章

- Liu et al. (2019)
  为什么重要：开放域事件抽取的代表工作，使用神经隐变量模型。
  核心贡献：提出不依赖预定义模式的开放域事件抽取方法。
  与当前论文关系：MODEE将其作为主要性能对比基线，旨在克服其依赖有限监督、泛化能力不足的问题。
- Tong et al. (2022)
  为什么重要：强调事件常跨多句描述，并采用‘一篇文档一个主事件’的设定。
  核心贡献：关注文档级事件抽取，指出句子级方法的局限。
  与当前论文关系：MODEE遵循其主事件抽取设定，并致力于解决其指出的文档级信息不完整问题。
- Cao et al. (2024)
  为什么重要：尝试将LLMs应用于事件抽取任务。
  核心贡献：通过对LLMs进行简单微调来处理事件抽取。
  与当前论文关系：MODEE指出这类工作未显式建模文档级推理，并通过引入图结构的多模态融合来弥补这一不足。

### 14. 总结

MODEE是一种新颖的开放域文档级事件抽取框架。它利用图神经网络（GraphSAGE）对文档令牌全连接图进行编码以捕获结构信息，与大语言模型（T5）的文本表示相结合，并通过注意力门控机制进行动态融合，最终以端到端方式生成事件的5Ws要素。在包含10,000篇印度新闻的数据集上，MODEE在开放域和封闭域任务上均超越了微调LLM、提示LLM及传统方法，证明了多模态融合与显式文档级推理建模的有效性。

## 3. From Codebooks to VLMs: Evaluating Automated Visual Discourse Analysis for Climate Change on Social Media

**Topics:** 计算机视觉, 视觉语言模型, 社交媒体分析, 气候变化传播
**Authors:** Katharina Prasse, Steffen Jung, Isaac Bravo, Stefanie Walter, Patrick Knab, Christian Bartelt, Margret Keuper
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.21786v1) | [PDF](http://arxiv.org/pdf/2604.21786v1.pdf) | [Code](https://github.com/KathPra/Codebooks2VLMs.git)

### 1. 一句话结论

本研究系统评估了视觉语言模型在社交媒体气候视觉话语分析中的表现，发现Gemini-3.1-flash-lite性能最佳，并提出了适用于社会科学研究的分布级评估指标与提示设计策略。

### 2. 为什么重要

社交媒体上的视觉内容是气候传播的关键，但海量图像难以人工分析。本研究为利用VLM进行大规模、自动化的视觉话语分析提供了首个系统性评估框架与实用指南，有助于社会科学研究者理解自动化方法的可行性与局限性。

### 3. 研究问题

1. VLM能否生成符合社会科学编码手册质量的视觉气候话语分析标注？
2. 分类法设计如何影响分类性能？
3. 计算机视觉模型的选择有多重要？
4. 提示工程对性能有何影响？

### 4. 背景与问题设定

社交媒体已成为气候传播的主要场域，视觉内容因其高参与度与情感影响力而尤为关键。然而，现有计算研究多集中于文本分析，对海量视觉内容的分析仍依赖小规模人工标注，成本高昂。尽管CLIP和VLM等模型为零样本图像分类提供了可能，但其在社会科学领域特定、细粒度且具文化背景的分类任务中的可靠性尚未得到系统评估。

### 5. 方法概览

- 任务 / 环境: 使用来自X（原Twitter）的两个气候相关图像数据集：ClimateCT（1,038张专家标注图像）和ClimateTV（超过120万张图像，其中5万张标签经过人工验证）。
- 变量 / 干预设计: 在五个标注维度（动物内容、气候变化后果、气候行动、图像场景、图像类型）上，对6个可提示的VLM和15个零样本CLIP类模型进行基准测试。系统消融了提示组件（类别描述长度、超类别定义、思维链推理、空间定位指令）的影响。
- 评价指标: 除了实例级准确率，还采用分布级评估指标（如Jensen-Shannon散度）来评估模型预测的标签分布与真实总体趋势的一致性。
- 模型比较: Gemini-3.1-flash-lite在所有超类别和两个数据集上均表现最佳。性能与中等规模开源模型之间的差距相对较小（0-8%）。
- 我的理解: 这是一个严谨的实证研究，旨在弥合先进计算机视觉技术与社会科学研究需求之间的鸿沟。其核心贡献不仅在于性能排名，更在于为社会科学研究者提供了一套从分类法设计、模型选择、提示工程到评估的完整方法论指南。

### 6. 关键发现

- 模型性能与分布级评估: Gemini-3.1-flash-lite综合性能最优。更重要的是，即使单图像准确率中等，VLM的预测也能可靠地恢复总体层面的趋势分布，这使其成为大规模话语分析的可行起点。
- 提示工程的影响: 思维链推理反而会降低性能。为每个标注维度设计特定的提示能提升性能。简单的任务需要更简单的提示。
- 分类法设计的影响: 社会科学编码手册中常见的层次化、语义重叠的类别（如“陆地哺乳动物”与“北极熊”）会给分类器带来系统性混淆，影响性能。
- 开源模型的竞争力: 中等规模的开源模型与顶级商业模型（如Gemini）的性能差距较小，为资源有限的研究提供了可行选择。

## 4. Who Defines "Best"? Towards Interactive, User-Defined Evaluation of LLM Leaderboards

**Topics:** LLM评估, 排行榜, 人机交互, 数据可视化, 负责任AI
**Authors:** Minji Jung, Minjae Lee, Yejin Kim, Sarang Choi, Minsuk Kahng
**Venue / Year:** FAccT '26
**Links:** [Abstract](http://arxiv.org/abs/2604.21769v1) | [PDF](http://arxiv.org/pdf/2604.21769v1.pdf)

### 1. 一句话结论

通过分析LMArena数据集的结构性偏差与排名异质性，提出并验证了一种支持用户交互式定义评估优先级（选择和加权提示切片）的可视化工具，以生成情境化模型排名。

### 2. 为什么重要

主流LLM排行榜（如LMArena）的单一聚合排名由基准设计者预设的数据构成和权重决定，掩盖了模型在不同任务类型上的表现差异，无法满足多样化用户的具体需求和约束，可能导致部署决策失准。

### 3. 研究问题

1. LMArena数据集的构成如何影响模型排名，存在哪些具体偏差？
2. 模型排名在不同提示类别（切片）中的异质性有多大？
3. 交互式、用户可定义权重的评估工具能否提升排行榜的透明度和情境适用性？

### 4. 背景与问题设定

LLM排行榜已成为模型比较和选型的核心依据，但其评估框架通常由少数设计者固定，排名结果被广泛用户被动消费。这种不对称性导致排名反映的是设计者的优先级，而非用户多样化的目标。单一聚合分数无法揭示模型在特定数据子集上的表现，而这些子集对许多实际场景至关重要。

### 5. 方法概览

- 任务 / 环境: 分析LMArena平台发布的Human Preference 140K数据集（含53个模型，约13.5万条人类偏好判断），并构建其语义主题层次。
- 变量 / 干预设计: 1. 数据集分析：通过两阶段流程（聚类生成400个细粒度主题，再经LLM归纳与人工修正构建3层主题层次）量化数据分布偏差与跨类别排名差异。2. 设计干预：开发交互式可视化界面作为设计探针，允许用户选择、加权提示切片并实时观察排名变化。
- 评价指标: 胜率（win rate）、斯皮尔曼等级相关系数（衡量跨类别排名一致性）、双比例z检验（识别模型在特定类别的异常表现）、贝叶斯平滑（处理小样本不确定性）。
- 模型比较: 对比模型整体排名与特定类别排名。关键案例：Claude系列模型在整体排名未进前20，但在‘数据处理与分析’类别中多个模型跃升至前5；minimax-m1在‘数学’类别胜率极高（z-score > 8）。
- 我的理解: 方法核心是将评估重构为‘感知构建’活动：用户不再是排名被动接受者，而是通过交互探索数据构成、自定义权重，主动生成符合自身情境的评估视图。

### 6. 关键发现

- 数据集存在显著主题偏差与噪声: LMArena数据严重偏向编程与AI开发主题（占约30%），过度代表开发者群体兴趣；包含大量简单问候语（>1000条，79%被武断判定胜负）和高度重复提示（如‘草莓中有几个r’出现205次），这些噪声可能扭曲聚合排名。
- 模型排名高度依赖提示类别，聚合排名误导性强: 不同提示类别下的模型排名差异巨大。例如，‘数据处理与分析’类别与整体排名的斯皮尔曼相关系数最低（0.60），‘Web前端’为0.70。这表明单一全局排名无法可靠预测模型在特定任务上的表现。
- 交互式、用户定义的评估工具能有效支持情境化决策: 定性研究（N=10）表明，提供的交互式可视化工具使用户能够：1）洞察数据构成偏差；2）根据自身需求（如更看重‘数学’或‘代码生成’）调整切片权重；3）观察并理解排名随权重变化的动态过程，从而做出更透明、更贴合自身目标的模型选择。

### 7. 最重要的图

- 图来源: 论文图2：跨中级别提示类别的模型性能热力图。
- 为什么重要: 该图是论文核心发现的视觉总结。它系统性地展示了模型性能的异质性：将每个模型在不同类别中的胜率并置，并按类别与整体排名的相关性排序，直观揭示了哪些类别导致排名‘反转’（如最右侧的‘数据处理与分析’），以及哪些模型在特定类别有突出表现（如minimax-m1在‘数学’的深绿色单元格）。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为特定任务选择Agent的底层LLM时，应优先考察其在相关任务切片（如工具调用、规划、特定领域QA）上的表现，而非全局排名。

- 对 Memory: N/A
- 对评测: 未来LLM评估体系应内置交互式探索能力，将数据集构成和聚合方法透明化，支持用户根据应用上下文进行动态、情境化的解读，推动评估从‘竞技体育’转向‘决策支持’。

### 11. 局限性

- 定性研究规模小（N=10），且参与者可能已有较强技术背景，工具对更广泛普通用户的可及性和效用有待验证。

### 12. 我的看法 / 个人笔记

这是一项极具洞察力和实践价值的工作。它没有提出新的评估指标或基准，而是犀利地指出了当前评估生态中的一个根本性错位：评估的制定者与使用者分离。其提出的交互式解决方案巧妙且务实，通过可视化将数据构成的‘黑箱’打开，将部分评估主动权交还给终端用户。这不仅是一个工具设计，更是一种评估范式的倡导——从寻求‘唯一真理排名’转向支持‘情境化决策’。对于促进LLM负责任和高效的应用部署具有重要意义。
