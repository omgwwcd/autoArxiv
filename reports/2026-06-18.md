## 2. Beyond the Current Observation: Evaluating Multimodal Large Language Models in Controllable Non-Markov Games

**Topics:** GUI Agent, Multimodal Large Language Models, Memory, Non-Markov Games
**Authors:** Shengyuan Ding, Xilin Wei, Xinyu Fang, Haodong Duan, Dahua Lin, Jiaqi Wang, Yuhang Zang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.19338v1) | [PDF](http://arxiv.org/pdf/2606.19338v1.pdf)

### 1. 一句话结论

RNG-Bench通过两个非马尔可夫游戏（Matching Pairs和3D Maze）系统评估多模态大模型在闭环交互中基于历史观测重建隐藏状态并采取行动的能力，发现前沿模型远未饱和，遗忘是主要瓶颈，且微调可提升性能并迁移至其他基准。

### 2. 为什么重要

现有基准要么暴露完整状态，要么将记忆与其他技能混淆，而RNG-Bench首次隔离了“记住-行动”能力，这对部署在具身控制、多轮工具使用等场景中的多模态模型至关重要。

### 3. 研究问题

1. 多模态大模型能否在闭环交互中从历史观测中重建隐藏状态并据此行动？
2. 模型失败的主要原因是遗忘还是决策错误？
3. 可控难度轴（规模、视觉模式、观测模态）如何影响记忆-行动能力？

### 4. 背景与问题设定

在长程交互中，正确行动常依赖多步前的观测而非当前视图，即非马尔可夫性。现有基准要么完全可见状态（如围棋），要么将隐藏状态重建与其他技能（如探索、规则理解）混杂，要么仅测试事后问答（remember-to-answer）。RNG-Bench设计为闭环游戏，要求模型在每一步基于历史重建隐藏状态并行动（remember-to-act），且错误会因果性地改变后续观测。

### 5. 方法概览

- 任务 / 环境: 两个互补游戏：Matching Pairs（静态离散隐藏状态，需记忆卡片身份-位置绑定）和3D Maze（动态空间隐藏状态，需从第一人称视图构建地图）。
- 变量 / 干预设计: 统一框架下控制三个难度轴：网格/地图大小、视觉模式（文本/2D/3D）、观测模态。引入决斗协议控制实例级方差，以及Memory Gap指标分离遗忘与决策。
- 评价指标: Matching Pairs：Score%（匹配对比例）和Resp./Score（每对所需响应数）；3D Maze：Success Rate (SR)、Efficiency (Eff)、Explore、Wall Hits，以及综合Game Score (GS)。Memory Gap：正常与注入真实隐藏状态（oracle）条件下的分数差距。
- 模型比较: 评估GPT-5.4、Gemini-3.1-Pro、Seed-2.0-Lite、Kimi-K2.5、Qwen3.5-397B等前沿多模态模型。
- 我的理解: 该方法巧妙地将记忆能力从其他认知技能中剥离，通过简单规则和可控难度确保失败可归因于隐藏状态跟踪而非规则误解。Memory Gap指标是亮点，直接量化遗忘的贡献。

### 6. 关键发现

- 模型在RNG-Bench上远未饱和: 在10×10 Matching Pairs图像模式下，GPT-5.4仅匹配62.3%的对，Qwen3.5-397B为25.3%，而最优策略每对仅需约3.24次响应，最强模型需8.01次。在13×13 3D迷宫中，Gemini-3.1-Pro最佳SR为50.0%，GS为49.7%，GPT-5.4和Seed-2.0-Lite SR为20.0%，Kimi-K2.5为10.0%，Qwen3.5-397B为0.0%。最困难配置需约128K token和350张图像，前沿模型仍有巨大提升空间。
- 遗忘是主要瓶颈: Memory Gap分析显示，大多数剩余错误源于遗忘早期观测，而非给定正确状态下的次优决策。例如，注入真实隐藏状态后模型性能显著提升。在决斗协议中，Gemini-3.1-Pro以100%胜率击败所有对手，进一步验证了其记忆优势。
- 微调可提升性能并迁移: 在最优策略轨迹和过滤的模型演示上微调Qwen3.5-9B，不仅提升RNG-Bench性能，还能迁移至现有记忆和空间基准（如EMemBench），且不损害通用多模态能力。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 部署多模态模型作为闭环策略时，必须重视其从历史中重建隐藏状态的能力，而非仅依赖当前观测。
- 对 Skill: 记忆-行动能力应作为独立技能评估，与推理、规划等分开。
- 对 Memory: 外部记忆（如小地图）可缓解遗忘，但模型仍需内部信念状态；微调可缩小记忆差距。
- 对评测: 应使用Memory Gap等指标分离遗忘与决策错误，并采用决斗协议控制实例方差。

### 11. 局限性

- 仅测试两种隐藏状态类型（静态离散和动态空间），未覆盖更复杂的隐藏状态（如部分可观测的连续状态）。
- 游戏规则简单，可能无法完全反映真实世界任务的复杂性。
- 微调实验仅使用Qwen3.5-9B，未验证在其他模型上的泛化性。

### 12. 我的看法 / 个人笔记

RNG-Bench是评估多模态模型记忆-行动能力的精心设计基准，其Memory Gap指标和可控难度轴为诊断模型失败提供了有力工具。未来工作可扩展至更多隐藏状态类型和更复杂任务，并探索如何通过架构改进（如显式记忆模块）缩小记忆差距。

## 3. AdsMind: A Physics-Grounded Multi-Agent System for Self-Correcting Discovery of Adsorption Configurations on Heterogeneous Catalyst Surfaces

**Topics:** 多智能体系统, 机器学习力场, 异相催化, 吸附构型搜索
**Authors:** Zongmin Zhang, Yuyang Lou, Bowen Zhang, Junwu Chen, Ryo Kuroki, Xuan Vu Nguyen, Edvin Fako, Lixue Cheng, Philippe Schwaller
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.19152v1) | [PDF](http://arxiv.org/pdf/2606.19152v1.pdf)

### 1. 一句话结论

AdsMind 通过闭环多智能体框架，利用 MLFF 弛豫反馈实现自主纠错，在 AA20 和 OCD-GMAE62 基准上分别达到 100% 和 98.8% 的成功率，且每个案例仅需约 4-5 次弛豫，相比启发式枚举减少约 14 倍计算成本。

### 2. 为什么重要

该工作解决了异相催化中吸附构型搜索的瓶颈问题，通过物理反馈机制显著提升了 LLM 代理的可靠性和自反思能力，为自动化化学工作流提供了新范式。

### 3. 研究问题

1. 如何利用 MLFF 弛豫反馈实现 LLM 代理的自主纠错？
2. 闭环搜索策略相比开环方法在吸附构型搜索中的优势是什么？

### 4. 背景与问题设定

异相催化中，确定最低能量表面-吸附物构型至关重要，但构型空间巨大，从头算计算成本过高。MLFF 加速了结构弛豫，但搜索策略仍是瓶颈。开环 LLM 代理缺乏物理反馈机制，无法纠正错误初始猜测。

### 5. 方法概览

- 任务 / 环境: 吸附构型搜索任务，涉及 AA20（20 个表面-吸附物案例）和 OCD-GMAE62（62 个氧化物、金属间化合物等表面）基准。

- 模型比较: 与开环基线 Adsorb-Agent、1-Shot 消融、非 LLM 启发式枚举基线比较。

### 6. 关键发现

- 高搜索可靠性: 在 AA20 和 OCD-GMAE62 上，AdsMind 分别达到 100% 和 98.8% 的成功率，且跨四个 LLM 后端（Gemini-2.5-Pro、GPT-5.4、Claude-Sonnet-4.6、Grok-4）一致。

## 4. Enhancing Decision-Making with Large Language Models through Multi-Agent Fictitious Play

**Topics:** LLM-based Multi-Agent Systems, Game Theory, Decision Making
**Authors:** Leyang Shen, Yang Zhang, Xiaoyan Zhao, Chun Kai Ling, Tat-Seng Chua
**Venue / Year:** arXiv preprint, 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.19308v1) | [PDF](http://arxiv.org/pdf/2606.19308v1.pdf)

### 1. 一句话结论

提出多智能体虚拟博弈（MAFP），通过迭代最佳响应解耦决策中的立场纠缠，在锦标赛强度和鲁棒性上均优于现有方法。

### 2. 为什么重要

现有LLM多智能体系统主要解决执行复杂度，而现实决策任务中的立场纠缠（相互依赖的决策）未被有效处理。MAFP将博弈论中的虚拟博弈引入语言空间，为LLM在竞争性决策场景中提供了一种无需训练、可扩展的均衡求解范式。

### 3. 研究问题

1. 如何形式化并解决LLM多智能体系统中的立场纠缠问题？
2. 虚拟博弈能否在语言空间中有效提升LLM决策的均衡性与鲁棒性？

### 4. 背景与问题设定

LLM多智能体系统通过分治范式处理执行复杂度（如长链推理、信息覆盖），但无法应对决策复杂度——立场纠缠，即各利益相关者的决策相互依赖，形成互锁循环。现有方法如递归心智理论（ToM）在单次推理中处理高阶信念，但LLM在高阶推理中准确率急剧下降。

### 5. 方法概览

- 任务 / 环境: 13个场景，包括竞争性游戏（如井字棋、尼姆游戏、四子棋）和自然语言谈判（如买卖、最后通牒博弈），每个场景由自然语言描述和多个利益相关者（立场）定义。
- 变量 / 干预设计: MAFP将每个立场建模为一个智能体，通过K轮迭代更新：每轮中，智能体聚合对手历史策略的实证混合，并生成最佳响应。最终输出各智能体历史策略的聚合作为最终策略。
- 评价指标: 锦标赛强度（平均效用）和鲁棒性（最差情况效用），通过循环赛和对抗性评估测量。
- 模型比较: 与单轮基线（Qwen3-1.7B、Llama-3.1-8B、GPT-5-nano、Qwen3.5-35B）和多轮基线（自反思、辩论、ToM、MAFP-Last）比较。
- 我的理解: MAFP通过将递归推理分解为迭代最佳响应，将LLM的推理负担从深度信念层次降低为单层条件预测，从而有效处理立场纠缠。

### 6. 关键发现

- MAFP在锦标赛强度上显著优于基线: 在13个场景中，MAFP的平均效用为0.527，高于所有单轮和多轮基线（如Q3.5-35B为0.500，ToM为0.527，但MAFP在多个场景中表现更稳定）。
- MAFP在鲁棒性上表现突出: MAFP的鲁棒性（最差情况效用）为0.426，远高于其他方法（如Q3.5-35B为0.374，ToM为0.389），表明其策略不易被对手利用。
- 迭代轮数提升效用: 随着迭代轮数K增加，MAFP的效用单调提升，并在约5轮后收敛，验证了虚拟博弈的收敛性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: MAFP为LLM多智能体系统提供了处理相互依赖决策的新范式，可扩展至谈判、市场模拟等场景。
- 对 Skill: 智能体需具备聚合历史策略和生成最佳响应的能力，这可通过提示工程实现，无需额外训练。
- 对 Memory: 历史策略的实证混合作为记忆形式，有效捕捉对手适应模式，避免遗忘。
- 对评测: 双轴评估（锦标赛强度与鲁棒性）为决策质量提供了更全面的度量，可推广至其他策略评估。

### 11. 局限性

- MAFP依赖LLM的聚合和最佳响应能力，在复杂场景中可能受限于LLM的推理质量。
- 当前评估仅基于13个场景，泛化性需在更多真实世界任务中验证。
- 迭代轮数K的选择可能影响性能，且计算成本随K线性增长。

### 12. 我的看法 / 个人笔记

MAFP巧妙地将博弈论经典算法适配到LLM语言空间，有效解决了立场纠缠问题。其无需训练、迭代收敛的特性使其具有实用价值。未来可探索更高效的聚合策略或结合强化学习进一步优化。

## 5. IndicContextEval: A Benchmark for Evaluating Context Utilisation in Audio Large Language Models Across 8 Indic Languages

**Topics:** Audio Large Language Models, Contextual ASR, Multilingual Benchmark, Indic Languages
**Authors:** Sakshi Joshi, Dhruv Subhash Rathi, Sanskar Singh, Eldho Ittan George, R J Hari, Kaushal Bhogale, Mitesh M. Khapra
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.19157v1) | [PDF](http://arxiv.org/pdf/2606.19157v1.pdf) | [Code](https://github.com/AI4Bharat/IndicContextEval)

### 1. 一句话结论

IndicContextEval 通过 7 级提示框架系统评估了 8 种印度语言中 AudioLLM 的上下文利用能力，发现模型在上下文利用上存在显著差异，其中 GPT-4o Transcribe 表现出平衡利用，而 Gemma-3N 则盲目依赖实体提示。

### 2. 为什么重要

现有基准无法区分 AudioLLM 是真正利用上下文还是依赖预训练参数知识，IndicContextEval 填补了这一空白，为评估和提升多语言语音识别中的上下文接地提供了关键工具。

### 3. 研究问题

1. AudioLLM 是否真正利用提供的上下文提示进行转录，还是主要依赖预训练参数知识？
2. 不同上下文类型（如元数据、自然语言描述、实体列表）对转录性能的影响有何差异？

### 4. 背景与问题设定

AudioLLM 支持基于文本提示的语音识别，但现有基准在固定提示条件下评估，无法揭示模型是否真正利用上下文。IndicContextEval 包含 56 小时自然语音、8 种印度语言和 23 个专业领域，通过 7 级提示框架（L0–L6）系统引入上下文信号，包括元数据、自然语言描述、英语和本地脚本实体列表以及对抗性错误实体，以分析上下文利用行为。

### 5. 方法概览

- 任务 / 环境: 多语言语音识别任务，涉及 8 种印度语言（印地语、孟加拉语、泰卢固语等），语音来自 555 名说话者，包含朗读和即兴演讲，覆盖 23 个专业领域如医学、工程、法律等。
- 变量 / 干预设计: 7 级提示框架：L0（无上下文）、L1（仅语言）、L2（语言+领域元数据，包括语音风格、地区、一句话描述）、L3（语言+音频自然语言描述）、L4（语言+英语实体列表，20–30 个）、L5（语言+本地脚本实体列表）、L6（对抗性错误实体，来自无关领域）。每级仅增加一种上下文信号。
- 评价指标: 词错误率（WER）和命名实体错误率（NEER），NEER 是实体偏置（L4–L6）的主要指标。
- 模型比较: 评估了 5 个模型：IndicConformer（600M 参数 ASR 基线，仅 L1）、GPT-4o Transcribe、Gemini 3 Flash、Sarvam Audio 和 Gemma-3N（8B-E4B）。
- 我的理解: 该基准通过逐步增加上下文信号并引入对抗性控制，系统性地归因性能变化，从而区分真正的上下文利用与参数记忆。

### 6. 关键发现

- 语言识别显著影响性能: 从 L0 到 L1（指定语言），Gemma-3N 的 WER 降低 12.48 点（从 51.21% 到 38.73%），Gemini 3 Flash 降低 5.40 点（从 24.30% 到 18.90%），而 GPT-4o Transcribe 仅降低 1.22 点（从 29.83% 到 28.61%），表明模型在无语言提示时需从声学推断脚本。
- 自然语言描述优于结构化元数据: L3（音频描述）一致优于 L2（结构化元数据）：GPT-4o Transcribe 在 L3 的 WER 改善 2.53 点（至 26.08%），而 L2 仅改善 0.24 点（至 28.37%）；Gemma-3N 在 L2 的 WER 增加 13.47 点（至 52.20%），而 L3 仅增加 1.49 点（至 40.22%）。
- 本地脚本实体偏置带来最大增益: L5（本地脚本实体）相比 L1 的 NEER 改善最大：GPT-4o Transcribe 改善 11.7%（从 35.59% 到 23.89%），Gemini 3 Flash 改善 8.5%（从 25.85% 到 17.39%），Gemma-3N 改善 8.6%（从 35.50% 到 26.90%），Sarvam Audio 改善 4.2%（从 25.93% 到 21.73%）。英语与本地脚本实体之间的差距高达 11 点。
- 模型上下文利用行为存在四种模式: GPT-4o Transcribe 平衡利用（L5 改善，L6 鲁棒，L6 WER 28.47% 接近 L1 的 28.61%）；Gemini 3 Flash 敏感利用（L5 显著改善，L6 适度退化，WER 增加 0.77 点至 19.67%）；Gemma-3N 不稳定利用（实体识别改善但 WER 增加，L6 严重退化，WER 增加 9.22 点至 47.95%）；Sarvam Audio 上下文盲（各水平变化极小，L5 WER 15.70% 仅比 L1 的 16.86% 改善 1.16 点）。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: AudioLLM 代理在部署时需谨慎设计提示，优先使用本地脚本实体列表和自然语言描述，并避免对抗性输入。
- 对 Skill: 模型应具备选择性利用上下文的能力，如 GPT-4o Transcribe 的平衡利用模式，而非盲目依赖提示。
- 对 Memory: 上下文提示可作为外部记忆，但需与声学证据交叉验证，避免参数记忆导致的错误。
- 对评测: 评估应包含对抗性控制以区分真正的上下文利用与参数记忆，并覆盖多语言和多领域。

### 11. 局限性

- 仅覆盖 8 种印度语言，未包括其他语系。
- 实体列表大小固定（20–30 个），未探索不同规模的影响。
- 未评估模型对上下文提示的泛化能力，如未见领域或实体。

### 12. 我的看法 / 个人笔记

IndicContextEval 是评估 AudioLLM 上下文利用的重要基准，其 7 级提示框架和对抗性控制设计精巧，揭示了模型行为的显著差异。未来工作可扩展至更多语言和动态上下文场景。
