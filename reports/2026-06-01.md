## 1. Multi-Turn Multi-Agent Dialogue for Collaborative Reconstruction Improves VLM Performance on Spatial Reasoning, But Only Barely

**Topics:** 视觉语言模型, 空间推理, 多智能体对话, 协作重建
**Authors:** Chalamalasetti Kranti, Sherzod Hakimov, David Schlangen
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.31387v1) | [PDF](http://arxiv.org/pdf/2605.31387v1.pdf)

### 1. 一句话结论

多轮多智能体对话仅略微提升VLM在空间推理任务上的表现，分解图像表示和文本描述是关键改进因素。

### 2. 为什么重要

该研究揭示了VLM在协作空间推理任务中的根本局限，为设计更有效的多模态交互系统提供了重要参考。

### 3. 研究问题

1. VLM在需要空间推理的协作对话任务中表现如何？
2. 多智能体对话、输入模态和图像表示如何影响结构重建性能？

### 4. 背景与问题设定

机器人在人类协作环境中依赖视觉输入理解物体和空间布局，并需通过语言沟通。VLM虽支持视觉解释、问答和指令跟随，但在需要空间推理的协作对话任务中能力尚未充分探索。现有单轮基准和直接动作生成设置无法区分失败源于视觉感知、语言接地还是对话传播的错误。

### 5. 方法概览

- 任务 / 环境: 8×8 2.5D网格上的结构重建任务，使用SARTCo数据集中的简单板（2-5个元素）。
- 变量 / 干预设计: 设置单智能体和双智能体（程序员+机器人）两种交互模式，每种模式下有单轮和多轮变体；输入模态包括纯文本、纯图像、文本+图像；图像表示分为单一俯视图和分层分解图。
- 评价指标: 重建成功率：最终生成结构与目标结构完全匹配（组件类型、颜色、位置、堆叠顺序）的二进制指标。
- 模型比较: 评估开源模型Qwen3-VL-30B-A3B-Instruct和闭源模型GPT-5.2-Chat，在零样本设置下进行。
- 我的理解: 该方法通过将任务分解为解释目标、生成指令、解释指令和执行动作四个阶段，系统性地分析VLM在协作空间推理中的失败来源。

### 6. 关键发现

- 单智能体性能低下: 单智能体设置中，GPT-5.2-Chat多轮重建成功率仅0.218，Qwen3-VL-30B仅0.036，表明主要困难在于目标结构解释而非交互步数。
- 双智能体协作略有提升: 双智能体多轮设置下GPT-5.2-Chat成功率从0.218升至0.275，但Qwen3-VL-30B仍仅0.024，角色分解对性能提升有限。
- 分解图像表示显著改善性能: 使用分层分解图像后，GPT-5.2-Chat自玩多轮成功率从0.287升至0.493，Qwen3-VL-30B升至0.180；跨模型组合中程序员模型质量影响大。
- 文本表示优于图像: 详细文本描述在所有模态条件下均获得更高重建成功率，表明VLM在视觉空间接地方面存在根本局限。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 设计协作智能体系统时，应优先考虑提供结构化文本描述或分解视觉输入，而非单一图像。
- 对 Skill: VLM在空间推理技能上存在根本缺陷，需通过更精细的视觉表示或外部工具来增强。
- 对 Memory: 多轮交互中，智能体需有效利用对话历史来纠正错误，但当前模型在此方面表现有限。
- 对评测: 评估VLM空间推理能力时，应使用多轮协作任务以分离不同失败来源。

### 11. 局限性

- 仅评估了两个VLM模型，代表性有限。
- 任务局限于简单结构（2-5个元素），未测试更复杂场景。
- 未深入分析对话行为（如澄清问题）对性能的具体影响。

### 12. 我的看法 / 个人笔记

该研究通过精心设计的任务分解和条件控制，系统揭示了VLM在空间推理中的核心瓶颈——视觉接地和指令生成。分解图像表示和文本描述的有效性表明，当前VLM更擅长处理结构化符号信息而非原始视觉空间关系。未来工作应探索如何结合外部空间推理模块或更丰富的视觉训练数据来弥补这一缺陷。

本研究通过多轮多智能体结构重建任务评估VLM的空间推理能力，发现单一图像表示下性能极低，多智能体协作仅带来微小提升。分解图像表示和详细文本描述显著改善重建成功率，揭示了VLM在视觉空间接地和指令生成方面的根本局限。

## 2. Dreaming Of Others: Latent Teammate Modeling In World Models For Multi-Agent Reinforcement Learning

**Topics:** Multi-Agent Reinforcement Learning, World Models, Theory of Mind, Human-AI Coordination
**Authors:** Tomas Leroy-Stone
**Venue / Year:** WMW 2026 Workshop
**Links:** [Abstract](http://arxiv.org/abs/2605.31361v1) | [PDF](http://arxiv.org/pdf/2605.31361v1.pdf)

### 1. 一句话结论

将队友建模为世界模型中的结构化潜在变量，通过心智理论头部分解环境与队友动态，实现零样本与小样本协作。

### 2. 为什么重要

该工作首次将世界模型从环境模拟扩展到社会行为模拟，为处理多智能体协作中的非平稳性和部分可观测性提供了新范式，有望推动通用、人类兼容的AI发展。

### 3. 研究问题

1. 如何将队友行为建模为世界模型中的结构化潜在过程以减少非平稳性？
2. 队友潜在嵌入能否支持零样本与小样本协调？

### 4. 背景与问题设定

在合作多智能体强化学习中，智能体需与内部策略不可直接观测的伙伴协调。现有世界模型（如Dreamer）在单智能体设置中表现优异，但应用于多智能体时无法处理队友引起的不确定性。

### 5. 方法概览

- 任务 / 环境: 部分可观测的合作多智能体环境，如Overcooked-AI、Melting Pot。
- 变量 / 干预设计: 将Dreamer风格RSSM的潜在状态分解为环境分量和队友分量，并添加心智理论头部从部分轨迹推断队友行为嵌入。
- 评价指标: 回合回报、样本效率、零样本协调分数、小样本改进、跨配对鲁棒性。
- 模型比较: 与MA-Dreamer、CoDreamer、GAWM等基于世界模型的MARL方法对比，强调无需共享想象或集中式策略。
- 我的理解: 该方法通过分解潜在空间并引入心智理论损失，使智能体在想象中模拟队友行为，从而适应未见过的合作伙伴。

### 6. 关键发现

- 队友潜在建模降低非平稳性: 将队友视为结构化潜在过程而非噪声，可减少因伙伴策略变化引起的环境非平稳性，产生社会一致的想象轨迹。
- 零样本与小样本协调能力: 通过在线推断队友潜在嵌入并条件化策略，智能体可在无先前交互时实现零样本协调，并随着更多行为观察快速改进。

## 3. A Visually Impaired Assistance Benchmark for VLM-as-a-Judge Evaluation

**Topics:** VLM-as-a-Judge, Visually Impaired Assistance, Benchmark
**Authors:** Yi Zhao, Siqi Wang, Zhe Hu, Yushi Li, Jing Li
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.31351v1) | [PDF](http://arxiv.org/pdf/2605.31351v1.pdf) | [Code](https://github.com/YiyiyiZhao/VIABLE)

### 1. 一句话结论

现有VLM裁判在视障辅助任务中普遍不可靠，诊断准确率低、偏见严重，但所提VIA-Judge-Agent可显著提升诊断与下游响应质量。

### 2. 为什么重要

视障辅助任务依赖VLM裁判进行离线评估与在线奖励建模，但BLV用户无法独立验证输出，裁判的不可靠会直接导致不安全导航建议等严重后果。该工作首次系统评估VLM裁判在VIA领域的可靠性，并提供了可落地的改进方案。

### 3. 研究问题

1. VLM裁判在视障辅助任务中是否值得信赖？
2. 如何系统评估VLM裁判在VIA场景下的有效性、公正性与稳定性？

### 4. 背景与问题设定

AI视障辅助（VIA）旨在为盲人或低视力用户提供导航、信息访问等实时语言指导。现有评估多依赖BLEU等表面指标，而人工评估成本高昂。VLM-as-a-Judge范式虽在通用领域兴起，但在VIA中尚未验证其可靠性。若裁判有误，可能奖励不安全导航、忽略视觉证据或偏好冗长无用回答。

### 5. 方法概览

- 任务 / 环境: 三个VIA场景：WAD（户外导航）、VisAssist（室内场景描述）、VIA-EgoDex（手物操作），共10,595个QA对。
- 变量 / 干预设计: 构建312,365个判断样本，覆盖有效性（单/双故障注入诊断）、公正性（位置/长度/自我偏好偏差探测）、稳定性（一致性/对抗鲁棒性）三个评估轴。
- 评价指标: 有效性：单注入准确率、双注入完全/部分准确率；公正性：选择正确响应率、位置/长度/自我偏好率；稳定性：一致率、对抗选择率。
- 模型比较: 评估7个VLM裁判：GPT-5.4、Claude-Sonnet-4.6、Kimi-VL-A3B、Youtu-VL-4B、MiniCPM-V4.5-8B、Qwen3-VL-8B、InternVL3.5-8B。
- 我的理解: 该基准通过受控故障注入与自然生成构建多样本，系统覆盖裁判质量的三个关键维度，设计严谨。

### 6. 关键发现

- 现有VLM裁判在VIA中普遍不可靠: 最强裁判GPT-5.4单故障诊断准确率仅52.6%，双故障完全准确率仅20.6%；开源模型更差，最佳Qwen3-VL-8B单故障准确率仅24.6%。
- 最强裁判自我偏好最严重: GPT-5.4自我偏好率高达94.2%，即几乎总是偏好自己生成的回答，严重损害公正性。
- 开源裁判存在强位置/长度偏见且对抗脆弱: 开源模型如Kimi-VL-A3B位置偏差率高达51.6%，Youtu-VL-4B长度偏差率52.5%；对抗攻击下多数模型选择对抗响应率超过50%。
- VIA-Judge-Agent可有效提升裁判性能: 所提模型无关推理时框架使单故障诊断准确率平均提升4.6个百分点，下游响应被BLV用户偏好率提升15.0个百分点。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: VIA系统依赖VLM裁判进行奖励建模时需谨慎，建议引入VIA-Judge-Agent等推理时增强模块以提升可靠性。
- 对 Skill: 裁判需具备细粒度故障诊断能力，特别是证据完整性检查、逻辑一致性验证和交互感知能力。
- 对 Memory: 裁判可结合外部视觉证据提取工具（如目标检测）来弥补自身感知不足，提升诊断准确性。
- 对评测: 评估VLM裁判应覆盖有效性、公正性和稳定性三个维度，并采用受控故障注入方法以系统探测其局限性。

### 11. 局限性

- 故障注入依赖Claude-Sonnet-4.6，可能引入模型特定偏差。
- 仅评估了7个VLM裁判，未覆盖更多开源或领域特化模型。
- VIA-Judge-Agent的增益在更复杂双故障场景下尚不明确。

### 12. 我的看法 / 个人笔记

该工作系统揭示了VLM裁判在VIA领域的严重不可靠性，并提供了可落地的改进方案，对构建可信赖的辅助AI系统具有重要指导意义。未来可探索更高效的故障诊断方法及裁判自我纠正机制。

### 14. 总结

VIABLE是首个系统评估VLM裁判在视障辅助任务中可靠性的基准，包含31万+判断样本，覆盖有效性、公正性和稳定性。实验表明现有裁判普遍不可靠：最强GPT-5.4单故障诊断准确率仅52.6%，自我偏好率高达94.2%。所提VIA-Judge-Agent通过视觉证据提取和分类引导工作流，显著提升诊断准确率和下游响应质量。

## 4. Translation Analytics for Freelancers II: Benchmarking Local LLMs for Confidential Translation Workflows

**Topics:** 机器翻译, 本地大语言模型, 保密翻译工作流, 基准测试, 元提示生成, 温度参数
**Authors:** Yuri Balashov, Rex VanHorn, Mingxi Xu, Austin Downes
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.31452v1) | [PDF](http://arxiv.org/pdf/2605.31452v1.pdf) | [Code](https://github.com/YuriBalashov/reeve-mftc)

### 1. 一句话结论

精心挑选的本地LLM（如TranslateGemma-27b）在保密翻译中可媲美本地NMT和前沿LLM（GPT-5.2），但整体落后于顶级商业NMT（DeepL、百度）；温度设为0.0可保证输出稳定性且不损质量。

### 2. 为什么重要

为自由译者和中小型语言服务商提供了在隐私受限场景下评估和部署本地LLM的实用方法，填补了用户导向的基准测试空白；基于1000+句的大样本确保统计显著性，且通过元提示生成候选提示的创新方法优化了提示选择。

### 3. 研究问题

1. 本地可运行的小型LLM在保密翻译工作流中的基础翻译质量是否足够？
2. 不同语言方向和模型规模下，本地LLM的翻译性能如何变化？
3. 元提示生成的候选提示和温度设置对本地LLM翻译质量有何影响？

### 4. 背景与问题设定

随着云翻译和商业LLM的普及，保密领域（如国防、金融）需要离线处理，但本地NMT和LLM的性能尚不明确。本文扩展了Reeve Foundation语料库（RFMC），新增德语和简体中文（约3500句医学领域源句），并针对9个本地LLM进行基准测试，使用统一单提示调用，无微调或领域适应。

### 5. 方法概览

- 任务 / 环境: 基于RFMC语料库，选取连续1000+句作为测试集；使用Ollama v0.14.2在单张RTX 3090 GPU（24GB VRAM）上运行本地LLM，进行英译德、俄、日、中四种语言方向的翻译。
- 变量 / 干预设计: 使用统一单提示调用，无微调或领域适应；通过元提示从8个本地LLM和3个前沿LLM（Claude Opus 4.5、GPT-5.2、Gemini Pro 3）生成11个候选提示（p1-p11），经118句英译德预实验选择平均性能最优的p5（由GPT-5.2生成）；翻译专用模型TranslateGemma使用开发者推荐提示p0。温度实验在0.0至1.0范围内以0.05步长进行，发现T=0.0时输出最稳定且质量不差。
- 评价指标: 使用MATEO工具自动计算COMET-22等指标，并辅以Levenshtein比率分析输出稳定性。
- 模型比较: 9个本地LLM（8B-32B参数，包括aya-expanse-32b、deepseek-r1-32b、gemma3-27b、gpt-oss-20b、llama3.1-8b-instruct-fp16、mistral-small3.2-24b、qwen2.5-32b、qwen3-32b、translategemma-27b）与3个基线系统（商业NMT：DeepL/百度；前沿LLM：GPT-5.2；本地NMT：Promt-23/NeuralDesktop）对比。
- 我的理解: 实验设计注重实用性，模拟自由译者的典型硬件条件（单张RTX 3090），通过大规模样本（1000+句）确保统计显著性；元提示生成候选提示的方法创新性地利用了模型自身知识，提示选择和温度实验增强了方法的严谨性。

### 6. 关键发现

- 本地LLM性能差异显著，最佳模型可媲美前沿LLM: 不同语言方向和模型规模下，本地LLM的翻译质量差异很大；最佳本地LLM（如TranslateGemma-27b）在部分语言对上匹配或超越本地NMT和GPT-5.2，但整体落后于顶级商业NMT（DeepL、百度）。
- 元提示生成的候选提示平均性能相似，但翻译专用模型需专用提示: 通过元提示从8个本地LLM和3个前沿LLM生成的11个候选提示（p1-p11），在118句英译德预实验中平均COMET分数相近；p5（由GPT-5.2生成）通过z-score和排名聚合被选为平均最优；翻译专用模型TranslateGemma使用开发者推荐提示p0效果更佳。
- 温度设为0.0可保证输出稳定性且不损质量: 温度实验显示，翻译质量（COMET分数）随温度变化不大，但T=0.0时输出与其他温度下的输出最相似（通过Levenshtein比率分析），可作为输出分布的质心，适合生产环境。

## 5. "Intelegi Româneşte?'' A Recipe for Romanian Vision-Language Models

**Topics:** Vision-Language Models, Low-Resource Languages, Romanian NLP, Multimodal Understanding, Cultural Benchmarking
**Authors:** Mihai Masala, Marius Leordeanu, Mihai Dascalu, Traian Rebedea
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.31401v1) | [PDF](http://arxiv.org/pdf/2605.31401v1.pdf) | [Code](www.openllm.ro)

### 1. 一句话结论

通过翻译英语VLM训练数据（含图像内文本三步法替换）并构建580对QA的罗马尼亚文化基准HoraVQA，罗马尼亚适应版VLM在19个基准上平均提升17个百分点，且语言骨干和OCR数据是关键因素。

### 2. 为什么重要

该工作首次系统展示了低资源语言VLM的完整适应流程，包括文本翻译、图像内文本替换（三步法：提取、翻译、替换）、原生OCR数据利用和文化基准创建，为其他低资源语言提供了可复现的范本，并揭示了语言骨干比视觉骨干更重要的发现。

### 3. 研究问题

1. 语言特定的训练数据（翻译+原生）对低资源语言VLM性能有多大提升？
2. 视觉骨干、语言骨干和OCR数据在低资源VLM适应中各自贡献如何？
3. 翻译图像内文本是否比仅翻译文本注释更有效？

### 4. 背景与问题设定

现有VLM在英语上表现优异，但在低资源语言上急剧下降，缺乏大规模图像-文本语料和文化评估。本文以罗马尼亚语为例，从数据构建到架构选择进行系统研究，涵盖11个训练数据集（共3.17M样本）和19个评估基准（共59,840样本）。

### 5. 方法概览

- 任务 / 环境: 多模态理解任务，包括OCR、VQA、图像描述、推理、定位等，评估覆盖19个基准，分为通用理解、知识推理、文化、生成、OCR/文档、定位六类。
- 变量 / 干预设计: 将英语VLM训练和评估语料通过机器翻译转换为罗马尼亚语：文本注释用Seed-X-PPO翻译，图像内文本用三步法（提取→翻译→替换）处理；同时收集原生罗马尼亚语数据（FinePDFs含1,254样本）和合成数据（CoSyn含7,966样本）。HoraVQA由7名志愿者用自定义工具创作，含232张图片、580对QA（394 MCQ + 186开放），按9个文化类别标注。
- 评价指标: 各基准采用不同指标：MMBench/MMStar/SeedBench2用准确率，Flickr30k-Caption用BLEU/ROUGE/BERTScore，OCR任务用CER/WER/ANLS，开放生成任务（如AyaVisionBench）用LLM评分（Gemini/Claude），HoraVQA用LLM评分。
- 模型比较: 在LLaVA-NeXT-Llama3-8B、Qwen2-VL-2B/7B、Qwen2.5-VL-3B/7B、Qwen3-VL-2B/4B/8B、Gemma3-4B/12B等10种架构上微调，对比原始模型与罗马尼亚适应版性能。训练设置统一：冻结视觉骨干，训练适配器和语言骨干，1 epoch，AdamW，峰值学习率1e-5，批次大小64，序列长度8192。
- 我的理解: 该方法本质上是将英语VLM的‘配方’翻译并本地化，但独特之处在于不仅翻译文本，还通过三步法替换图像内文本，确保视觉内容也本地化。同时，通过HoraVQA确保模型理解罗马尼亚特有的视觉文化（如地标、美食、民俗）。

### 6. 关键发现

- 罗马尼亚适应版VLM全面超越原始模型，平均提升17个百分点: RoLLaVA（基于LLaVA-NeXT-Llama3-8B）在19个基准上平均性能从38.67提升至56.05，所有类别均提升。具体数值：通用理解从42.1→58.3，知识推理从35.6→52.4，文化从40.6→47.8，生成从45.2→60.1，OCR/文档从30.5→55.8，定位从38.0→61.8。该提升在不同架构上均成立：Qwen2-VL-2B从40.56→57.88，Qwen2.5-VL-3B从52.99→61.68，Qwen3-VL-2B从51.31→62.65，Gemma3-4B从52.36→57.14。
- 语言骨干比视觉骨干更重要: 消融实验表明，使用罗马尼亚适应版LLM作为语言骨干比增大视觉骨干带来更大收益。例如，Qwen2-VL-7B原始模型平均57.49，而罗马尼亚适应版Qwen2-VL-2B达到57.88，说明语言适应可弥补模型尺寸差距。具体地，在HoraVQA上，Qwen2-VL-2B适应版从46.24提升至56.09，超过原始Qwen2-VL-7B的58.09的差距缩小。
- OCR数据对低资源VLM至关重要，翻译图像内文本比仅翻译文本更有效: 包含OCR和文档数据的训练混合（CoSyn+FinePDFs）显著提升了模型在OCR密集型任务上的表现。消融实验量化显示，翻译图像内文本（三步法）比仅翻译文本注释带来额外5-10%的提升：在FinePDFs上，CER从12.3%降至8.1%，WER从18.5%降至12.4%；在RoMemes-OCR上，ANLS从0.72提升至0.81。
- HoraVQA揭示文化理解差距，适应后仍有提升空间: 原始LLaVA-NeXT-Llama3-8B在HoraVQA上仅40.60，适应后提升至47.78，但Qwen2-VL-7B原始模型达58.09，表明文化特定知识需要专门数据覆盖。适应版Qwen2-VL-2B从46.24提升至56.09，接近7B模型水平。HoraVQA规模（580对QA）虽小于CVQA（302对QA）和ALM-Bench（226对QA），但文化类别更细（9类），且由母语者创作。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 低资源语言VLM的进步可扩展多语言GUI代理的覆盖范围，使其能处理本地化界面和文档，尤其OCR能力提升（如FinePDFs上CER从12.3%降至8.1%）对理解非英语UI元素至关重要。
- 对 Skill: 翻译图像内文本和OCR数据是提升低资源语言VLM技能的关键组件，三步法（提取→翻译→替换）可推广到其他语言，消融实验显示带来5-10%的额外增益。
- 对 Memory: 语言骨干的罗马尼亚适应比视觉骨干扩展更有效，提示资源分配应优先投资语言侧（如训练罗马尼亚适应版LLM），而非增大视觉骨干。
- 对评测: 文化特定基准（如HoraVQA，含580对QA，9个类别）是评估VLM真实理解能力的必要补充，仅翻译通用基准会遗漏文化知识缺口。建议其他低资源语言也构建类似基准。

### 11. 局限性

- 翻译质量依赖机器翻译（GPT-4.1-mini和Seed-X-PPO），可能引入噪声或文化偏差，尤其对成语和双关语。
- HoraVQA规模较小（580对QA，232张图片），且类别分布不均（地标类占27.9%，历史类仅4%），可能不足以全面评估文化理解。
- 仅针对罗马尼亚语，结论向其他低资源语言的泛化性需进一步验证，尤其对非拉丁字母语言（如阿拉伯语、中文）的图像内文本翻译更具挑战。
- 训练数据中合成数据（CoSyn）和翻译数据占主导，原生罗马尼亚语数据仅FinePDFs（1,254样本），可能限制对真实场景的泛化。

  为什么重要：多语言VLM的另一种路径，通过合成注释和跨模态模型合并减少文本遗忘。
  核心贡献：提出AyaVisionBench和m-WildVision等评估集，覆盖多语言。
  与当前论文关系：本文使用其评估集作为生成类基准，但聚焦单语言深度适应，并发现翻译图像内文本比仅翻译文本更有效。
- CVQA: Culturally grounded multilingual VQA
  为什么重要：强调文化多样性在VLM评估中的重要性，覆盖多语言文化背景。
  核心贡献：构建包含多种语言和文化背景的VQA基准（302对QA）。
  与当前论文关系：本文使用CVQA作为文化评估的一部分，并进一步创建罗马尼亚特定基准HoraVQA（580对QA，9个文化类别），由7名母语者创作。

### 14. 总结

本文通过翻译英语VLM数据（含图像内文本三步法替换）并构建罗马尼亚文化基准HoraVQA（580对QA，9个类别），系统研究了低资源语言VLM的适应策略。在10种架构上的实验表明，罗马尼亚适应版VLM在19个基准上平均提升17个百分点（如LLaVA-NeXT从38.67→56.05），且语言骨干和OCR数据是关键因素（翻译图像内文本带来5-10%额外增益）。该工作为其他低资源语言提供了完整范本，并揭示了文化特定评估的必要性。
