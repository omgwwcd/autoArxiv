## 1. Towards Direct Latent-Space Synthesis for Parallel Branches in LLM-Agent Workflows

**Topics:** LLM Agent, Parallel Synthesis, KV Cache Reuse
**Authors:** Shikun Liu, Mufei Li, Dongqi Fu, Haoyu Wang, Yinglong Xia, Hong Li, Hong Yan, Pan Li
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.14672v1) | [PDF](http://arxiv.org/pdf/2606.14672v1.pdf)

### 1. 一句话结论

提出Parallel-Synthesis框架，让合成器直接复用并行工作智能体的KV缓存，避免文本拼接和冗余预填充，在保持或提升质量的同时将首令牌延迟降低2.5-11倍。

### 2. 为什么重要

当前LLM智能体工作流中并行分支的合成依赖文本拼接，丢弃了并行结构并导致计算浪费。本文首次实现从并行分支的KV缓存直接合成，为高效、结构感知的智能体系统设计提供了新范式。

### 3. 研究问题

1. 能否让合成器直接消费并行工作智能体的KV缓存而非拼接文本输出？
2. 直接缓存合成能否在保持或提升下游任务质量的同时显著降低延迟？

### 4. 背景与问题设定

LLM作为智能体执行引擎，但仅支持顺序文本上下文接口。现代智能体工作流常包含并行分支（如独立子任务、候选解生成），现有方法将分支输出文本拼接后重新预填充，丢弃了DAG结构并引入冗余计算。

### 5. 方法概览

- 任务 / 环境: 涵盖数学推理、科学问答、代码生成、GAIA工具使用和多智能体数据库诊断等9个数据集。
- 变量 / 干预设计: 对比标准文本拼接合成与Parallel-Synthesis（直接复用KV缓存）。Parallel-Synthesis包含位置重编码、缓存映射器（MLP预测仿射系数）和合成器LoRA适配。
- 评价指标: 下游任务准确率/分数、首令牌时间（TTFT）。
- 模型比较: 与文本拼接合成基线对比，并在部分数据集上与RAG风格缓存复用方法（如APE）比较。
- 我的理解: 该方法本质上是将并行分支的中间表示（KV缓存）而非最终文本传递给合成器，通过位置对齐和轻量级校准解决缓存独立生成带来的不兼容问题，再通过LoRA微调让模型学会从这种非顺序缓存中推理。

### 6. 关键发现

- 质量持平或提升: 在9个数据集中，Parallel-Synthesis在7个上匹配或超越文本拼接合成，剩余2个上差距很小。
- 显著降低延迟: 首令牌时间（TTFT）降低2.5倍至11倍，因为避免了重新预填充工作智能体的输出。
- 推理任务上表现更优: 在多个推理密集型任务上取得增益，表明直接缓存合成不仅更快，有时还能更好地整合并行推理轨迹。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为智能体系统提供了一种更高效、结构感知的并行分支合成方式，可减少计算冗余并支持更复杂的DAG工作流。
- 对 Skill: 合成器需要学会从非顺序缓存中推理和聚合信息，这要求模型具备更强的跨分支注意力能力。
- 对 Memory: 缓存映射器和位置重编码使得不同分支的KV缓存可以对齐和校准，为长期记忆的并行访问提供了可能。
- 对评测: 需要新的评估指标来平衡合成质量与效率（如TTFT），并考虑并行分支数量对性能的影响。

### 11. 局限性

- 缓存映射器依赖分支长度和数量等元数据，可能无法泛化到未见过的分支配置。
- 训练数据构造复杂，需要生成并行分支轨迹和对应的文本合成监督。
- 当前仅验证了同质智能体（相同骨干模型）场景，异质智能体间的缓存复用尚未探索。

### 12. 我的看法 / 个人笔记

这是一项很有前景的工作，直接挑战了LLM必须通过文本通信的隐含假设。通过KV缓存接口保留并行结构，既提升了效率又可能带来质量收益。未来可探索更通用的缓存校准方法、扩展到异质模型，以及将类似思想用于更复杂的图结构工作流。

### 14. 总结

Parallel-Synthesis通过让合成器直接复用并行工作智能体的KV缓存，避免了文本拼接和冗余预填充。在9个数据集上，该方法在7个上匹配或超越文本合成，并将首令牌延迟降低2.5-11倍，为高效、结构感知的智能体合成提供了新方向。

## 2. CORA: Analyzing and bridging thinking-answer gap in Multimodal RLVR via Consistency-Oriented Reasoning Alignment

**Topics:** 多模态推理, 强化学习, 思维-答案一致性
**Authors:** Jiayue Cao, Zhicong Lu, Xuehan Sun, Wei Jia, Hongling Zheng, Changyuan Tian, Zichuan Lin, Wenqian Lv, Nayu Liu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.14691v1) | [PDF](http://arxiv.org/pdf/2606.14691v1.pdf)

### 1. 一句话结论

CORA通过轻量级一致性奖励模型（CRM，基于ModernBERT，在约90K NLI风格数据上训练，由GPT-5标注）和混合奖励优势分割（HRAS，超参数α,β控制任务与一致性学习强度），有效缓解多模态RLVR中思维-答案不一致问题，同时提升任务准确率。

### 2. 为什么重要

现有RLVR方法仅以最终答案正确性为奖励，忽视推理过程与答案的一致性，导致模型学习捷径而非忠实推理。CORA通过显式正则化思维-答案一致性，提升推理轨迹可靠性和任务性能，对构建可信多模态推理系统至关重要。

### 3. 研究问题

1. 在多模态RLVR中，思维-答案不一致问题是否普遍存在且持续？
2. 如何通过引入一致性奖励有效缓解这种不一致，同时保持或提升任务性能？

### 4. 背景与问题设定

RLVR已成功激发大语言模型推理能力，并扩展到多模态场景。现有方法主要关注提升推理轨迹视觉覆盖和减少视觉幻觉，但低估了推理过程与最终答案之间的语义不一致。本文通过分析GRPO训练过程中约719K个有效rollouts和评估输出，发现这种不一致在训练中持续存在（如Qwen2-VL-2B在MathVision上不一致率从早期46.96%升至晚期54.78%），并在推理时依然出现。在不一致样本中，最终答案正确率高于推理隐含答案，表明RLVR主要优化答案正确性而忽略推理过程。

### 5. 方法概览

- 任务 / 环境: 多模态推理任务，包括视觉感知（CVBench，训练集SAT）、多模态数学推理（MathVision、MathVista，训练集Math-40K）和视觉谜题推理（PuzzleVQA训练集6.5K、AlgoPuzzleVQA）。
- 变量 / 干预设计: 提出CORA方法：1) 一致性奖励模型（CRM），基于ModernBERT，在约90K NLI风格标注数据（由GPT-5标注）上训练，以判别方式输出思维-答案一致性概率；2) 混合奖励优势分割（HRAS），分别对任务奖励（格式+答案正确性）和一致性奖励进行组内归一化，再加权组合优势（超参数α,β控制相对强度）。格式奖励权重0.1，λ_cons控制一致性奖励贡献。
- 评价指标: 准确率（Accuracy）和不一致率（Inconsistency Rate, IR），IR = 不一致样本数 / 有效样本总数。
- 模型比较: 在Qwen2-VL-2B/7B和Qwen2.5-VL-3B/7B四个LVLM上，将CORA与标准GRPO基线（仅使用答案正确性和格式奖励）比较。训练设置：batch size=1，梯度累积2步，SAT和PuzzleVQA训练2个epoch，Math-40K训练1个epoch。
- 我的理解: CORA核心创新在于将思维-答案一致性作为额外奖励信号引入RLVR，并通过HRAS解决不同奖励分布特性（离散vs连续）导致的优化冲突。这类似于在强化学习中引入过程监督信号，而不仅是结果监督。CRM的轻量设计（ModernBERT）使其在推理时高效。

### 6. 关键发现

- 思维-答案不一致普遍存在且持续: 在GRPO训练和评估中，不一致现象广泛存在于多个模型和任务中，且随训练进行并未自然缓解，反而可能加剧。例如，Qwen2-VL-2B在MathVision上不一致率从早期46.96%升至晚期54.78%；Qwen2-VL-7B在PuzzleVQA上不一致率从早期约30%升至晚期约30%（具体数值见表1）。在不一致样本中，最终答案正确率高于推理隐含答案，表明RLVR主要优化答案正确性而忽略推理过程。不一致率任务依赖：视觉感知任务较低（如CVBench约8-18%），数学推理和谜题任务较高（如MathVision约10-55%）。
- CORA有效降低不一致率并提升准确率: 在五个基准测试上（见表1），CORA在大多数设置下显著降低不一致率，同时提升最终答案准确率。例如，Qwen2-VL-7B在PuzzleVQA上IR从30.21%降至5.03%，准确率从77.00%升至81.95%；Qwen2.5-VL-7B在MathVision上准确率从12.99%升至17.00%，IR从15.80%降至8.28%。在7B模型上效果更明显，且对需要深度推理的任务（如MathVision和AlgoPuzzleVQA）提升更大。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: CORA可提升多模态智能体推理的可靠性，确保内部推理过程与最终行动一致，减少因思维-答案不一致导致的错误决策。
- 对 Skill: 通过显式监督推理过程，CORA有助于模型学习更稳健的推理技能，而非仅记忆答案捷径。
- 对 Memory: 一致性奖励可视为一种过程记忆，促使模型在推理时保持逻辑连贯性，可能有助于长期记忆的构建。
- 对评测: 本文提出的不一致率（IR）可作为评估推理忠实度的新指标，补充传统准确率。

### 11. 局限性

- CRM训练依赖离线标注数据（约90K样本），由GPT-5标注可能引入标注偏差，且需要额外训练成本。
- 在小型模型（2B/3B）上效果有限，可能因其推理能力不足导致思维内容质量低，限制了一致性奖励的有效性。
- 实验仅在Qwen系列模型（Qwen2-VL和Qwen2.5-VL）上进行，泛化性需在更多模型族（如LLaVA、InternVL）上验证。
- 训练数据规模（SAT、Math-40K、PuzzleVQA 6.5K）相对较小，可能影响结论的普适性。

### 12. 我的看法 / 个人笔记

CORA通过引入过程级的一致性监督，有效弥补RLVR中结果监督的不足，思路清晰且实现轻量。HRAS设计巧妙解决异构奖励的优化冲突。未来工作可探索更高效的一致性评估方法（如无需额外模型）以及在更多模态和任务上的应用。

### 13. 这篇工作引用的近期高影响力相关文章

  为什么重要：这些工作将RLVR扩展到多模态场景，主要关注提升视觉覆盖和减少幻觉。
  核心贡献：提出了多模态RLVR的初步框架。
  与当前论文关系：本文指出现有方法低估了思维-答案不一致问题，并在此基础上提出改进。
- Concurrent works on reasoning-answer mismatch (Chen et al., 2025; Huang et al., 2025a)
  为什么重要：这些工作也认识到推理与答案不匹配的问题，但限于特定任务或依赖昂贵辅助奖励。
  核心贡献：启发式地识别了思维-答案不一致。
  与当前论文关系：本文通过系统分析揭示了该问题的普遍性，并提出了更通用的解决方案CORA。

### 14. 总结

本文系统分析了多模态RLVR中思维-答案不一致问题，发现其普遍存在且持续（如Qwen2-VL-2B在MathVision上不一致率从46.96%升至54.78%）。提出CORA方法，通过轻量级一致性奖励模型（基于ModernBERT，在约90K NLI数据上训练，由GPT-5标注）和混合奖励优势分割，在五个基准上显著降低不一致率（如Qwen2-VL-7B在PuzzleVQA上IR从30.21%降至5.03%）并提升准确率（如Qwen2.5-VL-7B在MathVision上准确率从12.99%升至17.00%），为构建更忠实可靠的多模态推理系统提供了有效方案。

## 3. ClinHallu: A Benchmark for Diagnosing Stage-Wise Hallucinations in Medical MLLM Reasoning

**Topics:** Medical Multimodal Large Language Models, Hallucination Diagnosis, Benchmarking
**Authors:** Sicheng Yang, Hangjie Yuan, Wenjun Zhang, Jinwang Wang, Yichen Qian, Weihua Chen, Fan Wang, Lei Zhu
**Venue / Year:** arXiv:2606.14697v1, June 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.14697v1) | [PDF](http://arxiv.org/pdf/2606.14697v1.pdf) | [Code](https://github.com/alibaba-damo-academy/ClinHallu)

### 1. 一句话结论

ClinHallu 通过结构化推理轨迹（视觉识别、知识回忆、推理整合）实现医学 MLLM 中幻觉的细粒度阶段诊断，并证明轨迹监督微调可减少幻觉。

### 2. 为什么重要

现有医学幻觉基准仅关注最终答案正确性，无法定位错误源头。ClinHallu 首次提供阶段级诊断，揭示不同数据集和模型中的视觉、知识或推理瓶颈，对构建可信临床决策支持系统至关重要。

### 3. 研究问题

1. 医学 MLLM 的幻觉主要源于视觉识别、知识回忆还是推理整合阶段？
2. 通过阶段替换干预，能否量化各阶段对最终答案正确性的影响？
3. 结构化轨迹监督微调能否有效减少阶段级幻觉？

### 4. 背景与问题设定

医学多模态大语言模型在临床决策支持中应用广泛，但常产生看似合理实则错误的输出（幻觉）。现有基准（如 Med-HALT、CARES）仅评估最终答案，无法区分错误来源。ClinHallu 通过将推理过程分解为视觉识别、知识回忆和推理整合三个阶段，实现源头诊断。

### 5. 方法概览

- 任务 / 环境: 医学视觉问答（VQA），涵盖 VQA-RAD、PathVQA、MedFrameQA、MedXpertQA 四个数据集，共 7,031 个实例。
- 变量 / 干预设计: 阶段替换干预：分别替换模型生成的视觉、知识或两者阶段为参考轨迹，让模型继续推理并生成新答案，以评估各阶段对最终答案的影响。
- 评价指标: 答案准确率（Acc）、各阶段幻觉率（HV、HK、HR）、替换后准确率增益（ΔAcc）。
- 模型比较: 评估 11 个模型，包括闭源（Qwen3-VL-Flash/Plus、Gemini-3-Flash）和开源（Qwen2.5-VL-7B、Qwen3-VL-8B/32B、Lingshu-7B、MedGemma-4B、InternVL3.5-8B、Qwen3.5-4B/9B）。
- 我的理解: 该方法通过结构化轨迹和替换干预，将幻觉诊断从二元分类提升为多阶段归因，能精确定位推理链中的故障节点。

### 6. 关键发现

- 视觉幻觉普遍严重，但数据集间存在差异: 所有子集视觉幻觉率平均超过 40%。VQA-RAD 以视觉瓶颈为主（平均幻觉率 42.9%），MedXpertQA 以知识瓶颈为主（43.1%），PathVQA 和 MedFrameQA 则相对平衡。例如，Gemini-3-Flash 在 MedXpertQA 上准确率达 85.0%，视觉幻觉率仅 27.6%，知识幻觉率 4.2%，表现最优。Qwen3.5-9B 整体准确率 69.1%，视觉幻觉率 41.9%，知识幻觉率 18.7%，推理幻觉率 4.8%。
- 阶段替换干预验证了瓶颈差异: 在 VQA-RAD 上，纠正视觉阶段使准确率提升 15.5%，远高于纠正知识的 4.6%；而在 MedXpertQA 上，纠正知识阶段提升 33.4%，高于视觉的 13.8%。
- 轨迹监督微调可减少阶段级幻觉: 在 VQA-RAD 和 PathVQA 上对 Qwen3.5-9B 进行轨迹监督微调后，各阶段幻觉率显著下降，验证了结构化轨迹作为训练监督的有效性。

## 4. From Shield to Target: Denial-of-Service Attacks on LLM-Based Agent Guardrails

**Topics:** LLM安全, 自主智能体, 拒绝服务攻击, 护栏机制
**Authors:** Yuguang Zhou, Xunguang Wang, Pingchuan Ma, Zhantong Xue, Zhaoyu Wang, Shuai Wang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.14517v1) | [PDF](http://arxiv.org/pdf/2606.14517v1.pdf)

### 1. 一句话结论

LLM护栏的推理能力可被利用，通过注入模仿其分析模式的对抗内容，引发长达148倍的延迟放大，导致系统瘫痪。

### 2. 为什么重要

首次揭示LLM护栏的可用性漏洞，证明其结构化推理模式可被反向利用为拒绝服务攻击放大器，威胁多智能体系统的关键路径安全。

### 3. 研究问题

1. LLM护栏的推理能力如何被利用来引发拒绝服务攻击？
2. 对抗性负载如何跨不同护栏架构和模型骨干迁移？

### 4. 背景与问题设定

LLM护栏通过结构化推理分析智能体上下文来防御提示注入和越狱攻击，但其每次动作前必须完成推理，成为关键路径。攻击者可通过注入模仿护栏分析模板的文本，诱使其陷入无限推理循环，消耗计算资源。现有DoS方法（如离题问题、对抗字符串）因护栏任务聚焦而失效，仅实现1.1-1.2倍放大。

### 5. 方法概览

- 任务 / 环境: 自主智能体系统（Web、桌面、代码、多智能体），护栏需在每次动作前输出安全裁决。
- 变量 / 干预设计: 设计两种攻击框架：1）束搜索优化框架，利用LLM提议器结合策略库生成自然语言负载；2）机制感知结构变异，直接修改护栏模板中的风险类别、枚举深度等槽位。
- 评价指标: 令牌放大倍数（输出长度比）、延迟放大倍数（端到端时间比）、吞吐量下降。
- 模型比较: 在8个模型骨干（Claude、GPT、Gemini、DeepSeek、Qwen等）上测试，与6种基线DoS方法对比。
- 我的理解: 攻击核心是逆向利用护栏的指令遵循能力：不是让其偏离任务，而是让其更忠实地执行任务但无限延长推理。束搜索通过护栏自身输出作为反馈，迭代优化负载结构。

### 6. 关键发现

- 跨模型迁移性: 在单一开源代理TS-Guard-8B上优化的负载可直接迁移至8个模型骨干，实现13-63倍令牌放大，而基线方法仅1.1-1.2倍。
- 端到端系统影响: 在OpenHands、LangGraph、BrowserGym、OSWorld四个部署中，分别实现36.3倍、148倍、131倍、18倍峰值延迟放大，LangGraph中共享护栏基础设施出现头阻塞导致吞吐量严重下降。
- 现有防御失效: 预推理过滤器无法检测流畅的安全分析式负载；硬令牌预算导致失败开放（安全绕过）或失败关闭（拒绝合法用户）；更强护栏模型因更忠实遵循注入模式而放大攻击。

- 图来源: 图1：端到端威胁示意图
- 为什么重要: 直观展示攻击流程：恶意内容注入环境→护栏将其解释为分析模式→陷入无限推理循环→无裁决输出→计算放大13-63倍（最高148倍）→共享护栏基础设施中协同智能体被饿死。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 护栏作为关键路径服务，其可用性漏洞可导致整个智能体系统瘫痪，需重新设计成本有界的推理机制。
- 对 Skill: 攻击利用护栏的指令遵循能力，提示技能设计需考虑推理深度与计算成本的平衡。

### 11. 局限性

- 攻击依赖护栏的特定模板结构，不同护栏设计可能需调整负载。
- 端到端部署中需针对场景定制负载（如代码智能体需矛盾因素，多智能体需变换鲁棒性）。
- 未探索对非LLM护栏（如规则或分类器）的适用性。

### 12. 我的看法 / 个人笔记

该工作揭示了LLM护栏的一个根本性悖论：其核心优势（结构化推理）也是其最大弱点。攻击不依赖模型漏洞，而是利用设计范式，因此防御需根本性重构，如引入推理预算、动态截断或混合架构。

## 5. Learning Coordinated Preference for Multi-Objective Multi-Agent Reinforcement Learning

**Topics:** Multi-Agent Reinforcement Learning, Multi-Objective Optimization, Preference Coordination
**Authors:** Pengxin Wang, Lihao Guo, Yi Xie, Bo Liu, Siyang Cao, Jingdi Chen
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.14693v1) | [PDF](http://arxiv.org/pdf/2606.14693v1.pdf)

### 1. 一句话结论

提出偏好协调多智能体策略优化（PCMA），通过学习智能体特定的协调偏好，在合作多目标多智能体强化学习中实现互补权衡，提升团队性能。

### 2. 为什么重要

现实多智能体系统（如交通信号控制、自动驾驶）常需平衡多个冲突目标，现有方法强制所有智能体使用相同偏好向量，导致协调不足。PCMA首次从理论上证明偏好多样性可带来团队一阶改进，并提供实用算法，为复杂多目标协作任务提供了新范式。

### 3. 研究问题

1. 如何协调多智能体系统中各智能体的目标偏好以实现团队最优？
2. 偏好多样性如何从理论上保证团队性能的一阶提升？

### 4. 背景与问题设定

合作多目标多智能体强化学习（MOMARL）建模多个智能体在多个潜在冲突目标下的联合决策。冲突不仅存在于目标之间，也存在于不同观测、角色和贡献的智能体之间。现有方法通常对所有智能体使用相同的标量化偏好向量，导致智能体在同一目标维度上竞争，限制协调效果。例如，两辆自动驾驶汽车通过无信号交叉口时，若都偏好效率则可能碰撞，若都偏好安全则可能都等待。因此需要一种机制让智能体学习互补的偏好。

### 5. 方法概览

- 任务 / 环境: 合作多目标多智能体环境，包括粒子世界（MOMPE）、无人机控制、步行器运动（MOMAwalker）和星际争霸（SMAC）等。每个环境提供稀疏团队奖励和智能体特定的向量引导奖励（如效率、安全、能量消耗）。
- 变量 / 干预设计: 提出偏好协调多智能体策略优化（PCMA），在集中训练分散执行（CTDE）框架下，为每个智能体配备随机偏好规划器（Dirichlet分布）和偏好条件化策略。偏好规划器根据局部观测输出偏好分布，采样偏好后输入策略网络。训练时通过团队优势反馈和偏好多样性正则化（避免偏好坍塌）优化规划器。
- 评价指标: 团队成功率、平均奖励（跨目标和智能体）、偏好多样性（成对偏好距离）、帕累托前沿覆盖度。
- 模型比较: 与MADDPG、IPPO、MAPPO对比，所有基线使用固定标量化权重[0.5,0.5]。PCMA在多数任务上取得最佳或并列最佳性能。
- 我的理解: PCMA的核心创新在于将偏好视为可学习的协调变量，通过团队优势信号引导偏好规划器产生多样化的偏好，从而让不同智能体专注于不同目标维度，实现角色分工。理论部分通过一阶改进分解证明了偏好多样性对团队性能的正面贡献，并利用隐函数定理保证偏好变化时策略可平滑跟踪均衡。

### 6. 关键发现

- 偏好多样性提升团队性能: 理论证明（定理4.2）：团队一阶改进可分解为直接团队梯度改进、平均偏好对齐和多样性诱导改进三项。其中多样性项ηNκDp（Dp为智能体间偏好成对距离）表明，当偏好多样性足够且与目标梯度对齐时，可带来额外的正面改进。
- 偏好协调实现角色分化: 实验显示，在SMAC中，PCMA使智能体在伤害输出与承受上分散分布（图4），形成攻击手和前线单位等不同角色；在粒子世界任务中，智能体偏好收敛到不同目标方向，覆盖帕累托前沿的不同区域（图3）。
- PCMA在多种任务上超越基线: 在Cooperative Spread、Safe Predator Prey、Catch、Escort、MOMAwalker和SMAC等任务中，PCMA的成功率或平均奖励均优于或持平于MADDPG、IPPO、MAPPO（表1）。

- 图来源: 图1：偏好协调示意图。左图：所有智能体使用相同偏好向量w，导致冲突；右图：不同智能体使用不同偏好w1,w2,w3，投影到帕累托前沿的不同点，团队共同覆盖前沿。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为多智能体系统提供一种自动角色分工机制，无需手动设计角色分配，可应用于自动驾驶、机器人协作、交通管理等场景。
- 对 Skill: 偏好协调可视为一种隐式技能学习，不同偏好对应不同行为模式，有助于智能体掌握多样化技能。
- 对 Memory: 偏好规划器基于局部观测输出偏好分布，不依赖长期记忆，但未来可结合记忆机制处理部分可观测性。
- 对评测: 提出团队成功率和平均奖励之外，偏好多样性和帕累托前沿覆盖度可作为评估协调质量的新指标。

### 11. 局限性

- 理论分析基于一阶近似和局部均衡跟踪，全局收敛性未证明。
- 偏好多样性正则化系数λ1需手动调节，可能影响稳定性。
- 实验环境为修改版，原始MOMALand任务中团队奖励与引导奖励的分离可能不自然。

### 12. 我的看法 / 个人笔记
