## 1. SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long-Horizon Deep Research

**Topics:** GUI Agent, Delegation Intelligence, Long-Horizon Deep Research
**Authors:** Pu Ning, Quan Chen, Kun Tao, Xinyu Tang, Tianshu Wang, Qianggang Cao, Xinyu Kong, Zujie Wen, Zhiqiang Zhang, Jun Zhou
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.09730v1) | [PDF](http://arxiv.org/pdf/2606.09730v1.pdf) | [Code](search-swarm.github.io)

### 1. 一句话结论

SearchSwarm通过引导框架合成监督微调数据，将委派智能内化到模型权重，在多个长时程研究基准上达到同规模最佳性能，并开源完整方案。

### 2. 为什么重要

该工作首次系统探索了委派智能的训练数据合成与模型训练方法，将委派智能定义为单代理上下文管理，解决了长时程任务中上下文管理的关键瓶颈，为开源社区提供了完整的引导框架、模型权重和训练数据。

### 3. 研究问题

1. 如何合成高质量的训练数据来培养大语言模型的委派智能？
2. 委派智能能否通过监督微调内化到模型权重中，从而提升长时程深度研究任务的性能？

### 4. 背景与问题设定

大语言模型在处理长时程真实世界任务时，上下文需求无限增长而窗口有限。现有被动式上下文管理方法（如截断、摘要）缺乏前瞻性规划。委派智能范式通过主代理分解任务并委派给子代理，仅返回摘要结果，可主动管理上下文预算，但训练数据稀缺。论文将委派智能视为单代理上下文管理：模型通过生成的简报和报告实现内容感知压缩，替代固定规则截断。

### 5. 方法概览

- 任务 / 环境: 深度研究任务，多轮交互环境，配备搜索、网页访问、学术搜索、Python执行等工具。
- 变量 / 干预设计: 设计引导框架（harness），包含主代理和子代理的系统提示，遵循四项原则：鼓励委派、全面简报、核心判断保留和引用报告。主代理通过call_sub_agent工具委派子代理，子代理在独立上下文中执行并返回带引用的报告。
- 评价指标: BrowseComp、BrowseComp-ZH、GAIA、xbench-DeepSearch-2505上的准确率。
- 模型比较: 与同规模（30B-A3B）模型（如MiroThinker、RedSearcher、LongSeeker）及更大规模闭源/开源模型（如GPT-5、Claude-4.5、DeepSeek V3.2）对比。
- 我的理解: 该方法本质上是单代理上下文管理：通过模型生成的简报和报告实现内容感知压缩，替代固定规则截断。子代理是同一模型在独立上下文中的调用，而非独立模型。

### 6. 关键发现

- 委派智能显著提升长时程任务性能: SearchSwarm在BrowseComp上达到68.1，较基线模型（43.4）提升24.7个绝对百分点，超越所有同规模模型。
- 引导框架可生成高质量SFT数据: 通过引导框架收集的轨迹经筛选后用于监督微调，成功将委派行为内化到模型权重，使模型在测试时展现委派智能。
- 与更大规模模型竞争: SearchSwarm在GAIA上（82.5）超越GPT-5（76.4）和Seed-2.0-Pro（78.6），表明委派智能可弥补模型规模差距。

- 对 Memory: 委派智能本质上是内容感知的上下文压缩，比固定规则截断更智能，可视为主动记忆管理策略。

### 11. 局限性

- 实验仅针对深度研究任务，委派智能在其他领域（如代码、对话）的适用性未验证。

### 12. 我的看法 / 个人笔记

该工作为委派智能的训练提供了首个完整开源方案，方法简洁有效。但单层委派和任务特定性可能限制其通用性，未来可探索多层委派和跨任务泛化。

## 2. SpatialWorld: Benchmarking Interactive Spatial Reasoning of Multimodal Agents in Real-World Tasks

**Topics:** GUI Agent, Spatial Reasoning, Multimodal Large Language Models, Embodied AI
**Authors:** Hongcheng Gao, Hailong Qu, Jingyi Tang, Jiahao Wang, Zihao Huang, Hengkang Qiao, Shihong Huang, Junming Yang, Yi Li, Hongyixuan Yuan, Wenjie Li, Bohan Zeng, Wenbo Li, Bo Wang, Jianhui Liu, Olive Huang, Haoyang Huang, Wentao Zhang, Guoqing Huang, Nan Duan, Yinpeng Dong
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.09669v1) | [PDF](http://arxiv.org/pdf/2606.09669v1.pdf) | [Code](https://spatial-world.github.io)

### 1. 一句话结论

SpatialWorld 基准测试表明，当前最强的多模态大模型在复杂真实世界空间任务中平均成功率仅 17.4%，揭示了主动探索与长程规划能力的严重不足。

### 2. 为什么重要

该工作首次在统一、模拟器无关的框架下，系统评估了多模态智能体在纯视觉部分可观测条件下的交互式空间推理能力，为未来空间智能体的发展提供了严格的测试平台。

### 3. 研究问题

1. 当前多模态大模型在真实世界复杂空间任务中的交互式空间理解能力究竟如何？
2. 任务成功与执行效率之间是否存在一致性？不同领域（如室内、户外、数字游戏）的模型表现有何差异？

### 4. 背景与问题设定

现有空间推理基准多采用被动评估（如静态 VQA）或与特定模拟器深度耦合，无法评估通用交互式空间理解。SpatialWorld 提出统一基准，要求智能体在纯视觉部分可观测条件下，通过主动探索和语言化高层决策完成跨多种 3D 环境的任务。

### 5. 方法概览

- 任务 / 环境: 8 种异构模拟后端（AI2-THOR、ProcTHOR、VirtualHome、CARLA、EmbodiedCity 及其多智能体变体，以及 3D 游戏环境），共 760 个人工标注任务，涵盖日常家务、工作学习、娱乐、旅行、社交协作和数字游戏六大领域。
- 变量 / 干预设计: 智能体仅接收第一人称 RGB 图像，无深度、地图等特权信息；通过统一文本动作接口（导航、视角/姿态、交互、任务控制）与环境交互；任务成功由终端状态验证器客观判定，而非匹配固定轨迹。
- 评价指标: 主要指标：任务成功率（TSR）；辅助指标：步骤效率（SE，成功任务中参考步数与实际步数之比）。
- 模型比较: 评估了 15 个先进多模态智能体，包括 GPT-5、Qwen-3.5-397B-A17B、Gemini-3.1-Pro 等。
- 我的理解: 该方法通过解耦环境执行与智能体决策，构建了标准化观测-动作接口，使得不同模拟器的任务可以在统一协议下比较，从而更纯粹地评估智能体的空间推理与规划能力。

### 6. 关键发现

- 当前模型空间任务解决能力远未可靠: 最强模型 GPT-5 平均 TSR 仅 17.4%，最佳开源模型 Qwen-3.5-397B-A17B 为 14.1%。
- 任务成功与执行效率存在明显脱节: 高 TSR 的模型不一定具有高 SE，表明成功常伴随冗余探索或任务依赖的捷径。
- 模型排名在不同领域差异显著: GPT-5 在日常生活、旅行和社交协作任务中领先；Qwen-3.5 在工作学习与娱乐任务中与 GPT-5 持平；Gemini-3.1-Pro 在数字游戏中得分最高。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 未来空间智能体需要更强的主动探索和长程规划能力，当前模型在部分可观测条件下的决策能力严重不足。
- 对 Skill: 模型在导航、交互和混合任务上的表现差异表明需要专门提升空间推理与物体操作技能。
- 对 Memory: 长程任务中，智能体需要有效维护空间信念和任务进度记忆，当前模型在此方面存在瓶颈。
- 对评测: SpatialWorld 提供的统一跨平台评估框架可作为标准测试床，推动空间推理研究从静态识别走向交互式任务解决。

- 基准任务数量有限（760 个），可能无法覆盖所有真实世界空间场景。
- 动作空间为高层符号接口，可能无法完全反映低层连续控制中的空间推理挑战。
- 当前仅评估了闭源和开源 MLLM，未涉及专门训练的具身模型或强化学习方法。

### 12. 我的看法 / 个人笔记

SpatialWorld 通过精心设计的统一接口和严格的部分可观测条件，为评估多模态智能体的空间推理能力提供了更真实、更全面的测试环境。其发现——即使最强模型成功率也极低——凸显了当前 MLLM 在主动探索和长程规划上的根本性短板，为未来研究指明了关键方向。

## 3. TABVERSE: Benchmarking Cross-Format Table Understanding in LLMs and VLMs

**Topics:** Table Understanding, LLM Evaluation, VLM Evaluation, Multimodal Benchmarking
**Authors:** Momina Ahsan, Sarfraz Ahmad, Ming Shan Hee, Roy Ka-Wei Lee, Preslav Nakov
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.09578v1) | [PDF](http://arxiv.org/pdf/2606.09578v1.pdf) | [Code](https://github.com/globeProject/databaseTABVERSE)

### 1. 一句话结论

TABVERSE 通过固定表格内容、对齐 HTML/LaTeX/Markdown 及其渲染图像，系统揭示了表示选择显著影响 LLM/VLM 的表格理解能力，HTML 通常是最稳健的文本格式，但结构化文本并非始终优于图像。

### 2. 为什么重要

现有表格评测常混淆内容、格式、布局和模态，无法隔离表示效应。TABVERSE 通过固定表格内容、对齐多种结构格式及其渲染图像，首次实现了对表示效应的可控评估，为可靠评测和模型设计提供了关键基准。

### 3. 研究问题

1. 表格的结构格式（HTML、LaTeX、Markdown）和输入模态（结构化文本 vs. 渲染图像）如何影响 LLM 和 VLM 的表格理解能力？
2. 不同任务（问答、结构理解、结构重建）下，表示效应的表现是否一致？

### 4. 背景与问题设定

LLM 和 VLM 在表格推理任务中表现日益重要，但同一表格内容可能以 HTML、Markdown、LaTeX 等结构格式或渲染图像形式出现。现有评测往往让内容、格式、布局和模态同时变化，难以分离表示效应。TABVERSE 通过固定表格内容、对齐多种表示，系统评估表示选择对表格理解的影响。

### 5. 方法概览

- 任务 / 环境: 基于 FEVEROUS、HybridQA、SQA、TabFact、WikiTableQuestions 的保留测试集，构建 6,097 个问答对（4,434 张唯一表格），并从中平衡采样 700 个样本（按难度和类别平衡）。
- 变量 / 干预设计: 每个表格生成 HTML、LaTeX、Markdown 三种结构格式及其渲染图像，形成对齐的多模态对。在 QA、结构理解能力（SUC）和结构重建（SR）三个任务上，通过 VLM-Image、VLM-Text、LLM-Text 三种流水线对比表示效应。
- 评价指标: QA 和 SUC 使用精确匹配（EM）准确率，SUC 额外报告字段准确率和宽松准确率；SR 使用 GriTS-Topology 和 GriTS-Content 评估结构相似性和单元格文本保真度，并报告语法可用性（HTML 解析、Markdown 渲染/解析、LaTeX 编译成功率）。
- 模型比较: 评估了多种 LLM（Qwen2.5-7B、Qwen3-30B-A3B、TableGPT2-7B、TAMA-QWen3）和 VLM（SmolVLM2-2.2B、Gemma-3-12B/27B、InternVL3.5-14B/30B-A3B、Qwen3-VL-8B/30B-A3B、Ministral-3-14B、LLaVA-1.6-7B/13B、TableLLaVA-v1.5-7B、GPT-5.2、Gemini-3-Flash-Preview），在零样本设置下进行对比。
- 我的理解: TABVERSE 的核心创新在于通过固定表格内容、变化表示方式，将表示效应从内容效应中分离出来。其设计类似于控制变量实验，但针对的是输入表示而非模型组件。三个任务（QA、SUC、SR）从不同维度（语义理解、结构感知、重建能力）全面评估表示效应，使得结论更具鲁棒性。

### 6. 关键发现

- 表示选择显著影响表格理解: 结构化文本通常优于渲染图像，但差距大小取决于任务、模型和格式。HTML 是最稳健的文本格式，而行敏感的结构任务和 LaTeX 重建仍具挑战性。
- 格式对文本推理影响更大: 渲染图像在不同格式（HTML/LaTeX/Markdown）下得分相似，而文本流水线在不同格式间差异更大，表明符号格式对基于文本的推理影响更显著。
- 在特定模型对比中，较小模型在严格 EM 下可能优于较大模型: 在 Qwen3-VL 系列中，8B 变体在严格 EM 下优于 30B-A3B 变体（例如 HTML 图像：50.29% vs 41.14%），但后者在宽松匹配下恢复明显（附录显示），表明部分差距源于答案格式化行为而非检索能力。此现象仅基于 Qwen3-VL 的特定对比，并非普遍规律。

- 为什么重要: 该图清晰展示了 TABVERSE 的核心设计：从平衡评测集出发，每个表格生成三种结构格式及其渲染图像，形成对齐的多模态对，并支持 QA、SUC、SR 三个任务。这直观体现了如何通过固定内容、变化表示来隔离表示效应。

### 8. 如何读这张图

从顶部平衡数据集开始，每个表格生成 HTML、Markdown、LaTeX 三种结构文件及其对应的渲染图像（共 629 组）。这些对齐的多模态对分别输入 QA、SUC、SR 任务，通过 VLM-Image、VLM-Text、LLM-Text 三种流水线进行评测，从而比较不同表示和模态下的性能差异。

### 9. 次重要图

- 为什么也重要: Table 3（QA 结果表）直接展示了不同模型在不同格式和模态下的 EM 准确率，是核心实验结果，揭示了表示效应和模型差异。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 智能体在处理表格时，应优先选择结构化文本格式（尤其是 HTML）而非渲染图像，以提高理解准确性。同时需注意格式敏感性，不同任务可能偏好不同格式。
- 对 Skill: 表格理解技能应包含格式感知能力，模型需适应不同结构格式和模态。训练时可引入多格式数据以增强鲁棒性。
- 对 Memory: 表格记忆系统应考虑表示效应：同一表格内容的不同表示可能导致不同的检索和推理性能，记忆存储时应保留原始格式信息。
- 对评测: 评测表格理解能力时，必须控制表示变量，避免内容、格式、模态混淆。TABVERSE 的设计可作为未来评测的参考范式。

### 11. 局限性

- 仅包含英文表格和问答对，未涉及多语言或跨文化表格。
- 表格来源限于五个 TableQA 数据集，可能未覆盖所有表格类型（如复杂层级表、多模态表）。
- SR 任务仅评估从图像到三种结构格式的重建，未考虑其他输出格式或端到端表格理解。
- 部分模型（如 TAMA-QWen3）在 QA 任务中性能异常低（EM 约 18-21%），可能源于其设计目标不同或对表格格式不敏感，但论文未深入分析原因。

### 12. 我的看法 / 个人笔记

TABVERSE 通过精巧的控制实验设计，有力地证明了表格表示是影响 LLM/VLM 表格理解的关键因素，而非仅仅是内容本身。其贡献在于提供了一个可复用的评测框架，而非单一性能榜单。未来工作可扩展至更多语言、表格类型和任务，并探索如何利用格式信息提升模型鲁棒性。

## 4. OmniGameArena: A Unified UE5 Benchmark for VLM Game Agents with Improvement Dynamics

**Topics:** GUI Agent, 游戏智能体, 视觉语言模型, 基准测试, 自我改进
**Authors:** Mingxian Lin, Shengju Qian, Yuqi Liu, Yi-Hua Huang, Yiyu Wang, Wei Huang, Yitang Li, Fan Zhang, Zeyu Hu, Lingting Zhu, Xin Wang, Xiaojuan Qi
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.09826v1) | [PDF](http://arxiv.org/pdf/2606.09826v1.pdf) | [Code](https://mxlin043.github.io/OmniGameArena/)

### 1. 一句话结论

OmniGameArena 提出了一个包含12个UE5游戏的统一基准，并引入改进动态曲线（IDC）来评估VLM游戏智能体在多次反思中的表现提升和技能迁移能力。

### 2. 为什么重要

现有游戏基准通常只报告单次尝试分数，忽略智能体在重复交互中的改进轨迹，且缺乏对单人、对抗和合作模式的统一评估。OmniGameArena 填补了这一空白，通过新构建的游戏和IDC框架，揭示了冷启动分数无法反映的改进动态和技能迁移差异。

### 3. 研究问题

1. VLM游戏智能体在多次反思交互中如何改进表现？
2. 原始任务上的改进是否能预测在保留任务变体上的迁移能力？

### 4. 背景与问题设定

VLM智能体越来越多地部署在交互式游戏环境中，但现有基准存在三个问题：仅报告单次分数、侧重单人模式、缺乏统一协议来评估不同类别的智能体（商业VLM、开源VLM、专用游戏策略）。OmniGameArena 通过12个新构建的UE5游戏（7个单人、3个对抗、2个合作）和IDC框架来解决这些问题。

### 5. 方法概览

- 任务 / 环境: 12个新构建的Unreal Engine 5游戏，涵盖单人、对抗和合作模式，具有统一的动作接口（键盘鼠标、游戏手柄）。
- 变量 / 干预设计: IDC框架：每个（智能体，游戏）对运行多轮反思，每轮包含K个回合，反射器LLM通过工具使用自主检查轨迹、诊断失败模式、验证并提炼技能提示，产生每轮分数序列和保留任务变体上的迁移分数。
- 评价指标: 游戏进度归一化为[0,1]连续分数；IDC报告每轮分数序列和迁移分数。
- 模型比较: 12个VLM智能体在冷启动排行榜上比较，4个顶级智能体在IDC下进一步分析。
- 我的理解: 该方法通过将反思过程分解为探索、诊断、验证和提炼四个阶段，并赋予反射器LLM自主选择检查内容的工具，实现了轻量级但功能完整的自我改进循环。

### 6. 关键发现

- 冷启动排行榜上无单一VLM主导: 商业VLM与开源VLM和专用策略之间存在较大差距，且领导力在不同游戏间轮换。
- IDC下所有智能体均能改进，但峰值出现在中期: 四个顶级智能体通过IDC反思均超过冷启动基线，但峰值性能通常出现在曲线中期而非最后一轮。
- 原始任务改进与保留变体迁移可能发散: 原始任务上的增益不能预测保留任务变体上的迁移表现，这一差异被单轮排行榜分数隐藏。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 需要设计能够从交互中持续改进的智能体，而不仅仅是单次尝试表现。
- 对 Skill: 技能提示的提炼应基于多轮反思和诊断，而非固定模板。
- 对 Memory: 持久化模块（笔记本和已验证技能）对于跨轮知识积累至关重要。
- 对评测: 评估应包含改进动态和迁移能力，而不仅仅是冷启动分数。

### 11. 局限性

- IDC框架中反射器LLM的自主性可能导致不一致的反思质量。
- 游戏数量有限（12个），可能无法完全覆盖所有能力维度。

### 12. 我的看法 / 个人笔记

该工作通过构建新游戏和引入IDC，显著推进了VLM游戏智能体的评估方法，特别是揭示了改进动态与迁移能力之间的差异，为未来研究提供了重要方向。

### 13. 这篇工作引用的近期高影响力相关文章

- BALROG
  为什么重要：一个2D网格游戏基准，用于评估VLM智能体，但缺乏3D和实时性。
  核心贡献：提供了多模态空间推理任务。

  与当前论文关系：IDC 扩展了Reflexion，通过多轮反思和工具使用实现更自主的改进。

### 14. 总结

OmniGameArena 是一个包含12个新UE5游戏的统一基准，覆盖单人、对抗和合作模式，并引入改进动态曲线（IDC）来评估VLM智能体在多次反思中的表现和技能迁移。实验表明，冷启动分数无法反映改进轨迹，且原始任务改进与迁移能力可能发散。

## 5. iOSWorld: A Benchmark for Personally Intelligent Phone Agents

**Topics:** GUI Agent, Mobile Agent Benchmark, Personalization
**Authors:** Lawrence Keunho Jang, Mareks Woodside, Geronimo Carom, Andrew Keunwoo Jang, Jing Yu Koh, Ruslan Salakhutdinov
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.09764v1) | [PDF](http://arxiv.org/pdf/2606.09764v1.pdf) | [Code](https://github.com/ljang0/iOSWorld)

### 1. 一句话结论

iOSWorld 是首个基于持久用户身份、跨 26 个原生 iOS 应用的交互式手机智能体基准，最佳配置整体通过率仅 52%，多应用任务仅 37%。

### 2. 为什么重要

现有手机智能体基准缺乏个性化，无法评估智能体对用户身份、历史记录和偏好的推理能力。iOSWorld 填补了这一空白，为构建真正有用的个人手机助手提供了关键评测平台。

### 3. 研究问题

1. 如何构建一个包含持久用户身份和跨应用关联数据的手机智能体基准？
2. 当前最先进的计算机使用模型在需要个性化推理的任务上表现如何？

### 4. 背景与问题设定

手机智能体需要具备“个人智能”，即能基于设备上用户的身份、历史和偏好进行推理，而非在无个性化数据的沙盒中执行孤立指令。现有基准（如 AndroidWorld、MobileWorld）均未植入用户身份，且缺乏跨应用数据连续性。iOSWorld 针对这一空白，构建了首个原生 iOS 模拟器基准。

### 5. 方法概览

- 任务 / 环境: 基于 iOS 模拟器，通过 Appium + XCUITest 驱动，环境建模为部分可观测马尔可夫决策过程（POMDP）。
- 变量 / 干预设计: 两种观测模态：纯视觉（仅截图）和视觉+XML（额外提供经筛选的无障碍树）。动作空间包括点击、滑动、输入等，XML 模态额外支持通过标识符精准点击和启动应用。
- 评价指标: 主要指标为二元通过率（pass rate），使用 GPT-5.4-Mini 作为轨迹级评判器，经人工验证一致性 κ=0.77。
- 模型比较: 评估了 Claude Opus 4.6、Sonnet 4.6、GPT-5.4、GPT-5.4 Mini、Gemini 3 Flash 和开源 Qwen3.5 35B-A3B，每种模型在两种模态下测试。
- 我的理解: 该基准通过精心设计的 26 个应用和 133 个任务，系统性地测试了智能体在单应用、多应用和记忆/个性化三个难度层次上的能力，尤其强调跨应用数据关联和隐含模式发现。

### 6. 关键发现

- 最佳配置整体通过率仅 52%，多应用任务表现最差: Opus 4.6 在视觉+XML 模态下整体通过率 51.9%，单应用 81.5%，记忆任务 54.3%，多应用任务仅 36.7%。多应用任务需要跨 2-8 个应用传递信息，对智能体的规划和信息整合能力要求最高。
- 视觉+XML 模态显著提升强模型性能，但对小模型帮助有限: Opus 4.6 从纯视觉的 26.3% 提升至 51.9%（+25.6 个百分点），Sonnet 4.6 提升 18.0 个百分点，GPT-5.4 提升 19.5 个百分点。但 GPT-5.4 Mini 和 Qwen3.5 在添加 XML 后性能反而下降或无明显提升。
- 记忆与个性化任务对当前模型极具挑战: 最佳模型 Opus 4.6 在记忆任务上仅 54.3% 通过率，纯视觉下仅 8.7%。这类任务要求智能体主动探索应用、发现隐含模式（如最常通勤路线、最常点餐餐厅），而非执行明确指令。

### 7. 最重要的图

- 图来源: Figure 3: 多应用任务（QuickBite → TeamChat）在纯视觉和视觉+XML 模态下的轨迹对比

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 当前智能体在多应用和个性化任务上表现薄弱，需要更强的跨应用信息整合和隐含模式发现能力。视觉+XML 模态对强模型有效，但小模型无法利用结构化信息，提示架构设计需考虑模型规模与输入模态的匹配。
- 对 Skill: 智能体需要发展“个人智能”技能，包括主动探索、模式识别、跨应用数据关联和基于用户历史的推理。
- 对 Memory: 记忆任务要求智能体从分散的应用数据中提取隐含模式，当前模型表现极差（最佳 54.3%），提示需要更有效的记忆机制和长期上下文管理。
- 对评测: LLM-as-a-Judge 与人工评判高度一致（κ=0.77），验证了自动评测的可靠性。但纯视觉模态下智能体易陷入视觉陷阱，评测需考虑模态差异。

### 11. 局限性

- 基准应用为自定义构建，与真实 iOS 应用在 UI 复杂性和数据多样性上存在差距。
- 仅评测了 6 个模型，且开源模型仅 Qwen3.5，代表性有限。
- 任务数量（133）相对有限，且未覆盖所有 iOS 应用类型和交互模式。
- 评测依赖 GPT-5.4-Mini 作为评判器，可能存在偏差。
- 未评测智能体在真实设备上的表现，模拟器环境与真实环境存在差异。

### 12. 我的看法 / 个人笔记

iOSWorld 是手机智能体评测的重要进步，首次将个性化推理纳入基准设计。其核心贡献在于构建了 26 个共享用户身份的应用生态，使跨应用数据关联和隐含模式发现成为可能。当前模型在记忆和个性化任务上的低通过率表明，个人智能仍是重大挑战。未来工作应关注如何让智能体更高效地利用结构化 UI 信息，以及如何在小模型上实现类似增益。
