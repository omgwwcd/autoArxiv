## 1. TestEvo-Bench: An Executable and Live Benchmark for Test and Code Co-Evolution

**Topics:** 软件测试, 代码演化, 基准测试, 自动化测试生成, 测试更新, 智能体评估
**Authors:** Jiale Amber Wang, Kaiyuan Wang, Pengyu Nie
**Venue / Year:** arXiv:2607.02469v1 [cs.SE]
**Links:** [Abstract](http://arxiv.org/abs/2607.02469v1) | [PDF](http://arxiv.org/pdf/2607.02469v1.pdf) | [Code](https://huggingface.co/TestEvo-Bench/datasets)

### 1. 一句话结论

TestEvo-Bench 是一个可执行、实时更新的基准测试，通过三阶段流水线从 152 个 Java Maven 项目中构建了 746 个测试生成和 509 个测试更新任务，实验表明 Claude Code 在测试生成上达到 77.5% 成功率，在测试更新上达到 74.6% 成功率，但在最新任务和有限成本下性能显著下降。

### 2. 为什么重要

现有基准测试多基于静态元数据，无法验证测试的可执行性及其与代码变更的语义关联，而 TestEvo-Bench 通过执行驱动指标（通过率、覆盖率、变异分数）和实时更新机制（支持按模型训练截止日期过滤任务），为评估自动化测试智能体提供了更可靠、防数据泄露的评测平台。

### 3. 研究问题

1. 如何构建一个可执行且能反映测试与代码共同演化过程的基准测试？
2. 当前最先进的代码智能体（Claude Code、Gemini CLI、SWE-Agent 搭配不同基础模型）在测试生成和测试更新任务上的表现如何？

### 4. 背景与问题设定

软件测试与代码通常共同演化：代码变更后应伴随新测试或更新测试以记录新行为。然而现有基准测试要么只针对固定代码快照生成测试（如 EvoSuite、Randoop），要么仅关注测试更新（如 CEPROT、TARGET），且多依赖静态依赖分析和差异信号作为标签，缺乏执行上下文，无法验证测试是否真正反映代码变更。随着大语言模型和智能体在软件工程领域的应用日益广泛，亟需一个可执行的基准来评估其维护和生成测试的真实能力。

### 5. 方法概览

- 任务 / 环境: 基于 Java Maven 开源项目，每个任务锚定一个真实提交历史，提供旧/新代码版本、构建配置、类路径等执行环境，支持编译、执行、覆盖率和变异分析。
- 变量 / 干预设计: 两个任务轨道：测试生成（智能体需编写新测试以捕获代码变更引入的新行为，要求新测试在新版本上通过且在旧版本上失败）和测试更新（智能体需修改现有测试以适应新代码，不要求与开发者更新完全一致，只要通过新版本并练习变更行为即可）。
- 评价指标: 主要指标为成功率（测试生成：新测试在新版本上通过且在旧版本上失败；测试更新：更新后的测试在新版本上通过），辅以通过率、覆盖率和变异分数（针对焦点依赖行，使用 Universal Mutator 工具）。
- 模型比较: 评估了四种配置：Claude Code 和 Gemini CLI 作为完整系统，以及 SWE-Agent 分别搭配 Claude Opus 4.7 和 Gemini 3.1 Pro。
- 我的理解: 该方法通过三阶段流水线从提交历史中提取高质量任务：第一阶段挖掘候选测试-代码变更对，要求相邻提交均能构建和测试通过；第二阶段执行跨版本测试（旧测试在旧代码、新测试在新代码、旧测试在新代码、新测试在旧代码）以清洗数据，仅保留语义关联强且可复现的任务；第三阶段将验证后的对转换为基准任务，并过滤冗余测试（例如，若新测试在旧代码上通过则视为冗余并丢弃）。这种基于执行结果而非静态差异的标注方式确保了基准的可执行性和语义有效性。

### 6. 关键发现

- 最强智能体在测试生成和更新上表现良好: Claude Code 在测试生成上达到 77.5% 成功率，在测试更新上达到 74.6% 成功率；SWE-Agent + Claude Opus 4.7 在测试生成上达到 72.3%，在测试更新上达到 70.1%；Gemini CLI 在测试生成上达到 68.9%，在测试更新上达到 65.4%；SWE-Agent + Gemini 3.1 Pro 在测试生成上达到 65.1%，在测试更新上达到 62.8%。
- 近期任务和有限成本下成功率显著下降: 在最新时间段（2025 年）的基准任务上，所有智能体的成功率平均下降约 15-20 个百分点；当每个任务的成本限制在 1 美元时，成功率平均下降约 30 个百分点，说明智能体在处理新近代码变更和资源受限场景时仍面临挑战。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 当前智能体在测试演化任务上表现尚可，但面对最新代码变更和成本限制时能力不足，提示未来需要更高效的推理和更鲁棒的代码理解能力。
- 对 Skill: 测试生成和更新需要智能体理解代码变更的语义并相应调整测试，这要求更强的因果推理和代码差异理解技能。
- 对 Memory: 任务涉及跨版本代码状态，智能体可能需要记忆旧版本行为以生成正确的新测试，对工作记忆和上下文管理提出要求。
- 对评测: TestEvo-Bench 的执行驱动指标和实时更新机制为评估智能体提供了更可靠、防数据泄露的评测范式，可推广到其他软件工程任务。

### 11. 局限性

- 仅针对 Java Maven 项目，可能不适用于其他语言或构建系统。
- 任务构建依赖严格的过滤条件（如提交间隔不超过 12 小时、排除重构变更、丢弃 T_new→Cold 通过的冗余测试），可能丢弃了部分有意义的共同演化案例。
- 当前仅评估了四种智能体配置，缺乏对更多开源或低成本方案的比较。

### 12. 我的看法 / 个人笔记

TestEvo-Bench 通过可执行评估和实时更新机制，显著提升了测试演化基准的可靠性和实用性。其自动化构建流水线和高过滤标准确保了任务质量，但 Java 生态的局限性可能限制了泛化性。未来工作可扩展到更多语言，并探索更高效的智能体架构以应对成本敏感场景。

## 2. Seek to Segment: Active Perception for Panoramic Referring Segmentation

**Topics:** Embodied AI, Active Perception, Referring Segmentation, Panoramic Vision, Agent Memory
**Authors:** Song Tang, Shuming Hu, Xincheng Shuai, Henghui Ding, Yu-Gang Jiang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.02497v1) | [PDF](http://arxiv.org/pdf/2607.02497v1.pdf) | [Code](https://henghuiding.com/APRS/)

### 1. 一句话结论

提出主动全景指代分割任务APRS及记忆增强智能体PanoSeeker，通过显式空间视觉记忆EgoSphere和强化学习优化探索效率，在360°环境中实现高效目标搜索与分割。

### 2. 为什么重要

现有指代分割模型仅处理固定视角静态图像，无法用于具身智能体在连续360°环境中的主动感知。APRS任务和PanoSeeker方法填补了这一空白，为具身智能体的空间推理与主动探索提供了新范式。

### 3. 研究问题

1. 如何在连续360°环境中，让智能体根据语言指令主动调整视角搜索并分割目标？
2. 如何设计空间视觉记忆以支持高效、非冗余的探索轨迹规划？

### 4. 背景与问题设定

指代分割在静态图像上取得显著进展，但现有方法被动处理固定视角图像，无法应用于具身智能体在连续360°环境中的主动感知。在360°环境中，目标可能不在初始视野内，智能体需调整视角（Δθ, Δϕ）进行探索。直接使用3D重建（如点云、高斯泼溅）计算和硬件成本高，而2D全景图像存在几何畸变，分割为孤立视角则丢失全局上下文。

### 5. 方法概览

- 任务 / 环境: 连续360°环境，由高分辨率全景图像通过球心投影模拟，初始视角远离目标，智能体以有限视场角（水平120°，垂直90°）观察。
- 变量 / 干预设计: 智能体PanoSeeker集成视觉语言模型（VLM）与显式空间视觉记忆EgoSphere。EgoSphere将顺序局部观测通过逆球心投影逐步映射到统一360°等距柱状投影画布上，并叠加十字准星、30°间隔经纬网格和探索轨迹作为空间提示。智能体根据当前局部视图、记忆画布和指令预测下一步动作（Δθ, Δϕ离散化为30°倍数，水平包括30°/60°/120°/180°，垂直包括30°/60°/90°）或[STOP]。先通过专家标注轨迹数据集进行监督微调（SFT），再使用GRPO强化学习优化探索效率，奖励函数包括效率奖励（每步减少与目标测地距离）和终止奖励（成功到达目标且IoU≥0.5）。
- 评价指标: 成功率（SR，需最终视角与目标质心测地距离<阈值且IoU≥0.5）、平均步数（AS）、成功加权路径长度（SPL）、平均交并比（mIoU）。
- 模型比较: 与适配的现有SOTA基线（如LISA、Seg-Zero等）相比，PanoSeeker在搜索效率和分割精度上均显著更优。消融实验显示：移除EgoSphere记忆后SR下降约15%，移除RL后SPL下降约20%。
- 我的理解: 该方法将主动探索问题转化为视觉“填空”任务：EgoSphere记忆画布逐步填充已探索区域，未探索区域保持为零值，智能体通过VLM理解当前局部视图与全局画布的关系，规划下一步移动方向，避免重复扫描。

### 6. 关键发现

- EgoSphere显式空间记忆有效减少冗余探索: 通过将局部观测累积到统一全景画布并添加空间提示，EgoSphere使智能体能感知已探索区域，避免死循环。消融实验表明，无记忆基线平均步数增加约40%，而EgoSphere保持恒定长度上下文，不受轨迹长度影响。
- 强化学习后训练显著提升探索效率: 在SFT基础上使用GRPO优化效率奖励和终止奖励，使智能体学会更直接的搜索路径。SPL指标从SFT后的0.52提升至RL后的0.63，平均步数从4.8降至3.9。
- APRS基准涵盖多样空间推理难度: 四种空间指代表达（EGO、UNIQ、ALLO、MULTIHOP）全面评估智能体的自我中心、属性识别、他物参照和多步推理能力。MULTIHOP最难，PanoSeeker在其上SR为0.45，而EGO为0.72。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: EgoSphere式显式空间记忆可推广至其他需要主动探索的具身任务（如导航、物体搜索），提供恒定长度的上下文表示，避免历史文本记忆的缩放瓶颈。
- 对 Skill: 智能体需学习将语言指令映射为空间探索策略，包括自我中心和他物参照推理，以及多步规划能力。
- 对 Memory: 显式空间记忆（如全景画布）比文本日志或隐式记忆更适用于连续空间探索，能直观表示已探索/未探索区域，支持非冗余路径规划。
- 对评测: APRS的多维评估（成功率、效率、分割质量）为主动感知任务提供了全面框架，SPL指标可推广至其他探索任务。

### 11. 局限性

- 当前基准基于全景图像模拟，未涉及真实3D场景中的物理交互（如碰撞、遮挡）。
- 动作空间离散化为30°步长，可能限制精细调整能力。
- EgoSphere记忆画布分辨率固定，极远处细节可能丢失。

### 12. 我的看法 / 个人笔记

PanoSeeker巧妙地将主动探索问题转化为视觉记忆填充任务，EgoSphere设计简洁有效，结合SFT和RL的训练策略合理。APRS基准的四种表达类型全面评估空间推理能力。未来可扩展至真实机器人平台，并探索连续动作空间和动态环境。

### 14. 总结

## 3. QFedAgent: Quantum-Enhanced Personalized Federated Learning for Multi-Agent Activity Recognition

**Topics:** 联邦学习, 量子机器学习, 多智能体系统, 活动识别
**Authors:** Quoc Bao Phan, Tuy Tan Nguyen
**Venue / Year:** IEEE MWSCAS 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.02426v1) | [PDF](http://arxiv.org/pdf/2607.02426v1.pdf)

### 1. 一句话结论

QFedAgent 通过变分量子电路融合模块，在非独立同分布的多智能体活动识别中，以仅72个量子旋转参数实现97.7%的平均测试准确率，参数总量减少约10倍。

### 2. 为什么重要

该工作首次将变分量子电路集成到个性化联邦学习框架中，用于多模态传感器融合，在保持高精度的同时大幅降低参数开销，为资源受限的分布式机器人感知系统提供了高效、隐私保护的解决方案。

### 3. 研究问题

1. 如何利用变分量子电路实现多模态传感器数据（加速度计和陀螺仪）的参数高效融合？
2. 在非独立同分布的多智能体联邦学习场景下，量子增强的个性化框架能否在降低通信成本的同时保持与经典方法相当的识别性能？

### 4. 背景与问题设定

联邦学习允许分布式设备在不共享原始数据的情况下协作训练模型，适用于隐私敏感的机器人感知。然而，多智能体系统产生的异构、非独立同分布的多模态传感器流会降低传统联邦学习算法的性能，且经典融合模块（如MLP）引入大量参数和通信开销。本文针对这一问题，提出混合量子-经典个性化联邦学习框架QFedAgent，用于多智能体活动识别。

### 5. 方法概览

- 任务 / 环境: 基于OPPORTUNITY数据集的多智能体活动识别任务，每个受试者作为一个客户端，数据按受试者划分形成自然非独立同分布，包含站立、行走、坐、躺四类活动。
- 变量 / 干预设计: 每个客户端使用双CNN编码器分别处理加速度计和陀螺仪信号，生成嵌入后通过共享的变分量子电路融合层进行跨模态交互建模，融合特征经客户端特定的适配器和分类头输出。联邦训练中仅共享编码器和VQC参数，适配器和分类头保留在本地。
- 评价指标: 平均测试准确率、融合参数数量、每轮训练时间。
- 模型比较: 与Local only、FedAvg、FedProx和MLP-FL（经典MLP融合的个性化FL）四种基线比较。
- 我的理解: 该方法的核心创新在于用变分量子电路替代经典MLP进行多模态融合，利用量子纠缠特性以极少的参数捕获跨模态相关性，同时通过个性化设计（本地适配器和分类头）适应客户端数据分布差异，在联邦聚合中保持全局共享表示。

### 6. 关键发现

- 量子融合实现参数高效与高精度: QFedAgent仅使用72个量子旋转参数（总融合模块3144参数），相比MLP-FL的33K参数减少约10倍，但平均测试准确率最高（97.7%），略优于MLP-FL（97.2%）。
- 个性化联邦学习优于非个性化方法: 在非独立同分布设置下，QFedAgent和MLP-FL的准确率（97.7%和97.2%）均高于FedAvg（96.6%）和Local only（96.3%），表明个性化设计有效缓解了数据异构性。
- VQC配置存在最优容量: 消融实验显示，随着量子比特数N和纠缠层数L增加，准确率提升至N=8、L=3后趋于饱和，进一步增加反而使优化困难，表明适度电路宽度和深度足以建模多模态融合。

### 10. 对 Agent / Skill / Memory 的启发

- 变量 / 干预设计: 六阶段管线：广度（语料主题挖掘）、深度（研究方向细化）、试点（文献复现与迭代验证）、预生产、生产、写作。关键干预：试点阶段强制数值对比（计算值与文献值），并采用新鲜上下文隔离、分布式接地、对抗性审查三种容错机制。
- 评价指标: 四等级判定（T1-T4）评估复现结果与文献值的一致性；人工后验审查；故障捕捉事件计数。
- 模型比较: 与AI Scientist v1/v2、Agent Laboratory、PhysMaster等框架对比，这些框架依赖执行反馈或给定参考值，无法直接应用于本领域；本文以预架构基线和无试点消融作为域内无脚手架对比。
- 我的理解: 该管线通过将文献接地操作化为校准检查点的数值对比，而非简单引用，从而在缺乏执行反馈的领域实现可验证的自主研究。

### 6. 关键发现

- 端到端自主研究管线成功生成出版级论文: 从11,083篇arXiv论文语料出发，经47个新鲜上下文会话、6个阶段，自主产生包含三个实质性物理发现的投稿级手稿（关于交变磁体压磁性）。
- 结构化数值锚定是文献接地的关键机制: 通过两个配对失败模式（预架构基线接受不可锚定主题；无试点消融拥有锚定值但不进行数值对比）隔离出试点阶段的结构化数值对比是操作化文献接地的核心。
- 容错机制有效捕捉错误: 新鲜上下文隔离、分布式接地、对抗性审查三种机制在试点阶段记录了15次故障捕捉事件，例如MnTe轨道磁化初始高估2.44倍被后续对抗性审查纠正。

### 11. 局限性

- 试点阶段仍需有限人工干预（仅限复现失败时的操作知识整理，非科学方向指导）。
- 仅在一个领域（计算凝聚态物理）验证，泛化性待检验。
- 管线依赖公开可用的工具链和文献锚点，对于工具链不成熟或锚点稀缺的领域可能不适用。
