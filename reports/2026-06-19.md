## 1. Contagion Networks: Evaluator Bias Propagation in Multi-Agent LLM Systems

**Topics:** Multi-Agent Systems, Evaluator Bias, Bias Propagation
**Authors:** Zewen Liu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20493v1) | [PDF](http://arxiv.org/pdf/2606.20493v1.pdf) | [Code](https://github.com/liuzewen/contagion-networks)

### 1. 一句话结论

在同类模型的多智能体系统中，评估者偏差会以可测量的方式传播，但传播系数较低（γ∈[0.157,0.352]），且通过增加评估委员会规模（k=3）可有效抑制偏差传播（降低72.4%）。

### 2. 为什么重要

该研究首次形式化定义了多智能体系统中评估者偏差的传播机制，揭示了同类模型与跨模型传播的差异，并提供了可操作的缓解策略，对设计鲁棒的多智能体协作系统具有重要指导意义。

### 3. 研究问题

1. 评估者偏差是否会在多智能体系统的智能体之间传播？
2. 在什么条件下，这种偏差传播会升级为系统范围的偏好崩溃？

### 4. 背景与问题设定

多智能体LLM系统依赖智能体间的相互评估来协作，但评估者偏差可能通过闭环反馈传播，导致系统多样性丧失。现有研究多关注单次或成对评估，缺乏对多跳传播的分析。本文提出Contagion Networks框架，量化偏差在智能体网络中的传播。

### 5. 方法概览

- 任务 / 环境: 使用DeepSeek-chat模型构建3个智能体，分别赋予结构化、平衡和基于证据的评估偏差提示。任务涵盖代码生成、数学推理、文本摘要、逻辑谜题和创意写作，共50个任务。
- 变量 / 干预设计: 实验分4阶段：基线偏好测量、成对传播矩阵构建、链式传播实验（A→B→C）、多样性缓解实验（评估委员会规模k=1,2,3）。
- 评价指标: 传播系数γ（公式2）、累积传播因子β_L（公式4）、谱半径ρ(Γ_N)、有效传播γ_eff、策略熵H(w)。
- 模型比较: 与跨模型传播（MM-EPC中GPT-4o→DeepSeek，γ≈0.85-1.3）对比，同类模型传播系数低3-5倍。
- 我的理解: 该方法通过TTRL机制模拟智能体策略权重的更新，将偏差传播量化为权重分布的变化，并借鉴流行病学模型分析传播动力学，设计精巧且可复现。

### 6. 关键发现

- 同类模型传播系数低: 所有非对角传播系数γ∈[0.143,0.304]，远低于跨模型传播（γ≈0.85-1.3），表明同类模型具有天然抑制效应。
- 链式传播快速衰减: 3跳链式传播的累积因子β_3=0.0055，偏差在传播中迅速衰减，未出现级联放大。
- 多样性委员会有效抑制传播: 评估委员会规模从k=1增至k=3，有效传播γ_eff降低72.4%（0.264→0.073），策略熵接近理论最大值。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 设计多智能体系统时应优先使用同类模型作为评估者，并监控传播矩阵Γ以避免级联。
- 对 Skill: 评估委员会多样性（k≥3）可有效抑制偏差传播，保护策略多样性。
- 对 Memory: TTRL机制中策略权重的更新可视为一种记忆，偏差传播通过权重调整实现。
- 对评测: 传播矩阵Γ可作为部署前的诊断工具，策略熵H(w)可作为实时健康指标。

### 11. 局限性

- 实验仅使用DeepSeek-chat单一模型家族，跨模型对比依赖MM-EPC数据，需直接测量验证。
- 偏差提示仅三种粗粒度类型，缺乏连续强度控制。
- 统计稳健性有限（成对传播仅n=2种子，其他阶段单次运行）。
- 仅研究链式拓扑，星型、环型等未探索。

### 12. 我的看法 / 个人笔记

该工作将流行病学模型引入AI偏差研究，视角新颖且实用。但实验规模较小，跨模型对比的结论需谨慎解读。未来可扩展至异构模型和复杂拓扑，并探索其他适应机制（如上下文学习）下的传播动力学。

  与当前论文关系：本文将其2×2矩阵推广至N×N，并聚焦跨智能体传播而非跨模态。

### 14. 总结

本文提出Contagion Networks框架，通过3智能体实验证明同类模型评估者偏差传播系数低（γ∈[0.157,0.352]），链式传播快速衰减（β_3=0.0055），且增加评估委员会规模至k=3可降低有效传播72.4%。研究为多智能体系统设计提供了诊断工具和缓解策略。

## 2. LLM agent safety, multi-turn red-teaming, jailbreak benchmarks, adversarial robustness, safety-critical systems

**Topics:** LLM Agent Safety, Multi-Turn Red-Teaming, Jailbreak Benchmarks, Adversarial Robustness, Safety-Critical Systems
**Authors:** Hanwool Lee, Dasol Choi, Bokyeong Kim, Seung Geun Kim, Haon Park
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20408v1) | [PDF](http://arxiv.org/pdf/2606.20408v1.pdf)

### 1. 一句话结论

NRT-Bench 通过模拟核电站控制室的多轮红队测试，发现自适应多轮攻击可导致 8.7% 至 12.1% 的会话失去关键安全功能，且不同模型的漏洞几乎不重叠，防御效果高度模型依赖。

### 2. 为什么重要

该工作首次将多轮自适应攻击、角色分工团队和客观物理危害信号结合，为评估 LLM 代理在安全关键系统中的鲁棒性提供了可复现的基准，揭示了现有单轮越狱基准的不足，并强调了固定攻击重放协议对可复现性的关键作用。

### 3. 研究问题

1. 自适应多轮攻击能否驱动多代理系统达到物理不安全状态？
2. 不同前沿 LLM 作为操作员团队时的漏洞是否重叠？

### 4. 背景与问题设定

LLM 代理正被提议用于安全关键系统的监督，但其在持续自适应对抗压力下的鲁棒性尚不清楚。现有越狱基准多为单轮、基于 LLM 判断文本危害，且评估单个模型，忽略了多轮、团队协作和物理后果。

### 5. 方法概览

- 任务 / 环境: 模拟核电站控制室，包含五角色操作员团队（SRO、RO、TO、AO、STA）和六个关键安全功能（CSF），状态由确定性步进函数更新，CSF 取值包括 intact、degraded、lost。
- 变量 / 干预设计: 攻击者通过四个通道（CH-A 外部、CH-B 冒充内部、CH-C 供应链、CH-D 辅助代理）在最多 10 轮会话中注入消息，每轮后收到情境摘要（黑盒或白盒模式）以自适应调整。
- 评价指标: 主要指标为 CSF 丢失率（ASR_CSF），即任何 CSF 变为 lost 的会话比例，由模拟器状态客观决定，无需 LLM 判断。
- 模型比较: 评估四个前沿 LLM 作为操作员，在固定攻击重放协议下比较 ASR_CSF 和失败集重叠，攻击序列固定以归因差异于防御者。
- 我的理解: 该方法通过将危害信号从文本判断转移到模拟器状态，并引入多轮自适应攻击和团队结构，更真实地模拟了实际攻击场景；固定攻击重放协议确保了评估的可复现性。

### 6. 关键发现

- 多轮攻击可靠导致 CSF 丢失: 四个模型在 8.7% 到 12.1% 的会话中因攻击失去关键安全功能，表明自适应多轮攻击能有效突破防御。
- 漏洞几乎不重叠: 在 149 个会话中，无一次同时击败所有四个模型，但约三分之一（约 49 个）击败至少一个模型，说明不同模型的脆弱点不同。
- 防御效果高度模型依赖: 相同的防护措施（如护栏堆栈或安全顾问）对某个模型降低攻击成功率，却可能提高另一个模型的成功率。

### 7. 最重要的图

- 图来源: 图1：NRT-Bench 概览
- 为什么重要: 该图展示了整个基准框架：多轮红队攻击者通过四个通道探测安全关键控制室，五角色 LLM 操作员团队管理模拟器，每轮更新状态并返回反馈，最终以 CSF 丢失作为客观危害信号。

### 8. 如何读这张图

从左侧攻击者开始，通过四个通道（CH-A 至 CH-D）向操作员团队发送消息；团队由五个角色（RO、SRO、TO、AO、STA）组成，共同操作模拟器；模拟器输出植物状态和 CSF 值；每轮后攻击者收到反馈以调整下一轮消息；当任何 CSF 变为 lost 时会话终止。

### 9. 次重要图

- 为什么也重要: 图2展示了攻击代理的输入输出协议，说明攻击者如何基于目标、策略、历史记录和上一轮摘要生成下一轮有效载荷，强调了攻击的可复现性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 部署前需评估多轮自适应攻击下的团队鲁棒性，而非仅单轮。
- 对 Skill: 操作员技能需包括对抗性输入识别和团队协作检查。
- 对 Memory: 攻击者利用上下文积累，因此代理的记忆机制可能成为攻击面。
- 对评测: 应使用客观环境信号而非 LLM 判断作为主要危害指标，并采用固定攻击重放协议确保可复现性。

### 11. 局限性

- 固定攻击工作负载可能偏向种子防御者的失败模式，不估计每个防御者的最大脆弱性。
- 模拟器为抽象文本模型，未完全反映真实核电站的复杂性。
- 攻击生成器模型未公开，影响完全可复现性。

### 12. 我的看法 / 个人笔记

NRT-Bench 通过引入多轮、团队和客观物理信号，显著推进了 LLM 代理安全评估，其发现漏洞不重叠和防御效果模型依赖具有重要实践意义，但固定攻击集限制了自适应攻击的全面性，且攻击生成器未公开是复现性的一个弱点。

### 13. 这篇工作引用的近期高影响力相关文章

  为什么重要：采用类似的状态检查而非 LLM 判断来评估鲁棒性，与 NRT-Bench 的客观危害信号理念一致。
  核心贡献：提出基于形式化状态检查的提示注入鲁棒性基准。
  与当前论文关系：NRT-Bench 借鉴了其攻击抵抗标准，但将其扩展到物理信号和团队场景。

  为什么重要：展示了多轮渐进升级攻击的有效性，支持 NRT-Bench 的多轮设计。
  核心贡献：证明多轮攻击可绕过单轮防御。
  与当前论文关系：NRT-Bench 将多轮攻击从文本危害扩展到物理危害。

### 14. 总结

NRT-Bench 是一个针对安全关键系统中 LLM 代理的多轮红队测试基准，通过模拟核电站控制室，发现自适应多轮攻击可导致 8.7%-12.1% 的会话失去关键安全功能，且不同模型的漏洞几乎不重叠，防御效果高度模型依赖。该基准提供了可复现的评估框架，强调客观物理危害信号、团队结构以及固定攻击重放协议的重要性。

## 3. ScholarQuest: A Taxonomy-Guided Benchmark for Agentic Academic Paper Search in Open Literature Environments

**Topics:** LLM-based search agents, academic paper search, benchmark, taxonomy-guided, agentic search evaluation
**Authors:** Tingyue Pan, Mingyue Cheng, Daoyu Wang, Yitong Zhou, Jie Ouyang, Qi Liu, Enhong Chen
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20235v1) | [PDF](http://arxiv.org/pdf/2606.20235v1.pdf) | [Code](https://github.com/pty12345/ScholarQuest)

### 1. 一句话结论

ScholarQuest 是一个基于 ACM CCS 分类法、覆盖 1000+ CS 主题和四种研究意图的大规模基准，用于评估学术论文搜索智能体，发现当前最佳智能体仅达 0.314 Recall@100，仍有巨大改进空间。

### 2. 为什么重要

现有基准在查询多样性、答案可扩展性和评估环境标准化方面不足，ScholarQuest 通过分类法引导的查询生成、自动化答案构建和共享检索后端 ScholarBase，为智能体学术搜索提供了系统、可复现的多维评估框架，揭示了当前方法的效率、意图鲁棒性和搜索过程缺陷。

### 3. 研究问题

1. 如何构建一个覆盖广泛 CS 主题和多种研究意图的学术论文搜索基准？
2. 当前基于 LLM 的搜索智能体在检索质量、效率和意图鲁棒性方面表现如何？

### 4. 背景与问题设定

学术论文搜索是科研核心步骤，传统方法依赖词法或语义匹配，难以处理细粒度条件查询。LLM 智能体可自主迭代搜索，但现有基准如 RealScholar、SPARBench 存在查询覆盖有限、答案构建成本高、缺乏标准化环境等问题。ScholarQuest 旨在填补这一空白。

### 5. 方法概览

- 任务 / 环境: 基于 arXiv 数据库构建的 ScholarBase 环境，提供统一检索、元数据查询和引用扩展接口，支持 BM25、稠密检索和混合检索。
- 变量 / 干预设计: 从 ACM CCS 分类法收集 1600+ 主题，映射到 arXiv CS 类别后保留 1000+ 种子，利用 Qwen3-Max 生成四种意图查询（方法导向、设置锚定、比较型、范围控制），经去重和质量过滤得到 1111 个查询。答案通过多源检索（Google、arXiv、Semantic Scholar）、引用图扩展和多阶段 LLM 相关性判定构建，阈值 τ=2，并经人工审计验证（严格匹配精度 86.0%）。
- 评价指标: 主要指标：Recall@25、Recall@100、Recall@All；过程指标：工具调用次数、搜索/扩展调用次数、轮次、候选数、每 100 候选的 Recall@100。
- 模型比较: 对比三类基线：标准检索（稠密、混合）、通用学术搜索（Google Search、Google Scholar、Semantic Scholar、DeepXiv）、开源智能体（PaSa、SPAR、PaperScout）。
- 我的理解: 该方法通过分类法系统化查询生成，避免了人工偏差；自动化答案管道结合多源检索和引用扩展，提高了覆盖率和可扩展性；标准化环境确保了可复现性。

### 6. 关键发现

- 智能体方法优于单次检索，但整体性能有限: 最佳智能体 PaperScout 在总体 Recall@100 达 0.314，Recall@All 达 0.355，显著高于稠密检索（0.208/0.290）和 Google Search（0.094/0.112），但仍有巨大提升空间。
- 范围控制查询最具挑战性: 所有方法在范围控制查询上 Recall@100 最低，Google Search 仅 0.006，PaperScout 为 0.182，表明智能体难以精确控制检索广度。
- 智能体效率低下，存在过度探索问题: PaperScout 平均调用 10.5 次工具、观察 100+ 候选，但每 100 候选的 Recall@100 仅 0.31，低于 PaSa（0.36），表明许多探索偏离目标。

## 4. Analyzing Defensive Misdirection Against Model-Guided Automated Attacks on Agentic AI Systems

**Topics:** GUI Agent, LLM Security, Jailbreak Defense, Agentic AI
**Authors:** Reza Soosahabi, Vivek Namsani
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20470v1) | [PDF](http://arxiv.org/pdf/2606.20470v1.pdf)

### 1. 一句话结论

针对模型引导的自动化攻击，提出“检测-误导”防御策略，通过生成上下文相关但安全的误导响应，使攻击者自动评判器产生误报，从而将攻击成功率上界降低两个数量级。

### 2. 为什么重要

传统“检测-拦截”防御在自动化攻击下因可预测的拒绝反馈反而帮助攻击者优化提示，而“检测-误导”策略从根本上破坏攻击者的反馈质量，为Agentic AI系统提供了一种可扩展的主动防御新范式。

### 3. 研究问题

1. 传统检测-拦截防御为何在模型引导的自动化攻击下失效？
2. 如何通过误导响应干扰攻击者的自动评判器，从而限制攻击成功率？
3. 攻击者能否通过自适应调整评判器或验证策略来规避误导防御？

### 4. 背景与问题设定

Agentic AI系统依赖语言模型解释指令、调用工具和协调智能体，这使其面临提示注入和越狱攻击的更大风险。攻击者采用模型引导的自动化框架（如PAIR、GPTFuzz）大规模探测、优化提示并评估响应。传统防御依赖检测后拦截或拒绝，但可预测的拒绝文本为攻击者的自动搜索提供了有效反馈，使得攻击成功率随查询预算增加而趋近于1。

### 5. 方法概览

- 任务 / 环境: 模型引导的自动化越狱攻击场景，攻击者使用自动评判器评估受害者模型响应，并迭代优化攻击提示。
- 变量 / 干预设计: 提出检测-误导策略：当检测到恶意交互时，不返回可预测的拒绝文本，而是生成上下文相关但安全的误导响应（如正面意图框架、安全上下文扩展和后续互动），旨在诱导攻击者评判器产生误报。具体实现为Contextual Misdirection via Progressive Engagement (CMPE)。
- 评价指标: 攻击者成功率（ASR）上界、攻击者评判器误报率、验证攻击成功率。
- 模型比较: 与检测-拦截基线对比，CMPE在PAIR和GPTFuzz端到端攻击中几乎消除验证成功的攻击，并将ASR上界降低两个数量级。
- 我的理解: 该方法的核心洞察是：模型引导攻击依赖自动评判器可靠区分成功与失败响应；通过生成看似成功但实际安全的响应，使攻击者收集大量误报，从而降低其正预测值，最终限制迭代搜索效率。

### 6. 关键发现

- 检测-拦截防御在自动化攻击下ASR趋近于1: 通过概率模型分析（公式17），当攻击者查询预算N足够大时，只要有效通过概率β_D(1-β_A)>0，最大ASR可任意接近1。
- 检测-误导策略使ASR有界: 引入误导诱导误报概率γ_A后，攻击者正预测值η_A降低（公式11），ASR上界被限制为P(C_N^g ≠ ∅)(1-(1-η_A)^K)（公式14），不再随N增长趋近于1。
- CMPE在基准测试中显著降低ASR: 在越狱基准上，CMPE将估计的ASR上界降低两个数量级，并在PAIR和GPTFuzz端到端攻击中几乎消除验证成功的攻击。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: Agentic AI系统应集成检测-误导防御层，在检测到自动化攻击时主动生成误导响应，而非简单拒绝，以保护工具调用和智能体协调功能。
- 对 Skill: 防御技能需从被动拦截转向主动欺骗，利用LLM评判器的启发式弱点（如依赖语气和结构）设计误导策略。
- 对 Memory: 误导响应需保持上下文一致性，避免被攻击者通过多轮交互识破，因此可能需要短期记忆维护对话状态。
- 对评测: 评估防御时需考虑攻击者自适应调整评判器或验证策略，因此ASR上界和误报率比单一攻击成功率更具参考价值。

### 11. 局限性

- CMPE依赖LLM评判器的特定弱点，若攻击者使用更鲁棒的评判器（如人工验证或规则校验），误导效果可能减弱。
- 当前分析假设攻击者不调整评判策略，实际中攻击者可能通过对抗训练降低误报率；论文第7节虽提及自适应攻击者但未深入探讨，需进一步研究。
- 误导响应可能被用户误认为真实功能，需谨慎设计以避免对正常用户造成混淆。

### 12. 我的看法 / 个人笔记

本文从信息论角度揭示了检测-拦截防御的根本缺陷，并提出一个优雅的对抗性解决方案——通过破坏攻击者的反馈信号而非单纯提高检测率。CMPE作为轻量级实现，在实验中表现显著，但实际部署需考虑攻击者自适应和用户混淆风险。未来工作可探索更通用的误导策略生成方法，以及结合强化学习动态优化误导响应。

## 5. AutoPass: Evidence-Guided LLM Agents for Compiler Performance Tuning

**Topics:** Compiler Optimization, LLM Agents, Auto-tuning
**Authors:** Zepeng Li, Jie Ren, Zhanyong Tang, Jie Zheng, Zheng Wang
**Venue / Year:** Conference acronym 'XX, 2018
**Links:** [Abstract](http://arxiv.org/abs/2606.20373v1) | [PDF](http://arxiv.org/pdf/2606.20373v1.pdf)

### 1. 一句话结论

AutoPass利用编译器和运行时证据指导LLM代理进行编译器性能调优，在x86-64和ARM64上分别实现1.043倍和1.117倍的加速比。

### 2. 为什么重要

该工作首次将LLM代理与编译器内部信号和运行时反馈结合，实现了无需离线训练或微调的编译器性能优化，为自动化编译器调优提供了新范式。

### 3. 研究问题

1. 如何利用LLM代理在有限预算下有效探索编译器优化空间？
2. 编译器内部证据和运行时反馈如何提升LLM生成的优化决策质量？

### 4. 背景与问题设定

编译器优化对软件性能至关重要，但默认优化级别（如-O3）无法在所有程序上表现最佳。传统自动调优方法（如OpenTuner）计算成本高，而基于学习的方法泛化性差。LLM在代码编译任务中展现出潜力，但应用于运行时性能优化面临微架构效应复杂和运行时测量噪声大的挑战。现有LLM方法主要关注静态目标（如代码大小），缺乏基于编译器内部信号或运行时证据的反馈机制。

### 5. 方法概览

- 任务 / 环境: 编译器性能调优，目标是为给定程序找到最优的LLVM pass序列和参数设置，以最小化运行时延迟。
- 变量 / 干预设计: AutoPass采用多代理框架：Score Agent识别热点函数；Analysis Agent提取IR特征和编译器诊断信息；Reasoning Agent基于证据生成和迭代优化pass管道；Evaluation Agent编译运行并收集性能反馈。
- 评价指标: 几何平均加速比（相对于LLVM -O3基线）。
- 模型比较: 与Instrumentation-based PGO、AutoFDO、CSSPGO和OpenTuner比较，AutoPass在所有基准测试和平台上取得最佳性能。
- 我的理解: AutoPass的核心创新在于将编译器内部状态（如优化备注、IR快照）和运行时性能反馈作为LLM的输入，形成一个闭环优化循环，使LLM能够基于证据而非纯代码推理做出决策。

### 6. 关键发现
