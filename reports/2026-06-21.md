## 1. QMFOL: Benchmarking Large Language Model Reasoning via Quantifiable Monadic First-Order Logic Test Case Generation

**Topics:** Deductive Reasoning, Benchmarking, Large Language Models, First-Order Logic
**Authors:** Xinyi Zheng, Ling Shi, Tianlong Yu, Yongxin Zhao, Lorenz Goette, Kailong Wang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20227v1) | [PDF](http://arxiv.org/pdf/2606.20227v1.pdf)

### 1. 一句话结论

QMFOL generates monadic first-order logic reasoning tasks with quantifiable complexity, enabling precise evaluation of LLM deductive reasoning across multiple dimensions.

### 2. 为什么重要

Existing benchmarks lack fine-grained control over logical complexity and struggle to balance semantic diversity with logical consistency. QMFOL addresses these gaps, providing a scalable and reliable framework for constructing deductive reasoning benchmarks with controllable complexity, which is crucial for high-stakes decision-making domains like law and healthcare.

### 3. 研究问题

1. How can we generate deductive reasoning tasks with quantifiable and controllable complexity across multiple dimensions (depth, width, label type, distractors)?
2. How do LLMs and LRMs perform under varying logical complexity and semantic variation?

### 4. 背景与问题设定

Large Language Models (LLMs) have made significant progress in deductive reasoning, which is essential for high-stakes decision-making. However, existing benchmarks lack fine-grained control over logical complexity and struggle to balance semantic diversity with logical consistency. This paper proposes QMFOL, an automated framework for generating monadic first-order logic (MFOL) reasoning tasks with quantifiable and controllable complexity, enabling more precise evaluation of reasoning capabilities in modern language models.

### 5. 方法概览

- 任务 / 环境: The framework generates MFOL reasoning tasks with configurable depth (5,10,15,20), width (5,10,15,20), label type (True/False/Unknown), number of distractors (0,5,10,15,20), and topic (Food, Animal, University, Mathematics).
- 变量 / 干预设计: The logic construction module builds basic rule sets using conjunction and disjunction patterns, then derives fact-conclusion pairs with specified labels. Distractor rules are added to increase difficulty. The FOL2NL module translates formal logic into natural language via LLMs, with round-trip verification using an external prover to ensure logical consistency.
- 评价指标: Accuracy and computational overhead (cost) are used to evaluate model performance.
- 模型比较: Six Large Reasoning Models (LRMs) and two LLMs are evaluated, including Gemini-3.1-Pro, GPT-5.4-High, Qwen3.5-27B, DeepSeek-V3.2-Thinking, and Claude-Sonnet-4-6.
- 我的理解: The method systematically controls logical complexity by separately expanding depth and width, and introduces distractors to increase difficulty. The round-trip verification ensures that natural language translations preserve logical consistency, addressing a key limitation of prior work.

### 6. 关键发现

- Performance degrades with increasing logical complexity: As depth and width increase, model accuracy decreases and computational overhead increases. This trend is consistent across all evaluated models.
- Models perform better on True-labeled tasks than on False or Unknown: Accuracy on True-labeled tasks is higher than on False or Unknown tasks, with many errors stemming from insufficient reasoning.
- Accuracy varies across topics under identical logical structures: Performance differs across topics (Food, Animal, University, Mathematics) even when logical structures are identical, indicating semantic dependence.

Future work could extend to polyadic predicates and explore more diverse semantic domains.

## 2. TimeProVe: Propose, then Verify for Efficient Long Video Temporal Reasoning in Activities of Daily Living

**Topics:** Long Video Question Answering, Vision-Language Models, Temporal Action Detection, Activities of Daily Living
**Authors:** Arkaprava Sinha, Dominick Reilly, Siddharth Krishnan, Hieu Le, Srijan Das
**Venue / Year:** arXiv:2606.20561v1
**Links:** [Abstract](http://arxiv.org/abs/2606.20561v1) | [PDF](http://arxiv.org/pdf/2606.20561v1.pdf) | [Code](https://thearkaprava.github.io/timeprove/)

### 1. 一句话结论

TimeProVe通过轻量级动作检测模块（ACE）生成查询相关的候选答案与证据窗口，再仅对少量选中的短片段调用大模型VLM进行验证，在专注于日常活动的开放式长视频问答基准OTB上以7.3%的性能提升，同时将VLM调用和推理成本分别降低75%和93%。

### 2. 为什么重要

该框架解决了长视频问答中高计算开销与细粒度证据缺失的矛盾——全视频VLM推理因令牌量巨大（如60分钟视频约需260万视觉令牌）而不可扩展，而纯字幕方法又难以保留动作细节。TimeProVe将探索与验证分离，使轻量级边缘设备完成大部分计算，云端VLM仅验证短片段，为实际低延迟高精度ADL应用铺平道路。

### 3. 研究问题

1. 如何在不处理完整长视频的情况下，准确回答涉及稀疏细粒度动作的ADL问题？
2. 能否设计一种混合架构：先用轻量级模块生成动作基假设，再让大模型仅作关键验证？
3. 现有LVQA基准（如VideoMME、MVBench）多为多项选择且不评价时间证据，如何构建开放式时间接地评估基准？

### 4. 背景与问题设定

长视频问答（LVQA）需要在数小时未裁剪的视频中找出少量查询相关证据。以60分钟视频按1 FPS采样、SigLIP编码器为例，会产生约260万视觉令牌，超过多数模型上下文限制且云端推理代价高昂。现有方法要么密集处理整段视频（高成本），要么依赖稀疏字幕（丢失时序细节如手物交互与细微动作）。作者提出TimeProVe，通过在边缘设备上轻量级定位动作并生成候选证据，再由云端VLM仅验证短片段，实现成本与精度的平衡。

### 5. 方法概览

- 任务 / 环境: 长视频ADL问答（LVQA），视频时长可达1小时以上，回答需要精确到数秒的动作证据；评估使用自建OTB基准（开放式问答，含1300余个问题）以及Charades-STA时间接地任务。
- 变量 / 干预设计: 先使用轻量级动作检测器（基于I3D或CLIP特征）对整视频单次处理，输出动作时间线A={(c_i, s_i, e_i)}；再通过边缘LLM（如轻量级LLM）根据查询q和时间线A生成候选答案与证据窗口（含原子窗和合并窗）；通过四准则排序（时间兼容性、语义相关性、覆盖度、长度惩罚）对候选排序；仅对排名最高的短片段（通常数秒）调用云端VLM（如GPT-4V或Qwen2-VL）进行验证，若确认则返回答案与语义/视觉证据，否则继续下一候选。
- 评价指标: OTB上报告开放式答案准确率（与人工标注精确匹配）；VLM调用次数减少百分比；推理成本降低百分比（基于API定价估算）；在Charades-STA上用召回率@K和mIoU评估时间接地性能。
- 模型比较: OTB上对比基线包括：全视频VLM（如VideoLLaMA2、TimeChat）、字幕基LLM（如VideoAgent、LangRepo）、以及帧选择+补全方法。TimeProVe在OTB上以7.3%绝对提升超越最强基线；同时将VLM调用数从数百次降至25次（降低75%），推理成本从$6.74降至$0.46（降低93%）。在Charades-STA上，无需专门训练即达与专用模型（如VidSTG-LLaVA）相当性能，结合时间接地VLM后mIoU达44.3，达到SOTA。
- 我的理解: 这是一种'先提案后验证'的混合范式：局部轻量模型负责粗粒度定位和假设生成，全局大模型仅做细粒度确认。ACE模块中的四准则排序（公式(5)-(8)）平衡了时间合理性、语义匹配、覆盖完整性和窗口紧凑性，是降低VLM调用的关键设计。

### 6. 关键发现

- 大幅降低计算成本且提升性能: 在OTB上，TimeProVe相比最强基线（VideoAgent）实现7.3%准确率提升，同时将VLM调用次数减少75%（从约100次降至25次），推理成本降低93%（从$6.74降至$0.46）。这一收益来自ACE模块的轻量级筛选——仅将最相关的短片段发送至VLM验证。
- ACE模块有效提炼动作证据: Action-based Candidate Evidence (ACE)模块将动作检测器输出的时间线（约200-400个动作实例）通过边缘LLM转化为查询相关的候选答案与证据窗口（平均每查询生成10-20个候选）。其中原子窗保留单动作精度，合并窗捕获跨时间上下文（如联合'阅读'和'喝水'窗口回答'边喝水边做什么'）。排序公式通过四准则确保高质量假设优先。
- 跨任务泛化能力强: 在Charades-STA时间接地任务上，TimeProVe无需任何时间接地训练即达到与专用模型竞争的性能（Recall@1=35.2, mIoU=44.3）。当将ACE中的动作检测器替换为时间接地VLM时，性能进一步提升至SOTA（Recall@1=37.8, mIoU=46.1），展示框架模块化的鲁棒性。

- 对 Agent 系统: 证明了混合智能体架构中将感知规划与验证解耦的有效性：轻量模块（动作检测+边缘LLM）负责探索与假设生成，云端大模型仅作确认。这种设计可推广至其他多步骤推理任务（如对话系统、工具使用）。
- 对 Skill: ACE模块中的提议生成与排序可视为一种元技能——根据查询从动作库中择取并组合最相关的证据窗口。排序公式中的时间兼容性、语义匹配、覆盖与长度惩罚可启发更通用的问题基证据检索。

### 11. 局限性

## 3. Sovereign Execution Brokers: Enforcing Certificate-Bound Authority in Agentic Control Planes

**Topics:** AI安全, 访问控制, 自主代理, 基础设施安全
**Authors:** Jun He, Deying Yu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20520v1) | [PDF](http://arxiv.org/pdf/2606.20520v1.pdf)

### 1. 一句话结论

Sovereign Execution Broker (SEB) 通过证书绑定的运行时强制边界，将自主代理的变异权限从非确定性推理过程中分离，确保只有经过认证、新鲜且可撤销的证书才能触发生产环境变更。

### 2. 为什么重要

随着LLM驱动的自主代理直接操作云基础设施，传统IAM无法防止幻觉或提示注入导致的破坏性操作。SEB提供了一种实用的运行时强制机制，将认证与执行分离，填补了现有安全架构中的关键空白。

### 3. 研究问题

1. 如何设计运行时强制边界，使自主代理无法持有或行使长期生产凭证？
2. 如何将证书认证与实时状态检查（如策略版本、撤销状态、状态漂移）结合，确保变异操作在执行时刻仍然安全？

### 4. 背景与问题设定

自主代理（如LLM驱动）越来越多地集成到云基础设施控制流程中，但其非确定性推理可能导致不安全操作。现有机制中，IAM授权身份，SAB认证提议动作，但两者都未在变异时刻提供强制执行点。代理可能绕过认证直接调用API，或使用过期/撤销证书。SEB旨在填补这一执行间隙。

### 5. 方法概览

- 任务 / 环境: 云基础设施（AWS、Kubernetes）中的自主代理控制平面，代理提议并执行基础设施变异操作（如修改安全组、部署容器）。
- 变量 / 干预设计: SEB作为运行时强制边界，所有变异请求必须通过SEB。SEB验证SAB颁发的证书Ω，执行形式化验证谓词V = Φ_sig ∧ Φ_match ∧ Φ_time ∧ Φ_policy ∧ Φ_rev ∧ Φ_drift ∧ Φ_replay ∧ Scopeable。其中Φ_drift检查实时状态St是否偏离认证时的证据E_admit；Scopeable确认目标平台能强制执行合约约束。通过后生成短期作用域凭证并执行变异。
- 评价指标: 执行延迟（p50/p95/p99）、密码学验证和漂移检测微基准、撤销传播延迟、故障注入（分区、崩溃恢复）下的行为。
- 模型比较: 与IAM（仅身份权限）、OPA（策略谓词）、审计日志（事后记录）、SAB（仅认证证书）对比，SEB是唯一在变异时刻强制证书绑定执行的机制。
- 我的理解: SEB是一个受信任的代理，持有生产凭证但仅在验证完整证书和实时状态后使用。它将授权从“谁”转移到“什么动作被认证”，通过短期凭证和原子性nonce预留防止重放和滥用。

### 6. 关键发现

- SEB执行模型形式化: 定义了Execute(Ω, req, St, Platform) → D|O接口，以及包含8个检查的验证谓词V：签名验证、合约匹配、有效性窗口、策略版本、撤销版本、实时状态漂移、重放检查、平台可作用域性。每个检查失败产生签名决策记录D，成功则生成结果记录O。
- 绕过预防部署模式: 要求目标平台拒绝所有非代理身份的生产变异请求，仅接受代理角色或代理颁发的会话凭证。在AWS中通过SCP和权限边界实现，确保代理运行时无长期凭证。
- 原型评估量化结果: 在AWS和Kubernetes上部署的原型显示：p50执行延迟约50ms，p95约120ms，p99约200ms；密码学验证和漂移检测微基准延迟<10ms；撤销传播延迟<1s；故障注入下行为符合预期，分区时fail-closed。

- 对 Memory: SEB的账本记录了所有决策和结果，可作为代理记忆的一部分，用于审计和回放。
- 对评测: 评估代理安全性时，需要检查是否部署了SEB类似的强制边界，以及证书验证和状态检查的完整性。

### 11. 局限性

- 依赖目标平台拒绝非代理身份，这需要云提供商或编排器的支持，可能不适用于所有环境。
- SEB本身成为单点故障或攻击目标，需要额外的安全加固（如安全enclave）。
- 实时状态漂移检测依赖于证据源的准确性和及时性，可能受限于最终一致性。

### 12. 我的看法 / 个人笔记

SEB提出了一种实用的、工程化的方法来解决自主代理的安全问题，将学术界的认证概念落地为运行时强制机制。其核心思想——将授权从身份转移到认证动作——对于构建可信的自主基础设施至关重要。未来工作可探索更高效的漂移检测和跨平台可作用域性。

### 14. 总结

Sovereign Execution Broker (SEB) 是一个运行时强制边界，通过证书绑定的方式确保自主代理的变异操作只有在经过认证、新鲜且可撤销的情况下才能执行。它填补了认证与执行之间的间隙，通过形式化的验证流水线（包括8个检查谓词）、短期作用域凭证和绕过预防部署模式，显著提升了自主代理控制平面的安全性。原型评估显示p50延迟约50ms，p99约200ms，撤销传播<1s。

## 4. S-Agent: Spatial Tool-Use Elicits Reasoning for Spatial Intelligence

**Topics:** GUI Agent, Spatial Intelligence, Vision-Language Models
**Authors:** Yalun Dai, Hao Li, Shulin Tian, Runmao Yao, Yuhao Dong, Fangzhou Hong, Zhaoxi Chen, Fangfu Liu, Baoliang Tian, Dingwen Zhang, Tao Wang, Kim-Hui Yap, Ziwei Liu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20515v1) | [PDF](http://arxiv.org/pdf/2606.20515v1.pdf) | [Code](https://ropedialab.github.io/S-Agent)

### 1. 一句话结论

S-Agent通过将VLM作为语义规划器，结合层次化空间工具和双记忆机制，将空间推理转化为时空证据累积过程，显著提升多视角和视频空间推理性能。

### 2. 为什么重要

现有VLM和工具增强智能体大多局限于静态、无状态的单帧推理，无法处理连续、演化的3D世界。S-Agent首次提出一种状态化的时空证据累积范式，使智能体能够跨视角和时间整合空间证据，为真实世界空间智能（如机器人、AR/VR、自动驾驶）提供了可扩展的解决方案。

### 3. 研究问题

1. 如何将连续多视角图像和视频中的空间推理从孤立帧级预测转变为场景级时空证据累积？
2. 层次化空间工具和双记忆机制如何协同提升VLM的空间推理能力？

### 4. 背景与问题设定

真实世界空间智能需要对连续演化的3D世界进行推理，但现有VLM和工具增强智能体主要依赖静态、无状态的单帧推理，缺乏跨视角和时间整合证据的能力。这导致语义-几何鸿沟：VLM擅长概率性语义推理，但难以忠实捕获高保真几何信息。

### 5. 方法概览

- 任务 / 环境: 多视角图像和视频空间推理任务，包括物体计数、测量、方向判断、相对位置等。
- 变量 / 干预设计: 将VLM作为语义规划器，决定需要何种证据；层次化空间工具（2D感知、2D到3D提升、空间知识聚合）提供场景特定证据；双记忆系统（场景记忆和智能体记忆）维护演化的场景状态和推理历史。
- 评价指标: 在MMSI-Bench、ViewSpatial-Bench、ReVSI、VSI-SUPER等基准上的准确率。
- 模型比较: 与Qwen3-VL-8B、GPT-5.4、Gemini 3 Pro等开源和闭源VLM对比，S-Agent在零样本和微调设置下均显著提升性能。
- 我的理解: S-Agent的核心创新在于将空间推理分解为语义规划和证据获取两个独立过程：VLM负责高层决策，工具负责具体感知和计算，记忆负责跨步骤的证据整合。这种分离使得VLM无需隐式内化整个3D推理过程，而是通过显式工具和记忆实现状态化的时空推理。

### 6. 关键发现

- 零样本设置下显著提升VLM空间推理能力: 直接应用S-Agent框架，GPT-5.4在MMSI-Bench上提升4.5%，Gemini 3 Pro提升1.2%。
- 微调后紧凑智能体S-Agent-8B性能媲美闭源模型: 在S-300K数据集上微调Qwen3-VL-8B，MMSI-Bench准确率从31.1%提升至41.6%（+10.5%），与GPT-5.4和Gemini 3 Pro相当。
- 层次化工具和双记忆机制是性能提升的关键: 2D感知、2D到3D提升、空间知识聚合三级工具逐步将原始2D观测转化为显式3D场景知识；场景记忆和智能体记忆分别维护可复用的场景证据和推理过程上下文，避免冗余和矛盾。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: S-Agent提供了一种可扩展的范式，将VLM作为语义规划器，工具作为证据提供者，记忆作为状态载体，可推广到其他需要持续感知和推理的智能体系统。

- 对 Memory: 双记忆机制（场景记忆和智能体记忆）分离了场景证据和推理过程，为长期推理和错误恢复提供了结构化支持。
- 对评测: S-Agent在多个基准上的显著提升表明，评估空间智能时应考虑时空证据累积能力，而非仅关注单帧推理。

### 11. 局限性

- 蒸馏数据集S-300K仅来自GPT-5.4，可能引入教师模型的偏差。

### 12. 我的看法 / 个人笔记

S-Agent通过将空间推理重新定义为时空证据累积，巧妙地将VLM的语义规划能力与工具的结构化感知能力结合，是迈向真实世界空间智能的重要一步。其训练-推理分离的设计使得零样本提升和蒸馏训练均有效，具有实用价值。未来可探索更鲁棒的工具集成和动态场景适应。

## 5. CRAX: Fast Safe Reinforcement Learning Benchmarking

**Topics:** Safe Reinforcement Learning, Hardware-Accelerated Benchmarking, Constrained MDP
**Authors:** Tristan Tomilin, Mourad Boustani, Mickey Beurskens, Thiago D. Simão
**Venue / Year:** arXiv preprint arXiv:2606.20376v1, 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20376v1) | [PDF](http://arxiv.org/pdf/2606.20376v1.pdf)

### 1. 一句话结论

CRAX 基于 JAX 和 MuJoCo XLA 实现硬件加速，相比 CPU 基准（如 Safety Gymnasium）实现约 100 倍速度提升，并揭示无单一安全 RL 算法在所有任务中占优，课程学习效果因环境而异。

### 2. 为什么重要

现有安全 RL 基准（如 Safety Gymnasium）依赖 CPU 模拟，速度慢，限制了大规模实验和快速原型开发。CRAX 通过 GPU 加速解决了这一瓶颈，使研究人员能够高效进行超参数调优、统计显著性测试和消融研究，从而加速安全 RL 领域的创新。

### 3. 研究问题

1. 如何利用 JAX 和 MuJoCo XLA 构建硬件加速的安全 RL 基准，并实现显著速度提升？
2. 七种安全 RL 算法（PPO、PPOCost、PPOLag、PPOPID、PPOSauté、P3O、FOCOPS）在多种任务和三个难度级别上的性能与安全性权衡如何？
3. 课程学习（从简单到困难任务顺序训练）和安全迁移（从无约束 PPO 策略初始化）能否提升在困难任务上的训练效果？

### 4. 背景与问题设定

安全是强化学习在机器人、自动驾驶等真实世界部署的核心问题。现有安全基准（如 Safety Gymnasium）依赖 CPU 模拟，速度慢，限制了大规模实验。CRAX 基于 MuJoCo XLA（MJX）和 JAX，利用向量化操作和硬件加速，提供快速、高保真 3D 物理模拟，包含六个环境套件（Safe Goal、Safe Push、Safe Circle、Safe Reacher、Safe Spider、Safe Pathway）和三个智能体特定任务（Safe Velocity、Safe Height、Safe Spider），每个任务有三个难度级别。

### 5. 方法概览

- 任务 / 环境: CRAX 包含六个环境套件和三个智能体特定任务，每个任务有三个难度级别。智能体包括 Ant、Humanoid、Spider、Half Cheetah、Walker2D、Hopper、Reacher 等，配备激光雷达传感器。成本信号基于五种约束类型：危险物接近/接触、速度阈值、高度约束、接触限制、目标导向安全。
- 变量 / 干预设计: 每个任务定义奖励和成本信号，形成性能与安全性的权衡。成本阈值可调，以改变任务难度。实验设置：500M 环境步，5 个随机种子，在单个 NVIDIA H100 GPU 和 72 核 AMD EPYC CPU 上运行。
- 评价指标: 主要指标包括奖励（Reward）和成本（Cost），以及安全性百分比（Safe%），即成本低于阈值（25）的环境比例。算法胜场数（Wins）定义为在安全前提下获得最高奖励的环境数。
- 模型比较: 评估了七种基线算法：PPO（无约束）、PPOCost（成本作为负奖励）、PPOLag（拉格朗日法）、PPOPID（PID 控制器更新乘子）、PPOSauté（状态增强）、P3O（渐进惩罚）、FOCOPS（信任域约束）。
- 我的理解: CRAX 通过将环境模拟和算法训练都放在 GPU 上，实现了端到端的硬件加速，这是其速度提升的关键。其设计借鉴了 Brax 和 Safety Gymnasium，但专注于安全约束，并提供了难度递进的任务，便于评估算法的泛化能力。

### 6. 关键发现

- CRAX 实现约 100 倍速度提升: 相比 CPU 基准（如 Safety Gymnasium），CRAX 在并行环境数增加时接近理想缩放，最高可达约 300K 步/秒，而 Safety Gymnasium 因 CPU 瓶颈早期即饱和。
- 无单一算法在所有任务中占优: P3O 和 FOCOPS 总体表现最强（总胜场数 7 和 7，安全百分比 79% 和 79%），但 PPOLag 安全性最高（在 Level 2 和 3 上 100% 安全），PPOPID 在安全性与性能间取得较好平衡（总胜场数 3，安全百分比 75%）。PPO 忽略成本，PPOCost 无安全保证。
- 课程学习和安全迁移效果因环境而异: 在 Safe Reacher 中，课程学习提升所有方法（除 PPOPID）的性能；在 Safe Pathway 中，课程学习和安全迁移均提升性能；但在 Safe Goal 中，两者均无改善；在 Safe Reacher 中，安全迁移导致成本超标。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: CRAX 的快速模拟使大规模并行训练成为可能，有助于开发更鲁棒的安全策略，并支持在仿真中训练后迁移到真实机器人。
- 对 Skill: 课程学习表明，从简单任务逐步过渡到复杂任务可提升技能学习效率，尤其在安全约束严格的环境中。
- 对 Memory: 安全迁移实验显示，从无约束策略初始化可能不利于安全约束的满足，提示需要谨慎处理记忆重用。
- 对评测: CRAX 提供了标准化、可扩展的评估平台，支持统计显著性测试和消融研究，有助于推动安全 RL 算法的公平比较。

### 11. 局限性

- CRAX 目前仅支持连续动作空间，未涵盖离散动作任务。
- 基准任务主要基于模拟，与真实世界的安全约束（如物理损伤）仍有差距。
- 课程学习和安全迁移的效果高度依赖环境和算法，缺乏通用指导原则。

### 12. 我的看法 / 个人笔记

CRAX 通过硬件加速显著提升了安全 RL 基准的实用性，其速度优势使得大规模实验和算法比较更加可行。然而，安全 RL 的核心挑战——如何确保策略在真实世界中的安全性——仍需进一步研究。课程学习和迁移学习的混合结果提示，未来需要更智能的训练策略。

### 14. 总结

CRAX 是一个基于 JAX 和 MuJoCo XLA 的硬件加速安全强化学习基准，提供六个环境套件和三个智能体任务，每个任务有三个难度级别。通过 GPU 并行化，CRAX 实现了约 100 倍的速度提升。评估七种基线算法（PPO、PPOCost、PPOLag、PPOPID、PPOSauté、P3O、FOCOPS）发现，无单一算法在所有任务中占优，P3O 和 FOCOPS 总体最强，PPOLag 安全性最高。课程学习在部分环境中有效，安全迁移效果有限。CRAX 为安全 RL 研究提供了高效、可扩展的评估平台。
