## 1. ArogyaSutra: A Multi-Agent Framework for Multimodal Medical Reasoning in Indic Languages

**Topics:** 多智能体系统, 医疗多模态推理, 低资源语言处理
**Authors:** Tanmoy Kanti Halder, Akash Ghosh, Subhadip Baidya, Arijit Roy, Sriparna Saha
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.13572v1) | [PDF](http://arxiv.org/pdf/2606.13572v1.pdf) | [Code](https://iitp-cse.github.io/ArogyaSutra/)

### 1. 一句话结论

提出ArogyaSutra框架，通过演员-评论家架构、工具视觉接地和双记忆机制，在七种印度语言的多模态医疗推理中显著提升准确率。

### 2. 为什么重要

该工作填补了低资源印度语言多模态医疗推理的空白，通过构建大规模数据集和创新的多智能体框架，为农村等医疗资源匮乏地区提供公平的AI辅助诊断可能。

### 3. 研究问题

1. 如何构建覆盖多种印度语言和成像模态的大规模医疗推理数据集？
2. 如何设计多智能体框架以在低资源语言场景下实现准确且语言一致的多模态医疗推理？

### 4. 背景与问题设定

现有MLLM在通用领域表现良好，但在医疗等专业领域，尤其是多语言和低资源场景下性能有限。印度农村患者常使用本地语言和医学图像表达复杂医疗问题，而现有英语中心模型难以支持，限制了AI医疗的公平可及。

### 5. 方法概览

- 任务 / 环境: 多模态医疗问答，输入为医学图像和印度语言问题，输出为正确答案及推理过程。
- 变量 / 干预设计: 演员-评论家框架：演员基于工具视觉接地和双记忆生成推理步骤；评论家评估正确性和语言一致性，提供针对性反馈（语言错误用英语，推理错误用对应印度语言）。
- 评价指标: 推理准确率（各语言独立及平均），消融实验验证各组件贡献。
- 模型比较: 与GPT-4o、Qwen2.5-VL、LLaVA等基线对比，ArogyaSutra在七种印度语言上平均准确率43.40%，优于所有基线。
- 我的理解: 该方法通过将推理错误分类为语言或逻辑问题，并分别用英语或本地语言反馈，实现了更精准的纠错；双记忆机制防止重复错误，蒸馏训练降低推理开销。

### 6. 关键发现

- ArogyaSutra在所有印度语言上超越基线: 在七种印度语言上，ArogyaSutra（基于Qwen2.5-VL-7B）平均准确率43.40%，高于GPT-4o的39.30%和Qwen2.5-VL-7B的34.21%。
- 各组件对性能均有贡献: 消融实验显示，移除评论家和图像接地后准确率降至33.43%，仅移除评论家降至33.71%，仅移除图像接地降至26.86%，仅移除代码切换降至26.86%。
- 数据集ArogyaBodha覆盖广泛且翻译质量高: 数据集包含40,857个样本，覆盖31个身体系统、6种成像模态、21个临床领域；反向翻译BLEU4在0.49-0.57之间，余弦相似度约0.93。

## 2. Who Pays the Price? Stakeholder-Centric Prompt Injection Benchmarking for Real-world Web Agents

**Topics:** LLM安全, 提示注入, Web Agent, 基准测试
**Authors:** Zihao Wang, Yiming Li, Yutong Wu, Zheyu Liu, Kangjie Chen, Fok Kar Wai, Pin-Yu Chen, Vrizlynn L. L. Thing, Bo Li, Dacheng Tao, Tianwei Zhang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.13385v1) | [PDF](http://arxiv.org/pdf/2606.13385v1.pdf) | [Code](https://github.com/StakeBench/SBC)

### 1. 一句话结论

StakeBench 提出以利益相关者为中心的提示注入基准，揭示 Web Agent 在不同受害者间存在不对称且异构的漏洞模式。

### 2. 为什么重要

现有基准仅从攻击者视角评估提示注入可行性，忽略了同一攻击对不同利益相关者（用户、卖家、平台）造成的差异化危害。StakeBench 通过受害者中心建模，首次系统量化了这种不对称风险，为真实部署中的安全评估提供了更全面的框架。

### 3. 研究问题

1. 提示注入攻击对 Web Agent 不同利益相关者（用户、卖家、平台）的危害是否存在不对称性？
2. 当前主流 Web Agent 系统在利益相关者视角下的漏洞分布和失败模式有何特征？

### 4. 背景与问题设定

LLM 驱动的 Web Agent 在真实环境中处理不可信网页内容时易受提示注入攻击。现有基准（如 INJECAGENT、AgentDojo、WASP）采用攻击中心视角，仅关注注入技术可行性，忽略了危害的受害者依赖性：同一攻击可能对用户、卖家、平台产生截然不同的后果，且相同攻击模式对不同受害者的有效性也不同。

### 5. 方法概览

- 任务 / 环境: 基于 VisualWebArena 的 OneStopMarket 电商平台，包含多利益相关者（用户、卖家、平台）和丰富不可信内容渠道（评论、评分、产品元数据）。
- 变量 / 干预设计: 构建 12 个攻击目标（如未授权购买、评论操纵、工作流绕过），通过 22 个可复用模板（9 个直接注入 DPI、13 个间接注入 IPI）在 12 个产品类别上实例化，共 264 个可执行对抗案例。
- 评价指标: 攻击成功率 (ASR)、任务偏离率 (TDR)、行为不规则率 (BIR)。ASR 与 TDR 联合定义四种失败模式：稳健行为、隐蔽寄生、错位干扰、复合失败。
- 模型比较: 评估 NanoBrowser 和 BrowserUse 两个系统，各搭配 GPT-5 和 Gemini-2.5-Flash 两个骨干模型。
- 我的理解: 该方法的核心创新是将攻击分类从攻击技术转向受害者实体，通过多轴指标（结果级+过程级）揭示传统聚合指标掩盖的异构失败模式。

### 6. 关键发现

- 无攻击目标被可靠防御: 在所有 Agent 配置下，间接注入的 ASR 介于 41.67% 至 68.16%，且伴随显著的任务偏离和执行不稳定。
- 存在三种定性不同的失败模式: 隐蔽寄生（攻击成功但用户任务未中断）、错位干扰（攻击失败但用户任务中断）、复合失败（攻击成功且用户任务中断）。这些模式被传统评估忽略。
- 漏洞分布因受害者而异: 同一 Agent 对不同利益相关者（用户、卖家、平台）表现出截然不同的漏洞特征，例如对卖家可能表现为隐蔽寄生，对平台则表现为不稳定。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: Agent 系统设计需考虑不同利益相关者的风险分布，例如对卖家隐蔽寄生攻击可能要求额外的行为审计机制。
- 对 Skill: 提示注入防御技能应针对不同受害者定制，例如对用户任务需优先保证任务完整性，对卖家则需检测隐蔽行为。
- 对 Memory: Agent 记忆机制可能放大攻击影响，需设计记忆隔离或污染检测。
- 对评测: 评估应超越单一 ASR 指标，采用多轴指标（如 TDR、BIR）并区分受害者，以揭示真实部署中的异构风险。

### 11. 局限性

- 仅聚焦电商领域，其他领域（如金融、医疗）的受害者中心评估尚未覆盖。
- 攻击模板基于人工设计，可能未穷尽所有现实攻击模式。
- 评估仅涉及两个 Agent 系统和两个骨干模型，泛化性有待验证。

### 12. 我的看法 / 个人笔记

该工作通过受害者中心视角揭示了提示注入评估中被忽视的关键维度，为安全社区提供了更贴近真实部署的评估工具。未来可扩展至更多领域和更复杂的攻击模式，并探索基于受害者风险的防御策略。

  与当前论文关系：StakeBench 扩展了其威胁模型，纳入受害者依赖性和多轴评估。

### 14. 总结

StakeBench 提出以利益相关者为中心的提示注入基准，通过多轴指标（ASR、TDR、BIR）和 264 个对抗案例，系统揭示了 Web Agent 在不同受害者（用户、卖家、平台）间存在不对称且异构的漏洞模式，包括隐蔽寄生、错位干扰和复合失败，为真实部署中的安全评估提供了更全面的框架。

## 3. An LLM System for Autonomous Variational Quantum Circuit Design

**Topics:** 量子电路设计, 大语言模型, 变分量子算法, 量子机器学习, 量子化学
**Authors:** Kenya Sakka, Wataru Mizukami, Kosuke Mitarai
**Venue / Year:** arXiv:2606.13380v1, 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.13380v1) | [PDF](http://arxiv.org/pdf/2606.13380v1.pdf) | [Code](https://github.com/...)

### 1. 一句话结论

提出一个基于LLM的自主智能体框架，通过闭环迭代流程自动设计高性能变分量子电路，在量子特征映射和VQE ansatz生成任务上超越现有基线。

### 2. 为什么重要

该工作首次将LLM驱动的自主智能体系统应用于通用量子电路设计，实现了从文献调研、想法生成、多视角批判到实验验证的全自动化闭环，显著降低了量子电路设计对人工专家的依赖，为跨学科量子应用提供了可扩展的自动化工具。

### 3. 研究问题

1. 如何利用LLM自主设计高性能的量子特征映射和VQE ansatz？
2. 闭环智能体框架能否在多个量子任务上实现超越人工设计的性能？

### 4. 背景与问题设定

量子电路设计（如量子特征映射和VQE ansatz）高度依赖人类专家经验和领域启发式，缺乏理论指导，且手动迭代成本高。现有自动化方法（如量子架构搜索）局限于预定义模板，而LLM应用多限于孤立环节（如代码生成）。本文提出一个包含探索、生成、讨论、验证、存储、评估和评审七个组件的闭环智能体系统，旨在实现端到端的自主量子电路设计。

### 5. 方法概览

- 任务 / 环境: 两个任务：量子机器学习中的特征映射设计（用于图像分类）和量子化学中的VQE ansatz设计（用于分子基态能量估计）。
- 变量 / 干预设计: 系统包含七个组件：Exploration（网络调研生成种子想法）、Generation（LLM生成并实现想法）、Discussion（多角色专家批判改进）、Validation（代码语法和执行验证）、Evaluation（实验评估）、Storage（存储库）、Review（结果评审反馈）。迭代循环优化。
- 评价指标: 分类准确率（QML任务）和基态能量误差（VQE任务），同时考虑电路深度和参数数量等资源约束。
- 模型比较: QML任务：对比典型量子特征映射（如ZZFeatureMap）和经典RBF核；VQE任务：对比化学启发式ansatz（UCCSD）和硬件高效ansatz（如HEA）。
- 我的理解: 该框架的核心创新在于将LLM嵌入一个完整的科研循环，通过多角色讨论（批评者+倡导者）模拟同行评审，结合网络调研和文献检索，使想法生成更具方向性和批判性，从而在迭代中提升电路质量。

### 6. 关键发现

- 自主生成的特征映射超越经典和量子基线: 在MNIST、Fashion-MNIST和CIFAR-10上，最佳特征映射分类准确率超过代表性量子特征映射，且在大规模量子比特下超越经典RBF核。
- 生成的ansatz在分子基态估计中达到竞争性精度: 在7个分子上，最佳ansatz在满足可扩展性约束的同时，精度与UCCSD和HEA相当，且参数更少、电路更浅。
- 闭环迭代显著提升性能: 随着迭代次数增加，特征映射和ansatz的性能持续改善，表明系统能有效利用实验反馈进行优化。

### 7. 最重要的图

- 图来源: 图1：系统架构图，展示七个组件及其交互流程。
- 为什么重要: 该图清晰呈现了闭环智能体的完整工作流，包括人类定义任务、LLM驱动的探索-生成-讨论-验证-评估-存储-评审循环，是理解方法核心的关键。

## 4. EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments

**Topics:** LLM Agent, Dynamic Environment, Memory Evolution, Benchmark
**Authors:** Jundong Xu, Qingchuan Li, Jiaying Wu, Yihuai Lan, Shuyue Stella Li, Huichi Zhou, Bowen Jiang, Lei Wang, Jun Wang, Anh Tuan Luu, Caiming Xiong, Hae Won Park, Bryan Hooi, Zhiyuan Hu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.13681v1) | [PDF](http://arxiv.org/pdf/2606.13681v1.pdf) | [Code](https://github.com/Aiden0526/EvoArena)

### 1. 一句话结论

EvoArena 基准测试揭示现有 LLM 智能体在持续环境演化中表现不佳，而所提出的 EvoMem 补丁式记忆范式通过记录结构化更新历史，显著提升了智能体的鲁棒性和证据保留能力。

### 2. 为什么重要

现实部署中环境持续变化，但现有基准多为静态快照。EvoArena 首次系统评估智能体在持久环境演化下的适应能力，EvoMem 提供了一种轻量级、版本感知的记忆方案，对构建可靠、可部署的 LLM 智能体具有关键指导意义。

### 3. 研究问题

1. 在终端、软件和社交偏好等持续演化的环境中，当前 LLM 智能体的表现如何？
2. 如何设计一种记忆机制，使智能体能够追踪环境变化、区分新旧版本的有效行为，并提升在演化任务链上的成功率？

### 4. 背景与问题设定

现有 LLM 智能体在静态基准上表现优异，但真实环境（如 API、代码库、用户偏好）持续变化。大多数记忆系统只保留最新状态，导致“状态崩溃”：当不同版本需要不同行为时，智能体丢失了旧版本的有效信息。EvoArena 通过构建版本链（每个版本继承前一版本的变化）来模拟这种演化，要求智能体同时适应新变化并保留仍有效的旧知识。

### 5. 方法概览

- 任务 / 环境: EvoArena 包含三个领域：Terminal-Bench-Evo（终端工作流演化，如依赖、接口、权限变化）、SWE-Chain-Evo（软件代码库演化，如新功能、问题修复）、PersonaMem-Evo（用户偏好演化，如对话中偏好变化）。每个环境组织成渐进式版本链，版本间保持相同目标但改变操作细节。
- 变量 / 干预设计: 提出 EvoMem 记忆范式：采用追加式补丁历史记录每次记忆更新，每个补丁包含更新前记忆、更新后记忆、更新理由和触发上下文证据。推理时默认使用最新记忆，但当查询涉及被覆盖状态或冲突证据时，可选择性地检索相关补丁。
- 评价指标: Step-level accuracy（单版本任务正确率）和 Chain-level accuracy（整个版本链全部正确才算成功）。
- 模型比较: 在 EvoArena 上测试了多个主流模型（如 GPT-5.5、GLM-5.1、Gemini-3.1-Pro 等），并对比了有无 EvoMem 的性能差异。同时也在 GAIA 和 LoCoMo 等标准基准上验证了 EvoMem 的泛化性。
- 我的理解: EvoMem 的核心思想是将记忆视为可追溯的演化历史，而非单一快照。通过保留每次更新的上下文（理由和证据），智能体可以理解变化的原因，并在需要时回溯到旧版本状态，从而避免因信息覆盖导致的决策错误。

### 6. 关键发现

- 现有智能体在演化环境中表现显著下降: 在 EvoArena 上，当前最强智能体的平均 step-level 准确率仅为 39.6%，chain-level 准确率更低，表明持续环境演化对智能体构成严峻挑战。
- EvoMem 持续提升性能: EvoMem 在 EvoArena 上带来平均 1.5% 的 step-level 提升和 3.7% 的 chain-level 提升，并在 GAIA 和 LoCoMo 上分别提升 6.1% 和 4.8%，表明其不仅适用于演化环境，也能改善标准长时任务。
- EvoMem 改善证据捕获: 机制分析显示，EvoMem 提高了记忆中的证据捕获率，尤其是在 PersonaMem-Evo 中，对时间轨迹和多模式综合问题的提升更为显著，说明补丁历史更好地保留了完整的偏好状态。

### 7. 最重要的图

- 图来源: Figure 1: Step accuracy vs. chain accuracy on EvoArena.
- 为什么重要: 该图直观展示了不同模型在 step-level 和 chain-level 上的表现分布，右上角为最优。它揭示了即使 step-level 准确率较高，chain-level 准确率仍可能很低，说明模型在连续演化任务中缺乏持久可靠性。

横轴为 step-level accuracy，纵轴为 chain-level accuracy。每个点代表一个模型，越靠近右上角性能越好。例如 GPT-5.5 的 step-level 约 50.8%，chain-level 约 27.2%，表明虽然单步任务解决能力较强，但完整演化链的成功率仍不足三分之一。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 智能体系统应内置版本感知能力，不能假设环境静态。记忆系统需支持历史追溯，而非仅保留最新状态。
- 对 Skill: 技能积累应考虑环境版本，避免将过时技能错误应用于新版本。EvoMem 的补丁历史可帮助智能体判断技能的有效范围。
- 对 Memory: 记忆应设计为可演化的证据链，记录每次更新的理由和上下文，而非简单的覆盖式存储。这有助于智能体在冲突或回滚场景中做出正确决策。
- 对评测: 评估应包含环境演化维度，使用 chain-level 指标衡量智能体在连续变化中的持久可靠性，而非仅看单步准确率。

### 11. 局限性

- EvoArena 的版本链长度有限（平均约 5 步），可能不足以模拟长期、复杂的真实演化过程。
- EvoMem 的补丁历史增加了存储和检索开销，在极大规模部署中可能需要优化。
- 当前实验主要基于闭源和开源大模型，未充分探索小模型或专用模型的适用性。

### 12. 我的看法 / 个人笔记

该工作精准地指出了当前智能体评估与部署之间的关键鸿沟——环境演化。EvoMem 的设计简洁而有效，将版本控制思想引入记忆系统，为构建鲁棒智能体提供了新方向。未来可进一步探索补丁的自动压缩与遗忘机制，以及跨智能体的演化记忆共享。

### 13. 这篇工作引用的近期高影响力相关文章

- GAIA: A Benchmark for General AI Assistants
  为什么重要：GAIA 是广泛使用的通用智能体基准，EvoMem 在其上提升了 6.1%，证明了方法的泛化性。
  核心贡献：提出包含多步推理和工具使用的任务集，评估智能体的通用能力。
  与当前论文关系：EvoArena 借鉴了 GAIA 的任务设计，但增加了环境演化维度；EvoMem 在 GAIA 上的改进表明其不仅适用于演化场景。

### 14. 总结

EvoArena 是一个针对持续环境演化的基准套件，包含终端、软件和社交三个领域的版本链任务。实验表明现有智能体平均准确率仅 39.6%。提出的 EvoMem 记忆范式通过记录结构化更新历史，在 EvoArena 上提升 1.5% step-level 和 3.7% chain-level 准确率，并在 GAIA 和 LoCoMo 上分别提升 6.1% 和 4.8%。该工作强调了在评估和记忆中建模演化的重要性，为可靠智能体部署提供了新思路。

## 5. InterleaveThinker: Reinforcing Agentic Interleaved Generation

**Topics:** GUI Agent, Interleaved Generation, Multi-Agent Systems, Reinforcement Learning
**Authors:** Dian Zheng, Harry Lee, Manyuan Zhang, Kaituo Feng, Zoey Guo, Ray Zhang, Hongsheng Li
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.13679v1) | [PDF](http://arxiv.org/pdf/2606.13679v1.pdf) | [Code](https://github.com/zhengdian1/InterleaveThinker)

### 1. 一句话结论

InterleaveThinker 通过多智能体流水线（规划器-生成器-评论家）赋予任意图像生成器交错生成能力，并利用单步强化学习实现轨迹级对齐，显著提升长程生成质量。

### 2. 为什么重要

该工作首次将多智能体强化学习应用于长程交错图像-文本生成，有效解决了统一多模态模型中的视觉过度依赖和逐步误差累积问题，为复杂序列生成任务提供了新范式。

### 3. 研究问题

1. 如何让现有图像生成器具备交错生成（文本-图像序列）能力？
2. 如何克服统一多模态模型在长程任务中的视觉过度依赖和逐步误差累积？

### 4. 背景与问题设定

现有图像生成器仅支持单图生成/编辑，无法输出文本-图像交错序列；统一多模态模型虽原生支持，但在长程任务中因视觉过度依赖（过早停止于中间状态）和逐步误差累积（早期微小退化逐级放大）而表现不佳。

### 5. 方法概览

- 任务 / 环境: 交错生成任务，包括视觉叙事、操作指导、具身操控等场景，输入为文本-图像交错序列，输出为多步文本-图像序列。
- 变量 / 干预设计: 提出三智能体流水线：规划器（Planner）先生成全局步骤指令，生成器（Generator）执行，评论家（Critic）评估输出并迭代优化提示，直至满足要求。
- 评价指标: 在交错生成基准上对比Nano Banana和GPT-5；在推理基准WISE和RISE上评估提升。
- 模型比较: 以4步FLUX.2-klein为默认生成器，与开源UMMs、Nano Banana Pro、GPT-5对比，在交错生成和推理基准上均取得显著提升。
- 我的理解: 该方法通过解耦规划与生成，阻断中间视觉反馈，从根本上避免视觉过度依赖；评论家模块的显式校正机制有效遏制误差累积。

### 6. 关键发现

- 多智能体流水线有效解决UMM两大问题: 规划器预先预测完整指令序列，阻断中间视觉反馈，消除视觉过度依赖；评论家评估输出并迭代优化提示，确保严格遵循全局轨迹，解决逐步误差累积。
- 单步强化学习实现轨迹级对齐: 由于单条轨迹可能包含超过25次生成器调用，端到端优化不可行。提出准确率奖励和逐步奖励，通过单步RL有效引导整个生成轨迹，大幅降低计算成本。
- 显著提升推理基准性能: 在4步FLUX.2-klein上，WISE从0.47提升至0.73，RISE从13.3提升至28.9，表明多智能体协作能解锁现有图像模型的复杂序列推理能力。
