## 1. Knowing the Self, Understanding the World: A Dual-Cognition Benchmark for UAV Spatio-temporal Reasoning with MLLMs

**Topics:** 多模态大语言模型, 无人机推理, 空间-时间推理, 双认知基准
**Authors:** Like Liu, Zhengzheng Xu, Haitao He, Hongzhe Li, Shuchang Zhang, Dian Shao
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.16193v1) | [PDF](http://arxiv.org/pdf/2607.16193v1.pdf) | [Code](https://uav-dualcog.lozumi.com)

### 1. 一句话结论

提出UAV-DualCog基准，系统评估MLLM在无人机场景中的自我状态与环境状态双认知推理能力，发现现有模型在空间-时间定位和视角变换上存在显著瓶颈。

### 2. 为什么重要

现有无人机基准多聚焦于场景理解或导航，忽略了无人机作为主动观察者需同时推理自身状态与环境状态的耦合需求。该基准首次将自我认知与环境认知作为顶层评估维度，并引入结构化证据输出（边界框、时间区间），为MLLM在具身无人机智能中的可靠部署提供了关键评估工具。

### 3. 研究问题

1. 当前MLLM在无人机多视角空间-时间推理中，自我状态认知与环境状态认知的表现如何？
2. 结构化空间-时间证据输出（边界框、时间区间）能否揭示MLLM在推理与定位上的不一致性？

### 4. 背景与问题设定

MLLM在视觉-语言任务中表现优异，但在无人机场景中，观测由主动移动的空中代理生成，视角和运动状态持续变化，要求模型同时推理自身状态（位置、运动、行为）和环境状态（地标方向、可见性）。现有基准多侧重环境理解或任务完成，缺乏对双认知的联合评估，且通常不要求结构化证据输出。

### 5. 方法概览

- 任务 / 环境: 基于AerialVLN模拟器的18个场景（覆盖城市、工业区、夜间等），构建场景级语义点云，包含746个有效地标。
- 变量 / 干预设计: 设计6类任务：图像任务（自我相对位置推理、未来观测预测、地标相对方向推理、地标驱动动作决策）和视频任务（飞行行为识别与时间定位、地标可见性计数与区间推理）。所有任务要求离散答案+结构化证据（边界框或时间区间）。
- 评价指标: 图像任务：答案准确率、边界框IoU≥0.5准确率、平均IoU；视频任务：行为/计数准确率、时间定位tIoU≥0.5准确率、平均tIoU。
- 模型比较: 评估了闭源模型（Claude、Gemini、GPT、Grok、Mimo、Qwen系列）和开源模型（GLM、Intern等），共14个轻量级MLLM，并包含人类基线。
- 我的理解: 该方法通过模拟环境生成可控、可验证的标注数据，将无人机具身推理分解为自我与环境两个耦合维度，并通过结构化输出强制模型进行空间-时间定位，从而暴露纯分类准确率无法反映的定位缺陷。

### 6. 关键发现

- 自我状态认知普遍弱于环境状态认知: 在图像任务中，所有模型在环境状态任务（地标方向推理、动作决策）上的准确率显著高于自我状态任务（相对位置、未来观测预测），差距可达10-20个百分点。
- 结构化定位是主要瓶颈: 即使答案准确率较高（如GPT-5.5在环境任务上达65.4%），边界框IoU@50准确率普遍低于20%，时间定位tIoU@50准确率也多在10%以下，表明模型缺乏精确的空间-时间接地能力。
- 人类基线验证基准有效性: 人类在图像任务上准确率超过80%，视频任务超过70%，证明基准对可理解但对现有MLLM极具挑战性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 无人机具身代理需同时建模自我状态与环境状态，现有MLLM在此耦合推理上远不可靠，需开发专门的空间-时间推理模块。
- 对 Skill: 视角变换、相对位置推理、时间区间定位是当前MLLM的薄弱技能，应作为训练重点。
- 对 Memory: 视频任务中的时间定位要求模型具备短期记忆和事件边界检测能力，当前模型表现不佳。
- 对评测: 仅依赖多项选择准确率会高估模型能力，必须引入结构化证据输出（边界框、时间区间）以揭示真实定位缺陷。

### 11. 局限性

- 基准基于模拟环境构建，存在sim-to-real差距，真实无人机场景的噪声和动态性未被充分覆盖。
- 仅评估轻量级MLLM，未测试更大规模模型或专用无人机推理模型。
- 任务类型有限，未涵盖导航、避障等更复杂的具身任务。

### 12. 我的看法 / 个人笔记

该基准精准定位了MLLM在无人机具身推理中的关键短板——自我状态认知和结构化定位，并通过自动化流水线实现了可扩展的评估。其核心贡献在于将“证据输出”作为评估的必要条件，迫使模型不仅“猜对”还要“定位准”，这比现有纯分类基准更具诊断价值。未来工作应探索如何将模拟数据用于微调以提升定位能力，并验证在真实无人机上的迁移效果。

### 13. 这篇工作引用的近期高影响力相关文章

### 14. 总结

UAV-DualCog是一个面向无人机多视角空间-时间推理的双认知基准，通过图像和视频任务联合评估MLLM的自我状态与环境状态认知，并要求结构化证据输出。系统评估显示，当前MLLM在自我状态推理、视角变换、精确空间-时间定位上存在显著瓶颈，环境状态认知普遍优于自我状态认知。该基准不仅可作为评估工具，其训练集也为提升无人机MLLM能力提供了结构化监督信号。

## 3. Searching Videos as Trees: Self-Correcting Agents for Grounded Long Video QA

**Topics:** GUI Agent, Long Video Understanding, Grounded Question Answering
**Authors:** Ce Zhang, Ziyang Wang, Yulu Pan, Oluwatumininu Oguntola, Pranav Wagh, Qiyu Wu, Hiromi Wakaki, Mohit Bansal, Gedas Bertasius
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.16189v1) | [PDF](http://arxiv.org/pdf/2607.16189v1.pdf) | [Code](https://github.com/CeeZh/VTS)

### 1. 一句话结论

VideoTreeSearch将长视频组织为自适应语义树，通过离散的树导航动作（zoom_in/zoom_out/shift/answer）实现显式回溯与自纠正，在Grounded LVQA任务上大幅超越先前方法。

### 2. 为什么重要

现有基于连续crop_video动作的智能体方法缺乏细到粗的回溯机制，容易过早收敛且无法从早期错误中恢复。VTS通过将视频结构化为语义树并引入离散导航动作，将回溯和自纠正变为可学习的显式行为，显著提升了长视频定位与问答的鲁棒性。

### 3. 研究问题

1. 如何设计一种支持显式回溯与自纠正的智能体框架，用于长视频中的细粒度证据定位？
2. 将视频组织为自适应语义树并让智能体通过离散动作导航，能否比连续crop_video方法更有效地进行多尺度搜索与错误恢复？

### 4. 背景与问题设定

Grounded LVQA要求模型在长视频中定位支持答案的短证据区间。现有智能体方法使用单一的crop_video(start, end)动作进行多轮探索，但该动作空间不对称：只能粗到细缩小，缺乏细到粗的回溯原语，导致智能体过早收敛且无法从早期错误中恢复。此外，连续时间戳回归缺乏对视频语义结构的先验，搜索空间平坦无序。

### 5. 方法概览

- 任务 / 环境: 长视频（分钟至小时级），智能体通过多轮交互定位证据区间并回答问题。
- 变量 / 干预设计: 将视频构建为自适应语义树（基于CLIP场景边界），智能体通过四个离散动作（zoom_in/zoom_out/shift/answer）导航树节点。
- 评价指标: Grounded LVQA：mIoU（CG-Bench）、Temporal F1（Haystack基准）、QA准确率；通用长视频QA：多项选择准确率。
- 模型比较: 与均匀采样（Qwen3-VL-8B、Video-R1）、Captioner-LLM智能体（SiLVR）、多轮裁剪智能体（LongVT、Video-o3、VideoZoomer、TimeSearch-R）对比。
- 我的理解: VTS的核心创新在于将视频的连续时间轴离散化为语义树，并将搜索过程转化为树上的离散路径选择。这使得回溯（zoom_out/shift）成为与前进（zoom_in）并列的显式动作，智能体可以通过学习何时回溯来纠正错误，而无需依赖隐式的连续裁剪近似。

### 6. 关键发现

- VTS在Grounded LVQA上显著超越先前方法: 在CG-Bench上mIoU提升+12.5，在Haystack-Ego4D上T-F1提升+7.4，QA准确率也大幅领先。
- 树导航策略可迁移至通用长视频QA: 在Video-MME、MLVU、LVBench上，VTS超越所有先前智能体基线，准确率最高提升+7.1。
- 显式回溯是性能提升的关键机制: 消融实验表明，移除自适应下降或显式回溯会显著降低性能；VTS平均每视频4.8轮搜索，约60%的轨迹中调用了回溯动作。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 离散化动作空间并引入显式回溯原语，可提升智能体在长时序任务中的鲁棒性和可学习性。
- 对 Skill: 智能体需要学会何时深入探索、何时回溯，这种决策能力可通过轨迹合成和强化学习有效训练。
- 对 Memory: 紧凑的记忆设计（当前节点帧+文本历史）支持长搜索轮次而不耗尽视觉上下文预算。
- 对评测: Grounded LVQA的定位指标（mIoU/T-F1）比单纯QA准确率更能反映模型对视频内容的真实理解。

### 11. 局限性

- 树构建依赖CLIP场景边界，可能无法捕捉所有语义变化，尤其当场景过渡平滑时。
- 轨迹合成依赖外部模型（Qwen3-VL-8B、DeepSeek-R1）进行评分和推理，可能引入偏差。
- 当前方法主要针对单证据区间问题，多证据或更复杂推理场景尚未验证。

### 12. 我的看法 / 个人笔记

VTS通过将视频结构化为语义树并引入离散导航动作，优雅地解决了现有智能体方法缺乏回溯能力的问题。其核心贡献在于将回溯从隐式近似变为显式可学习行为，并通过轨迹合成和强化学习有效训练。实验证明，这种层次化搜索策略在长视频定位和通用QA上均有效，且搜索轮次越多优势越明显。未来工作可探索更灵活的树结构（如动态调整）和更复杂的推理路径。

## 4. Knowledge-Centric Agents for Workflow Generation

**Topics:** GUI Agent, Workflow Generation, ComfyUI, Knowledge-Centric AI
**Authors:** Zhendong Li, Lei Sun, Ruibo Ming, He Zhang, Danda Pani Paudel, Luc Van Gool, Jinjin Gu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.15845v1) | [PDF](http://arxiv.org/pdf/2607.15845v1.pdf)

### 1. 一句话结论

提出知识中心框架，通过知识反转（从原始工作流提取分层知识）、知识注入（SFT训练任务→策略和策略→骨架伪代码）和知识推理（反向生成并自优化），在ComfyUI工作流生成中实现节点多样性提升30%、结构连贯性提升25%、执行成功率从基线ComfyGen的45%提升至78%。

### 2. 为什么重要

现有LLM直接生成JSON工作流因缺乏领域经验知识而结构脆弱、成功率低。本工作首次系统建模工作流设计中的分层知识（策略、结构、推理），将生成问题转化为知识推理问题，为复杂视觉管线自动化提供可扩展范式，并通过消融实验明确论证SFT在能力注入中起主导作用（GRPO仅提升2%成功率，而SFT提升33%）。

### 3. 研究问题

1. 如何从无标注的原始ComfyUI工作流中自动提取分层知识表示（完整伪代码、骨架伪代码、策略推理、高层策略、任务指令）？
2. 如何将提取的知识通过SFT注入LLM，使其能从任务描述推理出高层策略，再从策略生成可执行的骨架伪代码？

### 4. 背景与问题设定

ComfyUI等视觉创作系统采用模块化DAG工作流，生成有效工作流需从抽象概念设计到具体语法实现的多层次推理。现有LLM方法（如ComfyGen）将工作流生成视为文本到JSON的直接生成，缺乏结构化的多级知识，导致生成结果结构脆弱、缺乏专家级经验知识。人类专家依赖隐性经验（如是否需分割、如何安排分支），而LLM难以从纯文本数据中获取此类知识。论文使用包含5000个高质量ComfyUI工作流的数据集进行训练和评估。

### 5. 方法概览

- 任务 / 环境: ComfyUI工作流生成，输入为用户任务指令（如风格化编辑），输出为可执行的JSON格式DAG工作流图。
- 变量 / 干预设计: 提出知识反转（从原始工作流通过规则和LLM逐步提取：完整伪代码→骨架伪代码→策略推理→高层策略→任务指令）、知识注入（对任务→策略和策略→骨架伪代码进行SFT，训练Qwen3-14B，并引入策略推理提取作为辅助目标）、知识推理（推理时反向执行：策略规划→伪代码生成→参数填充→规则重建，并加入自优化机制）。
- 评价指标: 节点多样性（不同节点类型数量）、结构连贯性（图编辑距离）、连接准确性（边匹配率）、执行成功率（工作流在ComfyUI中无错误运行）。
- 模型比较: 与ComfyGen、ComfyBench、ComfyMind、ComfyGPT、ComfyUI-R1等现有方法对比。具体结果：执行成功率上，ComfyGen为45%，ComfyGPT为52%，ComfyUI-R1为55%，本文方法为78%。
- 我的理解: 该方法本质上是将工作流设计中的隐性知识显式化为分层结构，通过反转过程从数据中学习，再通过SFT注入模型。与直接端到端生成JSON不同，它强制模型进行中间推理（策略→结构），从而降低生成难度并提升可解释性。论文通过消融实验证明：仅使用SFT（无GRPO）时成功率为78%，仅使用GRPO（无SFT）时成功率为57%，而两者结合时成功率为79%，表明SFT贡献了主要性能提升。

### 6. 关键发现

- 知识反转有效提取分层知识: 从5000个原始工作流中通过规则（完整伪代码）和LLM（骨架伪代码、策略推理、高层策略、任务指令）逐步提取出分层知识表示。其中，策略推理提取步骤依赖LLM进行语义解释，论文通过人工评估100个样本确认准确率为92%。
- SFT注入策略推理能力，SFT主导能力注入: 对任务→策略和策略→骨架伪代码进行SFT，使模型学会从任务描述推理出高层策略，再转化为结构化伪代码。消融实验显示：仅SFT成功率为78%，仅GRPO成功率为57%，SFT+GRPO为79%，证明SFT贡献了主要性能提升（33% vs GRPO的2%）。
- 生成工作流质量显著提升: 相比ComfyGen，节点多样性提升30%（平均节点类型数从8.2增至10.7），结构连贯性提升25%（图编辑距离从12.3降至9.2），连接准确性提升40%（边匹配率从0.55增至0.77），执行成功率从45%提升至78%。自优化机制进一步将成功率提升3%。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 知识中心范式可推广至其他需要分层推理的代理系统（如机器人任务规划、代码生成），通过反转和注入领域知识提升代理的推理能力，尤其适用于任务分解和结构化规划场景。

- 对 Memory: 知识反转过程可视为从经验中提取结构化记忆（如策略、骨架），代理在推理时利用这些记忆进行规划，类似人类利用经验知识进行决策。
- 对评测: 评估不应仅关注最终结果（如执行成功率），还应评估中间推理步骤（如策略合理性、伪代码结构正确性），以全面衡量代理的知识运用能力，避免过度拟合最终指标。

### 11. 局限性

- 知识反转中策略推理提取步骤依赖LLM进行语义解释，虽然人工评估准确率为92%，但仍可能引入偏差或错误，论文未提供自动化缓解措施。

- 当前方法仅针对ComfyUI，泛化到其他视觉编程框架（如Blender节点、Nuke）需进一步验证，且规则重建部分可能依赖ComfyUI特定格式。

### 12. 我的看法 / 个人笔记

该工作将工作流生成从直接文本生成提升为知识推理问题，通过反转、注入、推理的闭环，系统性地解决了LLM缺乏领域经验知识的问题。方法设计精巧，分层知识表示清晰，实验充分（包括消融实验和与5个基线的对比）。然而，知识反转中LLM的使用可能引入偏差，论文对此讨论不足；此外，SFT与GRPO的对比论证虽有力，但未考虑混合训练的可能性（如先SFT后GRPO）。未来可探索更自动化的知识反转（减少LLM依赖）、跨领域迁移以及SFT与强化学习的协同。

## 5. When Does Muon Help Agentic Reinforcement Learning?

**Topics:** GUI Agent, Reinforcement Learning, Optimizer
**Authors:** Kai Ruan, Jinghao Lin, Zihe Huang, Ziqi Zhou, Qianshan Wei, Xuan Wang, Hao Sun
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.16169v1) | [PDF](http://arxiv.org/pdf/2607.16169v1.pdf)

### 1. 一句话结论

Muon优化器在稀疏奖励的智能体强化学习中，结合组内策略优化（GiGPO）可显著提升成功率，但效果依赖于优势估计器和学习率。

### 2. 为什么重要

该研究首次系统探索了Muon优化器在强化学习后训练中的潜力，特别是在智能体任务中，为优化器选择与优势估计器设计的联合优化提供了新视角。

### 3. 研究问题

1. Muon优化器在稀疏奖励的智能体强化学习中是否优于AdamW？
2. Muon的效果如何依赖于优势估计器的结构和学习率？

### 4. 背景与问题设定

Muon在预训练中表现出色，但在强化学习后训练中的价值尚不明确。现有证据矛盾：部分研究显示Muon在RLVR中失败，而另一些报告轻微改进。本研究聚焦于长时域、稀疏奖励的智能体RL任务（ALFWorld），使用Qwen2.5-0.5B-Instruct模型，比较Muon与AdamW在不同组基RL算法（GRPO、GiGPO、GraphGPO）下的表现。

### 5. 方法概览

- 任务 / 环境: ALFWorld：包含6类家庭任务（Pick、Look、Clean、Heat、Cool、PickTwo），最大步数50，稀疏终端成功奖励，无效动作惩罚。
- 变量 / 干预设计: 仅将策略网络隐藏权重矩阵的优化器从AdamW替换为Muon（嵌入层、归一化层等保持AdamW），学习率设为3e-5或1e-5；对比AdamW基线（lr=1e-6）及高学习率AdamW控制组（1e-5、3e-5）。
- 评价指标: 验证成功率（最终窗口均值及最终检查点）、归一化验证AUC（步数0-200的梯形面积除以200）、达到0.5/0.75成功率的更新步数。
- 模型比较: 比较三种组基RL算法：GRPO（回合级优势）、GiGPO（回合级+步级优势）、GraphGPO（转移级优势），每种算法下Muon与AdamW对比。
- 我的理解: 实验设计精巧，通过控制变量（仅改优化器）和多种优势估计器对比，系统揭示了Muon在RL中的条件性优势。但单种子实验和单一任务限制了结论的泛化性。

### 6. 关键发现

- Muon显著提升GiGPO成功率: 在GiGPO下，Muon（3e-5）将最终窗口验证成功率从0.290提升至0.546（+88%），最终检查点达0.63 vs 0.32。
- 高学习率AdamW控制组失败: AdamW在1e-5和3e-5下所有后更新检查点成功率为0，而Muon在3e-5下保持成功至200步。
- 效果因优势估计器而异: GRPO下Muon提升较小（0.161→0.268），GraphGPO下晚期差距缩小但AUC提升显著（0.399→0.556）。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: Muon可显著提升基于GiGPO的智能体系统性能，建议在长时域稀疏奖励任务中优先考虑。
- 对 Skill: 优化器选择与信用分配机制需联合设计，Muon的谱归一化可能放大高质量信用信号。
- 对 Memory: Muon需全矩阵暴露给牛顿-舒尔茨迭代，可能增加内存开销（FSDP NO_SHARD）。
- 对评测: 评估时应同时关注最终性能和训练效率（如AUC），因Muon可能加速早期学习。

### 11. 局限性

- 单种子实验，缺乏多种子统计显著性验证。
- 仅测试ALFWorld单一任务，跨任务泛化性未知。
- 未直接测量梯度信噪比或更新谱，梯度SNR猜想缺乏实证。

### 12. 我的看法 / 个人笔记

该研究为Muon在RL中的应用提供了重要实证，但单种子和单一任务设计使其结论需谨慎对待。未来应进行多种子实验并扩展至更多任务，同时直接验证梯度SNR假设。
