## 1. Agentic Chain-of-Thought Steering for Efficient and Controllable LLM Reasoning

**Topics:** LLM推理, 推理效率, 推理控制
**Authors:** Yu Xia, Zhouhang Xie, Xin Xu, Byungkyu Kang, Prarit Lamba, Xiang Gao, Julian McAuley
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.03965v1) | [PDF](http://arxiv.org/pdf/2606.03965v1.pdf) | [Code](https://github.com/Andree-9/ACTS)

### 1. 一句话结论

ACTS通过一个控制器智能体在推理过程中自适应选择推理策略和引导短语，在匹配或接近全思考准确率的前提下，将token消耗降低37%-64%。

### 2. 为什么重要

现有高效推理方法主要通过缩短、早停或压缩来控制推理长度，但ACKS首次将推理控制提升到策略层面，通过显式选择推理策略（如理解、计划、执行、检查等）引导步骤行为，实现了更精细、更可控的准确率-效率权衡，且具有理由器无关性。

### 3. 研究问题

1. 如何将LLM推理过程建模为马尔可夫决策过程，通过一个控制器智能体自适应地选择推理策略和引导短语，在给定token预算下实现高效推理？
2. 如何训练控制器智能体使其学会在准确率和token消耗之间进行最优权衡，避免过早终止或过度思考？

### 4. 背景与问题设定

LLM通过长链式思维推理提升准确率，但推理轨迹中常包含冗余推导、重复检查和延迟终止，导致token浪费且难以控制。现有高效推理方法主要控制推理长度（如缩短、早停、压缩），但忽略了步骤级推理行为的显式控制。

### 5. 方法概览

- 任务 / 环境: 数学推理（MATH-500、AIME24、AMC、OlympiadBench）和科学问答（GPQA Diamond）任务。
- 变量 / 干预设计: 将推理引导建模为MDP：控制器在每个步骤观察推理历史与剩余预算（bt），输出一个推理策略（ut ∈ {UNDERSTAND, PLAN, EXECUTE, EXPLORE, CHECK, SUMMARIZE, CONCLUDE}）和一个自然语言引导短语（pt），冻结的推理器据此生成下一步推理。
- 评价指标: 准确率（Accuracy）和总token数（#Tokens，包含控制器和推理器token）。
- 模型比较: 与Vanilla（完整推理）、NoThink（跳过思考）、CoD（思维草稿）、BudgetGuidance（预算引导）、DEER（早停）等基线比较，并在DeepSeek-R1-Distill-Qwen-1.5B/7B和Qwen3-8B上验证。
- 我的理解: ACTS的核心创新在于将推理控制从长度控制提升到策略控制，通过一个独立的控制器智能体在推理过程中动态选择推理策略，实现了更细粒度、更可解释的推理引导，并在预算条件奖励塑造中同时惩罚过度思考（正确但超预算）和过早终止（错误且预算未用尽），避免了控制器通过提前终止来逃避惩罚的作弊行为。

### 6. 关键发现

- ACTS在多个基准上匹配或接近全思考准确率，同时大幅节省token: 在DeepSeek-R1-Distill-Qwen-7B上，ACTS在MATH-500准确率85.2%（全思考92.6%，降低7.4个百分点），token减少57.0%（从4,339降至1,866）；在AIME24准确率60.0%（全思考51.3%，提升8.7个百分点），token减少37.1%（从13,031降至8,200）。
- ACTS实现了可控的准确率-效率权衡: 通过调整预算条件奖励塑造中的惩罚系数α，ACTS可以在不同预算下实现不同的准确率-效率权衡，适应不同应用场景。例如，在更紧的预算下，准确率下降幅度有限，但token节省更显著。
- ACTS的控制器策略分布与专家推理轨迹中的自然策略分布一致: 控制器在推理早期倾向于UNDERSTAND和PLAN，中期以EXECUTE为主，后期CHECK和CONCLUDE增加，与从OpenR1-Math专家轨迹中提取的策略分布高度吻合（见图2）。

### 7. 最重要的图

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: ACTS的控制器-推理器架构为构建更高效的智能体系统提供了新范式：通过一个轻量级控制器动态引导主模型的行为，可以在保持性能的同时大幅降低计算成本，且控制器可独立优化。
- 对 Skill: ACTS显式建模了推理策略（如理解、计划、执行、检查等），为技能学习提供了可解释的控制接口，有助于将复杂任务分解为可管理的子技能，并支持在推理过程中动态切换技能。
- 对 Memory: ACTS的控制器需要维护推理历史状态（包含之前的策略、步骤和预算），这为记忆机制的设计提供了新思路：控制器需记住过去的推理步骤和策略选择，以做出更好的后续决策。
- 对评测: ACTS的预算条件奖励塑造为评估推理效率提供了新视角：不仅关注准确率，还关注token使用效率及准确率-效率的权衡曲线，避免了单一指标可能带来的误导。

### 11. 局限性

- 控制器训练需要大量专家推理轨迹进行行为初始化（从OpenR1-Math中筛选了7,500条），这些轨迹的获取和标注成本较高；策略集U={UNDERSTAND, PLAN, EXECUTE, EXPLORE, CHECK, SUMMARIZE, CONCLUDE}是预定义的，可能无法覆盖所有推理场景或任务类型。
- 控制器-推理器架构增加了系统复杂度；虽然通过异步流水线（SGLang服务器）缓解了延迟问题，但部署和维护成本增加，且控制器调用本身也会消耗少量额外token。
- 在部分基准上（如MATH-500上DeepSeek-R1-Distill-Qwen-7B），ACTS准确率（85.2%）与全思考（92.6%）仍有差距，表明策略级控制有时会牺牲一定准确率，此权衡是否可接受取决于应用场景。

### 12. 我的看法 / 个人笔记

ACTS是一个很有前景的推理效率优化框架，其核心创新在于将推理控制从长度控制提升到策略控制，通过显式选择推理策略来引导推理过程。这种方法不仅更精细、更可控，且具有更好的可解释性。特别值得关注的是，其预算条件奖励塑造设计巧妙，同时惩罚过度思考和过早终止，有效避免了常见的作弊行为。实验验证了该方法在不同模型和任务上的有效性，且token节省幅度显著（37%-64%）。不过，控制器训练依赖高质量专家轨迹，且预定义策略集可能限制了泛化能力。未来工作可探索更自动化的策略集学习、更高效的控制器训练方法，以及将ACTS扩展到更广泛的任务领域（如编程、逻辑推理等）。

### 13. 这篇工作引用的近期高影响力相关文章

- Chain-of-Thought Reasoning without Thinking (Ma et al., 2025a)
  为什么重要：提出了NoThink方法，通过预填充空思考块强制推理器跳过思考阶段，是ACTS的重要基线之一。
  核心贡献：证明了跳过思考阶段可以大幅节省token，但准确率显著下降（例如在MATH-500上从92.6%降至78.8%）。
  与当前论文关系：ACTS通过策略级控制实现了比NoThink更好的准确率-效率权衡（准确率85.2% vs 78.8%，token消耗更少）。
- DEER: Dynamic Early Exiting for Efficient Reasoning (Yang et al., 2026)
  为什么重要：提出了基于早停的高效推理方法，是ACTS的另一个重要基线。
  核心贡献：通过动态早停机制在推理过程中提前终止，节省token。
  与当前论文关系：ACTS在更多基准上实现了比DEER更好的准确率-效率权衡（例如在AIME24上准确率60.0% vs 50.0%，token减少更多）。

### 14. 总结

ACTS提出了一个智能体链式思维引导框架，通过将推理引导建模为马尔可夫决策过程，训练一个控制器智能体在推理过程中自适应地选择推理策略和引导短语。该框架在保持准确率接近全思考水平的同时，在多个数学推理和科学问答基准上实现了37%-64%的token节省，并提供了可控的准确率-效率权衡。实验在三种不同大小的推理模型上验证了其有效性和理由器无关性。

## 2. Benchmarking Visual State Tracking in Multimodal Video Understanding

**Topics:** 视觉状态追踪, 多模态大语言模型, 视频理解基准
**Authors:** Sihyun Yu, Nanye Ma, Pinzhi Huang, Hyunseok Lee, Shusheng Yang, June Suk Choi, Ellis Brown, Oscar Michel, Boyang Zheng, Jinwoo Shin, Saining Xie
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.03920v1) | [PDF](http://arxiv.org/pdf/2606.03920v1.pdf) | [Code](https://github.com/vision-x-nyu/vstat)

### 1. 一句话结论

当前最先进的多模态大语言模型在视觉状态追踪任务上远低于人类水平，主要瓶颈在于视觉感知而非推理能力。

### 2. 为什么重要

该研究揭示了现有视频理解基准的局限性，强调了视觉状态追踪作为视频理解核心能力的重要性，为未来模型改进提供了明确方向。

### 3. 研究问题

1. 当前多模态大语言模型在视觉状态追踪任务上的表现如何？
2. 模型失败的根本原因是视觉感知不足还是推理缺陷？

### 4. 背景与问题设定

视频理解需要持续追踪实体、状态和事件，但现有基准多依赖关键帧或显著时刻，无法评估模型的视觉状态追踪能力。VSTAT基准通过设计无法从单帧或短片段回答的问题，强制模型整合整个视频流的信息。

### 5. 方法概览

- 任务 / 环境: 包含合成（Blender渲染）和真实世界（YouTube及自录）视频，共834个片段，涵盖9种合成环境和多种真实场景。
- 变量 / 干预设计: 通过对比原始视频与文本转录条件下的模型表现，以及时间拉伸实验，隔离感知与推理的影响。
- 评价指标: 采用MCQ准确率和NQ的平均相对准确率（MRA），并报告人类表现和随机/频率基线。
- 模型比较: 评估了Gemini、Qwen3VL、LLaVA-OV等10余种模型，包括不同规模和思考模式，以及AVP和Claude Code等智能体框架。
- 我的理解: 该方法通过精心设计的基准和对比实验，系统性地诊断了MLLMs在视觉状态追踪上的缺陷，特别是通过文本转录实验直接证明了感知瓶颈。

### 6. 关键发现

- MLLMs在VSTAT上表现远低于人类: 最佳模型（Gemini-3.1 Pro）平均准确率仅44.4%，而人类达90.5%，模型仅略高于频率基线（37.8%）。
- 失败根源在于视觉感知而非推理: 在文本转录条件下，模型几乎完美解决任务，但视频条件下表现差，表明视觉感知是主要瓶颈。
- 智能体框架无法缓解失败: AVP和Claude Code等智能体方法在VSTAT上仍表现不佳，未能显著提升性能。

- 为什么重要: 直接证明了模型失败源于视觉感知而非推理，为后续研究指明了改进方向。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 当前智能体框架（如AVP）无法解决视觉状态追踪问题，需要更强的感知模块。
- 对 Skill: 模型需要提升对连续视觉流中事件的感知能力，而非仅依赖推理。
- 对 Memory: 状态更新和跨帧实体关联是失败关键，需改进记忆机制。
- 对评测: 现有基准高估了模型能力，VSTAT提供了更严格的测试。

### 11. 局限性

- 基准覆盖的感知挑战有限，可能未涵盖所有真实场景。
- 文本转录实验仅适用于简单任务，复杂任务无法手动转录。
- 未探索训练数据或模型架构对状态追踪能力的影响。

### 12. 我的看法 / 个人笔记

该工作精准定位了MLLMs在视频理解中的核心短板——视觉感知，而非推理，为未来研究提供了清晰路线图。但基准的合成视频比例较高，真实世界多样性有待加强。

### 13. 这篇工作引用的近期高影响力相关文章

- CP-Bench
  为什么重要：首个专门评估状态追踪的基准，但限于单一合成任务（计数立方体）。
  核心贡献：提出了状态追踪评估的概念。
  与当前论文关系：VSTAT扩展了其范围，涵盖多样化的真实世界场景和更复杂的追踪需求。
- VET-Bench
  为什么重要：评估视觉追踪能力，但仅基于shell game单一任务。
  核心贡献：强调了追踪隐藏对象的能力。
  与当前论文关系：VSTAT提供了更全面的任务集和更深入的分析。

### 14. 总结

VSTAT基准揭示了当前MLLMs在视觉状态追踪上的严重不足，主要瓶颈在于视觉感知而非推理，且现有智能体方法无法解决，为视频理解研究指明了关键改进方向。

## 3. EvoDS: Self-Evolving Autonomous Data Science Agent with Skill Learning and Context Management

**Topics:** GUI Agent, Data Science Agent, Reinforcement Learning, Skill Acquisition, Context Management
**Authors:** Zherui Yang, Fan Liu, Yansong Ning, Hao Liu
**Venue / Year:** KDD '26
**Links:** [Abstract](http://arxiv.org/abs/2606.03841v1) | [PDF](http://arxiv.org/pdf/2606.03841v1.pdf) | [Code](https://github.com/usail-hkust/EvoDS)

### 1. 一句话结论

EvoDS通过自主技能获取和自适应上下文压缩，在四个基准测试上平均提升28.9%，并消除了令牌溢出失败。

### 2. 为什么重要

现有数据科学代理受限于静态动作集和缺乏长期上下文管理，EvoDS通过层次化多智能体架构和强化学习实现了自我进化，显著提升了自动化数据科学任务的性能和可靠性。

### 3. 研究问题

1. 如何设计能够从经验中自主获取可复用技能的数据科学代理？
2. 如何有效管理数据科学工作流中不断增长的上下文，避免长上下文干扰？

### 4. 背景与问题设定

大型语言模型代理在自动化数据科学方面取得了进展，但现有方法依赖预定义工作流和固定动作空间，无法积累可复用经验，且缺乏有效的长期上下文管理，导致在多阶段迭代任务中表现不佳。

### 5. 方法概览

- 任务 / 环境: 数据科学任务，包括数据预处理、特征工程、模型选择、评估和可视化，涉及多步骤迭代工作流。
- 变量 / 干预设计: EvoDS采用层次化多智能体架构，包含一个Manager代理和多个专业子代理（如Cleaner、Featurizer等），通过自主技能获取（ASA）机制合成新技能，并通过自适应上下文压缩（ACC）策略动态管理上下文。
- 评价指标: 任务完成质量（如与参考答案比较或LLM评判）、技能获取效率、上下文压缩效果、令牌溢出失败率。
- 模型比较: 与DS-Agent、AutoKaggle、DeepAnalyze等开源数据科学代理在四个基准测试上比较。
- 我的理解: EvoDS通过将技能获取和上下文管理视为可学习的问题，利用强化学习联合优化，使代理能够像人类数据科学家一样从经验中学习并适应新任务。

### 6. 关键发现

- 性能显著提升: EvoDS在四个基准测试上平均优于最先进的开源数据科学代理28.9%，并完全消除了令牌溢出失败。
- 技能获取有效: 自主技能获取机制使代理能够合成、验证和复用可执行技能，随着经验积累持续改进。
- 上下文管理稳健: 自适应上下文压缩策略动态决定何时压缩和保留哪些信息，在长上下文任务中保持高性能。

### 7. 最重要的图

- 图来源: 图1：EvoDS代理框架概览
- 为什么重要: 该图展示了EvoDS的整体架构，包括层次化多智能体设计、自主技能获取和自适应上下文压缩的集成，以及两阶段训练流程，是理解方法核心的关键。

**Topics:** GUI Agent, Agentic Reinforcement Learning, Large Language Models, Tool Use
**Authors:** Hongye Cao, Nuo Yan, Haoyuan Deng, Ziwei Wang, Tianpei Yang, Jing Huo, Yuyao Zhang, Yang Gao
**Venue / Year:** arXiv 2026

### 1. 一句话结论

TAO-RL通过工具感知轨迹过滤和熵引导探索，显著提升了智能体强化学习的训练稳定性和推理性能。

### 2. 为什么重要

该工作解决了工具集成强化学习中训练不稳定和探索不足的核心矛盾，为构建更可靠的智能体系统提供了实用框架。

### 3. 研究问题

1. 如何同时稳定工具集成强化学习的训练并增强探索？
2. 现有方法为何无法充分处理工具交互带来的分布偏移和噪声？

### 4. 背景与问题设定

智能体强化学习通过工具使用增强LLM推理能力，但工具集成导致输入分布偏移和训练不稳定；过度保守的工具使用则限制探索。现有方法要么仅基于格式过滤轨迹，要么忽略工具执行成功与否，未能全面处理工具交互效应。

### 5. 方法概览

- 任务 / 环境: 数学推理基准测试（如AIME 24/25），模型需通过代码解释器执行程序并获取运行时反馈。

- 评价指标: Avg@16（平均准确率）、梯度范数、有效代码率。
- 模型比较: 与TIR、SimpleTIR、Curriculum方法对比，在Qwen2.5-7B等3种模型规模上评估。
- 我的理解: TAO-RL通过过滤无效轨迹（工具失败且答案错误）和退化查询（全对或全错）构建高质量训练分布，再在关键决策点（工具调用后高熵令牌）施加熵奖励，实现稳定训练与定向探索的协同。

### 6. 关键发现

- 双准则轨迹过滤提升训练稳定性: 过滤掉工具执行失败且答案错误的轨迹，以及全对或全错的查询，保留工具有效且答案多样化的轨迹，显著降低梯度范数波动。
- 熵引导探索增强推理性能: 在工具调用后高熵令牌处施加归一化熵奖励，鼓励策略探索多样化推理路径，在AIME 24/25上Avg@16分别提升至约30和25。
- TAO-RL优于现有方法: 在7个数学推理基准和3种模型规模上，TAO-RL一致超越TIR、SimpleTIR和Curriculum方法，且训练更稳定。

## 5. Skill-RM: Unifying Heterogeneous Evaluation Criteria via Agent Skill

**Topics:** Reward Modeling, LLM Post-training, Agent Skill, Evaluation Criteria
**Authors:** Tao Chen, Gangwei Jiang, Pengyu Cheng, Siyuan Huang, Yihao Liu, Jingwei Ni, Jiaqi Guo, Mengyu Zhou, Kai Tang, Junling Liu, Qinliang Su, Xiaoxi Jiang, Guanjun Jiang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.03980v1) | [PDF](http://arxiv.org/pdf/2606.03980v1.pdf) | [Code](https://github.com/Qwen-Applications/Skill-RM)

### 1. 一句话结论

Skill-RM 将奖励建模重构为可复用的智能体技能执行过程，通过动态编排异构评估资源，在多种下游任务中一致超越传统评判基线。

### 2. 为什么重要

当前奖励模型依赖规则验证器、参考答案、程序化检查表和复杂评分标准等异构标准，缺乏统一整合机制。Skill-RM 首次提出以智能体技能为抽象层，将评估逻辑外化为可复用的技能包，实现评估的实例自适应、证据可追溯和模块化，为 LLM 后训练提供可扩展的高质量反馈信号。

### 3. 研究问题

1. 如何统一整合规则验证、参考答案、检查表和评分标准等异构评估标准？
2. 将奖励计算建模为结构化智能体任务能否提升评估的一致性和透明度？

### 4. 背景与问题设定

LLM 后训练（如强化微调 RFT 和强化学习 RL）依赖奖励模型提供反馈信号。但现有评估标准异构（规则验证器、参考答案、程序化检查表、复杂评分标准），缺乏统一机制整合所有证据。标量奖励模型将复杂证据压缩为不透明分数，LLM-as-a-Judge 系统依赖非结构化扁平提示，资源选择、证据追踪和信号聚合隐式且不可控。近期工作仅关注单一资源模态，亟需统一、可复用的抽象层来无缝合成异构资源。

### 5. 方法概览

- 任务 / 环境: LLM 后训练中的奖励建模任务，包括点式评分、成对比较和最佳-of-N 选择，覆盖数学、代码、安全、智能体轨迹等多样化评估场景。
- 变量 / 干预设计: 提出 Reward-Evaluation Skill，包含技能规范（SKILL.md）和资源库（评分标准、参考答案、检查表、验证器、校准规则）。智能体评判模型按诊断→选择→验证→聚合的流程执行技能，动态检索和调用相关资源，生成结构化证据集合并最终读出奖励分数。
- 评价指标: 在奖励基准和下游应用（最佳-of-N 选择、强化学习）上，以胜率或准确率与基线（传统评判模型）比较。
- 模型比较: 与传统标量奖励模型、LLM-as-a-Judge 系统（如 GPT-4 评判）以及仅增加上下文或工具可用性的消融变体对比。
- 我的理解: Skill-RM 的核心创新在于将评估知识从模型参数或扁平提示中外化为可复用的技能包，通过显式的资源选择、执行和聚合协议，使奖励计算变得可解释、可适应和可审计。

### 6. 关键发现

- 统一框架超越传统基线: 在奖励基准和下游应用（最佳-of-N 选择、RFT）上，Skill-RM 一致优于传统评判模型，证明技能介导的证据编排显著提升奖励质量。
- 增益源于技能编排而非上下文增加: 消融实验表明，性能提升来自技能介导的资源选择与执行逻辑，而非单纯增加上下文长度或工具可用性。
- 结构化证据提升可解释性: Skill-RM 生成包含每个标准局部证据的结构化判断，使最终奖励可追溯至具体证据，增强了透明度和可信度。
