## 1. PsyScore: A Psychometrically-Aware Framework for Trait-Adaptive Essay Scoring and ZPD-Scaffolded Feedback

**Topics:** Automated Essay Scoring, Psychometrics, Large Language Models, Educational Feedback
**Authors:** Wei Xia, Jin Wu, Haoran Shi, Xiangyu Wang, Chanjin Zheng
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20287v1) | [PDF](http://arxiv.org/pdf/2606.20287v1.pdf)

### 1. 一句话结论

PsyScore integrates psychometric IRT with LLM-based feedback to produce interpretable scores and ZPD-aligned instructional feedback.

### 2. 为什么重要

It bridges the gap between scoring and feedback in AES, providing a principled framework that enhances both interpretability and pedagogical alignment.

### 3. 研究问题

1. How can latent ability from IRT be used to unify scoring and feedback generation?
2. Does ZPD-adaptive feedback improve revision outcomes compared to generic feedback?

### 4. 背景与问题设定

Current AES systems treat scoring and feedback separately: neural scorers lack interpretability, and LLM feedback is often ability-agnostic, leading to cognitive mismatch. PsyScore addresses this by using a shared latent ability representation from Item Response Theory to guide both assessment and instruction.

### 5. 方法概览

- 任务 / 环境: Automated essay scoring and feedback generation on the ASAP++ dataset, which includes holistic and multi-trait scores.
- 变量 / 干预设计: PsyScore uses a Trait-Adaptive Neural GPCM Scorer to estimate latent ability θ, then a ZPD-Conditional Feedback Generator that maps θ to pedagogical strategies (explicit correction, scaffolding, Socratic questioning) via multi-agent fusion.
- 评价指标: Quadratic Weighted Kappa (QWK) for scoring; pairwise preference win rate and normalized revision gain for feedback quality.
- 模型比较: Compared against STL-LSTM, HISK, MTL-BiLSTM, DualTrans, ArTS, SaMRL-large, T-MES, and Shibata & Uto (2022).
- 我的理解: The framework cleverly uses IRT's latent trait θ as a bridge: it makes scoring interpretable and simultaneously conditions feedback on the same ability estimate, ensuring coherence.

### 6. 关键发现

- State-of-the-art scoring performance: PsyScore-AES achieves an average QWK of 0.747 across prompts, outperforming all baselines, especially on narrative prompts (Prompt 8: 0.730 vs. previous best 0.649).
- IRT module improves scoring: Ablation without IRT drops average QWK from 0.747 to 0.705, confirming the psychometric component's contribution.
- ZPD-adaptive feedback yields higher revision gains: PsyScore-AEF feedback leads to higher normalized revision gains compared to generic LLM feedback, indicating better pedagogical scaffolding.

- 为什么重要: It visually contrasts the fragmented pipeline (black-box scorer + generic feedback) with PsyScore's integrated loop (interpretable scoring → ZPD-aligned feedback), highlighting the core innovation.

### 8. 如何读这张图

Left panels show traditional AES: a black-box scorer outputs a holistic score, and a separate LLM generates generic feedback, leading to mismatch. Right panel shows PsyScore: the Trait-Adaptive Neural IRT Scorer produces a latent ability θ and diagnostic vector Dx, which directly informs the ZPD-Conditional Feedback Generator to produce scaffolded feedback, forming a cohesive formative loop.

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: The multi-agent fusion approach (heterogeneous generators + expert aggregator) can be adapted for other educational tasks requiring personalized output.
- 对 Skill: The ZPD strategy mapping provides a template for adaptive skill scaffolding in intelligent tutoring systems.
- 对 Memory: The latent ability θ serves as a compact memory of learner state, enabling consistent personalization across interactions.
- 对评测: The multi-perspective evaluation (pairwise comparison, simulated revision, human expert) offers a robust framework for assessing open-ended feedback quality.

### 11. 局限性

- The feedback evaluation relies on simulated student agents, which may not fully capture real learner behavior.
- The ZPD boundaries (−1.0, 1.0) are heuristic and may need calibration for different domains or age groups.
- The framework is tested only on ASAP++ dataset; generalizability to other writing tasks or languages is not verified.

### 12. 我的看法 / 个人笔记

PsyScore is a well-designed integration of psychometrics and LLMs, addressing a real gap in AES. The use of IRT latent ability as a shared representation is elegant and principled. However, the reliance on simulated revision for feedback evaluation is a limitation; real-world user studies would strengthen the claims. The multi-agent fusion is computationally expensive but likely necessary for quality.

## 2. SPOT-E: Test-Time Entropy Shaping with Visual Spotlights for Frozen VLMs

**Topics:** Vision-Language Models, Test-Time Adaptation, Visual Grounding, Entropy Optimization
**Authors:** Bo Yin, Xiaobin Hu, Chengming Xu, Ruolin Shen, Mo Yang, Jiangning Zhang, Peng-Tao Jiang, Cheng Tan, Shuicheng YAN
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20244v1) | [PDF](http://arxiv.org/pdf/2606.20244v1.pdf) | [Code](https://github.com/YinBo0927/SPOT-E)

### 1. 一句话结论

SPOT-E uses test-time entropy shaping with visual spotlights to improve evidence utilization in frozen VLMs, achieving consistent gains across benchmarks.

### 2. 为什么重要

It addresses the evidence utilization gap in VLMs without retraining, offering a plug-and-play solution that enhances performance on evidence-intensive tasks and robustness to visual corruptions.

### 3. 研究问题

1. How can we design a test-time intervention that is both instance-adaptive and self-verifying?
2. Can answer entropy serve as a label-free signal for evidence utilization in frozen VLMs?

### 4. 背景与问题设定

VLMs often fail on evidence-intensive tasks because decisive visual evidence is small and localized, leading to a gap between high-level reasoning and fine-grained evidence extraction. Prior inference-time visual interventions are open-loop and lack verification of evidence usage.

### 5. 方法概览

- 任务 / 环境: Evidence-intensive visual question answering tasks (e.g., chart reading, document parsing) with frozen VLMs.
- 变量 / 干预设计: A question-conditioned visual spotlight module generates soft masks to highlight relevant regions; the intervened image is optimized per instance via GRPO using an entropy-shaping reward.
- 评价指标: Accuracy on benchmarks (TextVQA, DocVQA, ChartQA, MathVista, MMMU, GQA, MMBench, POPE) and robustness under visual corruptions.
- 模型比较: Compared against inference-time visual prompting baselines (e.g., FGVP, Set-of-Mark) and applied to both open-source (Qwen2.5-VL, LLaVA, InternVL) and closed-source (GPT-4o, Gemini) VLMs.
- 我的理解: The method treats the VLM as a black box and optimizes a lightweight spotlight module to reduce answer entropy while preserving baseline confident tokens, using GRPO for per-instance adaptation.

### 6. 关键发现

- Entropy reduction is ambiguous: Naive entropy minimization can lead to shortcut collapse (confident but wrong) as well as evidence-grounded confidence; low-entropy anchors help disambiguate.
- SPOT-E yields consistent gains across backbones: On 8 benchmarks, SPOT-E improves accuracy for both open-source and closed-source VLMs, with gains up to +6.2% on TextVQA for LLaVA-NeXT-7B.
- Improved robustness under visual corruptions: SPOT-E maintains performance under controlled visual corruptions, indicating better evidence utilization.

### 7. 最重要的图

- 图来源: Figure 2: Entropy reduction can be misleading
- 为什么重要: It illustrates the core ambiguity of entropy reduction: low entropy can arise from either evidence-grounded confidence or shortcut collapse, motivating the need for entropy shaping.

### 8. 如何读这张图

The figure shows answer entropy (Hans) as a function of blur strength α on the most entropy-sensitive region. Hans initially rises as evidence becomes ambiguous, then drops when evidence is erased, indicating shortcut-driven overconfidence.

### 9. 次重要图

- 为什么也重要: Figure 3 demonstrates how low-entropy anchors distinguish evidence-revealing from destructive interventions, supporting the anchor-preservation reward.

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: SPOT-E can be integrated into GUI agents to improve visual grounding in dynamic environments without retraining.
- 对 Skill: The entropy-shaping principle can be applied to other skills requiring fine-grained evidence extraction, such as document parsing.
- 对 Memory: The method does not require memory of past instances; each instance is optimized independently.
- 对评测: Answer entropy can serve as a proxy for evidence utilization, enabling label-free evaluation of grounding quality.

### 11. 局限性

- Per-instance optimization adds computational overhead at test time.
- The method relies on a structured output format (e.g., 'Final answer: ...') to compute answer entropy.
- The spotlight module uses CLIP, which may have its own biases and limitations.

### 12. 我的看法 / 个人笔记

SPOT-E presents a clever use of entropy as a feedback signal for test-time adaptation, effectively addressing the ambiguity of entropy reduction with anchor preservation. The method is practical for frozen VLMs and shows strong empirical results, though the per-instance optimization cost may limit real-time applications.

### 13. 这篇工作引用的近期高影响力相关文章

- FGVP: Fine-Grained Visual Prompting
  为什么重要：A prior inference-time visual intervention method that highlights regions but is open-loop.
  核心贡献：Uses fixed heuristics to overlay marks on images to improve grounding.
  与当前论文关系：SPOT-E extends this by adding closed-loop feedback via entropy shaping and per-instance optimization.
- Set-of-Mark Prompting
  为什么重要：Another visual prompting approach that uses marks to guide VLMs.
  核心贡献：Overlays marks on objects to improve segmentation and grounding.
  与当前论文关系：SPOT-E uses soft masks instead of discrete marks and optimizes them per instance.
- ViCrop: Visual Cropping for VLMs
  为什么重要：A method that crops images to focus on relevant regions.
  核心贡献：Selectively crops or zooms into candidate areas to preserve small details.
  与当前论文关系：SPOT-E uses a spotlight mask that can highlight multiple regions without cropping, preserving global context.

### 14. 总结

SPOT-E is a test-time visual adaptation framework that uses entropy shaping with low-entropy anchors to improve evidence utilization in frozen VLMs. It optimizes a question-conditioned spotlight via GRPO, achieving consistent accuracy gains across diverse benchmarks and backbones while enhancing robustness to visual corruptions.

## 3. A Multi-Agent system for Multi-Objective constrained optimization

**Topics:** 多智能体强化学习, 约束优化, 边缘计算资源分配
**Authors:** Federica Filippini
**Venue / Year:** AAMAS 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20236v1) | [PDF](http://arxiv.org/pdf/2606.20236v1.pdf) | [Code](https://github.com/FFede0/RL4CC)

### 1. 一句话结论

MAMO通过分层多智能体框架将奖励权重选择转化为学习问题，自动平衡成本与约束违规，无需手动调参。

### 2. 为什么重要

传统RL解决约束优化时依赖手动设定权重，在动态环境中难以适应。MAMO首次将权重调整作为独立学习任务，提升了系统的自主性和鲁棒性。

### 3. 研究问题

1. 如何自动学习奖励权重以平衡成本最小化与约束满足？
2. 分层多智能体框架能否在非平稳环境中自适应调整优化目标？

### 4. 背景与问题设定

计算与网络系统中的决策问题常建模为带性能约束的成本最小化问题。RL通过加权惩罚项将约束融入奖励，但权重需手动设定，在动态环境中难以找到合适平衡。

### 5. 方法概览

- 任务 / 环境: 边缘FaaS环境中的副本缩放问题：动态工作负载下选择函数副本数，最小化启动成本并满足响应时间约束（拒绝概率<0.05）。
- 变量 / 干预设计: 分层架构：任务执行（TE）智能体使用加权奖励学习控制策略；权重适应（WA）智能体基于TE的长期性能指标（平均成本与拒绝概率）调整权重，每15k步更新一次。
- 评价指标: 平均执行成本、请求拒绝概率（需低于容忍阈值0.05）。
- 模型比较: 与离线最优解（Gurobi求解P1）、固定权重极端值（w=0.99和0.1）对比，MAMO在噪声负载下能自适应保持约束满足。
- 我的理解: MAMO将权重选择视为外循环的序贯决策问题，WA智能体通过观察TE的聚合性能来调整权重，从而解耦任务执行与目标设计，使系统能根据经验自动演化最优权衡。

### 6. 关键发现

- MAMO自动收敛到满足约束的权重配置: 在噪声负载下，WA智能体训练过程中拒绝概率逐渐接近0.05阈值，权重收敛至0.8-0.9之间，表明系统能自主找到成本与约束的平衡点。
- 固定权重导致极端行为: w=0.99时过度保守（高成本低违规），w=0.1时过度激进（低成本高违规），而MAMO能动态调整避免此类问题。

- 为什么重要: 直观展示了MAMO如何通过迭代学习使拒绝概率逼近容忍阈值，验证了权重自适应机制的有效性。

- 对 Agent 系统: 提供一种分层架构，使系统能自主适应目标权衡，减少人工干预。

## 4. Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages

**Topics:** Code Generation, Multilingual Benchmark, Large Language Models
**Authors:** Maria Ivanova, Pavel Zadorozhny, Rodion Levichev, Ivan Petrov, Adamenko Pavel, Ivan Lopatin, Alexey Kutalev, Dmitrii Babaev
**Venue / Year:** ICLR 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20517v1) | [PDF](http://arxiv.org/pdf/2606.20517v1.pdf) | [Code](https://github.com/Multi-LCB/Multi-LCB)

### 1. 一句话结论

Multi-LCB将LiveCodeBench扩展至12种编程语言，通过统一STDIN/STDOUT格式和自动转换管道，揭示LLM在Python之外存在显著性能差距和过拟合现象，且语言特定污染因语言而异。

### 2. 为什么重要

现有代码生成基准多局限于Python，无法反映真实软件工程中多语言需求。Multi-LCB通过保持污染控制和评估协议，系统评估LLM跨语言泛化能力，暴露了当前模型的关键短板，为开发更鲁棒的多语言代码模型提供了基准。

### 3. 研究问题

1. LLM能否将代码生成能力泛化到多种编程语言？
2. 模型是否存在Python过拟合和语言特定数据污染？
3. 推理型与指令型模型在多语言任务上表现有何差异？

### 4. 背景与问题设定

LiveCodeBench (LCB) 是广泛采用的代码生成基准，通过持续更新和日期过滤实现污染感知评估，但仅支持Python。真实软件开发涉及多种语言，LLM在Python上的表现不能代表其他语言。Multi-LCB将LCB任务转换为12种语言，保持相同评估协议，以系统评估跨语言能力。

### 5. 方法概览

- 任务 / 环境: 12种编程语言（C++, C#, Python, Java, Rust, Go, TypeScript, JavaScript, Ruby, PHP, Kotlin, Scala），任务来自LeetCode（函数式）、AtCoder和Codeforces（STDIN/STDOUT），使用统一STDIN/STDOUT格式评估。
- 变量 / 干预设计: 自动转换管道：对LeetCode函数式任务，解析问题示例和测试用例，转换为STDIN/STDOUT格式（标量直接输出，一维数组空格分隔，二维数组首行指定行数后逐行空格分隔）；AtCoder/Codeforces任务保持原格式。零样本提示，模型需生成完整程序。
- 评价指标: Pass@1（首次生成通过所有隐藏测试的比例），10次运行平均。
- 模型比较: 评估24个LLM（7B-685B参数），包括推理型（如DeepSeek-R1-0528、Qwen3-235B-A22B-Thinking-2507）和指令型（如Qwen3-235B-A22B-Instr-2507、Qwen2.5-Coder-32B-Instr）。推理型模型在Python上平均高出指令型约15%，但在非Python语言上优势缩小，如OpenReasoning-Nemotron-32B在Python上64.4%，但在JavaScript上仅10.8%。
- 我的理解: 该方法巧妙地将Python任务转化为多语言任务，通过统一I/O格式避免语言特定模板，确保公平比较。自动转换和持续更新设计使其可扩展。

### 6. 关键发现

- Python不是非Python语言的可靠代理: 模型在Python上的优势不能推广到其他语言，存在显著性能差距。例如，OpenReasoning-Nemotron-32B在Python上64.4%，但在JavaScript上仅10.8%；DeepSeek-R1-Distill-Qwen-32B在Python上39.4%，但在Scala上仅7.0%。
- Python过拟合普遍存在: 许多模型在Python上表现强，但在其他语言上性能急剧下降。例如，GPT-OSS-120B（Medium）在Python上71.1%，但在Scala上仅54.1%；Qwen3-30B-A3B-Thinking-2507在Python上64.0%，但在Ruby上仅42.1%。
- 语言特定数据污染: 不同语言的数据泄漏程度不同。例如，DeepSeek-R1-0528在C++上68.0%（接近Python的66.3%），而OpenReasoning-Nemotron-32B在C++上44.2%（远低于Python的64.4%），表明某些模型在C++上可能受益于预训练语料中的高比例。类似地，DeepSeek-R1-Distill-Qwen-14B在Ruby上34.5%（高于其他非Python语言），暗示Ruby特定污染。
- 多语言性能差异大: 模型在静态类型或较少使用语言（如Scala、Ruby）上表现更弱。平均Pass@1在Python上最高（约50%），Scala最低（约30%），差距达20%以上。推理型模型在Python上平均64.0%，在Scala上仅42.2%；指令型模型在Python上平均37.5%，在Scala上仅28.1%。

## 5. Probe-and-Refine Tuning of Repository Guidance for Coding Agents

**Topics:** LLM-based coding agents, repository guidance, probe-and-refine tuning
**Authors:** Asa Shepard, Jeannie Albrecht
**Venue / Year:** arXiv preprint, June 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.20512v1) | [PDF](http://arxiv.org/pdf/2606.20512v1.pdf) | [Code](https://github.com/asashepard/probe-and-refine-tuning)

### 1. 一句话结论

Probe-and-refine tuning, a lightweight iterative procedure using synthetic bug-fix probes, significantly improves coding agent resolve rates on SWE-bench Verified by enhancing evaluation coverage rather than patch precision.

### 2. 为什么重要

This work resolves a critical debate on whether repository guidance files (e.g., AGENTS.md) help or harm coding agents, showing that the method of producing guidance is decisive. It introduces a practical, low-cost method to generate effective guidance, demonstrating that instruction quality is a key bottleneck in current coding agents.

### 3. 研究问题

1. Does the method of producing repository guidance determine its effectiveness for coding agents?
2. Can a lightweight, single-shot LLM-based procedure iteratively refine guidance to improve agent performance?
3. Does improved guidance enhance agent performance through better patch coverage or higher patch precision?

### 4. 背景与问题设定

LLM-based coding agents require operational knowledge about a repository (e.g., file organization, test procedures) that is not present in the code itself. Engineers use AGENTS.md files to provide this context, but studies disagree on whether such guidance helps. This paper hypothesizes that the production method is the key variable and introduces probe-and-refine tuning to iteratively improve guidance using synthetic bug-fix probes.

### 5. 方法概览

- 任务 / 环境: SWE-bench Verified benchmark: real GitHub issues from popular open-source Python repositories; agent must produce a patch that passes held-out tests.
- 变量 / 干预设计: Three conditions: no_context (no guidance), static_kb (two-layer knowledge base with structural and generic procedural guidance), and probe_refined (guidance refined via probe-and-refine tuning from static_kb). All other components (scaffold, model, harness) fixed.
- 评价指标: Resolve rate (percentage of instances where agent patch passes tests), evaluable patch rate, per-patch precision.
- 模型比较: Primary model: Qwen3.5-35B-A3B (agent step budget of 200). Cross-model experiment: NVIDIA-Nemotron-3-Nano-30B-A3B.
- 我的理解: The method isolates the effect of guidance by holding the agent scaffold and model constant across conditions. Probe-and-refine tuning uses only single-shot LLM calls (no agent loop or tool use) to iteratively diagnose and patch guidance based on synthetic probe failures, producing a compact text artifact per repository.

### 6. 关键发现

- Probe-and-refine tuning significantly improves resolve rates: On SWE-bench Verified with Qwen3.5-35B-A3B at an agent step budget of 200, probe-refined guidance achieves 33.0% mean resolve rate vs. 28.3% for static_kb and 25.5% for no_context (p < 0.001 for both contrasts).
- Improvement stems from increased coverage, not precision: Refined guidance produces evaluable patches for 14.5 percentage points more instances, while per-patch precision remains statistically constant (~59%, p = 0.119). This indicates better guidance helps agents reach the correct file, not improve patch quality.
- Guidance enables productive use of larger step budgets: A step-budget experiment shows that guidance allows the agent to use a larger step budget productively; without guidance, unstructured exploration saturates quickly regardless of budget.
- Cross-model generalization depends on model capacity: With a weaker model (NVIDIA-Nemotron-3-Nano-30B-A3B), the tuning loop degrades because the model cannot generate sufficiently diagnostic output, though per-patch precision remains constant.

The refined guidance (probe_refined) is then used by the coding agent on SWE-bench Verified issues. No evaluation instances are used during refinement.

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: Guidance quality is a critical bottleneck; lightweight iterative refinement of instructions can substantially improve agent performance without changing the model or scaffold.
- 对 Skill: The procedure distills operational knowledge (e.g., which test to run, which files to trace) into compact guidance, effectively transferring implicit repository expertise to the agent.
- 对 Memory: Guidance serves as persistent memory for repository-level knowledge, reducing the need for the agent to reconstruct it on the fly.
- 对评测: Improvement in coverage (more evaluable patches) rather than precision suggests evaluation should consider both metrics; guidance helps agents localize issues correctly.

### 11. 局限性

- The procedure is evaluated only on SWE-bench Verified with Python repositories; generalization to other languages or benchmarks is untested.
- Guidance length increases with refinement (63% longer on average), and the effects of content vs. length are not fully disentangled.
- The procedure requires a model capable of generating diagnostic output; weaker models may not sustain the tuning loop.
- The compact single-file guidance may not scale to very large repositories (>100k lines).

### 12. 我的看法 / 个人笔记

This paper provides a compelling resolution to a practical debate in coding agent research, showing that how guidance is produced is crucial. The probe-and-refine method is elegantly simple and practical, using only single-shot LLM calls. The finding that improvement comes from coverage rather than precision is insightful, suggesting that guidance primarily helps agents navigate the codebase correctly. The cross-model limitation is important for deployment. Future work could explore scaling to larger repositories and integrating with structured knowledge representations.
