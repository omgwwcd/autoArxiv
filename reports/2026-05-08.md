## 1. NeuroAgent: LLM Agents for Multimodal Neuroimaging Analysis and Research

**Topics:** LLM智能体, 多模态神经影像, 医学图像分析, 自动化预处理
**Authors:** Lujia Zhong, Yihao Xia, Jianwei Zhang, Shuo huang, Jiaxin Yue, Mingyang Xia, Yonggang Shi
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.06584v1) | [PDF](http://arxiv.org/pdf/2605.06584v1.pdf)

### 1. 一句话结论

NeuroAgent是一个基于LLM的智能体框架，能够自动化处理多模态神经影像数据（sMRI、fMRI、dMRI、PET）的完整分析流程，从预处理到下游分类，显著减少人工干预。

### 2. 为什么重要

多模态神经影像分析流程复杂、工具链异构且高度依赖专家手动配置与质量控制，这是阻碍大规模、可重复性研究的主要瓶颈。该工作首次将LLM智能体系统应用于从原始数据到科学发现的完整神经影像分析生命周期，有望大幅降低领域专家的操作负担。

### 3. 研究问题

1. 如何利用LLM智能体自动化处理异构、复杂的多模态神经影像预处理与分析流程？
2. LLM驱动的智能体系统能否在保证准确性的前提下，有效减少神经影像研究中对专家手动劳动的依赖？

### 4. 背景与问题设定

神经影像分析正朝着多模态、大规模队列和精准医学方向发展。然而，将原始采集数据转化为可用于分析的图像，需要经过漫长、模态特异且依赖多种工具（如FSL、SPM、ANTs）的预处理流程，并涉及大量手动配置、质量控制和参数调整。现有自动化框架（如fMRIPrep）在处理非标准数据或工具失败时仍需人工干预，且分析流程的差异可能导致研究结论不一致。

### 5. 方法概览

- 任务 / 环境: 处理来自ADNI（阿尔茨海默病神经影像倡议）数据集的1,470名受试者的多模态神经影像数据（sMRI、fMRI、DTI、Tau-PET）及表格数据，目标包括自动化预处理、群体统计分析和AD分类。
- 变量 / 干预设计: 提出NeuroAgent框架：采用分层多智能体架构，包含中央协调器（负责意图解析和任务分解）和多个模态专用智能体（负责执行特定工具链）。核心是“生成-执行-验证”反馈引擎，能自动生成代码、执行工具、捕获错误并自我纠正。
- 评价指标: 意图解析准确率、端到端预处理步骤正确率、阿尔茨海默病分类的ROC-AUC。
- 模型比较: 在多个LLM后端上进行消融实验，性能最强的后端（Qwen3.5-27B）达到100%意图解析准确率和84.8%的预处理步骤正确率。
- 我的理解: 该工作并非用LLM直接分析图像，而是将其作为高层“规划者”和“协调器”，利用其推理能力来编排和调用底层的确定性专业工具（如ANTs、FSL），并通过闭环反馈机制处理执行中的异常，实现了从静态脚本到情境感知、自适应流程的转变。

### 6. 关键发现

- 高效的自动化预处理: 在ADNI数据集上，使用Qwen3.5-27B后端，系统实现了84.8%的端到端预处理步骤正确率，自动化恢复机制将人工干预限制在需要通过人机交互（HITL）接口处理的边缘情况。
- 强大的下游任务性能: 使用自动预处理后的四模态数据，智能体集成在AD分类任务上达到了0.9518的AUC，超越了所有单模态基线，证明了自动化流程产出数据的有效性。
- 系统具备临床一致性验证: 在自动预处理的sMRI和DTI数据上进行的群体分析，成功复现了AD相关的皮层变薄和与年龄相关的各向异性分数下降等临床预期效应，表明流程输出符合既定的临床认知。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 证明了分层多智能体架构与“生成-执行-验证”反馈引擎在复杂、长周期科学工作流中的有效性，为将LLM智能体应用于其他需要编排专业工具链的领域（如计算生物学、天文物理）提供了范本。
- 对 Skill: 系统将领域专家的预处理、质量控制和流程编排技能编码到了智能体的规划、工具调用和错误恢复逻辑中，实现了专家技能的“封装”与“自动化”。

### 11. 局限性

- 性能高度依赖底层LLM后端的能力，如文中所示，不同后端在预处理步骤正确率上差异显著。
- 系统虽然能处理异构数据，但其工具库和知识库的覆盖范围可能仍无法应对所有边缘情况或最新出现的影像模态/分析方法，需要持续维护和扩展。

### 12. 我的看法 / 个人笔记

这是一项具有重要实用价值的前瞻性工作。它没有追求用LLM替代专业影像处理工具，而是巧妙地将其定位为“智能协调员”，这避开了LLM在确定性计算和医学可靠性上的短板，放大了其在规划、理解和适应复杂情境上的优势。如果此类系统成熟，将极大降低神经影像研究的入门门槛并提升分析的可重复性。

## 2. Beyond Task Success: Measuring Workflow Fidelity in LLM-Based Agentic Payment Systems

**Topics:** GUI Agent, 多智能体系统, 评估指标, 支付系统
**Authors:** Donghao Huang, Joon Kiat Chua, Zhaoxia Wang
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.06457v1) | [PDF](http://arxiv.org/pdf/2605.06457v1.pdf)

### 1. 一句话结论

本文提出了Agentic Success Rate (ASR)，一种用于评估LLM多智能体支付系统工作流保真度的轨迹级指标，揭示了传统指标无法检测的系统性流程偏差。

### 2. 为什么重要

在支付等受监管领域，审计追踪和流程合规性与最终结果同等重要。现有指标（如任务成功率TSR）仅关注最终结果，无法捕捉智能体执行工作流的顺序偏差，可能隐藏合规风险。

### 3. 研究问题

1. 如何量化评估LLM多智能体系统对预期工作流序列的遵循程度？
2. 在支付等关键领域，传统评估指标在捕捉流程偏差方面存在哪些盲点？

### 4. 背景与问题设定

LLM多智能体系统正被部署于端到端的支付工作流自动化中。然而，现有评估框架（如HMASP系统使用的任务成功率TSR和智能体交接F1分数HF1）仅关注最终结果或无序的路由决策，无法衡量工作流保真度，即智能体对预期步骤序列的遵循程度。这在需要严格审计的支付领域是一个关键缺陷。

### 5. 方法概览

- 任务 / 环境: 在Hierarchical Multi-Agent System for Payments (HMASP) 框架中评估，包含四个场景：卡片注册、卡片检索、支付处理、无关输入拒绝。
- 变量 / 干预设计: 引入Agentic Success Rate (ASR) 指标，基于流程挖掘中的一致性检查思想，在转移（二元组）级别比较观察到的与预期的工作流轨迹。
- 评价指标: Task Success Rate (TSR), Agent Handoff F1-Score (HF1), Agentic Success Rate (ASR) 及其分解指标 Transition Recall (TR) 和 Transition Precision (TP)。
- 模型比较: 评估了18个LLM（包括开源和专有模型，如GPT-4.1, GPT-5.2, Qwen系列, Llama系列等），在90,000个任务实例上进行。
- 我的理解: 该方法的核心创新是将工作流视为有序的智能体转移序列，并通过计算预期转移集和观察转移集之间的重叠度来量化保真度，从而直接衡量流程合规性。

### 6. 关键发现

- ASR揭示了传统指标无法检测的隐藏工作流捷径: 在支付处理场景中，10个模型（包括GPT-41）系统性地跳过了支付前的一个用户确认检查点，尽管它们达到了完美的TSR和HF1。这种偏差对传统指标不可见，但ASR能捕捉到（T3 ASR存在0.01%-5.45%的差距）。
- 模型表现存在明显分化: 18个模型中，有8个模型（包括GPT-5.2、所有Gemma4变体和GPT-OSS模型）在所有指标上均达到100%，完美执行了工作流。其余10个模型则表现出相同的系统性捷径模式。
- ASR诊断能指导系统优化并大幅提升性能: 基于ASR分析进行的提示工程和确定性路由守卫，使先前表现不佳的模型（如Llama3.1:8b）的任务成功率获得了高达+93.8个百分点的提升，证明了工程干预的有效性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为复杂、多步骤的智能体系统评估提供了新的维度，强调了对过程而不仅仅是结果的监控。
- 对 Skill: 表明即使智能体具备完成任务的“技能”，也可能缺乏遵循特定安全或合规流程的“纪律”，这需要被单独评估和优化。
- 对 Memory: 工作流保真度评估可能需要对执行历史（轨迹）进行更结构化的记录和比对，这对智能体的记忆或系统的日志提出了要求。
- 对评测: 在安全关键或受监管的领域，评估框架必须包含过程一致性指标，ASR为此提供了一个可操作的范式。

### 11. 局限性

- 当前工作主要在一个特定的支付系统（HMASP）和数据集上进行验证，其普适性有待在其他领域和架构中检验。
- ASR将每个转移视为同等重要，未来可能需要加权变体以区分安全关键转移和常规转移。

### 12. 我的看法 / 个人笔记

这项工作敏锐地指出了当前LLM智能体评估中的一个关键缺口：对过程合规性的忽视。ASR指标简单而有力，其基于流程挖掘的灵感来源使其具有扎实的理论基础。在AI智能体日益渗透到金融、医疗等高风险领域的背景下，这种对“工作流如何执行”而不仅仅是“任务是否完成”的关注，至关重要且及时。

## 3. StraTA: Incentivizing Agentic Reinforcement Learning with Strategic Trajectory Abstraction

**Topics:** 强化学习, 大语言模型智能体, 长视野决策
**Authors:** Xiangyuan Xue, Yifan Zhou, Zidong Wang, Shengji Tang, Philip Torr, Wanli Ouyang, Lei Bai, Zhenfei Yin
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.06642v1) | [PDF](http://arxiv.org/pdf/2605.06642v1.pdf) | [Code](https://github.com/xxyQwQ/StraTA)

### 1. 一句话结论

StraTA框架通过引入显式的轨迹级策略抽象，将长视野智能体强化学习分解为策略生成与动作执行，显著提升了样本效率与最终性能。

### 2. 为什么重要

当前基于大语言模型的智能体在长视野决策任务中多为纯反应式，导致探索短视、信用分配困难。引入显式策略指导是提升其长期规划与行为一致性的关键。

### 3. 研究问题

1. 如何为智能体强化学习引入有效的轨迹级抽象以指导长视野决策？
2. 如何设计训练框架来联合优化策略生成与动作执行，并改善探索与信用分配？

### 4. 背景与问题设定

大语言模型越来越多地被用作交互式智能体，但在长视野决策任务上的优化仍然困难。现有基于强化学习的方法多为纯反应式，智能体需同时决定即时动作与整体行动路线，导致探索短视、行为不一致及信用分配复杂化。

### 5. 方法概览

- 任务 / 环境: 在ALFWorld、WebShop和SciWorld等长视野智能体交互基准上进行评估。
- 变量 / 干预设计: 在每轮交互开始时，从初始任务状态采样一个紧凑的自然语言策略，随后所有动作生成均以该策略为条件。
- 评价指标: 任务成功率（ALFWorld, WebShop）和总体得分（SciWorld）。
- 模型比较: 与提示基线和GRPO等强化学习方法进行比较。

## 4. Coordination Matters: Evaluation of Cooperative Multi-Agent Reinforcement Learning

**Topics:** 多智能体强化学习, 协调评估, 任务分配, 基准测试
**Authors:** Maria Ana Cardei, Matthew Landers, Afsaneh Doryab
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.06557v1) | [PDF](http://arxiv.org/pdf/2605.06557v1.pdf) | [Code](https://github.com/mariacardei/coordination_aware_MARL)

### 1. 一句话结论

本文提出并实例化了‘协调感知评估’框架，证明在多智能体强化学习中，仅靠回报等聚合指标会掩盖关键的协调失败模式，必须补充过程层面的诊断指标。

### 2. 为什么重要

现有协作MARL基准（如StarCraft、粒子世界）依赖回报、成功率等结果指标，但这些指标无法揭示智能体‘如何’协调。在智能体、任务和联合决策组合性扩展的场景中，相似的回报可能源于完全不同的协调机制（如高冲突或低多样性），导致算法评估失真，阻碍对协调本质的理解和改进。

### 3. 研究问题

1. 在组合性扩展的协作任务中，仅靠聚合回报能否准确评估和区分不同MARL方法的协调能力？
2. 如何设计可量化的过程诊断指标来暴露任务分配中的协调失败模式，如冗余分配和低效分工？

### 4. 背景与问题设定

协作多智能体强化学习（MARL）的性能高度依赖智能体间的协调。随着智能体与任务数量增加，联合动作空间呈组合性爆炸，协调愈发困难。现有主流基准测试虽推动了算法发展，但其环境复杂性（如部分可观测、导航、稀疏奖励）常将协调失败与其他挑战混淆，难以孤立地研究协调本身在规模扩展下的变化规律。

### 5. 方法概览

- 任务 / 环境: 使用STAT（空间任务分配测试平台），一个完全可观测的2D网格环境。智能体从固定原点出发，需分配并完成空间分布的任务。环境通过动作掩码和有限状态承诺机制，抽象掉低层路径规划和碰撞，将‘任务分配协调’隔离为核心挑战。
- 变量 / 干预设计: 沿三个轴进行受控组合性扩展：1) 智能体数量，2) 任务数量，3) 环境空间大小。同时保持任务规则和观测完全固定，以纯粹研究协调如何随规模变化。
- 评价指标: 主要绩效指标：平均回报。协调感知过程诊断指标：总任务分配冲突数、冲突率、每任务冲突数（衡量冗余分配）；分配多样性（衡量分工广度）；任务完成吞吐量（衡量效率）。
- 模型比较: 评估六种基于价值的MARL方法，覆盖不同集中化范式：集中训练集中执行（CTCE）的DQN；集中训练分散执行（CTDE）的VDN、QMIX、QTRAN、FDQN；分散训练分散执行（DTDE）的IQL。以此探究训练与执行结构对协调的影响。
- 我的理解: 研究核心是方法论创新：构建一个‘协调显微镜’（STAT+过程指标）。它不提出新算法，而是提供更精细的工具来诊断和比较现有算法在受控协调问题上的真实表现，弥补评估盲点。

### 6. 关键发现

- 回报相似性掩盖协调机制异质性: 实验显示，不同MARL方法（如VDN与QMIX）在STAT中可获得相似的平均回报，但其冲突率、分配多样性等过程指标差异显著。这表明它们通过不同的协调机制（有的冗余多，有的分工窄）达到了相近的结果产出。
- 过程指标是回报的必要补充诊断: 冲突率等指标提供了回报无法揭示的信息。例如，Figure 1表明，在回报相近的方法中，冲突率分布分散，直接暴露了回报指标所隐藏的协调失败程度。
- 规模扩展下的性能由多因素塑造: 在承诺约束的任务分配中，性能随规模的变化不仅取决于名义上的联合动作空间大小，更受‘分配压力’（待分配任务与可用智能体的比例）、‘稀疏决策机会’（仅在特定状态可选择任务）以及智能体间‘冗余选择’的相互依赖等因素共同影响。

### 7. 最重要的图

- 图来源: Figure 1: Conflict rate provides a complementary diagnostic beyond return.
- 为什么重要: 该图是论文核心论点的直接视觉证据。它用散点图展示了六种MARL方法的回报与冲突率关系，清晰表明：在横轴（回报）上位置接近的点，在纵轴（冲突率）上却高度分散。这无可辩驳地证明，仅看回报会错误地将协调质量迥异的方法视为等同，凸显了引入协调感知指标的必要性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 设计大规模多智能体系统时，不能只优化最终产出，必须将过程协调效率（如最小化冲突、最大化合理分工）作为核心设计目标之一。
- 对 Skill: 智能体需要习得高阶协调技能，包括在稀疏决策点做出非冗余的分配判断，并能推理其他智能体的承诺状态以进行有效分工。
- 对 Memory: 在承诺约束环境中，高效的协调可能要求智能体具备对团队任务分配状态的记忆或显式推理能力，以避免冲突和重复劳动。
- 对评测: 为MARL领域提出了明确的评估范式转变：未来的基准测试必须将协调感知的过程指标（如本文定义的冲突率、多样性）作为与回报同等重要的报告内容，以实现更全面、更真实的算法性能衡量。

### 11. 局限性

- STAT环境高度抽象和简化，专注于高层任务分配，忽略了真实场景中常伴随的路径规划冲突、动态障碍物、通信延迟等问题，其结论向更复杂环境的泛化性需进一步验证。
- 评估范围局限于离散动作、基于价值的MARL方法。未来需要将协调感知评估框架扩展到连续动作、基于策略的方法以及涉及通信或学习的算法。

### 12. 我的看法 / 个人笔记

这是一项极具洞察力且实用的‘评估科学’研究。它精准地指出了当前MARL研究中的一个关键盲点——我们过于关注‘做得怎么样’（回报），而忽视了‘怎么做的’（协调过程）。通过精心设计的STAT和一套可操作的过程指标，论文为社区提供了一套宝贵的‘诊断工具’。其价值不仅在于具体发现，更在于倡导了一种更严谨、更深入的评估文化，这对于任何追求鲁棒、可扩展多智能体系统的研究都至关重要。

### 14. 总结

本文批判性地指出，当前协作多智能体强化学习（MARL）的评估过度依赖回报等聚合指标，无法揭示智能体间的协调质量。为此，作者提出了‘协调感知评估’新视角，并构建了STAT测试平台来受控地研究任务分配中的协调问题。论文设计了一套过程诊断指标（如冲突率、分配多样性），并通过对六种主流MARL方法的实验证明：回报相似的算法，其协调机制（冗余程度、分工效率）可能截然不同。研究最终论证，协调感知指标是回报基准的必要补充，应成为未来MARL评估的标准组成部分，以更准确地引导算法发展。

## 5. BAMI: Training-Free Bias Mitigation in GUI Grounding

**Topics:** GUI Agent, 多模态大语言模型, 视觉定位, 推理优化
**Authors:** Borui Zhang, Bo Zhang, Bo Wang, Wenzhao Zheng, Yuhao Cheng, Liang Tang, Yiqiang Yan, Jie Zhou, Jiwen Lu
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.06664v1) | [PDF](http://arxiv.org/pdf/2605.06664v1.pdf) | [Code](https://github.com/Neur-IO/BAMI)

### 1. 一句话结论

提出无需训练的BAMI方法，通过由粗到精聚焦和候选选择两大操作，针对性缓解GUI视觉定位中的精度偏差与模糊性偏差，显著提升多种骨干模型性能。

### 2. 为什么重要

GUI视觉定位是智能体执行点击、拖拽等原子操作的基础，但在高分辨率、元素密集的真实场景（如ScreenSpot-Pro）中，现有模型性能常低于50%。BAMI无需额外训练即可将TianXi-Action-7B的准确率从51.9%提升至57.8%，为低成本挖掘模型潜力、提升智能体可靠性提供了高效途径。

### 3. 研究问题

1. GUI视觉定位模型错误的主要根源是什么？如何量化归因？
2. 能否在不重新训练模型的前提下，通过设计推理时干预来纠正这些错误？

### 4. 背景与问题设定

基于多模态大语言模型（MLLMs）的GUI智能体需完成视觉定位任务：给定自然语言指令和屏幕截图，输出目标元素坐标。然而，面对高分辨率、元素密集的GUI界面（如专业软件），现有模型在ScreenSpot-Pro等基准上表现不佳，准确率常低于50%。

### 5. 方法概览

- 任务 / 环境: 输入：自然语言查询q和高分辨率GUI截图I∈R^(H×W×3)。输出：目标边界框坐标(x1, y1, x2, y2)。正确性判定：预测框中心点位于真实框内即为正确。
- 变量 / 干预设计: BAMI将单步定位转化为结构化多步推理（算法1）。外层循环（N次迭代）执行由粗到精聚焦：每轮基于上轮预测框按比例λ裁剪图像，缩小搜索范围。内层循环（每轮M次预测）执行候选选择：通过掩码已预测区域生成M个互斥候选框，再使用校正模型（如GPT-5或Qwen3-VL-8B）依据GUI先验规则提示（如功能偏好、典型UI模式、交互组件优先）选择最优框。
- 评价指标: 主要使用准确率（Accuracy）。在ScreenSpot-Pro和ScreenSpot-V2等数据集上评估。
- 模型比较: 在OS-Atlas-7B、UI-TARS-7B、TianXi-Action-7B等多个开源骨干模型上验证。例如，BAMI使TianXi-Action-7B在ScreenSpot-Pro上的准确率从51.9%提升至57.8%（+5.9%）。在ScreenSpot-V2等其他数据集上也观察到一致提升。
- 我的理解: BAMI的核心是“诊断-干预”。首先通过MPD方法诊断出74%的错误源于归纳偏差（精度偏差20%，模糊性偏差54%）。然后设计两种针对性干预：1）空间上的由粗到精聚焦，通过迭代裁剪缓解高分辨率下的离散化精度损失；2）语义上的候选选择，通过规则提示纠正模型因优化编辑距离（而非欧氏距离）产生的选择偏好不一致。

### 6. 关键发现

- 错误根因：归纳偏差为主: 使用提出的掩码预测分布（MPD）方法对50个错误样本分析显示：知识缺陷占14%，精度偏差占20%，模糊性偏差占54%，其他占12%。归纳偏差（精度+模糊性）合计占74%，是主要优化目标。
- BAMI有效提升性能与通用性: BAMI在ScreenSpot-Pro上一致提升所有测试骨干模型的性能（见图3）。消融实验证实“由粗到精聚焦”和“候选选择”均贡献显著。方法在ScreenSpot-V2等其他数据集上也有效，展现了跨数据集通用性。
- 语言增强无效的深层原因: 实验发现扩展语言序列（查询扩展、上下文描述）对提升定位精度无效甚至有害。根本原因在于MLLMs优化的是坐标文本序列的编辑距离，而非真实的欧氏距离。例如，坐标789与189的编辑距离为1（小于与801的编辑距离3），但欧氏距离为600（远大于与801的12），目标不一致导致语言增强难以纠正空间错误。

### 7. 最重要的图

- 图来源: Figure 5. Illustration of BAMI.
- 为什么重要: 该图完整展示了BAMI的三步推理流程：1）由粗到精聚焦（裁剪），2）掩码生成多候选，3）基于规则的候选选择。它直观体现了如何将单步黑盒预测转化为可控、可解释的结构化推理，以同时应对空间精度和语义模糊两大挑战。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 提供了一种即插即用、无需重训练的方案，可快速提升现有GUI智能体系统的定位鲁棒性和成功率，降低部署成本。
- 对 Skill: 展示了如何通过结合空间操作（裁剪）和语义引导（规则提示）的结构化推理，来弥补模型在精确空间任务上的原生不足，此范式可推广至其他需要细粒度空间推理的任务。
- 对 Memory: 候选选择中注入的GUI先验规则（如“交互组件优先”）是一种轻量化的领域知识（记忆）整合方式，使模型能做出更符合人类常识的决策。
- 对评测: MPD错误归因方法为模型评估提供了新维度，不仅能看准确率，还能诊断错误类型，有助于针对性改进模型或推理流程。

### 11. 局限性

- BAMI引入了额外的推理步骤（迭代N次，每轮生成M个候选）和外部选择模型，增加了计算开销和延迟，可能影响实时性。
- 对于14%的知识缺陷错误，BAMI无法解决，这仍需依赖模型本身知识库的扩展或更基础的训练数据增强。
- 方法的性能提升依赖于超参数（如迭代次数N、裁剪比例λ、候选数M）的设置，需针对不同场景进行调整。

### 12. 我的看法 / 个人笔记

这是一项非常扎实的“工程科学”工作。它没有追逐更大模型或更多数据，而是深入剖析现有模型在特定任务上的失败模式，并设计出精巧、低成本的推理时“补丁”。其最大亮点在于：1）用MPD方法量化了错误根源（74%归纳偏差），使干预有的放矢；2）设计的两种干预分别针对空间离散化（精度偏差）和优化目标错位（模糊性偏差）两大本质问题。这为如何“榨干”现有模型潜力提供了优秀范例，尤其适合资源有限或需要快速迭代的场景。

### 14. 总结

本文首先提出MPD方法，诊断出GUI视觉定位中74%的错误源于模型的归纳偏差（精度偏差20%，模糊性偏差54%）。为此，作者提出了无需训练的BAMI方法，它通过迭代的由粗到精聚焦操作缓解高分辨率下的坐标离散化精度损失，并通过融入GUI先验规则的候选选择操作纠正模型因优化编辑距离（而非欧氏距离）产生的语义模糊。实验表明，BAMI能一致、显著地提升多种SOTA骨干模型在ScreenSpot-Pro等复杂基准上的性能（如TianXi-Action-7B从51.9%升至57.8%），证明了通过精细化推理设计挖掘模型潜力的巨大价值。
