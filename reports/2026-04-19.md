## 1. Discovering Novel LLM Experts via Task-Capability Coevolution

**Topics:** 大语言模型, 协同进化, 开放终结算法, 模型合并, 集体智能, 质量-多样性
**Authors:** Andrew Dai, Boris Meinardus, Ciaran Regan, Yingtao Tian, Yujin Tang
**Venue / Year:** ICLR 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.14969v1) | [PDF](http://arxiv.org/pdf/2604.14969v1.pdf) | [Code](acdc-llm.github.io)

### 1. 一句话结论

提出AC/DC框架，通过模型与任务的协同进化，在单一运行中自动发现一个由小型LLM组成的专家集合，其集体知识覆盖率超越更大的单体模型及GPT-4o，且无需对下游基准进行显式优化。

### 2. 为什么重要

当前LLM开发范式依赖手动、静态的数据集和训练流程，成本高昂且难以持续、自主地扩展能力多样性。AC/DC借鉴开放终结进化思想，通过模型与任务的双向驱动，为自动化、持续地发现多样化、互补的模型能力提供了一种新范式，更贴近集体智能的本质。

### 3. 研究问题

1. 如何设计一个开放终结的系统，使LLM的能力能够像自然进化一样持续、自主地创新和扩展，而无需手动干预或针对特定基准的显式优化？
2. 能否通过协同进化，自动发现一个由小型、高效模型组成的集合，使其在集体层面的问题解决广度（覆盖率）上超越参数规模大得多的单体前沿模型？

### 4. 背景与问题设定

前沿大模型开发追求持续获得新兴且多样化的能力，但现有预训练/后训练范式每次都需要基于静态数据集或奖励函数手动启动训练，成本高昂且过程不连续。集体智能表明，多样化个体组成的集体能完成远超任何单一个体的任务。开放终结算法旨在通过永不停止的进化实现持续创新。

### 5. 方法概览

- 任务 / 环境: 协同进化两个档案库：1）LLM模型档案库，通过质量-多样性算法筛选；2）合成任务档案库，由大型‘科学家LLM’生成并验证，涵盖科学、工程、创意写作等领域，且任务难度和 novelty 随时间进化。
- 变量 / 干预设计: 核心是AC/DC算法循环：a) 模型进化阶段：通过进化模型合并（交叉）与权重矩阵奇异值加噪（变异）产生子代模型，评估其在当前任务集上的技能向量，应用‘胡言乱语过滤器’（MC），最后使用主导新颖性搜索更新模型档案库。b) 任务进化阶段：定期由科学家LLM基于现有任务（根据通过率判断难度）生成新任务，经过新颖性过滤、自我反思验证和质量过滤（如移除无人能解的任务）后更新任务档案库。
- 评价指标: 核心评估指标是覆盖率（Coverage），衡量模型集合中至少有一个模型能正确回答的问题占总问题的比例。次要指标包括单模型在MMLU等基准上的性能，以及多智能体最佳N选1（Best-of-N）场景下的表现。
- 模型比较: 与四种基线对比：1）手动挑选的专家模型组合（Experts）；2）同一基础模型多次采样（Control）；3）单个大型模型（Big Model）；4）GPT-4o。评估在多个不同领域的下游基准（OOD）上进行。
- 我的理解: 这是一个精心设计的开放终结进化模拟系统。任务复杂度的提升‘逼迫’模型进化出新能力以维持生存（通过率），而模型能力的多样化又‘激励’任务生成器提出更新颖、更难的挑战以区分模型，形成正反馈循环。质量-多样性算法确保进化方向不是单一的性能最优，而是行为（技能）的多样化，最终自动涌现出一个互补的专家模型生态系统。

### 6. 关键发现

- 集体覆盖率超越大型单体模型及GPT-4o: 进化出的由多个小型LLM（如3个7B模型）组成的‘任务特遣队’，其总参数量远小于对比的大型单体模型（如72B），但在多个下游基准上的集体覆盖率（Coverage）显著更高。例如，Qwen2.5 7B的N=8集合覆盖率比72B模型高9.78%。在某些配置下（如Qwen2.5 7B, N=3），其覆盖率甚至超过了GPT-4o。
- 实现无需显式基准优化的开放终结改进: AC/DC的覆盖率随时间（进化代数）持续提升，单模型在MMLU上的性能也随进化提高。这表明系统能在不针对任何下游评估基准进行显式优化的情况下，自主、持续地创新任务和模型能力。
- 能力多样性可转化为更优的单一答案选择: 进化出的模型集合在Best-of-N（从N个候选答案中选择最佳）评估中也优于基线，表明其多样化的能力在需要产生单一可靠答案的实际部署场景中同样具有价值。
- 跨模型家族的有效性与参数高效性: 方法在Qwen2、Qwen2.5、Qwen3、DeepSeek等多个不同规模的基础模型家族上均验证有效。通过分布式专业化，小型模型集合能以更少的总体参数实现更广的知识覆盖，展示了高参数效率。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 提供了一种自动生成多样化专家智能体种群的方法。无需手动设计，可通过协同进化获得一组能力互补的智能体，通过集体决策（如Best-of-N或更复杂的仲裁机制）解决复杂问题，构建更强大的多智能体系统。
- 对 Skill: 展示了一种动态、涌现式的技能发现与扩展机制。技能（定义为解决特定任务的能力）不是在固定数据上训练出来的，而是在与不断进化的任务环境的互动中竞争、分化和深化的。
- 对 Memory: 进化过程中保存的‘历史模型档案库’构成了一个不断增长的外部、可检索的‘技能记忆库’。针对新问题，可以通过检索（选择任务特遣队）快速组装具备相关技能的模型组合。
- 对评测: 强力推广了‘覆盖率（Coverage）’作为评估模型集合或生态系统能力广度的核心指标。这推动评估焦点从单一模型的‘绝对性能’转向模型群体的‘能力覆盖范围’和‘互补性’，更适合评估面向开放世界的AI系统。

### 11. 局限性

- 任务进化的质量严重依赖作为‘科学家’的大型LLM的能力、创造力和偏见，这可能限制任务空间的探索范围并引入不可控的偏差。
- 协同进化过程计算开销大，涉及多轮模型评估（前向传播）和复杂的任务生成、验证循环，成本仍高于传统单次训练。
- 当前工作主要证明了能力广度（覆盖率）的提升，但对每个进化出的‘专家’在特定领域的能力深度缺乏深入分析和理论保证。
- 实验虽覆盖多个模型家族，但均为Decoder-only架构，方法在其它架构（如MoE）上的泛化能力有待验证。

### 12. 我的看法 / 个人笔记

这是一项将开放终结进化、质量-多样性优化和前沿模型合并技术深度融合的杰出工作。其最大价值在于范式转换：从追求‘一个更大更强的万能模型’转向培育‘一个持续进化、能力互补的专家模型生态系统’。AC/DC不仅提供了一种自动化扩展LLM能力的新工具，更启发我们以更生物化、生态化的视角来思考AI的发展路径。尽管仍有计算成本和依赖大模型等限制，但它无疑为通往更自主、更持续的AI创新迈出了关键一步。

### 14. 总结

AC/DC是一个创新的开放终结协同进化框架，用于自动发现多样化的LLM专家集合。它通过进化模型合并（交叉/变异）来进化LLM种群，同时利用大模型生成并筛选日益复杂的任务来进化任务环境。双方在最低标准过滤和质量-多样性算法引导下协同进化。最终，该系统能在单一运行中自动产生一组小型LLM，其集体知识覆盖率超越参数规模大得多的单体模型甚至GPT-4o，且无需对下游基准进行任何显式优化，展示了持续自主改进的能力，为LLM开发提供了全新的生态化范式。

## 2. Enhancing Large Language Models with Retrieval Augmented Generation for Software Testing and Inspection Automation

**Topics:** 软件工程, 大语言模型, 检索增强生成, 软件测试, 代码审查
**Authors:** Zoe Fingleton, Nazanin Siavash, Armin Moin
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.15270v1) | [PDF](http://arxiv.org/pdf/2604.15270v1.pdf)

### 1. 一句话结论

通过检索增强生成（RAG）为LLM注入外部知识，能有效减少其在自动化代码审查和测试用例生成中的幻觉，提升输出准确性，部分场景性能接近或超越人类基线。

### 2. 为什么重要

软件验证与确认（V&V）是保障软件质量的核心，但人工执行成本高昂。LLM虽能自动化这些任务，但其固有的‘幻觉’问题（包括事实性幻觉和忠实性幻觉）会生成不准确或与事实/指令相悖的输出，严重阻碍了其在关键工程任务中的可靠部署。本研究通过引入RAG管道，为LLM动态提供领域特定的外部知识作为生成依据，旨在从根源上缓解幻觉，提升自动化V&V的可靠性和效率，对实现可信赖的AI辅助软件工程具有重要实践意义。

### 3. 研究问题

1. 不结合RAG管道，LLM能否有效且高效地进行代码审查？
2. 不结合RAG管道，LLM能否有效且高效地进行测试用例生成？

### 4. 背景与问题设定

软件测试与代码审查是软件开发生命周期中至关重要但劳动密集的验证活动。LLM的出现为自动化这些任务带来了希望，但其存在严重的‘幻觉’问题，导致输出不可靠。传统人工代码审查平均能发现约60%的缺陷，但过程耗时。本研究旨在探索如何通过RAG技术增强LLM，使其在自动化V&V任务中达到或超越人类基准水平，同时减少因知识不足或指令误解而产生的幻觉。

### 5. 方法概览

- 任务 / 环境: 两个具体任务：1) 自动化代码审查：使用Bug In the Code Stack数据集，目标是检测代码中的缺陷，并以人类审查员约60%的平均检出率为基线。2) 自动化测试用例生成：使用Wang等人提出的TestEval基准数据集，目标是生成高质量、可覆盖代码的测试用例。
- 变量 / 干预设计: 设计了一个统一的RAG增强框架。核心干预是：在LLM生成前，使用检索器（基于all-MiniLM-L6-v2句子Transformer模型）从外部知识库（如MBPP数据集）中，通过余弦相似度搜索获取与当前查询最相关的Top-K文档片段。将这些检索到的上下文与原始提示词拼接，一并输入LLM，从而引导其生成更准确、有据可依的输出。
- 评价指标: 代码审查：以人类审查员约60%的平均缺陷检出率作为性能基线进行对比。测试用例生成：在TestEval基准数据集上评估生成测试用例的质量。幻觉量化：将模型的整体输出准确率（正确输出的比例）作为衡量幻觉程度的代理指标。
- 模型比较: 实验评估了多个不同LLM在‘有RAG’和‘无RAG’两种配置下的性能，以分离并确认RAG干预带来的净效应，避免结论受单一模型特性偏差的影响。
- 我的理解: 本研究是一项典型的‘增强型’工程工作。它不追求发明新模型，而是采用RAG这一成熟的系统架构方案，为现有LLM配备一个可按需访问的外部知识存储器。这相当于让LLM在回答专业问题时能够‘查阅资料’，从而有效弥补其在特定领域知识不足和易产生幻觉的短板，使其在软件工程这类对精确性要求严苛的任务中变得更可靠、更可用。

### 6. 关键发现

- RAG普遍提升双任务性能: 实验结果表明，引入RAG管道、整合外部上下文后，LLM在代码审查和测试用例生成两项任务上的表现均得到普遍改善。这证实了外部知识注入对于减少领域特定幻觉、提升输出准确性的有效性。
- 实现成本节约与效率提升: 该自动化方法能够显著节省人工测试员和代码审查员的时间，从而降低项目的总体V&V成本。同时，由于LLM结合RAG后输出质量提高，这些自动化活动的整体有效性和效率也得以增强。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为构建可靠的专业领域智能体提供了关键模块（RAG）的示范。表明通过集成可检索的外部知识库，可以显著提升智能体在需要事实准确性和领域专业知识任务中的表现，降低其产生幻觉的风险。
- 对 Skill: 具体展示了如何通过‘基础LLM + 领域知识增强’的模式，为LLM赋予更可靠的‘软件工程技能’（代码审查、测试生成）。此模式可推广至法律、医疗等其他专业技能的构建。
- 对 Memory: RAG管道实现了一种高效的、可按需访问的外部非参数化记忆。它扩展了模型的‘工作记忆’，使其能够突破训练数据的限制，利用最新、最相关的领域知识，这对于处理动态变化的软件项目至关重要。
- 对评测: 研究采用了多维度评估策略：与人类专家基线（60%缺陷检出率）对比（衡量实用性），在标准基准数据集（TestEval）上测试（确保可复现性），并用输出准确率作为幻觉的代理指标（突出问题针对性）。这种综合评估框架对衡量AI辅助工程工具的成熟度具有参考价值。

### 11. 局限性

- 研究结论的有效性依赖于特定的技术选型（LLM、嵌入模型、检索算法、知识库）。不同的技术组合可能导致效果差异，其普适性需要在更广泛的技术栈和更多样的软件项目上进行验证。
- RAG管道的性能上限受限于外部知识库的质量、覆盖度和时效性。构建和维护一个高质量、与目标任务高度相关且持续更新的知识库本身是一项具有挑战性和持续成本的工作。

## 4. VisPCO: Visual Token Pruning Configuration Optimization via Budget-Aware Pareto-Frontier Learning for Vision-Language Models

**Topics:** 视觉语言模型, 视觉令牌剪枝, 帕累托优化, 可微分搜索, 计算效率
**Authors:** Huawei Ji, Yuanhao Sun, Yuan Jin, Cheng Deng, Jiaxin Ding, Luoyi Fu, Xinbing Wang
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.15188v1) | [PDF](http://arxiv.org/pdf/2604.15188v1.pdf) | [Code](https://github.com/JHW5981/VisPCO)

### 1. 一句话结论

VisPCO是一个可微分框架，通过帕累托优化自动寻找视觉令牌剪枝的最优层间配置，避免了传统网格搜索的高昂成本。

### 2. 为什么重要

高分辨率图像和视频帧导致VLM计算成本呈二次增长。现有剪枝方法（如FastV、SparseVLM）依赖预定义或启发式配置，无法保证在给定计算预算下达到最优性能，而通过穷举网格搜索寻找最优配置成本过高。

### 3. 研究问题

1. 现有的视觉令牌剪枝配置是否达到了计算与性能的最优权衡？
2. 如何高效地自动调整这些剪枝配置以达到最优？

### 4. 背景与问题设定

为提升VLM性能，常需输入高分辨率图像或多视频帧，导致视觉令牌数量激增，计算成本（FLOPs）呈二次增长。现有剪枝方法通过在单层或多层按预定义比率剪枝冗余令牌来加速，但其配置（剪枝位置和比率）是否最优未知。通过网格搜索寻找最优配置需进行大量实验，成本高昂。

### 5. 方法概览

- 任务 / 环境: 给定一个计算预算B，为VLM（如Qwen2.5VL-3B）自动寻找一组层级的令牌保留比率配置 r = [r1, r2, ..., rL]，以最小化剪枝导致的性能损失。
- 变量 / 干预设计: 1. 将问题形式化为帕累托优化：最小化性能损失（KL散度），约束为计算成本FLOPs(r) ≤ B。2. 使用增广拉格朗日法求解该带约束优化问题。3. 为使得离散的剪枝操作可微分，引入连续松弛（高斯核软插值阈值）和直通估计器。4. 使用一个轻量级代理网络（Ratio Predictor）根据预算B预测配置r。
- 评价指标: 优化目标使用剪枝前后模型输出logits的KL散度（蒸馏损失）。最终在8个视觉基准（涵盖VQA、多模态推理、图表理解）上测试任务准确率，以验证配置的有效性。主要评估VisPCO找到的配置能否逼近经验帕累托前沿。
- 模型比较: 与通过网格搜索获得的经验帕累托前沿进行对比，衡量逼近程度。同时在多种基础VLM架构（Qwen2.5VL-3B, LLaVA-NeXT-34B, CLIP-ViT）、多种底层剪枝方法（FastV, FitPrune, SparseVLM）和不同数据集上验证其泛化能力。
- 我的理解: 核心创新是将离散的剪枝配置搜索问题，通过数学重构（帕累托优化+约束求解）和工程技巧（连续松弛+STE），转化为一个可端到端梯度优化的连续问题，从而用高效的可微分搜索替代昂贵的网格搜索。

### 6. 关键发现

- 有效逼近帕累托前沿，揭示显著性能差距: 在8个基准测试上，VisPCO自动找到的配置能有效逼近网格搜索得到的经验帕累托前沿。论文图1(a)显示，在相同计算预算（1.7 TFLOPs）下，不同剪枝配置的性能差距可达23.6%，这直接驱动了本研究的必要性。
- 帕累托前沿动态变化，VisPCO泛化性强: 帕累托前沿随VLM架构、底层剪枝方法和数据集复杂度不同而变化（论文图1 b-d）。VisPCO在不同设置（包括更大模型LLaVA-NeXT-34B）下均能生成有效的帕累托前沿，展示了良好的泛化性。
- 多步渐进式剪枝模式在严格预算下最优: 通过比较不同可学习核函数（指数衰减、线性、sigmoid、多步sigmoid）的性能，发现多步渐进式剪枝（由多步sigmoid核建模）能捕捉VLM的层次压缩结构，在严格计算预算下比单层一次性剪枝获得更优的精度-效率权衡。

### 4. 背景与问题设定

传统AI公平性研究聚焦于静态预测模型的输出指标。MAAI系统由多个基于基础模型的自主或半自主智能体组成，在动态环境中协作，其公平性应被视为一种‘社会技术性结果’，产生于智能体之间、智能体与人类利益相关者之间的复杂交互。现有研究对此新兴挑战的关注零散且不深入，缺乏系统性梳理与批判性分析。

### 5. 方法概览

- 任务 / 环境: 对截至2025年初关于MAAI系统公平性与伦理的学术文献进行系统性范围综述。
- 变量 / 干预设计: 遵循Arksey & O‘Malley的范围综述方法论，在Scopus和arXiv数据库使用结构化检索字符串进行系统检索，并结合前向与后向滚雪球抽样，最终纳入23项研究（20篇来自arXiv，3篇来自Scopus）。
- 评价指标: 通过定性内容分析，构建了一个五维形态分析框对文献进行编码与分析。五个维度包括：公平性概念（规范奠基 vs. 简化概念）、评估方法（输出分析、形式化游戏、人类评估等）、公平性引入阶段（请求/提示、模型训练/微调）、研究目标（记录、检查、改进）、系统自主性类型（纯AI、人机混合、纯人类）。
- 模型比较: 不适用（本文为综述性研究，不涉及模型比较）。
- 我的理解: 作者采用了三重分析透镜：1）人本视角：将人类利益相关者（发起者、参与者、受试者）置于中心；2）期望驱动视角：依据Deck等人提出的八个具体公平性期望来评估文献贡献；3）系统级视角：关注多智能体交互产生的涌现属性，而非单一模型的孤立行为。

### 6. 关键发现

- 归纳出五种典型的研究原型: 通过分析文献在形态分析框中的分布模式，识别出五种反复出现的研究原型：1）规范性委托：将公平判断委托给系统内特定智能体；2）公平性门面：通过提示词等表面化、事后介入方式引入公平性；3）公平性教化：通过训练数据或微调影响基础模型的公平性倾向；4）培养皿公平性：在高度简化、可控的实验室环境（如形式化游戏）中测试公平性；5）公平性有效性：重点评估特定干预措施对公平性结果的实际影响。
- 揭示当前研究存在三大核心缺陷: 首先，概念基础薄弱：大多数研究（13篇中的7篇）使用‘简化’的公平性概念，基于直觉且定义模糊，缺乏坚实的伦理或政治哲学理论奠基（‘规范性贫血’）。其次，评估环境脱离实际：绝大多数研究在简化的实验室环境中进行，缺乏真实世界部署或包含真实人类利益相关者的混合人机协作环境验证。最后，忽视系统复杂性：研究普遍聚焦于单个智能体或静态输出，未能充分考察多智能体交互、协商和系统级动态所引发的独特公平性风险。
- 提出公平性必须‘结构性嵌入’开发周期: 批判了将公平性作为‘事后附加项’或通过简单提示实现的普遍做法。主张公平性必须作为核心设计原则，结构性地嵌入MAAI系统从概念设计、开发、部署到监控的整个生命周期。有意义的公平性评估需要三个要素：明确的人类监督角色与问责机制、清晰的规范性选择（阐明‘对谁公平’、‘何种公平’）、以及针对具体公平性期望的精确评估框架。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 系统架构设计必须预先集成公平性机制与问责接口，考虑智能体间的规范性协商与冲突解决协议，并在关键决策点设置明确的人类监督接入点。
- 对 Skill: 智能体可能需要发展理解和遵循特定公平性规范的能力，能够在不同利益相关者的需求间进行协调。
- 对 Memory: 系统可能需要维护决策与交互的审计追踪，以支持事后公平性分析、归责与解释。
- 对评测: 评估范式需要从评估单一模型的静态输出，转向评估动态、交互系统中的涌现行为公平性，并整合形式化指标与利益相关者感知测量，在更贴近实际的环境中进行。

### 11. 局限性

- 领域处于极早期阶段，符合严格纳入标准的高质量文献数量有限（仅23篇），且时间跨度短（2023-2025），这可能影响综述结论的全面性。
- 纳入文献中绝大部分（20篇）来自预印本平台arXiv，且截至2026年3月多数尚未经传统同行评审，可能包含了不成熟或未经充分验证的观点。

### 12. 我的看法 / 个人笔记

这是一篇极具洞察力与批判性的奠基性综述。其核心价值在于为一片方兴未艾但杂乱无章的新兴领域绘制了第一张精细的‘学术地图’，并提供了强有力的诊断工具。文章最尖锐的批判在于指出了该领域核心的‘规范性贫血’问题——许多研究在技术操作层面探讨‘公平’，却刻意回避或无力深究‘何为公平’这一根本性的伦理、社会与政治哲学问题。这为未来所有研究者设立了一个高标准的起点。

### 14. 总结

本文对多智能体AI系统公平性这一新兴领域进行了首次系统性范围综述。通过对23项研究的定性分析，作者构建了一个五维形态分析框架，并据此识别出五种典型的研究原型。综述批判性地指出，当前研究普遍存在三大问题：公平性概念基于直觉且简化、缺乏坚实的规范理论奠基（‘规范性贫血’）、以及严重忽视多智能体交互带来的系统级动态与风险。基于此，文章主张公平性必须从“事后附加项”转变为“结构性嵌入”整个系统开发生命周期的核心原则，并提出了一个融合人本视角、期望驱动与系统级考量的未来研究议程。
