## 1. LLMs and Agentic AI Systems for Smart Grids: A Tutorial on Architectures and Applications

**Topics:** LLM Agent, Smart Grids, Power Systems
**Authors:** Daniela Rojas, Abdulwahab Albassam, Aidan G. Leung, Jett Ngo, Ryan Luo, Peter R. Quawas, Junpyung Kim, Kangkai Liang, Mansi Nanavati, Jonathan Mai, Meng-Chi Tsai, Yun-Tong Tsai, Yize Chen, Yuanyuan Shi
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.18147v1) | [PDF](http://arxiv.org/pdf/2607.18147v1.pdf)

### 1. 一句话结论

提出求解器接地设计原则：智能电网中LLM仅作为编排层，数值结果必须来自可信工具并通过显式验证，案例显示EVAgent减少LLM未满足能量7.5-9.5倍，GridDebugAgent修复17/39个故障案例并减少52.3%违规。

### 2. 为什么重要

智能电网是安全关键系统，LLM直接输出数值可能违反物理定律导致事故。该工作首次形式化求解器接地原则，提供统一评估框架和可复现案例，为LLM在工程领域的可靠部署奠定基础。

### 3. 研究问题

1. 如何设计LLM/智能体系统使其在智能电网中产生物理可行且数值正确的输出？
2. 如何统一评估此类系统在任务效用、求解器接地正确性、忠实性与安全失败、成本与延迟四个维度的表现？

### 4. 背景与问题设定

LLM在智能电网中应用于预测、优化和控制，但存在幻觉、违反物理定律（如基尔霍夫定律）、输出难以验证等问题。现有系统缺乏统一设计原则和评估协议，数值正确性与物理可行性无法保证。

### 5. 方法概览

- 任务 / 环境: 智能电网任务，包括风电预测、电动汽车充电调度、潮流分析和故障诊断，涉及异构数据（文本、数值）和混合计算栈（预测模型、优化求解器、仿真器）。
- 变量 / 干预设计: 提出求解器接地设计：LLM解释请求、规划任务、调用工具、生成响应；可信工具执行数值计算；验证门检查结果是否符合物理和运行约束。在四个案例中对比纯LLM基线与求解器接地版本。
- 评价指标: 四组评估框架：任务效用（如预测误差、调度成本）、求解器接地正确性（数值是否来自可信工具）、忠实性与安全失败（是否报告未验证结果）、成本与延迟。
- 模型比较: 每个案例中比较纯LLM基线与求解器接地版本（如EVAgent调用CVXPY求解器，GridDebugAgent调用潮流求解器），使用相同数据和指标。
- 我的理解: 该方法本质上是将LLM从数值计算者转变为编排者，通过工具调用和验证门确保数值可信，类似于将LLM作为智能接口而非计算核心。

### 6. 关键发现

- 求解器接地设计显著提升数值正确性: EVAgent复现CVXPY最优解，同时将纯LLM未满足能量减少7.5-9.5倍；GridDebugAgent修复17/39个故障案例，总违规减少52.3%。
- 一致的分工模式出现: 智能体系统可靠地编排、检索和解释，可信工具负责计算，验证门决定报告内容。LLM不应直接输出数值。
- 纯LLM基线产生物理不可行输出: 在风电预测、潮流分析等任务中，纯LLM输出数值上看似合理但违反物理约束，验证门可有效拦截。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 智能体系统应明确划分LLM与工具职责，LLM负责编排和解释，工具负责计算，验证门确保输出可信。
- 对 Skill: LLM技能应聚焦于任务分解、工具调用、结果解释和异常处理，而非直接数值计算。
- 对 Memory: 短期记忆用于跟踪多步工具调用状态，长期记忆可存储电网知识、历史案例和验证规则。
- 对评测: 评估需覆盖任务效用、求解器接地正确性、忠实性与安全失败、成本与延迟四个维度，避免仅关注任务效用。

### 11. 局限性

- 验证门设计细节未充分展开，如验证规则如何自动生成和更新。
- 修复策略限于有限重试，复杂故障可能无法自动修复。
- 缺乏统一的智能体基准测试，案例规模较小（如故障诊断仅39个案例）。

### 12. 我的看法 / 个人笔记

该工作为LLM在安全关键工程领域的应用提供了清晰的设计原则和评估框架，求解器接地思想具有普适性。未来可扩展至更多电网任务（如动态安全评估），并探索验证门的自动化设计和修复策略的强化学习优化。

## 2. SEE: Structure-aware Exploring \& Exploiting for Long-horizon GUI Agent Trajectory Synthesis

**Topics:** GUI Agent, Data Synthesis, MLLM, Smartphone Control
**Authors:** Zhuohang Fan, Beichen Zhang, Yuanfa Li, Changqiao Wu, Wei Liu, Jian Luan, Weigang Zhang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.18046v1) | [PDF](http://arxiv.org/pdf/2607.18046v1.pdf)

### 1. 一句话结论

SEE通过构建屏幕-元素转移图并基于图规划合成轨迹，生成长达14.8步、无虚假循环的高质量GUI训练数据，微调后的智能体在任务成功率和跨屏幕泛化上均有提升。

### 2. 为什么重要

现有GUI数据集多依赖人工演示或在线探索，覆盖短路径、低复杂度区域，缺乏长程、元素丰富的真实移动应用轨迹。SEE提出结构感知的探索与利用框架，首次显式解耦结构获取与轨迹合成，通过图规划实现可控、可复现的长程数据生成，填补了该空白。

### 3. 研究问题

1. 如何高效获取移动应用中高覆盖率的UI状态转移结构？
2. 如何基于显式图结构合成可执行的长程、多步GUI轨迹？

### 4. 背景与问题设定

GUI智能体依赖高质量训练轨迹，但现有数据集（如AndroidControl、AMEX）轨迹短（平均5.5-12.8步）、页面元素少（13-17个），且多来自静态或简单应用。真实移动应用具有元素密集、动态变化、依赖前置状态、含中断（如权限弹窗）等特点，需要长程（>10步）且覆盖稀有转移的数据。现有方法要么依赖昂贵人工，要么在线探索偏向常见流，忽略长尾状态。

### 5. 方法概览

- 任务 / 环境: 真实移动应用（如购物、社交媒体等），通过Android模拟器交互，观测为截图和UI解析后的元素列表。
- 变量 / 干预设计: 两阶段框架：探索阶段构建屏幕-元素二分图（节点为屏幕状态和元素，边为转移和包含关系）；合成阶段基于图规划生成多步轨迹。探索时结合语义对齐、布局先验和历史惩罚选择动作，并引入随机探索；合成时先采样子目标屏幕，用BFS搜索路径，再基于节点/边语义生成低层指令。
- 评价指标: 轨迹平均长度、页面元素数量、下游任务成功率（在SEE测试集及跨基准测试上的泛化）、虚假循环避免率。
- 模型比较: 与AndroidControl、AMEX、AndroidLab、UI-Genie-Agent、OS-Genesis等数据集对比，SEE在轨迹长度（14.8步）和页面复杂度（平均24.63个元素）上显著领先。
- 我的理解: SEE的核心创新在于将数据生成从‘在线逐步推理’转变为‘先建图、后规划’的离线方式。探索阶段通过视觉匹配和语义描述构建结构化图，合成阶段利用图搜索避免重复推理，从而低成本生成可控长度、覆盖长尾状态的长程轨迹。反射和精炼机制进一步保证了图的质量。

### 6. 关键发现

- SEE生成长轨迹、高复杂度数据: SEE生成的轨迹平均长度14.8步，远超现有数据集（5.5-12.8步）；页面平均元素24.63个，高于其他数据集（13-17个）。
- 基于图合成避免虚假循环: 通过图搜索和路径约束，SEE生成的轨迹无虚假循环，而在线探索方法（如OS-Genesis）易产生循环。
- 微调后智能体性能提升: 在SEE数据上微调的GUI智能体在任务成功率和跨屏幕泛化上优于使用其他数据集训练的模型。

- 图来源: Figure 1（左图：SEE数据集与现有数据集在轨迹长度和页面复杂度平面上的对比；右图：SEE提供多级对齐监督）

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: SEE提供了一种可复现、低成本的数据生成范式，使得训练长程、鲁棒的GUI智能体成为可能，尤其适用于需要多步导航和中断处理的真实场景。
- 对 Skill: 通过图规划，智能体可以学习到从任意状态到目标的路径，提升技能组合和泛化能力。
- 对 Memory: 显式图结构可作为外部记忆，辅助智能体在长程任务中回溯和规划，减少对内部记忆的依赖。
- 对评测: SEE数据集提供了多级监督（高层指令、子目标、低层动作），可用于更细粒度的评估，例如子目标达成率和路径效率。

### 11. 局限性

- 探索阶段依赖UI解析器和VLM生成元素描述，可能引入噪声或遗漏元素。
- 图构建需要多次交互，对于频繁更新的应用可能成本较高。
- 合成轨迹基于静态图，无法覆盖应用更新后的新状态。

### 12. 我的看法 / 个人笔记

SEE通过结构感知的探索与图规划合成，优雅地解决了GUI数据生成中的覆盖率和长程问题。其将数据生成从‘在线推理’转为‘离线规划’的思路具有启发性，但探索阶段的效率和图维护的鲁棒性仍需进一步验证。未来可探索动态图更新和跨应用图融合。

### 14. 总结

SEE提出两阶段框架：先通过结构感知探索构建屏幕-元素转移图，再基于图规划合成可执行的长程GUI轨迹。生成的轨迹平均14.8步、页面元素24.63个，无虚假循环，微调后的智能体在任务成功率和泛化上优于现有数据集。该方法为训练鲁棒的长程GUI智能体提供了可扩展的数据生成方案。

## 3. Sparse Evidence Can Suffice: Agentic Evidence Seeking for Multimodal Video Misinformation Detection

**Topics:** 多模态视频虚假信息检测, 智能体证据获取, 强化学习
**Authors:** Haochen Zhao, Yongxiu Xu, Xinkui Lin, Dong Xie, Jiarui Lu, Yuqi Qian, Yubin Wang, Hongbo Xu, Gaopeng Gou
**Venue / Year:** AAAI 2027
**Links:** [Abstract](http://arxiv.org/abs/2607.18080v1) | [PDF](http://arxiv.org/pdf/2607.18080v1.pdf)

### 1. 一句话结论

提出SIEVE框架，通过智能体在预算约束下主动获取稀疏但关键的多模态证据（OCR、ASR、视觉帧），在多个基准上以紧凑证据包达到接近饱和的Macro-F1，并生成可审计的证据获取轨迹。

### 2. 为什么重要

传统方法处理整个视频导致冗余并可能掩盖关键证据，SIEVE通过解耦证据获取与验证，提升了检测的透明性和可解释性，为开放世界中的可审计虚假信息检测提供了新范式。

### 3. 研究问题

1. 视频虚假信息检测中，是否只需少量关键证据即可做出可靠判断？
2. 如何训练智能体在有限预算下主动获取互补且关键的多模态证据？

### 4. 背景与问题设定

多模态视频虚假信息检测通常被视为整体视频理解任务，但真实虚假信息往往具有稀疏且组合的证据结构：可靠决策仅依赖少数耦合线索，大部分视频内容冗余。现有方法多为端到端分类器，缺乏显式、可解释的多步证据推理能力。SIEVE将检测分解为预算约束下的证据获取与验证两个阶段。

### 5. 方法概览

- 任务 / 环境: 视频虚假信息检测，输入为视频及其平台提供的描述字段（Description），输出为Fake/Real标签。数据集FakeSV和FakeTT提供Event和Description字段，但SIEVE仅使用Description作为初始声明上下文，排除Event和评论。
- 变量 / 干预设计: 将检测分解为证据获取与验证两个阶段：智能体在预算约束下从视频的OCR、ASR、视觉帧（CTX和KEY）等通道中主动选择证据，构建紧凑证据包，再由验证器基于描述和证据包做出判断。证据空间通过TransNetV2进行镜头分割，PaddleOCR提取OCR文本，Whisper生成ASR转录，DINOv2 ViT-S/14编码视觉候选。最大交互次数设为10步，视觉图像累计上限为24张。
- 评价指标: Macro-F1
- 模型比较: 在FakeSV和FakeTT数据集上与多种基线方法比较，包括传统多模态融合方法（如FakeSV、FakingRecipe）和基于大语言模型的方法（如Fact-R1、ExMRD）。SIEVE在FakeSV上Macro-F1达到0.876，优于Fact-R1（0.851）和ExMRD（0.838）；在FakeTT上达到0.842，优于Fact-R1（0.821）和ExMRD（0.809）。
- 我的理解: 该方法将证据获取建模为预算约束下的序列决策问题，通过强化学习（GRPO）训练智能体在有限交互次数内获取最具判别力的证据。奖励函数结合验证结果（正确时基础奖励B(s)=+1.0，错误时B(s)=-1.2）、证据覆盖度（0.80权重）和交互成本（惩罚重复/无效动作），避免了全量视频处理的冗余。

### 6. 关键发现

- 稀疏证据足以实现可靠检测: 在FakeSV和FakeTT上，SIEVE使用平均仅3.2个证据槽（来自补充材料）即可达到接近饱和的Macro-F1（FakeSV: 0.876, FakeTT: 0.842），表明少量关键线索足以做出准确判断。
- 证据获取过程可解释: SIEVE的决策过程产生显式的证据获取轨迹，包括每个READ动作的槽位和通道来源，提高了检测的透明度和可审计性。
- 跨数据集零样本泛化能力: 在FakeTT训练的模型直接应用于FakeVV（零样本设置），Macro-F1达到0.793，优于基线方法Fact-R1（0.771）和ExMRD（0.758），表明证据获取策略具有一定泛化性。

## 4. MADA-RL: Multi-Agent Debate-Aware Reinforcement Learning for Parameter-Efficient Reasoning in Compact Models

**Topics:** cs.LG, cs.AI, cs.CL, cs.MA
**Authors:** Martino M. L. Pulici, Cuong Xuan Chu, Evgeny Kharlamov, Zifeng Ding, Volker Tresp, Yunpu Ma
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.18006v1) | [PDF](http://arxiv.org/pdf/2607.18006v1.pdf)

### 1. 一句话结论

MADA-RL通过反事实评论优势信号和LoRA微调，在紧凑模型上以极少可训练参数实现了数学推理性能的提升。

### 2. 为什么重要

该工作展示了在资源受限的紧凑模型上，通过角色专门化的多智能体强化学习与辩论机制结合，可以高效提升推理能力，为低资源场景下的模型优化提供了新思路。

### 3. 研究问题

1. 如何设计轻量级的后训练框架，使紧凑模型在数学推理任务上达到更高准确率？
2. 反事实优势信号能否有效引导评论者智能体纠正生成者错误，而非简单模仿？

### 4. 背景与问题设定

大型语言模型在推理任务上表现优异，但训练成本高昂，尤其对于参数不超过4B的紧凑模型。现有强化学习方法常需全模型微调，存在信用分配不稳定和测试时推理利用不足的问题。MADA-RL旨在结合多智能体辩论与参数高效微调，解决上述局限。

### 5. 方法概览

- 任务 / 环境: 数学推理基准测试，包括Math-500、AIME 2024/2025、AMC-23和Minerva-Math。
- 变量 / 干预设计: 将紧凑模型（DeepSeek-R1-Distill-Qwen-1.5B）微调为生成者和评论者两种角色，采用LoRA适配器仅更新少量参数。训练分为两阶段：先生成者独立训练，再基于生成者输出构建评论者数据集，并使用反事实优势信号训练评论者。
- 评价指标: 准确率（accuracy），基于符号等价性判断答案正确性。
- 模型比较: 与DeepSeek-R1、Still-3、DeepScaleR、Open-RS系列、Tina等基线模型比较，并对比了LoRA版本（DeepScaleR-LoRA、Still-3-LoRA）。
- 我的理解: 该方法的核心创新在于反事实优势：评论者的奖励减去生成者平均准确率的两倍，从而鼓励评论者在生成者失败时给出正确答案，而非简单复制正确结果。

### 6. 关键发现

- MADA-RL在准确率-可训练参数前沿上表现优异: 在五个数学推理基准上，MADA-RL将DeepSeek-R1-Distill-Qwen-1.5B的平均准确率从39.9%提升至41.9%（+2.0个百分点，p<0.001），且可训练参数仅为全微调基线的1/16。
- 反事实优势信号是性能提升的关键: 控制实验表明，反事实优势产生了所有评估模型中最高评论者改进率，说明训练后的评论者学会了纠正生成者错误，而非模仿它们。
- MADA-RL接近但未超越最强基线: 与DeepScaleR和Still-3相比，MADA-RL准确率略低，但这些基线使用了更大的训练数据集；论文分析了这一差距及相关的推理时间成本。

MADA-RL通过精巧的反事实优势信号设计，在参数高效微调下实现了紧凑模型推理能力的提升，思路新颖且实用。然而，其性能提升幅度有限（+2个百分点），且依赖多智能体辩论带来的额外推理开销，实际部署时需权衡成本与收益。未来可探索更高效的优势信号或结合更大规模数据。

## 5. Human Grounded Evaluation of Large Language Models for Optical Network Automation

**Topics:** GUI Agent
**Authors:** Kiarash Rezaei, Omran Ayoub, Paolo Monti, Carlos Natalino
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2607.18068v1) | [PDF](http://arxiv.org/pdf/2607.18068v1.pdf)

### 1. 一句话结论

HuGLEN 提出一种结合 LLM-as-a-judge 与少量专家标注的自动化评估流水线，在光网络 QoT 解释任务中，12B 参数的 Gemma3 模型在质量与效率间取得最佳平衡。

### 2. 为什么重要

该工作解决了 LLM 在网络自动化中评估缺乏可扩展性与人类对齐的问题，引入质量-效率得分 (QES) 为资源受限场景下的模型选择提供了实用指南。

### 3. 研究问题

1. 如何设计一种可扩展且与人类判断一致的 LLM 评估框架？
2. 在光网络 QoT 解释任务中，不同规模 LLM 在解释质量与推理效率之间如何权衡？

### 4. 背景与问题设定

LLM 正被广泛用于网络自动化，但其输出质量和推理成本差异巨大。现有评估方法如 BLEU/ROUGE 仅捕捉表面相似性，人工评估虽准确但成本高、难扩展。LLM-as-a-judge 方法可自动化评估，但在专业领域与专家一致性不足。光网络 QoT 估计中，XAI 输出的 SHAP 值需转化为操作员可理解的解释，这为评估 LLM 提供了具体场景。

### 5. 方法概览

- 任务 / 环境: 光网络质量传输 (QoT) 估计任务，使用 XGBoost 模型预测误码率，SHAP 生成局部特征重要性及交互值，需 LLM 将其转化为自然语言解释。
- 变量 / 干预设计: HuGLEN 流水线包含四阶段：任务定义（标准化提示生成）、LLM 推理（候选模型输出）、LLM 评判选择（基于与专家评分的一致性选出最佳评判模型）、自动评估（用选定评判模型计算 QES 排名）。
- 评价指标: 正确性 (Correctness, 0/1)、范围 (Scope, 0/1)、有用性 (Usefulness, 0-5)，聚合为复合质量指数 (CQI)，再结合参数规模效率因子计算 QES。
- 模型比较: 比较 DeepSeek-R1 (1.5B/14B/32B)、Gemma3 (4B/12B)、Qwen3 (4B/14B) 共 7 个模型，Prometheus-7B-v2.0 被选为 LLM 评判模型。
- 我的理解: HuGLEN 的核心创新在于用少量专家标注校准 LLM 评判，从而在保持与人类判断一致性的同时实现大规模自动评估；QES 将质量与效率统一为单一指标，便于实际部署决策。

### 6. 关键发现

- Prometheus-7B-v2.0 与人类专家一致性最高: 在正确性、范围、有用性三个指标上，Prometheus-7B-v2.0 的评分与四位人类专家的平均一致性均优于其他候选评判模型，因此被选为自动评估阶段的 LLM 评判。
- Gemma3 (12B) 取得最高 QES: 在质量-效率得分上，Gemma3 (12B) 以约 78 分领先，表明 10-15B 参数的中等规模 LLM 在解释质量与计算效率间达到最佳平衡；更大模型 (如 32B) 质量提升有限但成本显著增加，更小模型 (4B) 则质量不足。
