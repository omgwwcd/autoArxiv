## 1. MaskClaw: Edge-Side Personalized Privacy Arbitration for GUI Agents with Behavior-Driven Skill Evolution

**Topics:** GUI Agent, Privacy, Edge Computing
**Authors:** Yanqiu Zhao, Dongying Zheng, Kaibo Huang, Yukun Wei, Zhongliang Yang, Linna Zhou
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.28646v1) | [PDF](http://arxiv.org/pdf/2605.28646v1.pdf) | [Code](https://github.com/Theodora-Y/MaskClaw)

### 1. 一句话结论

MaskClaw在边缘侧通过本地证据提取、策略记忆检索和Allow/Mask/Ask三级仲裁，在原始截图离开可信环境前决定隐私暴露，并利用用户反馈经沙箱验证演化可复用的隐私技能。

### 2. 为什么重要

GUI代理依赖截图执行任务，但截图常含私密信息；云端VLM在决定保护前已上传原始截图，造成隐私悖论。MaskClaw将隐私决策置于边缘侧，结合上下文和用户策略，实现细粒度、可演化的隐私保护，填补了静态PII检测与云端推理之间的空白。

### 3. 研究问题

1. 边缘侧隐私仲裁能否在截图离开设备前准确判断Allow/Mask/Ask？
2. 用户反馈能否转化为可复用的隐私技能并提升策略准确性？

### 4. 背景与问题设定

GUI代理通过截图理解界面并执行任务，但截图常包含私密信息。现有方法要么依赖静态PII检测（忽略上下文），要么将原始截图上传至云端VLM后再决定保护（造成先暴露后保护）。隐私决策需考虑任务、接收者、应用状态和用户角色，但缺乏统一框架。MaskClaw在边缘侧（用户或组织可控环境）进行仲裁，避免原始截图外泄。

### 5. 方法概览

- 任务 / 环境: GUI代理交互场景，包括移动端和桌面端任务，覆盖医疗、金融、办公等多领域。
- 变量 / 干预设计: MaskClaw作为边缘侧隐私仲裁层，包含在线循环（感知→检索→仲裁→安全截图构建）和演化循环（反馈→技能候选→沙箱验证→合并到策略记忆）。仲裁输出Allow、Mask或Ask三种动作。
- 评价指标: 联合决策准确率、Mask F1、Ask召回率、泄露率、过度保护率、原始上传率。技能演化使用加权评分：规则覆盖0.4、测试决策准确0.3、安全惩罚0.2、技能质量0.1。
- 模型比较: 与模式匹配、云端推理、路由等基线对比，MaskClaw在相同协议下减少过度确认、过度屏蔽和原始截图暴露。具体基线包括：模式匹配（基于正则和PII列表）、云端推理（GPT-4o等VLM）、路由（基于规则或LLM的决策路由）。
- 我的理解: MaskClaw的核心创新在于将隐私决策从云端前移到边缘，并通过策略记忆和技能演化实现个性化。其Allow/Mask/Ask三级决策比简单的二值化（允许/屏蔽）更灵活，Ask机制处理边界情况。演化循环通过沙箱验证确保技能安全，避免恶意或错误策略生效。

### 6. 关键发现

- 边缘仲裁优于云端或静态方法: 在P-GUI-Evo基准的L2全量832条策略行上，MaskClaw的联合决策准确率显著高于模式匹配、云端推理和路由基线，且泄露率和原始上传率更低。具体地，MaskClaw的泄露率低于5%，而云端推理基线超过20%。
- 用户反馈可有效演化隐私技能: 在五个技能演化场景（iCloud清理、应用权限、高价值转账、日程合并、驾驶模式）中，MaskClaw通过用户修正、取消等反馈，经沙箱验证后更新策略记忆，使技能覆盖率和决策准确率逐步提升。20次迭代后，平均评分从初始的0.6提升至0.85以上。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为GUI代理提供可部署的边缘隐私层，使代理能在不泄露敏感信息的前提下安全执行任务，尤其适用于医疗、金融等隐私敏感领域。
- 对 Skill: 技能演化机制使隐私策略能随用户行为自适应调整，减少人工配置成本，提升长期可用性。
- 对 Memory: 策略记忆作为可审计的规则库，支持个性化隐私决策，且演化过程保持可追溯性。
- 对评测: P-GUI-Evo基准提供了多角色、多场景的隐私决策评估框架，可推动GUI代理隐私保护研究的标准化。

### 11. 局限性

- 当前基准P-GUI-Evo基于合成数据和模拟用户，真实部署中的用户行为多样性和反馈噪声可能影响技能演化效果。
- MaskClaw依赖本地感知模块（OCR/VLM），在低质量截图或复杂布局下证据提取可能不准确，影响仲裁质量。
- 技能演化中的沙箱验证依赖LLM-Judge，其判断可能受模型偏见影响，且验证成本较高。

### 12. 我的看法 / 个人笔记

MaskClaw提出了一个实用且创新的边缘隐私仲裁框架，将隐私决策从云端前移到边缘，并通过技能演化实现个性化。其Allow/Mask/Ask三级决策和沙箱验证机制设计合理，但真实部署中的鲁棒性和可扩展性仍需进一步验证。未来可探索更高效的感知模块和更轻量的验证方法。

### 14. 总结

MaskClaw是一个边缘侧隐私仲裁系统，用于GUI代理。它通过本地感知、策略记忆检索和Allow/Mask/Ask决策，在原始截图离开可信环境前控制隐私暴露。同时，它利用用户反馈通过沙箱验证演化隐私技能。在P-GUI-Evo基准上，MaskClaw优于模式匹配、云端推理和路由基线，展示了边缘隐私仲裁和技能演化的有效性。

## 2. Do Agents Need Semantic Metadata? A Comparative Study in Agentic Data Retrieval

**Topics:** GUI Agent, Data Retrieval, Semantic Metadata, FAIR Principles, LLM-as-a-Judge
**Authors:** Shiyu Chen, Tarfah Alrashed, Alon Halevy, Natasha Noy
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.28787v1) | [PDF](http://arxiv.org/pdf/2605.28787v1.pdf)

### 1. 一句话结论

语义元数据（如schema.org）对于自主智能体可靠检索可操作数据仍不可或缺，相比无结构网页搜索，其FAIR合规精度高出65.7%。

### 2. 为什么重要

随着LLM驱动的智能体兴起，该研究直接挑战了“语义元数据是否仍必要”这一核心问题，为构建可靠、可执行的自主数据工作流提供了关键证据。

### 3. 研究问题

1. 语义元数据是否在发现可操作数据集方面优于无结构网页搜索？
2. 无结构网页搜索在“最后一英里”数据获取中失败的原因是什么？
3. 两种智能体在探索广度与检索精度之间如何权衡？

### 4. 背景与问题设定

自主智能体时代，机器可操作数据对数据驱动工作流至关重要。传统上，schema.org等语义元数据支撑了FAIR原则和Google Dataset Search等工具。但LLM能直接导航无结构网页，引发疑问：语义元数据是否仍必要？本文对比了基于无结构网页搜索的基线智能体和基于90M数据集语义元数据的语义智能体。

### 5. 方法概览

- 任务 / 环境: 使用NTCIR-16 Data Search基准中的58个英文关键词查询，模拟真实数据发现意图。基线智能体查询Google Search索引（数十亿网页），语义智能体查询Google Dataset Search索引（9000万数据集元数据记录）。
- 变量 / 干预设计: 控制变量：相同智能体框架（ADK）和底层模型（Gemini 2.5 Pro），温度设为0，最多返回3个数据集。唯一变量是目标索引和搜索工具。基线智能体附加数据关键词（如“dataset”），语义智能体直接提取核心关键词。
- 评价指标: 基于FAIR原则的三维评估：相关性（Findable）、数据可访问性（Accessible）、数据集页面类型（Interoperable & Reusable）。使用LLM-as-a-judge自动评估，并经过人工验证（线性加权Cohen's Kappa分别为0.73、0.78、0.74）。
- 模型比较: 基线智能体 vs 语义智能体。基线智能体覆盖更广（多回答40%的问题），但语义智能体精度更高（整体FAIR合规精度高65.7%）。
- 我的理解: 该方法巧妙地将FAIR原则转化为可自动评估的指标，并通过严格控制变量隔离了语义元数据的影响。使用真实在线索引而非静态语料库增强了生态效度。

### 6. 关键发现

- 语义智能体在可操作数据检索上精度显著更高: 语义智能体在元数据丰富注册表上的精度高44.9%，在机器可读下载页面上的精度高46.6%。
- 基线智能体频繁遭遇“最后一英里”效用失败: 基线智能体返回的结果中，20.1%为散文密集型页面，8.5%为门户登录页，而非实际数据页面。
- 基线智能体覆盖更广但精度更低: 基线智能体多回答40%的问题，但语义智能体的整体FAIR合规精度高出65.7%。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 构建可靠的数据驱动智能体时，应优先集成结构化语义元数据源（如Google Dataset Search），而非仅依赖通用网页搜索。
- 对 Skill: 智能体应具备识别和利用语义元数据的能力，以提升数据检索的精度和可操作性。
- 对 Memory: 语义元数据可作为智能体长期记忆的一部分，用于缓存和索引已知的可操作数据集。
- 对评测: 评估智能体数据检索时，应超越传统相关性指标，纳入FAIR原则（可访问性、互操作性、可重用性）的自动化评估。

### 11. 局限性

- LLM-as-a-judge评估虽经人工验证，但仍可能引入模型偏差。

### 12. 我的看法 / 个人笔记

该研究有力地证明了语义元数据在自主智能体数据检索中的核心价值，尤其强调了“可操作性”而非单纯的“可发现性”。其将FAIR原则转化为可自动评估指标的方法具有重要实践意义。未来工作可探索混合策略，结合无结构搜索的广度与结构化元数据的精度。

## 3. Agent Explorative Policy Optimization for Multimodal Agentic Reasoning

**Topics:** GUI Agent, Reinforcement Learning, Vision-Language Models
**Authors:** Minki Kang, Shizhe Diao, Ryo Hachiuma, Sung Ju Hwang, Pavlo Molchanov, Yu-Chiang Frank Wang, Byung-Kwan Lee
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.28774v1) | [PDF](http://arxiv.org/pdf/2605.28774v1.pdf)

### 1. 一句话结论

AXPO通过固定思考前缀并重采样工具调用，有效缩小了多模态智能推理中思考与行动之间的差距，在9个基准上平均Pass@1和Pass@4分别提升1.8个百分点，且8B模型在Pass@4上超越32B基线，参数减少4倍。

### 2. 为什么重要

该工作揭示了标准RL方法（如GRPO）在训练智能体工具使用时的结构性缺陷，并提出一种轻量级、高效的解决方案，显著提升了模型在需要外部工具的真实世界任务中的表现，为多模态智能推理的RL训练提供了新范式。

### 3. 研究问题

1. 如何解决多模态智能推理中思考与工具使用之间的结构性不对称（Thinking-Acting Gap）？
2. 如何通过定向探索工具调用，在固定计算预算下提高智能体RL训练中工具使用的学习信号？

### 4. 背景与问题设定

视觉-语言模型在复杂推理任务上表现优异，但许多现实问题需要外部工具（如代码解释器、网络搜索）辅助，而内部推理无法解决。智能推理需要交替进行思考（默认模式）和工具使用（高方差辅助行为），两者存在结构性不对称，即Thinking-Acting Gap。标准RL方法（如GRPO）下，工具使用仅出现在约20-35%的rollout中，且当尝试时，工具使用子组在约40%的问题上全部失败，导致工具调用令牌的学习信号被抑制。

### 5. 方法概览

- 任务 / 环境: 多模态智能推理任务，模型需根据文本和图像输入，自主决定何时思考、何时使用工具（Python解释器、网络搜索、图像放大），最终输出答案。
- 变量 / 干预设计: AXPO针对GRPO中工具使用子组全部失败的情况，固定其思考前缀，重采样工具调用及其后续生成，并基于不确定性（低置信度）选择前缀，优先处理最不确定的样本。
- 评价指标: Pass@1和Pass@4（%），在9个多模态基准上评估，涵盖推理、感知和搜索三类任务。
- 模型比较: 与Base（无训练）、SFT、SFT+GRPO对比，在Qwen3-VL-Thinking的2B/4B/8B规模上测试。
- 我的理解: AXPO的核心创新在于识别出工具调用是思考与行动之间的关键分歧点，通过固定已产生工具调用意图的思考前缀，仅重采样工具调用及其后续，将探索集中在最需要学习信号的位置，从而高效提升工具使用的成功率。不确定性前缀选择进一步优化了重采样效率。

### 6. 关键发现

- Thinking-Acting Gap的诊断: GRPO训练中，工具使用仅占rollout的20-35%，且工具使用子组全部失败的概率（约40%）远高于无工具子组（约25%），导致工具调用令牌获得非正优势。
- 工具调用重采样的有效性: 固定思考前缀后重采样工具调用，16次采样平均产生2.9-3.4个语义簇，证明工具调用是真正的分歧点，重采样能有效探索多样化的工具使用策略。
- AXPO的性能提升: 在9个多模态基准上，SFT+AXPO平均Pass@1和Pass@4比SFT+GRPO分别提升1.8个百分点；8B模型在Pass@4上超越32B Base，参数减少4倍。

## 4. VeriTrip: A Verifiable Benchmark for Travel Planning Agents over Unstructured Web Corpora

**Topics:** GUI Agent, Travel Planning, Benchmark, Multimodal Retrieval, Factual Verification
**Authors:** Yuting Xu, Jiayi Tian, Jian Liang, Xin Xiong, Hang Zhang, Mu Xu, Xiao-Yu Zhang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.28683v1) | [PDF](http://arxiv.org/pdf/2605.28683v1.pdf)

### 1. 一句话结论

VeriTrip 通过构建多模态检索库（MRB）和可验证知识库（VKB），引入细胞级事实验证，揭示了当前 MLLM 在无约束网页环境中进行检索增强规划时存在严重的“检索-推理权衡”，且地理合理性（AM）普遍较差。

### 2. 为什么重要

现有旅行规划基准过度依赖 API 接口，忽略了真实网页中的信息噪声、多源矛盾以及视觉感知与逻辑规划的结合。VeriTrip 首次将评估焦点转向基于证据的推理，并引入细粒度的事实验证协议，为下一代鲁棒规划代理的评估提供了更严格的基准。

### 3. 研究问题

1. 当代理必须从非结构化、多模态的网页证据中主动发现事实时，它们能否生成事实正确的旅行计划？
2. 自主检索的认知负荷如何影响代理对指令的保持和高层偏好满足？

### 4. 背景与问题设定

现有旅行规划基准（如 TravelPlanner、TripTailor）将规划简化为对结构化 API 的固定参数调用，忽略了信息噪声、多源矛盾以及视觉感知与逻辑规划的结合。同时，深度研究基准（如 DeepResearchBench）虽注重信息广度，但缺乏对合成结果的事实验证。VeriTrip 旨在弥合这一差距，评估代理在无约束、多模态网页环境中的检索增强规划能力。

### 5. 方法概览

- 任务 / 环境: 静态多模态检索库（MRB），包含 8,210 份文档和 4,146 张图片，来自真实旅游网站和论坛（如 Trip.com），冻结为沙盒环境。代理通过文本搜索（docSearch）、图像搜索（imgSearch）、文档获取（getDocument）以及专用函数（如 getRecommendRestaurant、getRestaurantByName、getRestaurantByFood）与 MRB 交互。
- 变量 / 干预设计: 代理接收包含文本约束和模糊视觉锚点（如裁剪的社交媒体照片）的查询，必须自主检索并合成结构化 JSON 行程。查询分为简单（2-5天，1-2个偏好）、中等（1-6天，2-5个偏好）、复杂（3-7天，2-8个偏好）三个难度级别，共 228 个查询，分布均衡。
- 评价指标: 交付率（DR，格式正确性）、事实可靠性（FR，细胞级事实验证，精确匹配数值字段，语义相似度匹配文本字段）、通过率（PR，硬约束如预算）、偏好满足率（PFR，软约束如餐饮偏好）、平均边际（AM，基于 TSP 的地理合理性度量，计算代理路线与最优路线的归一化距离差）。
- 模型比较: 评估了非推理 MLLM：GPT-5-mini、GPT-4o、GPT-4o-mini、Gemini-2.5-flash、Claude-4.5-Sonnet、Claude-3.7-Sonnet、Qwen-VL-Max、Qwen3-VL-235B；以及推理 MLLM：o3、o4-mini、Gemini-2.5-pro。检索工具使用 Qwen3-Embedding-8b（文本）和 DINOv2（图像）。
- 我的理解: VeriTrip 通过构建一个冻结的、包含真实噪声的多模态网页快照，迫使代理进行主动检索和跨模态消歧，同时利用可验证知识库（VKB）实现细粒度的事实验证，从而严格区分逻辑推理与参数幻觉。VKB 的构建采用双阶段专家协议：首先使用 Qwen3-Max 进行语义过滤（去除广告但保留自然噪声）和结构化解析，随后由人工交叉校验所有条目，确保事实准确性。评估协议包括格式检查、细胞级事实检查、常识与偏好检查以及地理一致性检查，提供了多维度的能力评估。

### 6. 关键发现

- 系统性失败：所有 MLLM 在无约束检索规划中表现不佳: 即使是最先进的模型（如 o4-mini）在复杂任务上的事实可靠性（FR）也仅为 53.08%，偏好满足率（PFR）低至 12.16%，表明当前模型难以同时满足事实准确性和用户偏好。地理合理性（AM）普遍较差，例如 GPT-4o 在简单任务中 AM 为 0.45 km/POI，复杂任务中升至 1.12 km/POI，表明路线规划缺乏空间常识。
- 检索-推理权衡：自主检索的认知负荷侵蚀指令保持: 强制代理使用视觉工具解决歧义会耗尽推理能力，导致高层偏好满足率崩溃。例如，o4-mini 在简单任务中 PFR 为 32.05%，但在复杂任务中降至 12.16%。同时，复杂任务中工具使用次数增加（如 o4-mini 从简单任务的 12.3 次升至复杂任务的 18.7 次），但 PFR 反而下降，表明检索与推理之间存在竞争。
- 不确定性驱动的检索改善事实基础: 尽管检索增加了认知负荷，但它抑制了模型对参数记忆的依赖，从而提高了事实可靠性。例如，o4-mini 在复杂任务中的 FR（53.08%）高于简单任务（44.34%），表明更复杂的任务迫使代理进行更多检索，减少了幻觉。类似地，Claude-4.5-Sonnet 的 FR 从简单任务的 41.23% 提升至复杂任务的 49.87%。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 未来代理系统需要设计更高效的检索-推理协调机制，例如将检索结果缓存或结构化，以减少认知负荷对推理的干扰。同时，应集成地理空间推理模块以改善路线规划。
- 对 Skill: 代理需要发展更强的跨模态消歧技能，特别是从模糊视觉信息中识别目标实体，并整合多源文本证据。此外，需要提升空间常识推理能力。
- 对 Memory: 代理应减少对参数记忆的依赖，转而依赖外部检索，但需注意检索可能带来的认知负荷，需平衡记忆与检索的使用。
- 对评测: 评估应超越简单的格式检查和 LLM 评判，采用基于 ground truth 的细粒度事实验证（如细胞级检查），并纳入地理合理性等维度，以全面衡量代理能力。

### 11. 局限性

- VeriTrip 基于静态沙盒环境，无法评估代理在实时动态环境（如实时票价更新、售罄情况）中的交互执行能力。

### 12. 我的看法 / 个人笔记

VeriTrip 通过精心设计的沙盒环境和细粒度验证协议，为评估检索增强规划代理提供了更严格的基准。其揭示的“检索-推理权衡”是当前 MLLM 代理的核心瓶颈，提示未来研究需关注如何在不牺牲推理能力的前提下增强检索效率。然而，静态环境与真实动态网页的差距仍需进一步弥合，且地理合理性（AM）的发现表明代理在空间推理方面存在显著不足。

## 5. An LLM-Based Assistance System for Intuitive and Flexible Capability-Based Planning

**Topics:** GUI Agent, LLM, Automated Planning, Industrial Automation, Hybrid AI
**Authors:** Luis Miguel Vieira da Silva, Nicolas König, Felix Gehlhoff
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2605.28666v1) | [PDF](http://arxiv.org/pdf/2605.28666v1.pdf)

### 1. 一句话结论

结合LLM的自然语言交互与符号规划器的形式化保证，显著提升了工业自动化中基于能力规划的可用性和适应性。

### 2. 为什么重要

该工作解决了传统符号规划器在工业动态环境中难以被非专家用户使用的问题，通过LLM层实现自然语言查询、结果解释和规划自适应，同时保持形式化正确性，为实际工业部署提供了可行方案。

### 3. 研究问题

1. 如何利用LLM实现基于能力规划的自然语言交互，包括知识查询和规划问题表述？
2. 如何以用户可理解的方式解释可满足和不可满足的规划结果？
3. 如何从不可满足的求解器反馈和报告的环境变化中推导出可操作的适应建议以支持规划调整？

### 4. 背景与问题设定

现代工业自动化系统面临动态环境、模块化可重构资源的复杂性，需要自动规划流程序列。基于能力的方法通过语义知识模型自动生成计划，但实际使用受限：求解器反馈（尤其不可满足时）难以解释，且知识模型需随运行条件变化而调整。

### 5. 方法概览

- 任务 / 环境: 模块化生产系统，包含异构资源（如制造机器、自主机器人），需处理动态变化的需求、资源故障等。
- 变量 / 干预设计: 提出混合辅助系统，在已有基于能力SMT规划方法上增加LLM层，用于自然语言交互、解释和适应。系统分为四个组件：能力接地、符号规划、结果解释、规划适应，实现为路由代理工作流，中央路由器委托给五个专门代理。
- 评价指标: 测试用例正确率：知识查询（9/10正确）、可满足规划（4/4正确）、不可满足规划（3/4产生具体修复建议）、自适应规划（5/5通过迭代用户批准的知识模型修改得到可满足计划）。
- 模型比较: 与纯LLM规划（缺乏形式保证、可能幻觉）和纯符号规划（难以解释、适应性差）对比，混合方法结合两者优势。
- 我的理解: 该方法本质上是将LLM作为符号规划器的‘前端’，处理自然语言输入输出和知识模型调整，而规划的正确性仍由SMT求解器保证，通过人机协同确保模型修改的可靠性。

### 6. 关键发现

- 混合架构有效提升可用性: 在23个测试用例中，知识查询和可满足规划全部正确，不可满足规划中75%产生具体修复建议，自适应规划全部成功转化为可满足计划。
- 路由代理工作流优于单体和全自主多智能体: 路由代理工作流避免了提示过载和不可预测的执行路径，同时保持灵活性，并确保人机协同检查点位于固定位置。
- LLM能有效推导不可满足问题的修复建议: 通过分析最小不可满足约束集，LLM能确定根本原因并提出具体知识模型修改建议，经用户批准后重新规划成功。

从用户请求开始，经过能力接地组件（LLM解释意图并查询知识图谱），进入符号规划组件（SMT求解器生成计划或返回不可满足集），结果解释组件将输出转为自然语言，若不可满足或条件变化则进入规划适应组件（LLM分析并建议修改，经用户批准后更新知识图谱并重新规划）。

### 9. 次重要图

- 为什么也重要: Figure 2: Agentic workflow 展示了三种交互路径（知识查询、规划请求、运行时故障报告）及人机协同检查点，说明了路由代理工作流的具体实现。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 路由代理工作流为构建可解释、可追溯的LLM代理系统提供了有效模式，尤其适用于需要人机协同和形式保证的场景。
- 对 Skill: LLM可被用于技能（能力）的语义理解和动态调整，但需符号规划器确保执行正确性。
- 对 Memory: 知识图谱作为长期记忆存储能力模型，LLM负责查询和更新，但更新需用户批准以保证一致性。
- 对评测: 评估需覆盖多种场景（知识查询、可满足/不可满足规划、自适应规划），并关注人机交互的准确性和效率。

### 11. 局限性

- 评估基于单一模块化生产系统，泛化性需进一步验证。
- LLM可能产生不准确或误导性的解释或建议，尽管有人机协同检查，但用户可能缺乏足够知识判断。
- 系统依赖预定义能力本体，构建和维护本体仍需专家知识。

### 12. 我的看法 / 个人笔记

该工作巧妙地将LLM的灵活性与符号规划器的可靠性结合，通过路由代理工作流实现可解释、可适应的工业规划辅助系统。其核心贡献在于不仅处理了自然语言交互，还解决了不可满足规划的自适应问题，且通过人机协同确保安全性。未来可探索更复杂的自适应场景和跨领域泛化。
