## 1. Reinforcement Learning without Ground-Truth Solutions can Improve LLMs

**Topics:** 强化学习, 可验证奖励, 无真实答案训练, 代码生成, 优化问题
**Authors:** Yingyu Lin, Qiyue Gao, Nikki Lijing Kuang, Xunpeng Huang, Kun Zhou, Tongtong Liang, Zhewei Yao, Yi-An Ma, Yuxiong He
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27369v1) | [PDF](http://arxiv.org/pdf/2606.27369v1.pdf)

### 1. 一句话结论

RiVER 通过实例级排名和胜者偏重奖励塑造，在无真实答案的12个评分型优化任务上训练 LLM，不仅提升评分型基准 ALE-Bench 排名（Qwen3-8B 提升8.9%，GLM-Z1-9B-0414 提升9.4%），还显著泛化到精确求解基准 LiveCodeBench（绝对提升2.4%）和 USACO（绝对提升3.5%），而原始评分基线无法泛化。

### 2. 为什么重要

该工作将 RLVR 从依赖真实答案的传统狭窄场景扩展到无真实答案的开放优化任务，为利用丰富的可执行反馈训练通用编码能力开辟了新途径，并揭示了奖励校准在迁移学习中的核心作用。

### 3. 研究问题

1. 无真实答案的评分型优化任务能否为 LLM 提供有效的训练信号？
2. 原始评分作为奖励时存在尺度主导和频率主导问题，如何设计奖励校准方法以提升训练效果和泛化能力？

### 4. 背景与问题设定

传统 RLVR 依赖真实答案进行二元奖励，限制了其在无唯一正确答案的优化任务（如算法竞赛）中的应用。这些任务中，候选解可通过执行验证可行性和计算目标值，但缺乏真实最优解。本文研究如何利用这种可执行反馈训练 LLM，并解决原始评分带来的两个关键挑战：尺度主导（不同实例间评分幅度差异扭曲策略更新）和频率主导（频繁采样的次优解梯度累积压倒罕见的强解）。

### 5. 方法概览

- 任务 / 环境: AtCoder Heuristic Contest 的12个评分型优化任务，每个任务包含问题描述、确定性评估器和隐藏测试实例集。训练时对每个问题采样G个候选解（G为组大小），并执行在M个隐藏实例上。
- 变量 / 干预设计: RiVER 框架：首先对每个隐藏实例，对 G 个候选解进行实例级排名（使用平均秩处理平局）；然后应用胜者偏重奖励塑造：无效解得 -1，最佳有效解得 1，其他有效解得 clip((G-1-r)/(G-3),0,1)-0.5 的梯度反馈（范围 [-0.5,0.5]）；最后跨实例聚合排名塑造奖励，并用 GRPO 优化策略。
- 评价指标: ALE-Bench 评分排名、LiveCodeBench 和 USACO 的通过率。
- 模型比较: 基线包括使用原始评分的 GRPO 和直接使用原始评分聚合的 RLVR。RiVER 在 Qwen3-8B 和 GLM-Z1-9B-0414 上分别提升 ALE 排名 8.9% 和 9.4%，并在 LiveCodeBench 和 USACO 上分别绝对提升 2.4% 和 3.5%，而基线仅提升 ALE 排名但无法泛化。
- 我的理解: RiVER 的核心创新在于将原始评分转化为实例级排名，消除了跨实例的尺度差异，并通过胜者偏重塑造放大最佳解的信号，同时保留非胜者的有限信息，从而避免频率主导问题。这使得模型能从优化任务中学习到可迁移的编码能力。

### 6. 关键发现

- RiVER 提升评分型基准性能: 在 ALE-Bench 上，RiVER 使 Qwen3-8B 的评分排名提升 8.9%，GLM-Z1-9B-0414 提升 9.4%。
- RiVER 泛化到精确求解基准: 尽管仅在评分型任务上训练，RiVER 在 LiveCodeBench 上绝对提升 2.4%，在 USACO 上绝对提升 3.5%，而原始评分基线无法泛化。
- 奖励校准是关键因素: 原始评分基线虽提升 ALE 排名但无法泛化，表明仅靠可执行反馈不足，必须通过实例级排名和胜者偏重塑造进行校准才能实现迁移。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: RiVER 使智能体能从无真实答案的交互反馈中学习，例如在规划或资源分配任务中，通过比较不同策略的执行结果来优化行为，拓宽了训练信号的来源。
- 对 Skill: 该方法可提升 LLM 的通用编码技能，包括算法设计、调试和优化，因为训练信号来自执行反馈而非固定答案，未来可结合经验回放或长期记忆来存储高质量候选解以增强学习效率。
- 对 Memory: 当前方法未显式利用记忆机制，但未来可结合经验回放或长期记忆来存储高质量候选解，以增强学习效率。
- 对评测: 提示需要更丰富的评估基准，包括评分型任务和精确求解任务，以全面衡量模型在无真实答案场景下的泛化能力。

### 11. 局限性

- 训练任务仅限 AtCoder Heuristic Contest 的12个任务，领域覆盖有限，未来需验证在更广泛优化问题上的有效性。
- 方法依赖可执行环境，不适用于无法自动验证的任务（如开放式写作或主观推理）。
- 胜者偏重塑造可能过度强调最佳解，但这是方法论本身的设计权衡，旨在缓解频率主导问题；未来可探索更柔软的奖励分配策略。

### 12. 我的看法 / 个人笔记

RiVER 巧妙地利用实例级排名和胜者偏重奖励解决了连续评分反馈中的尺度与频率问题，展示了无真实答案训练的巨大潜力。未来工作可探索更丰富的奖励塑造策略和更广泛的任务领域，并融入记忆机制以增强学习。

### 13. 这篇工作引用的近期高影响力相关文章

- GRPO (Shao et al., 2024)
  为什么重要：GRPO 是 RLVR 的常用算法，通过组内相对优势估计实现策略优化，并有效降低 KL 散度。
  核心贡献：提出组相对策略优化，在无需价值模型的情况下利用组内奖励归一化计算优势，并结合裁剪与 KL 正则化训练 LLM。
  与当前论文关系：RiVER 基于 GRPO 框架，但将其标准二元或原始评分奖励替换为排名诱导的胜者偏重奖励，以处理连续评分型反馈。

  为什么重要：该工作探索将优化问题作为推理任务，与本文使用优化任务作为训练环境的方向一致。
  核心贡献：提出将 NP-hard 优化问题转化为 LLM 推理基准，并评估模型求解能力。
  与当前论文关系：本文进一步将优化问题用作训练环境，而非仅作为评估基准，并展示了向其他编码任务的泛化。

### 14. 总结

RiVER 提出一种无真实答案的 RLVR 框架，通过实例级排名和胜者偏重奖励塑造，将评分型优化任务转化为有效的训练信号。在12个 AtCoder 任务上训练后，不仅提升了评分型基准 ALE-Bench 的性能（Qwen3-8B 提升8.9%，GLM-Z1-9B-0414 提升9.4%），还泛化到精确求解基准 LiveCodeBench（绝对提升2.4%）和 USACO（绝对提升3.5%），而原始评分基线无法实现泛化。这证明适当的奖励校准对于利用可执行反馈提升通用编码能力至关重要。

## 2. Advancing Omnimodal Embodied Agents from Isolated Skills to Everyday Physical Autonomy

**Topics:** GUI Agent, Embodied Agent, Long-Horizon Task Planning
**Authors:** Junhao Shi, Zezheng Huai, Siyin Wang, Jia Chen, Yubang Wang, Zhaoye Fei, Hechang Chen, Jingjing Gong, Xipeng Qiu, Yu-Gang Jiang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27251v1) | [PDF](http://arxiv.org/pdf/2606.27251v1.pdf) | [Code](https://github.com/EmbodiedForge/RAS_interactivate_planner)

### 1. 一句话结论

OmniAct通过层次化异步架构融合统一技能路由、事件边界压缩记忆与异步视觉抢占，在40项真实长期任务中实现了端到端成功率的显著提升并保持近线性Token消耗。

### 2. 为什么重要

现有体现系统在跨域工具协调、长期上下文管理与物理失败自主恢复上存在割裂，而OmniAct首次联合解决这些挑战，为持久物理自主性提供了工程可行的框架，且性能增益在不同机器人平台上一致迁移。

### 3. 研究问题

1. 如何统一离散式网络物理工具与连续物理控制在单一决策流程中？
2. 如何设计记忆机制在长期部署中保持亚线性上下文增长并完成失败反思？
3. 如何通过语义级闭环验证实现VLA执行物理失败的自主检测与重规划？

### 4. 背景与问题设定

现有体现系统将API/IoT调用与物理操作分开处理，缺乏统一动作空间；长期部署中线性累积历史导致语义冲突和内存膨胀；VLA策略开环执行无法检测自身失败并恢复。

### 5. 方法概览

- 任务 / 环境: 两个真实机器人平台（UR5e六轴机械臂和Keenon轮式移动机器人）与4个IoT设备、12个虚拟API构成异构网络物理环境；40项长期任务分为三级复杂度（纯物理、网络物理、全模态）。
- 变量 / 干预设计: OmniAct由多模态语义规划器（技能路由）、自适应层次记忆（感官/情节/反思三层，事件边界压缩）和异步视觉抢占引擎（VLM低频检测异常，触发即时重规划）组成。
- 评价指标: 端到端成功率（E2E）用于评估完整任务完成，子任务成功率（Sub）用于捕捉部分进展，两者间的差距反映错误传播程度。
- 模型比较: 对比方法：Direct Policy（无规划/记忆）、SayCan（基于LLM+情感打分的规划）、Code as Policy（一次性代码生成）、RoboBrain2（32B体现VLM）。除RoboBrain2外均使用Gemini-3.1-Pro以确保架构差异孤立。
- 我的理解: 核心创新在于将技能路由、记忆压缩和视觉前馈闭环解耦为异步模块，而非端到端模型。事件边界压缩是关键：物理任务固有语义分割点（如子任务完成）可抑制无效Token增长。

### 6. 关键发现

- OmniAct在L3复杂任务上E2E成功率显著领先: 在UR5e和Keenon平台上，L3 E2E分别达到50%和54.2%，最佳基线SayCan在UR5e上的L3 E2E仅为43.8%，Code as Policy在导航任务上仅为37.5%。
- 子任务成功率体现恢复能力: L3子任务成功率在UR5e和Keenon上分别达到80.5%和80.6%，显著高于最佳基线（60.9%和57.9%），说明视觉抢占和反思记忆将灾难性失败转化为可恢复延迟。
- 背景记忆防止上下文增长失控: 在40k+累积Token的长期部署中，OmniAct保持近平坦Token消耗，而Raw History和Sliding Window基线则线性增长。同时记忆机制在长期偏好冲突场景中保持95%以上的约束满足率。
- 开放权重模型可达到专有模型性能: OmniAct能将中规模开放权重模型（如LLaMA-3）性能提升至与Gemini-3.1-Pro相当的水平，表明框架对模型规模具有泛化性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 提供了一种层次化异步架构模板，可将技能路由、记忆压缩与闭环验证解耦，适用于需要持久自主能力的机器人系统。
- 对 Skill: 技能路由应支持离散与连续动作的统一调度，并允许并行技能执行。异步视觉抢占是物理技能可靠性的关键补齐。
- 对 Memory: 事件边界驱动的压缩策略显著优于固定窗口或等距摘要，且反思记忆能实现错误累积学习。
- 对评测: E2E与Sub的双指标体系可量化错误传播程度，对长期任务评估具有参考意义。

### 11. 局限性

- 评估仅在实验室环境进行，部署环境未知；视觉抢占依靠VLM，存在推理延迟和误判风险。
- 依赖预定义技能库和工具模式，对新技能的零样本学习能力未明确评估。
- 主要考虑物理失败恢复，未涉及社交或人机交互中的失败场景。

### 12. 我的看法 / 个人笔记

OmniAct通过明确解耦规划、记忆和验证机制，展现了在长期自主任务中的鲁棒性。事件边界记忆扩展和异步视觉前馈是实用且有效的设计，但对VLM的依赖可能限制部署效率。未来可考虑将视觉抢占轻量化至小模型。

## 4. TOPS: First-Principles Visual Token Pruning via Constructing Token Optimal Preservation Sets for Efficient MLLM Inference

**Topics:** Multimodal Large Language Models, Visual Token Pruning, Efficient Inference
**Authors:** Tinghao Wang, Yichen Guo, Rui Huang, Zheng Lu, Qizhe Zhang, Chenxi Li, Yuan Zhang, Jiajun Cao, Zhirong Shen, Yaosong Du, Guangyan Gan, Wenya Wang, Lin William Cong, Shanghang Zhang
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27161v1) | [PDF](http://arxiv.org/pdf/2606.27161v1.pdf)

### 1. 一句话结论

TOPS 从信息论第一性原理出发，将视觉令牌剪枝形式化为构建最优保留集，在移除 77.8% 令牌时保持甚至提升 MLLM 性能，并缓解幻觉。

### 2. 为什么重要

MLLM 因大量视觉令牌导致推理开销巨大，TOPS 提供了一种无需训练、模型无关的剪枝方法，显著降低计算成本且不损失性能，甚至能缓解幻觉，对实际部署和未来轻量化设计有重要价值。

### 3. 研究问题

1. 如何从第一性原理出发，形式化视觉令牌剪枝的固有目标？
2. 剪枝后的令牌子集应满足哪些原则才能最大化保留原始信息？
3. TOPS 能否在不同 MLLM 架构和剪枝比例下一致优于现有启发式方法？

### 4. 背景与问题设定

多模态大语言模型（MLLM）将视觉输入编码为大量令牌，与文本令牌一起通过所有 Transformer 层，自注意力复杂度随序列长度二次增长，导致高计算和内存开销。现有剪枝方法（如基于注意力、多样性或覆盖度）多为启发式评分，缺乏对剪枝内在目标的原理性表述，常保留冗余或丢弃关键令牌。

### 5. 方法概览

- 任务 / 环境: MLLM 视觉-语言推理任务，包括图像 VQA、高分辨率输入和视频理解，涉及 LLaVA-1.5、LLaVA-NeXT、LLaVA-Video、Qwen2.5-VL-7B、InternVL3-8B 等架构。
- 变量 / 干预设计: 提出 TOPS 模块，基于信息论推导的三个原则（任务相关性、信息覆盖、语义多样性），通过贪心选择构建最优令牌保留集。分为两阶段：Stage I 在投影后使用 CLS 注意力粗剪枝，Stage II 在 LLM 内部指定层使用文本-视觉注意力细剪枝。
- 评价指标: 相对性能（Rel.%）：剪枝后模型准确率与基线之比；FLOPs、延迟、内存占用。
- 模型比较: 在 LLaVA-1.5、LLaVA-NeXT、LLaVA-Video、Qwen2.5-VL-7B、InternVL3-8B 上，与 FastV、PyramidDrop、SparseVLM、DivPrune、DART、VisionZip、TRIM、PruMerge+、SCOPE、CDPruner 等 10 种方法对比。
- 我的理解: TOPS 的核心创新在于从互信息最大化 I(S;V,Q) 出发，通过链式法则分解为 I(S;Q)（任务相关性）和 I(S;V|Q)（信息覆盖），并进一步推导出语义多样性作为必要条件。该分解为贪心选择提供了理论依据，避免了传统方法的启发式缺陷。

### 6. 关键发现

- TOPS 在多种 MLLM 上一致最优: 在 LLaVA-1.5-7B 上，移除 77.8% 令牌保留 98.3% 性能；在 LLaVA-NeXT-7B 上，移除 77.8% 令牌达到 100.0% 性能；在 Qwen2.5-VL-7B 和 InternVL3-8B 上，80% 剪枝率下分别保留 96.3% 和 96.7% 性能，优于所有对比方法。
- 多原则组合优于单一原则: 实验表明，基于注意力（任务相关性）在低剪枝率下有效，而多样性和覆盖度在高剪枝率下更重要。TOPS 联合三者，在所有剪枝率下均取得最佳结果，验证了信息论分解的正确性。
- 剪枝可缓解幻觉: 在 HallusionBench 上，TOPS 在所有设置下均取得最高分数，表明去除冗余视觉令牌有助于减少模型幻觉。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: TOPS 可显著降低 MLLM 驱动的智能体系统的推理延迟和内存占用，使其更适用于实时交互场景。

- 对 Memory: 通过减少令牌数量，TOPS 降低了 KV 缓存大小，有利于长序列和视频任务。
- 对评测: TOPS 在 HallusionBench 上的优势提示，剪枝可能成为缓解幻觉的一种新途径，评估时应关注此效应。

### 11. 局限性

## 5. When are likely answers right? On Sequence Probability and Correctness in LLMs

**Topics:** stat.ML, cs.LG, GUI Agent
**Authors:** Johannes Zenn, Jonas Geiping
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27359v1) | [PDF](http://arxiv.org/pdf/2606.27359v1.pdf)

### 1. 一句话结论

序列概率与正确性的正向关联仅在数据集内跨问答对时成立；改变解码方法或超参数提高概率不能稳定提升准确性，且同一提示的多次响应相关性弱。

### 2. 为什么重要

揭示了概率-正确性关系的边界，为解码方法选择、自洽性和无验证器自改进提供指导。尽管论文未直接研究GUI Agent，但其发现对依赖高概率输出（如自改进迭代）的Agent系统有警示意义。

### 3. 研究问题

1. 序列概率与正确性在哪些条件下一致？
2. 不同解码方法和超参数下，提高序列概率能否稳定提升准确性（within-method和across-method相关性）？
3. 对于同一提示的不同响应（within-sample），序列概率是否可指示正确性？

### 4. 背景与问题设定

许多LLM解码方法（如低温采样、截断采样、Beam search、Best-of-N和Power sampling）可视为将概率质量转移到模型更可能的输出上。然而，Beam search常降低性能，暗示概率与正确性的关系并非简单线性。本研究系统量化八个解码方法、十四个模型和六个基准数据集中，序列概率与正确性在四个层级的关联：跨方法（across-method）、方法内（within-method）、数据集内（within-dataset）、样本内（within-sample）。

### 5. 方法概览

- 任务 / 环境: 六个基准数据集：MATH500（数学推理），GPQA（研究生水平问答），MMLU（多任务理解），HumanEval（代码生成），MedQA（医学问答），IFEval（指令遵循）。
- 变量 / 干预设计: 比较八种解码方法：SPS（可扩展功率采样）、Power-SMC、低温采样（LTS）、Beam search、Best-of-N（BoN）、Top-k、Top-p、ε-sampling，在四种粒度分析相关性：跨方法（across-method：不同方法用典型超参数）、方法内（within-method：同一方法不同超参数如温度、束宽）、数据集内（within-dataset：固定方法和超参数跨问答对）、样本内（within-sample：同一提示多次响应）。
- 评价指标: Spearman等级相关系数（ρ）和分箱Pearson相关系数（r）衡量log概率与二元正确性标签的关联。
- 模型比较: 使用Qwen3系列（0.6B、1.7B、4B、8B的基座和后训练模型）、Qwen2.5-8B（基座和后训练）、Olmo3-7B（基座和后训练），总计14个模型。
- 我的理解: 作者构建了多层级分析框架，将解码方法分类为局部（修改逐token分布）和全局（优化完整序列概率），并通过变分角度统一理解全局方法。实验通过对比模型系列和数据集，剥离了方法、超参数和样本的混淆影响。

### 6. 关键发现

- 数据集内（within-dataset）一致正相关，但方法间不传递: 在固定数据集内，log概率与正确性在问答对间呈正相关（MATH500最强，r=0.96；IFEval因基座模型指令遵循能力弱而负相关）。然而，within-method相关性表明：同一方法调整超参数提高序列概率时，准确性变化不一致。across-method相关性表明：不同方法生成的更高概率序列并不稳定对应更高准确性。
- 样本内（within-sample）相关性弱且对称: 同一提示的多次响应间，log概率与正确性的Spearman相关系数分布以零为中心，宽度窄，表明不能仅靠概率进行可靠的自改进。只有当模型已有较高任务准确率时，概率可能提供一定信息。
- 基座模型 vs 后训练模型的差异: 基座模型在IFEval上呈一致负相关，而后训练模型在多数数据集上表现更强且正相关，表明指令调优可能增强概率与正确性的对齐。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 在开发涉及多次生成自改进的Agent系统时（如GUI Agent通过历史动作概率选择），本文发现概率-正确性在样本内弱相关，因此单纯依赖高概率响应可能不可靠；建议结合多数投票或外部验证器。
- 对 Skill: 对于需要高可靠性的技能（如数学推理），数据集内样本间的概率-正确性关联较强，可辅助过滤低概率输出；但对于指令遵循（IFEval），需谨慎。
- 对 Memory: within-sample相关性弱，意味着同一提示的多次生成中，高概率回答未必正确，因此自洽性方法需结合其他验证机制（如多数投票或外部验证器）。
- 对评测: 评估时，应将序列概率作为分析工具而非性能提升手段；在开发无验证器自改进时需考虑within-sample相关性不足的限制。

### 11. 局限性

- 研究未涵盖所有可能解码方法和超参数空间（如逐步高温退火），可能忽略某些能稳定改善准确性的设置。

### 12. 我的看法 / 个人笔记

本工作对LLM解码领域常见假设（高概率→正确）提供了严谨的量化校准，其多层级分析框架具有方法论价值。核心洞察是：概率与正确性的关联是数据集内现象而非跨方法恒等式，这解释了早期Beam search的悖论。实践者应警惕将概率直接作为正确性代理，尤其在方法选择时。对于GUI Agent等依赖自改进的领域，本文结果提示需要谨慎。
