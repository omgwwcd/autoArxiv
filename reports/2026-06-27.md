## 1. OpenRCA 2.0: From Outcome Labels to Causal Process Supervision

**Topics:** Root Cause Analysis, LLM Evaluation, Causal Reasoning, Microservice Systems
**Authors:** Aoyang Fang, Yifan Yang, Jin'ao Shang, Qisheng Lu, Junjielung Xu, Rui Wang, Songhan Zhang, Yuzhong Zhang, Boxi Yu, Pinjia He
**Venue / Year:** NeurIPS 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27154v1) | [PDF](http://arxiv.org/pdf/2606.27154v1.pdf)

### 1. 一句话结论

OpenRCA 2.0 引入过程级因果标注，揭示 LLM 代理在根因分析中常给出正确服务但缺乏可验证因果路径的“无根基诊断”问题。

### 2. 为什么重要

现有 RCA 基准仅评估结果标签，掩盖了代理推理中的关键缺陷。OpenRCA 2.0 提供步骤级因果真值，推动更可信的 LLM 代理评估。

### 3. 研究问题

1. LLM 代理在根因分析中能否同时识别正确根因并构建可验证的因果传播路径？
2. 过程级评估与结果级评估在揭示代理推理失败模式上有何差异？

### 4. 背景与问题设定

根因分析（RCA）是 LLM 代理能力的综合测试，但现有数据集仅标注根因，忽略从根因到症状的传播路径，导致任务简化为模式匹配。本文提出 PAVE 协议，利用故障注入中的已知干预，通过前向验证重建因果传播路径，并构建跨系统基准 OpenRCA 2.0。

### 5. 方法概览

- 任务 / 环境: 三个微服务系统（TrainTicket 44服务、OTel Demo 15服务、Hotel Reservation 9服务），涵盖不同语言栈和通信模式，注入 27 种故障类型，原始池 8137 次运行，经筛选得 500 实例。
- 变量 / 干预设计: PAVE 协议利用故障注入的干预信息（目标组件、故障类型、时间窗口）进行前向验证：阶段1基于拓扑和传播规则集（如 RPC 超时级联、资源争用）进行结构剪枝，枚举候选路径；阶段2通过基线对比验证统计偏差（Z-score 或百分位数检验）和时间对齐（下游异常不早于上游），仅当三者（结构、统计、时间）均满足时确认路径。
- 评价指标: 结果层：精确匹配（EM）、F1、精确率、召回率、AnySvc（至少一个正确服务）；过程层：路径可达性（PR，至少一个正确服务且存在有效路径）、节点 F1、边 F1、SQL 执行率。
- 模型比较: 11 个前沿 LLM（Claude Opus 4.7、Gemini 3.1 Pro、Claude Sonnet 4.6、MiMo 2.5 Pro、GLM 5.1、Kimi K2.6、DeepSeek V4 Pro、Qwen3.6-Max、Hy 3.0 Preview、Seed 2.0 Pro、MiniMax M2.7），共享 ReAct 风格工具集和提示模板，无依赖图提供。
- 我的理解: PAVE 的关键创新在于利用干预信息将逆向推理转为前向验证，通过结构、统计、时间三重约束确保因果路径可靠性，避免了从症状反向推断的歧义性。

### 6. 关键发现

- 结果级指标低估失败程度：无根基诊断普遍存在: 平均 AnySvc 达 76.0%，但 PR 仅 61.5%，14.5 个百分点案例中代理给出正确服务却无有效路径。例如 Claude Sonnet 4.6 的 PR/AnySvc 比率仅 66%，而 Qwen3.6-Max 达 91%，表明结果级评分相近的模型在推理质量上差异显著。
- 边 F1 普遍低于节点 F1，差距显著: 平均节点 F1 62.2%，边 F1 43.4%，差距 18.8 个百分点。Gemini 3.1 Pro 差距最大（67.9% vs 36.8%），表明代理识别服务比推理依赖关系更可靠。
- 无单一模型主导，排名依赖评估维度: 各列最优值分布在四个不同模型：Gemini 3.1 Pro 在 EM（29.4%）和 F1（43.8%）最优，Qwen3.6-Max 在 PR（71.8%）最优，GLM 5.1 在 Edge F1（50.4%）最优，Claude Opus 4.7 在 AnySvc（82.6%）最优。系统间差异大于模型间差异：Hotel Reservation 在结果层接近饱和，而 TrainTicket 过程层差距超 30 个百分点。
- 三种典型失败模式: 基于分层抽样分析，代理常犯：1）幻觉边（添加不存在的依赖）；2）跳过节点（遗漏中间服务，如图1中跳过 Travel）；3）错误排序（方向颠倒）。这些模式在结果级评估中完全隐藏。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: RCA 代理需从结果导向转向过程可验证，确保推理路径的因果一致性，例如通过显式输出因果图并附证据。
- 对 Skill: 代理需提升因果推理能力，特别是将服务识别转化为有向依赖图，避免幻觉边和跳过节点。
- 对 Memory: 长期记忆可存储已验证的因果模式（如特定故障类型的典型传播路径），辅助未来诊断。
- 对评测: 评估应包含过程级指标（如路径可达性、边 F1），避免结果级指标掩盖推理缺陷。

### 11. 局限性

- PAVE 依赖故障注入的干预信息，无法应用于无干预的真实场景，限制了其在实际运维中的直接使用。
- 基准仅覆盖三个微服务系统（TrainTicket、OTel Demo、Hotel Reservation），且 Hotel Reservation 在结果层接近饱和，泛化性有限。
- 因果验证的统计阈值（如 Z-score 阈值）可能遗漏弱信号传播，导致假阴性。

### 12. 我的看法 / 个人笔记

本文通过过程级标注揭示了 LLM 代理在 RCA 中的关键弱点，为未来研究提供了更严格的评估标准。PAVE 协议的设计巧妙，利用干预信息进行前向验证，但实际部署中干预信息的可用性可能限制其应用。实验设计严谨，多系统对比和失败模式分析增强了结论的可靠性。

### 14. 总结

OpenRCA 2.0 通过 PAVE 协议构建首个带步骤级因果标注的跨系统 RCA 基准，发现 LLM 代理在 76.0% 案例中识别正确服务，但仅 61.5% 能提供可验证因果路径，结果级评估掩盖了这一“无根基诊断”问题。过程级评估揭示边 F1（43.4%）显著低于节点 F1（62.2%），且存在幻觉边、跳过节点等典型失败模式，为可信 RCA 代理评估提供了新标准。

## 2. Joint Learning of Experiential Rules and Policies for Large Language Model Agents

**Topics:** LLM Agent, Reinforcement Learning, Experience Reuse, GUI Agent
**Authors:** Shicheng Ye, Chao Yu
**Venue / Year:** arXiv 2606.27136v1, 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27136v1) | [PDF](http://arxiv.org/pdf/2606.27136v1.pdf)

### 1. 一句话结论

JERP通过联合更新经验规则库和策略参数，在AlfWorld和WebShop上持续提升了LLM智能体在多步交互任务中的决策性能。

### 2. 为什么重要

该工作首次将外部规则池和策略参数在同一个训练循环中联合更新，解决了以往规则与策略不同步、局部错误修正不足的问题，为稀疏奖励下的复杂交互任务提供了更稳定有效的学习范式。

### 3. 研究问题

1. 如何在同一交互轨迹上同时更新显式可读的规则池和隐式的模型策略参数？
2. 稀疏奖励环境下，联合更新能否比单独使用规则或参数更新带来更一致的性能提升？

### 4. 背景与问题设定

现有LLM智能体通常将交互经验以两种方式利用：一是外部化为自然语言规则供后续提示（如Reflexion、ExpeL），二是通过轨迹反馈直接更新模型参数（如RLHF、GRPO）。然而，前者在策略进化时规则可能过时，后者在稀疏奖励下对局部错误修正不足。JERP提出了一个联合学习框架，使同批轨迹同时用于策略优化和规则库修订。

### 5. 方法概览

- 任务 / 环境: AlfWorld（具身家居环境）和WebShop（在线购物环境），都是多步交互、稀疏奖励的POMDP任务。
- 变量 / 干预设计: 每个任务实例维护一个长期经验规则库K(d)，包括文本规则和效用分数；每轮episode开始时按效用分数选出top-k规则作为工作集；policy同时基于任务描述、历史交互和当前规则集生成动作；episode结束后，用GRPO进行参数更新，同时通过对比当前轨迹和参考成功轨迹生成规则编辑操作（新增/更新/合并/投票等）来更新规则库。
- 评价指标: 任务成功率、轨迹折扣回报G(τ)。
- 模型比较: 基线包括单独使用规则学习的ExpeL和单独使用参数优化的GRPO/GiGPO，以及它们的结合。实验结果显示JERP在所有环境上一致优于单独优化范式。
- 我的理解: JERP的核心创新在于“联合循环”：每次episode收集的轨迹既用于训练策略（GRPO损失），又通过对比成功与失败轨迹产生规则编辑（对比反思），使规则与策略共同进化，避免规则陈旧或策略忽略局部信息。

### 6. 关键发现

- 联合更新规则与策略效果优于单独任意一种: 在AlfWorld和WebShop上，JERP的任务成功率均高于仅用规则学习（ExpeL）或仅用策略优化（GRPO/GiGPO）的方法，且增益统计显著。
- 规则库质量随策略进化动态改善: 通过效用分数和合并操作，规则库保持精简且与当前策略对齐，避免了规则过时带来的负迁移。
- 合并操作有效控制规则库规模: 在训练过程中，规则库规模并未无限增长，而是通过语义合并维持了一个合理数量，提高了检索效率。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 未来的LLM智能体可以内建一个自更新的规则库，使其在持续交互中既提升内部策略，又积累可解释的外部知识，尤其适合需要长期自我改进的部署场景。
- 对 Skill: 规则库的合并操作提示我们，可以设计一种“技能压缩”机制，将多次验证成功的规则固化为更抽象的技能或子程序，融入模型本体。
- 对 Memory: 该工作展示了“效用分数+合并”的规则管理策略，为长时记忆的维护提供了可行的工程方案：避免记忆无限膨胀，同时保证记忆与策略同步。
- 对评测: 评估联合学习系统时需要同时衡量任务成功率、规则库的时效性和策略的一致性，而不仅仅是最终性能。

### 11. 局限性

- 规则更新依赖于LLM的反思能力，生成编辑操作的质量可能受限，且增加了推理开销。
- 当前仅按任务级别维护规则库，未进行实例级细粒度检索，对跨任务迁移支持有限。
- 实验仅覆盖两个环境，未在更多复杂交互任务（如WebArena、Minecraft）上验证。

### 12. 我的看法 / 个人笔记

JERP巧妙地弥补了规则学习和策略学习之间的鸿沟，为稀疏奖励场景提供了一个协调统一的学习范式。其实出亮点在于“共同数据源”和“双向反馈”，不过规则编辑的自动化部分可能产生噪声，未来可以探索如何用强化学习信号直接驱动规则编辑。总体而言，这是一个很有潜力的研究方向，尤其适合需要长期自我进化的自主智能体。

## 3. Application of LLMs to Threat Assessment of Foreign Peacekeeping Missions

**Topics:** LLM, threat assessment, peacekeeping missions, OSINT
**Authors:** Gerhard Backfried, Christian Schmidt, Diego Pilutti, Michael Suker
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.27106v1) | [PDF](http://arxiv.org/pdf/2606.27106v1.pdf)

### 1. 一句话结论

LLMs can effectively extract and ground threats from multilingual media for peacekeeping missions, achieving high agreement with human experts on core relevance.

### 2. 为什么重要

This work demonstrates a practical, semi-automated LLM pipeline for real-world threat assessment in complex geopolitical settings, potentially improving scalability and timeliness while preserving human oversight.

### 3. 研究问题

1. Can LLMs reliably extract mission-relevant threats from multilingual OSINT data?
2. How well do LLM-generated threat assessments align with expert human judgment?

### 4. 背景与问题设定

EU peacekeeping missions (e.g., EUMM Georgia) require continuous threat assessment across diverse socio-political, demographic, and environmental factors. Manual analysis of vast, multilingual open-source media is slow and resource-intensive. Existing AI approaches are limited; LLMs offer multilingual, semantic reasoning capabilities but lack deployment in this domain.

### 5. 方法概览

- 任务 / 环境: Threat assessment for EUMM Georgia using OSINT media from 300+ sources (TV, radio, social media) in English, Russian, and other languages.
- 变量 / 干预设计: LLMs (few-shot, reasoning mode) process media documents to extract structured threat information (threat description, justification, actors, locations, severity, immediacy). A second LLM stage grounds and filters threats for mission relevance.
- 评价指标: Accuracy score (0-1) based on expert ratings of 11 binary/partial questions per threat (e.g., threat relevance, mission relevance, location relevance).
- 模型比较: Not explicitly compared to other models; evaluated against human expert judgment.
- 我的理解: The pipeline uses indicator-based prompts derived from a risk model, translates non-English documents, then applies LLMs for extraction and grounding. Threats are clustered and visualized for analysts.

### 6. 关键发现

- High agreement on core threat aspects: Average expert agreement score of 0.82 (0.79 strict) across 48 threats. Threat relevance, mission relevance, and location relevance scored 0.89-0.95.
- Lower scores for threat level and actors: Threat level estimation scored lower (perceived too low); actor identification limited by mission context (e.g., breakaway regions, Russia).
- LLMs enable scalable multilingual processing: Successfully processed English and Russian documents, extracting 8340 threats across two periods (2023, 2024) for four threat types.

- 图来源: Figure 2: Evaluation Scores
- 为什么重要: Quantifies LLM performance across 11 expert-rated questions, showing strengths (threat relevance) and weaknesses (threat level, actors).

### 9. 次重要图

- 为什么也重要: Figure 1 (EUMM mission map) provides geographic context for the use-case, grounding the threat assessment in a real operational setting.

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: LLM agents can be designed to autonomously monitor and extract threats from OSINT, but require human-in-the-loop for validation.
- 对 Skill: Enables analysts to focus on high-level interpretation rather than manual data sifting.
- 对 Memory: Structured threat storage (JSON) supports temporal and geographic clustering for situational awareness.
- 对评测: Expert-in-the-loop evaluation framework (11 questions) can be reused for similar LLM-based extraction tasks.

### 11. 局限性

- Small evaluation set (48 threats) from a single mission; generalizability uncertain.
- Threat level and actor identification need improvement; human validation essential.
- No comparison with other LLMs or traditional NLP methods; reverse evaluation (expert-generated threats vs. LLM) not performed.

## 5. RolloutPipe: Overlapping Pipelined Rollout and Training in Disaggregated On-Policy LLM Reinforcement Learning

**Topics:** LLM Post-Training, Reinforcement Learning with Verifiable Rewards, Disaggregated Systems, Pipeline Parallelism
**Authors:** Rongjian Chen, Jianmin Hu, Kejiang Ye, Minxian Xu
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.26997v1) | [PDF](http://arxiv.org/pdf/2606.26997v1.pdf)

### 1. 一句话结论

RolloutPipe通过完整组流水线和前沿组调度，在保持策略正确性的同时，将解耦的on-policy GRPO训练中训练GPU的空闲等待时间转化为有效的训练重叠，缩短端到端时间30.7%-42.3%。

### 2. 为什么重要

LLM推理后训练中，GRPO等on-policy RL方法要求组级统计量，导致传统串行rollout-then-train模式中训练GPU长时间空闲。RolloutPipe首次在不引入陈旧数据的前提下，实现了rollout与训练的有效重叠，显著提升了资源利用率，对大规模LLM后训练系统设计具有重要指导意义。

### 3. 研究问题

1. 如何在不违反on-policy正确性的前提下，让GRPO训练在完整rollout完成之前就开始？
2. 如何通过调度策略使最早完成的组更早、更稳定地到达训练端，从而最大化流水线重叠收益？

### 4. 背景与问题设定

LLM后训练越来越多地采用基于可验证奖励的强化学习（RLVR），其中GRPO因移除critic而流行。现代RLVR系统采用解耦架构，将rollout生成和策略训练分离到独立GPU池。然而，现有同步on-policy GRPO系统（如Slime）必须等待所有rollout组完成后才开始训练，导致训练GPU在整个rollout阶段空闲。异步方法虽能重叠阶段，但会引入陈旧数据。RolloutPipe旨在解决这一矛盾。

### 5. 方法概览

- 任务 / 环境: 解耦的LLM后训练系统，包含独立的rollout GPU池（运行SGLang）和训练GPU池（运行Megatron-LM），使用GRPO算法在数学、逻辑和科学推理任务上微调模型（如Qwen3-1.7B）。
- 变量 / 干预设计: 提出两种机制：完整组流水线（CGP）在训练端，一旦某个组完成物化（所有K个响应完成奖励计算和组统计量计算），立即将其送入训练器FIFO，当累积U个组时开始一次训练更新；前沿组调度（FGD）在rollout端，维护一个大小为Fw的前沿集合，只调度属于该集合的组的请求，优先完成最早提交的组，使训练端能更早获得完整批次。
- 评价指标: 主要指标：rollout到训练结束时间（rollout-to-train-end time）和训练器等待比率（trainer waiting ratio，即训练器空闲等待时间占端到端时间的比例）。
- 模型比较: 与Slime（当前最先进的rollout和训练系统）对比，在Qwen3-1.7B上使用四个推理和科学基准测试，覆盖12种rollout设置。
- 我的理解: RolloutPipe的核心洞察是：GRPO的训练单位是完整组而非单个样本，但组完成时间存在差异。CGP利用这一差异，将串行边界从整个round细化到组级别，使训练能随组完成而逐步开始；FGD则通过主动控制组完成顺序，进一步压缩首个训练批次就绪的时间。两者结合，在不改变训练语义的前提下，将空闲等待转化为有效计算。

### 6. 关键发现

- CGP显著降低训练器空闲时间: 在Qwen3-1.7B和LSAT-AR任务上，Slime的训练器等待比率高达47%-52%，而CGP通过让训练在第一个U个组完成后立即开始，将等待比率降低37%-76%。
- FGD进一步提前首个训练批次就绪时间: FGD通过优先调度前沿组，使最早完成的组更早、更稳定地到达训练端。在CGP基础上叠加FGD，首个U组就绪时间进一步提前，端到端时间缩短更多。
- RolloutPipe在多种设置下均优于Slime: 在12种rollout设置（不同R、K、任务）下，RolloutPipe（CGP+FGD）将rollout到训练结束时间缩短30.7%-42.3%，且保持on-policy正确性。

- 对 Skill: 对于需要多步推理的技能学习，更快的训练循环意味着更高效的策略迭代。
- 对 Memory: CGP和FGD不增加额外内存开销，因为组级流水线不改变每个组的数据量。
- 对评测: 训练器等待比率成为评估RL系统效率的新指标，RolloutPipe展示了如何量化空闲时间。

### 11. 局限性

### 12. 我的看法 / 个人笔记

RolloutPipe是一个优雅且实用的系统优化，它精准识别了GRPO训练中组级依赖带来的空闲问题，并通过巧妙的流水线设计解决了它。CGP和FGD的设计简洁有效，且不牺牲训练正确性。未来工作可探索自适应Fw调整、扩展到更大模型和更多样化的RL算法。
