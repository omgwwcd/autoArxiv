## 1. Brief Is Better: Non-Monotonic Chain-of-Thought Budget Effects in Function-Calling Language Agents

**Topics:** Agent, Skill, Memory, RL, Evolve
**Authors:** Xuan Qi
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2604.02155v1) | [PDF](http://arxiv.org/pdf/2604.02155v1.pdf)

### 1. 一句话结论

简短的思维链（32 tokens）显著提升函数调用语言代理的准确性，而过长的思维链会导致性能下降，提出结构化简短思维链（FR-CoT）可有效防止函数幻觉并实现可靠路由。

### 2. 为什么重要

挑战了“更多推理总是更好”的假设，揭示了在结构化工具使用场景中推理长度与准确性的非单调关系，为语言代理系统设计提供了重要指导，特别是通过结构化提示实现可靠函数路由。

### 3. 研究问题

1. 在函数调用代理中，最优的思维链（CoT）推理令牌数量是多少？
2. 推理长度如何影响准确性？
3. 是否可以预测哪些任务从额外推理中受益？

### 4. 背景与问题设定

语言代理在使用工具和API时，通常采用链式思维（CoT）来选择正确的函数及其参数。实践中，通常选择固定的推理预算（如512个令牌），假设更多推理总是更好。然而，这种假设在结构化工具使用场景中尚未得到充分理解。

### 5. 方法概览

- 任务 / 环境: 使用Berkeley Function Calling Leaderboard v3 Multiple基准，包含200个任务，每个任务提供用户查询和2-4个候选函数模式，代理需输出指定函数名和参数的JSON对象。
- 变量 / 干预设计: 在Qwen2.5-1.5B-Instruct模型上，测试六种令牌预算（0, 32, 64, 128, 256, 512）下的CoT效果。提出Function-Routing CoT (FR-CoT)，通过模板化推理阶段（"Function: [name] / Key args: [...]"）强制在推理开始时承诺一个有效函数名。
- 评价指标: 准确率（Accuracy），有效性失败率（Validity failure rate），内容错误率（Content error rate），函数幻觉率（Hallucination rate），错误分解（三类错误：幻觉函数、错误有效函数、错误参数）。
- 模型比较: 比较了Qwen2.5-1.5B-Instruct和Qwen2.5-7B-Instruct两个模型在不同预算下的表现，以及FR-CoT与自由形式CoT（d=32）和约束解码基线的比较。
- 我的理解: 研究揭示了在函数调用代理中，推理长度与准确性的非单调关系，即简短的推理（32 tokens）能显著提升性能，而过长的推理（256 tokens）反而会因函数选择错误和幻觉而降低性能。FR-CoT通过结构化提示强制函数路由，有效解决了这一问题，实现了与自由形式简短推理相当的准确率，但消除了函数幻觉，提供了结构上的可靠性保证。

### 6. 关键发现

- 非单调的CoT效应: 在Qwen2.5-1.5B-Instruct上，简短推理（32 tokens）将准确率从44.0%提升至64.0%（+45%相对），而扩展推理（256 tokens）将准确率降至25.0%，低于无CoT基线（McNemar p < 0.001）。
- Oracle预算分布: 88.6%的可解任务的Oracle最优预算d* ≤ 32 tokens，平均为27.6 tokens，表明简短推理对绝大多数任务足够。
- 三类错误分解: 在d=0时，错误主要来自错误的函数选择（30.5%）；简短CoT（d=32）将此降至1.5%，但长CoT（d=256）导致错误选择（28.0%）和幻觉（18.0%）回升。
- FR-CoT方法: 提出的Function-Routing CoT (FR-CoT) 通过模板化推理阶段，强制在推理开始时承诺一个有效函数名，实现了与自由形式d=32 CoT相当的准确率，但将函数幻觉降至0.0%。
- 多模型验证: 该非单调效应在Qwen2.5-7B-Instruct上得到复制，7B模型在d=32时达到82.5%的峰值准确率，在d=256时降至18.0%。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 应避免盲目增加推理预算，而应采用简短的、结构化的推理（如FR-CoT）来提高函数调用的准确性和可靠性。
- 对 Skill: 函数路由是语言代理的一项关键技能，可以通过结构化提示来增强，而非依赖于更长的自由形式推理。
- 对 Memory: 研究结果表明，模型在处理任务时，其内部记忆和推理过程可能在长推理中发生偏移，导致错误，因此需要更有效的记忆和推理机制。
- 对评测: 评估语言代理时，应考虑推理长度的影响，而不仅仅是最终输出的准确性，以更全面地理解模型性能。

### 11. 局限性

- 研究主要在Qwen2.5系列模型上进行，其结论在其他模型家族（如Llama）上的普适性有待验证。
- FR-CoT虽然有效，但其性能仍略低于自由形式d=16的最优预算，表明可能存在更优的结构化方法。

本研究系统地研究了函数调用语言代理中思维链（CoT）预算的影响，发现了一个显著的非单调模式：简短的推理（32 tokens）能大幅提升准确率，而过长的推理（256 tokens）会导致性能严重下降。通过错误分解，揭示了其机制：简短推理作为函数路由步骤，有效解决了错误函数选择问题；而长推理则导致函数选择错误和幻觉。受此启发，提出了结构化的Function-Routing CoT (FR-CoT) 方法，通过模板化推理阶段强制函数路由，实现了与自由形式简短推理相当的准确率，但将函数幻觉降至0.0%，提供了结构上的可靠性保证。

## 2. MTI: A Behavior-Based Temperament Profiling System for AI Agents

**Topics:** Agent Skill, Memory, RL, Evolve
**Authors:** Jihoon Jeong
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2604.02145v1) | [PDF](http://arxiv.org/pdf/2604.02145v1.pdf)

### 1. 一句话结论

MTI 是一个基于行为的 AI 代理气质评估系统，通过四个轴（反应性、合规性、社会性和韧性）测量代理的气质，揭示了气质与模型规模无关，并且 RLHF 会重塑气质特征。

### 2. 为什么重要

现有方法要么借用人类人格维度依赖自我报告（与 LLM 实际行为不符），要么将行为差异视为缺陷而非特质。MTI 提供了一种标准化的行为测量工具，能够区分能力与气质，为 AI 代理的表征、模型选择和对齐评估提供基准。

### 3. 研究问题

1. 如何标准化地测量 AI 代理的气质差异？
2. AI 代理的行为差异是能力问题还是气质问题？
3. RLHF 如何影响代理的气质？
4. 不同气质维度之间是否存在关联？

### 4. 背景与问题设定

AI 模型在等效能力下表现出根本不同的行为模式，但缺乏标准化的测量工具。现有方法或借用人类人格维度依赖自我报告（与 LLM 实际行为不符），或视行为差异为缺陷。MTI 基于行为而非自我报告，旨在测量 AI 代理的气质。

### 5. 方法概览

- 任务 / 环境: 使用结构化检查协议，通过两阶段设计（能力检查和冲突条件）分离能力与气质。所有测量在温度=0下进行，确保确定性输出。
- 变量 / 干预设计: 每个轴使用特定的测试电池：反应性电池（环境条件变化）、合规性电池（指令-行为冲突）、社会性电池（无指令的社交资源分配）、韧性电池（逐步压力下的性能维持）。
- 评价指标: 使用李克特量表相似度评分、嵌入差异、关键词和长度差异作为评分指标。
- 模型比较: 对 10 个小型语言模型（1.7B-9B 参数，6 个组织，3 个训练范式）进行气质分析，比较不同模型在四个轴上的表现。
- 我的理解: MTI 通过行为而非自我报告来测量 AI 代理的气质，其设计原则是行为、代理级测量、两阶段设计、标准温度和成对差异。这使得 MTI 能够区分能力与气质，提供了一个标准化的基准。

### 6. 关键发现

- 四个轴 largely 独立: 在指令调优模型中，四个轴（反应性、合规性、社会性、韧性）之间相关性较低（所有 |r| < 0.42），表明它们是独立的气质维度。
- 轴内特征分离: 合规性分解为完全独立的正式和立场特征（r = 0.002），韧性分解为逆相关的认知和对抗特征（r = -0.481）。
- 合规性-韧性悖论: 意见产出和事实脆弱性通过独立通道运作，对安全部署有直接意义。
- RLHF 的影响: RLHF 不仅改变轴分数，还创造了基线模型中不存在的轴内特征差异。
- 气质与模型大小无关: 气质与模型大小（1.7B-9B）无关，证实 MTI 测量的是气质而非能力。

## 3. APEX: Agent Payment Execution with Policy for Autonomous Agent API Access

**Topics:** Agent, API Access, Payment, Security, Policy
**Authors:** Mohd Safwan Uddin, Mohammed Mouzam, Mohammed Imran, Syed Badar Uddin Faizan
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2604.02023v1) | [PDF](http://arxiv.org/pdf/2604.02023v1.pdf)

### 1. 一句话结论

APEX实现了一个基于HTTP 402的支付门控系统，将加密货币支付范式适配到UPI等实时法币系统，同时保持策略控制、令牌验证和防重放的安全性。

### 2. 为什么重要

随着自主代理成为经济实体，API提供商需要请求级别的货币化和支出治理。现有HTTP 402实现多依赖加密货币，与许多国家的法币系统（如印度UPI）不兼容。APEX填补了这一空白，提供了一个可复现的、基于法币的代理支付基础设施，确保安全和策略保证不被牺牲。

### 3. 研究问题

1. 如何将HTTP 402风格的访问控制适配到法币支付流程中？
2. 如何在法币支付系统中实现策略驱动的支出控制？
3. 如何在代理支付中保证安全的令牌生命周期和防重放？

### 4. 背景与问题设定

自主代理正从简单的检索任务演变为调用API、编排工作流并实时决策的经济实体。API提供商需要请求级别的货币化和程序化支出治理。HTTP 402协议将支付作为协议事件处理，但大多数实现依赖于加密货币。在许多部署环境中，特别是像印度这样拥有强大实时法币系统（如UPI）的国家，这种假设与监管和基础设施现实不符。

### 5. 方法概览

- 任务 / 环境: 一个受保护的API端点，需要成功支付后才能返回数据。系统包含客户端代理、受保护的API、支付API、记账存储和策略引擎。
- 变量 / 干预设计: 系统支持三种基线：无策略（直接访问）、无策略支付（有支付但无支出控制）、有策略支付（有支付和支出控制）。实验在六种场景（正常、超额支出、重放攻击、无效令牌、令牌过期、幂等性）下进行。
- 评价指标: ['成功率', '被阻止的请求数', '平均延迟', '95%置信区间', 'p95延迟', '吞吐量', '总支出']
- 模型比较: 与现有x402系统相比，APEX在法币支持、策略控制、重放保护和实验验证方面具有优势。
- 我的理解: APEX通过一个挑战-结算-消费的生命周期，使用HMAC签名的短期令牌、幂等结算处理和策略感知的支付批准，将HTTP 402风格的支付门控适配到UPI等法币工作流。它在保持策略控制、令牌化访问验证和防重放的同时，实现了法币结算。

### 6. 关键发现

- 策略执行显著降低支出: 策略执行将总支出减少了27.3%（从$550降至$400），同时保持了52.8%的合法请求成功率。
- 安全机制实现100%阻断率: 安全机制对重放攻击（20/20被阻止）和无效令牌（20/20被阻止）实现了100%的阻断率，平均延迟开销为19.6ms。
- 系统具有高可复现性: 多次试验运行显示各场景间方差低（标准差±2.7-8.9ms），在95%置信区间内表现出高可复现性。
