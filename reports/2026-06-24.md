## 1. Automated Summarization of Software Documents: An LLM-based Multi-Agent Approach

**Topics:** LLM-based Multi-Agent Systems, Software Documentation Summarization, Prompt Optimization
**Authors:** Duc S. H. Nguyen, Minh T. Nguyen, Phuong T. Nguyen, Juri Di Rocco, Davide Di Ruscio
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.24689v1) | [PDF](http://arxiv.org/pdf/2606.24689v1.pdf) | [Code](https://github.com/MDEGroup/Metagente)

### 1. 一句话结论

Metagente, a Teacher-Student multi-agent LLM framework with dynamic iteration, outperforms baselines in summarizing software documents (READMEs and HTML app descriptions) by iteratively refining prompts via ROUGE-L feedback.

### 2. 为什么重要

Automating the generation of concise 'About' summaries from lengthy software documents (e.g., GitHub READMEs, Google Play descriptions) can significantly reduce the time and effort for users to understand a repository or app, addressing a common gap where such summaries are missing or poorly written.

### 3. 研究问题

1. Can an LLM-based multi-agent system (Metagente) generate concise and accurate summaries of software documents?
2. How does Metagente's dynamic iteration strategy compare to non-dynamic approaches in terms of efficiency and summary quality?
3. Can Metagente generalize across different document types (Markdown READMEs and HTML app descriptions)?

### 4. 背景与问题设定

Software documents like GitHub READMEs and Google Play app descriptions are often lengthy, and their short 'About' summaries are frequently missing or inadequate. Existing summarization tools like GitSum (based on BART/T5) struggle with mixed-content inputs (code and text). LLMs offer promise but face limitations in domain-specific tasks; multi-agent systems can combine specialized LLMs to overcome these limitations.

### 5. 方法概览

- 任务 / 环境: Summarization of software documents: GitHub README files (Markdown) and Google Play app descriptions (HTML) into short 'About' summaries.
- 变量 / 干预设计: Metagente uses a Teacher-Student architecture with four agents: Extractor (filters noise), Summarizer (generates summary), Teacher (evaluates via ROUGE-L and refines prompt), Prompt Creator (aggregates best prompts). A dynamic iteration strategy halts optimization when ROUGE-L plateaus or declines.
- 评价指标: ROUGE-1, ROUGE-2, ROUGE-L (F1 scores) comparing generated summaries to ground-truth 'About' descriptions.
- 模型比较: Baselines include GitSum (BART/T5), and single LLM variants (GPT-4o-mini, GPT-3.5-turbo, Gemma, Mistral, Llama). Metagente is evaluated with GPT-4o-mini for Extractor/Summarizer and GPT-4o for Teacher/Prompt Creator.
- 我的理解: The approach leverages a feedback loop where the Teacher agent uses ROUGE-L scores to iteratively improve the Summarizer's prompt, effectively automating prompt engineering. The dynamic stopping reduces computational cost by avoiding unnecessary iterations on low-potential samples.

### 6. 关键发现

- Metagente outperforms baselines in ROUGE scores: On both GitHub and Google Play datasets, Metagente achieves higher ROUGE-1, ROUGE-2, and ROUGE-L F1 scores compared to GitSum and single LLM baselines, indicating better semantic alignment and n-gram overlap.
- Dynamic iteration reduces computational cost without quality loss: The dynamic stopping mechanism (halting when ROUGE-L plateaus or declines) achieves comparable or better ROUGE scores while requiring fewer training iterations than a fixed-iteration approach.
- Generalization across document types: Metagente effectively summarizes both Markdown READMEs and structured HTML app descriptions, demonstrating robustness to heterogeneous input formats.

### 9. 次重要图

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: Demonstrates that multi-agent collaboration with a Teacher-Student loop can effectively automate prompt optimization, reducing the need for manual prompt engineering.
- 对 Skill: The approach can be adapted to other SE tasks requiring domain-specific summarization or text generation, such as code comment generation or bug report summarization.
- 对 Memory: The dynamic iteration strategy reduces computational overhead, making multi-agent systems more practical for resource-constrained environments.
- 对评测: ROUGE-L as a feedback metric for prompt refinement is effective; future work could explore other metrics (e.g., BERTScore) for semantic evaluation.

### 11. 局限性

- Evaluation is limited to two datasets (GitHub and Google Play); generalizability to other software document types (e.g., API docs, technical reports) is not tested.
- The approach relies on GPT-4o for Teacher and Prompt Creator agents, which may be costly and not accessible to all researchers.
- Dynamic iteration threshold is not extensively tuned; optimal stopping criteria may vary across tasks.

### 12. 我的看法 / 个人笔记

Metagente presents a practical and effective application of multi-agent LLMs for software documentation summarization. The dynamic iteration strategy is a smart efficiency improvement. However, the reliance on proprietary LLMs (GPT-4o) limits reproducibility and cost-effectiveness. Future work should explore open-source LLMs and more diverse document types.

### 13. 这篇工作引用的近期高影响力相关文章

  为什么重要：First approach to summarize GitHub READMEs into 'About' descriptions using BART/T5, but struggles with mixed code/text inputs.
  核心贡献：Proposes a transformer-based summarization model for software documents.
  与当前论文关系：Metagente directly builds on GitSum's problem setting and aims to overcome its limitations using multi-agent LLMs.

### 14. 总结

Metagente is an LLM-based multi-agent system for summarizing software documents (READMEs and app descriptions) into concise 'About' summaries. It uses a Teacher-Student architecture with four agents (Extractor, Summarizer, Teacher, Prompt Creator) and a dynamic iteration strategy that halts optimization when ROUGE-L plateaus. Evaluated on GitHub and Google Play datasets, Metagente outperforms baselines (GitSum, single LLMs) in ROUGE scores while reducing computational cost. The work highlights the potential of multi-agent systems for automated prompt engineering in SE tasks.

## 2. Reinforcement Learning for Computer-Use Agents with Autonomous Evaluation

**Topics:** GUI Agent, Reinforcement Learning, Autonomous Evaluation
**Authors:** Marta Sumyk, Oleksandr Kosovan
**Venue / Year:** GLOW @ IJCAI 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.24515v1) | [PDF](http://arxiv.org/pdf/2606.24515v1.pdf) | [Code](https://github.com/martasumyk/rl with autonomous feedback)

### 1. 一句话结论

通过显式建模视觉语言模型评估器的噪声并推导无偏奖励估计器，强化学习微调计算机使用代理在三个操作系统上的成功率平均提升12.6个百分点。

### 2. 为什么重要

该工作解决了桌面GUI环境中强化学习缺乏可扩展奖励信号的瓶颈，为无需人工标注的自主代理训练提供了实用框架。

### 3. 研究问题

1. 自主VLM评估能否作为计算机使用代理强化学习微调的可扩展奖励信号？
2. 建模和校正评估器噪声是否比直接使用原始评估器奖励能改善策略学习？

### 4. 背景与问题设定

计算机使用代理（CUA）通过感知和操作图形用户界面执行高级用户目标，但桌面环境缺乏机器可读的奖励信号，任务成功通常依赖视觉判断且难以手工设计奖励函数。现有方法依赖任务特定启发式或人工标注，无法跨应用和操作系统扩展。

### 5. 方法概览

- 任务 / 环境: 三个桌面环境：macOSWorld、Windows Agent Arena、OSWorld，覆盖macOS、Windows和Linux操作系统。
- 变量 / 干预设计: 使用Qwen2-VL-7B作为自主评估器，基于最终截图和指令输出二元完成信号；将评估器噪声建模为混淆矩阵，推导噪声校正奖励估计器用于PPO优化。
- 评价指标: 任务成功率（success rate），对比零样本基线、原始评估器奖励和校正奖励。
- 模型比较: 比较五种配置：零样本基线、统一模型+原始奖励、统一模型+校正奖励、每OS模型+原始奖励、每OS模型+校正奖励。
- 我的理解: 该方法将评估器视为有噪声的二元奖励通道，通过校准集估计假阳性和假阴性率，然后对PPO中的奖励进行线性校正，使得校正后的奖励期望等于真实任务完成度。

### 6. 关键发现

- 校正奖励显著优于原始奖励和零样本基线: 在三个基准上，校正奖励微调的平均成功率比零样本提升12.6个百分点，比原始奖励微调提升5.1个百分点。
- 每OS微调优于统一模型: 每OS校正奖励微调在macOSWorld上从8.4%提升至20.3%，在Windows Agent Arena上从33.1%提升至44.2%，在OSWorld上从28.3%提升至43.2%。
- 原始奖励微调存在不一致性: 原始奖励在某些设置下可提升域内性能，但泛化能力较差，尤其在OSWorld上表现不如校正奖励。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 提供了一种无需人工标注的RL微调方法，可提升CUA在真实桌面环境中的鲁棒性和任务成功率。

## 3. Agentic Collaborative Cognition for Zero-Shot 3D Understanding

零样本3D理解通常将任务转化为视频关键帧理解，但视频视角固定且有限，且现有方法隐式推断空间布局，难以应对视角变化。本文提出协作式多智能体框架，通过规划智能体主动补充视角，感知智能体构建结构化认知地图，实现显式、全面的3D场景理解。

## 4. VisCritic: Visual State Comparison as Process Reward for GUI Agents

**Topics:** GUI Agent, Visual Process Reward, VLM, Action Verification
**Authors:** Jiachen Qian
**Venue / Year:** arXiv 2026
**Links:** [Abstract](http://arxiv.org/abs/2606.24525v1) | [PDF](http://arxiv.org/pdf/2606.24525v1.pdf)

### 1. 一句话结论

VisCritic通过直接比较动作前后的截图在视觉特征空间中进行过程奖励验证，显著提升了GUI智能体在长程任务中的性能。

### 2. 为什么重要

现有GUI智能体缺乏步骤级验证，导致错误级联；VisCritic首次利用视觉状态比较作为过程奖励信号，弥补了文本验证与视觉状态变化之间的模态鸿沟，为提升智能体可靠性提供了新范式。

### 3. 研究问题

1. 如何利用视觉状态比较作为GUI智能体动作验证的过程奖励信号？
2. 能否在不依赖人工标注的情况下，从现有轨迹中自动构建批评者训练数据？

### 4. 背景与问题设定

GUI智能体在长程任务中常因单步错误而失败，现有过程奖励模型仅通过文本推理验证动作，忽略了GUI状态变化的视觉本质。VisCritic提出直接比较动作前后的截图，在视觉特征空间中检测动作是否产生预期效果。

### 5. 方法概览

- 任务 / 环境: 五个基准测试：Mind2Web（网页）、AndroidWorld（移动）、OSWorld（桌面）、WebArena（网页）、AITW（移动离线诊断，仅用于记录截图一致性诊断，非交互设置）。
- 变量 / 干预设计: VisCritic作为即插即用模块，主要采用后执行验证模式（默认），在智能体执行动作后，通过Siamese视觉差异编码器提取前后截图的语义差异，结合动作和任务指令，由多任务批评头预测动作成功概率、任务进度和错误类型。可选地，在Best-of-N预执行模式中，利用状态预测器生成候选动作并评分。
- 评价指标: 任务成功率（AndroidWorld/OSWorld/WebArena）、步骤成功率（Mind2Web）、动作准确率（AITW，离线诊断）。
- 模型比较: 与无批评者基线、文本PRM（GUI-PRA、GuidNav）、文本错误检测（BacktrackAgent）、预执行批评者（GUI-Critic-R1）比较。VisCritic在多数设置中优于文本基线，但在OSWorld上Qwen2.5-VL的文本批评者略优（+5.4% vs +5.1%），表明文本密集型桌面环境可能更适合文本验证。
- 我的理解: VisCritic的核心创新在于将视觉变化检测（如Siamese ViT）应用于GUI智能体的过程验证，通过弱监督数据构建避免了人工标注，且与智能体架构解耦，易于集成。其与文本PRM的对比揭示了模态互补性：视觉方法在动态UI变化中更可靠，而文本方法在文本密集型场景中仍有优势。

### 6. 关键发现

- 视觉比较优于文本验证: 在控制模型容量的消融实验中，VisCritic（视觉）的F1为84.6%，而文本批评者仅66.2%，表明视觉状态比较提供了更可靠的验证信号。
- 即插即用提升多智能体性能: 在四个交互式基准测试中，VisCritic提升了SeeClick、ShowUI、CogAgent和Qwen2.5-VL的平均任务成功率，其中Qwen2.5-VL提升5.4个百分点，ShowUI提升4.6个百分点。
- 零样本跨平台泛化: VisCritic在未参与训练的OSWorld（桌面）和WebArena（网页）上仍表现良好，证明其跨环境泛化能力。
- 文本批评者在特定场景略优: 在OSWorld上，Qwen2.5-VL的文本批评者（+5.4%）略优于VisCritic（+5.1%），表明文本密集型桌面环境可能更适合文本验证。

- 对 Agent 系统: VisCritic可作为标准模块集成到任何截图型GUI智能体中，提供步骤级验证，减少错误级联，提升任务成功率。
- 对 Skill: 通过检测动作是否产生预期视觉效果，VisCritic有助于智能体学习更精确的交互技能，如点击目标元素而非相邻元素。
- 对 Memory: VisCritic的视觉差异特征可作为记忆线索，帮助智能体回忆历史状态变化，改进决策。

- 依赖截图质量，在渲染噪声或动画干扰下可能降低性能。

- 在文本密集型桌面环境（如OSWorld）中，文本批评者有时略优，表明视觉方法并非万能，需与文本PRM结合。

### 12. 我的看法 / 个人笔记

VisCritic巧妙地将视觉变化检测应用于GUI智能体验证，弱监督数据构建是其亮点。未来可探索与文本PRM的深度融合，以及更鲁棒的视觉特征提取。

## 5. DeepBD: A Grounded Agentic Workflow for Variant Prioritization and Diagnosis of Genetic Birth Defects

**Topics:** GUI Agent, 基因变异优先级排序, 出生缺陷诊断
**Authors:** Shiyu Li, Ziqi Yan, Zhihao Wu, Jielong Lu, Weiran Liao, Jiajun Yu, Genjie Li, Zeyu Chu, Jiajun Bu, Haishuai Wang
**Venue / Year:** arXiv:2606.24779v1
**Links:** [Abstract](http://arxiv.org/abs/2606.24779v1) | [PDF](http://arxiv.org/pdf/2606.24779v1.pdf)

### 1. 一句话结论

DeepBD 通过将可训练的证据引擎、专业模块和 LLM 辅助诊断审查分离，在遗传性出生缺陷的变异优先级排序中显著优于现有方法。

### 2. 为什么重要

该工作解决了产前和婴儿期遗传性出生缺陷诊断中，如何整合异质性证据（群体遗传、变异效应、基因-疾病关联、表型本体等）进行患者特异性变异排序的关键瓶颈，为临床决策提供了可审计、可基准测试的智能工作流。

### 3. 研究问题

1. 如何为遗传性出生缺陷设计一个将可学习证据整合、工具化精炼和 LLM 辅助审查分离的接地智能体工作流？
2. 在胎儿和婴儿队列中，该工作流在回顾性变异优先级排序上的表现是否优于现有工具（如 Exomiser、DeepRare）和 LLM 重排序基线？

### 4. 背景与问题设定

出生缺陷是胎儿丢失和新生儿发病的主要原因。对于疑似遗传病因的病例，测序后瓶颈在于：临床医生需在不完整的胎儿/婴儿表型及异质性证据下，对患者特异性候选变异进行排序。现有工具（如 Exomiser）整合了部分证据，但缺乏针对出生缺陷场景的、可学习的证据加权机制。

### 5. 方法概览

- 任务 / 环境: 遗传性出生缺陷的变异优先级排序与诊断解释，输入为患者表型（HPO 术语或自由文本）和测序候选变异表，输出为排序后的候选变异列表及诊断报告。
- 变量 / 干预设计: DeepBD 工作流包含四层：LLM 辅助病例结构化、预训练证据引擎（学习规则证据、序列/变异效应表示和表型条件生物上下文）、专业证据模块（SHEPHERD、Exomiser、结构精炼）和接地诊断代理（约束重排序、证据审计、综合报告）。
- 评价指标: Recall@1/3/5/10（病例级，衡量真实致病变异在 top-K 中的召回率）。
- 模型比较: 与 Exomiser、DeepRare、提示 LLM 重排序基线（均基于 Exomiser top-20 候选）比较，DeepBD 在 Recall@1/3/5/10 上达到 0.658/0.882/0.912/0.929，全面超越。
- 我的理解: 该方法的核心创新在于将变异排序任务分配给可训练的证据引擎，使其能从大量胎儿/婴儿病例中学习证据权重，而将语言密集的表型归一化、证据审计和综合留给 LLM，将计算昂贵的专业分析（如蛋白质结构建模）作为可调用模块，实现了模块化、可基准测试的智能体设计。

### 6. 关键发现

- DeepBD 在变异优先级排序上显著优于现有基线: 在内部保留的已解决病例基准（549 例，1307 个阳性变异）上，DeepBD 的 Recall@1/3/5/10 分别为 0.658/0.882/0.912/0.929，全面超过 Exomiser、DeepRare 和 LLM 重排序基线。
- 规则证据、机制上下文和专业精炼提供互补信号: 消融和重叠分析表明，规则证据（如等位基因频率、基因-疾病有效性）、机制上下文（表型-细胞-通路-基因图）和专业模块（如 SHEPHERD、结构建模）各自贡献了独特的排序能力，共同提升了整体性能。
- 接地智能体设计有效分离了不同计算特征的任务: 将可学习的证据整合（证据引擎）、计算昂贵的专业分析（专业模块）和语言密集的审查（LLM 代理）分离，使得系统既能从大规模病例中学习，又能保持模块的可更新性和可审计性。

### 10. 对 Agent / Skill / Memory 的启发

- 对 Agent 系统: 为智能体系统在医疗诊断中的应用提供了一个接地设计范例：将可学习的核心任务（如排序）与语言任务（如审查）分离，避免通用智能体循环在关键决策上的不稳定性。
- 对 Skill: 强调了模块化技能（如专业工具调用）在智能体系统中的价值，这些技能作为可调用模块而非固定输入，提高了系统的灵活性和可扩展性。
- 对 Memory: 预训练证据引擎可视为一种从大规模病例中学习的“记忆”机制，为每个患者提供稳定的证据基底，而 LLM 代理则负责短期的工作空间管理。
- 对评测: 提出了病例级 Recall@K 作为临床有意义的评估指标，并强调了在真实竞争变异分布下进行基准测试的重要性，为遗传诊断系统的评估提供了参考。

### 11. 局限性

- 评估基于内部队列，缺乏外部公共基准验证，泛化性未知。
- 数据集因包含敏感临床信息而无法公开，限制了可重复性。
- 当前评估聚焦于回顾性变异优先级排序，前瞻性临床效用尚未验证。

### 12. 我的看法 / 个人笔记

DeepBD 通过精心设计的模块化智能体架构，有效解决了遗传诊断中证据整合的难题，其“接地”设计理念（将可学习任务与语言任务分离）为 AI 在临床决策支持中的应用提供了重要思路。未来需在外部数据集和前瞻性研究中进一步验证。
